# Exécuter l'exemple

## Installer les dépendances

```sh
npm install
```

## Construire le projet

```sh
npm run build
```

## Lancer le projet

```sh
npm start
```

Vous devriez voir le texte :

```text
Registering tools...
Starting server...
```

Parfait, cela signifie qu'il démarre correctement.

## Tester le serveur

Testez les fonctionnalités avec la commande suivante :

```sh
npx @modelcontextprotocol/inspector node build/app.js
```

Cela devrait lancer l'interface web de l'outil d'inspection.

Exécuter un outil :

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

Vous devriez voir une réponse similaire à :

```text
{
  "content": [
    {
      "type": "text",
      "text": "Tool add called with arguments: {\"a\":1,\"b\":2}, result: {\"content\":[{\"type\":\"text\",\"text\":\"3\"}]}"
    }
  ]
```

Essayez de tester un outil qui n'existe pas, comme "add2", avec cette commande :

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add2 --tool-arg a=1 --tool-arg b=2
```

Vous devriez maintenant voir ce message indiquant que votre validation fonctionne :

```text
{
  "content": [],
  "error": {
    "code": "tool_not_found",
    "message": "Tool add2 not found."
  }
```

Essayez également d'envoyer un paramètre `c` qui devrait être rejeté par le schéma, comme ceci :

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg c=2
```

Vous devriez maintenant voir une erreur "arguments invalides" :

```text
{
  "content": [],
  "error": {
    "code": "invalid_arguments",
    "message": "Invalid arguments for tool add: [\n  {\n    \"code\": \"invalid_type\",\n    \"expected\": \"number\",\n    \"received\": \"undefined\",\n    \"path\": [\n      \"b\"\n    ],\n    \"message\": \"Required\"\n  }\n]"
  }
}
```
