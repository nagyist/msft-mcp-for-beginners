# Mintapéldák futtatása

## Függőségek telepítése

```sh
npm install
```

## Fordítás

```sh
npm run build
```

## Futtatás

```sh
npm start
```

A következő szöveget kell látnod:

```text
Registering tools...
Starting server...
```

Nagyszerű, ez azt jelenti, hogy helyesen indult el.

## A szerver tesztelése

Próbáld ki a funkciókat az alábbi parancs segítségével:

```sh
npx @modelcontextprotocol/inspector node build/app.js
```

Ez elindítja az ellenőrző eszköz webes felületét.

Egy eszköz futtatása:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

Egy ehhez hasonló válasznak kell megjelennie:

```text
{
  "content": [
    {
      "type": "text",
      "text": "Tool add called with arguments: {\"a\":1,\"b\":2}, result: {\"content\":[{\"type\":\"text\",\"text\":\"3\"}]}"
    }
  ]
```

Próbálj ki egy nem létező eszközt, például "add2"-t az alábbi parancs segítségével:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add2 --tool-arg a=1 --tool-arg b=2
```

Most ezt az üzenetet kell látnod, amely azt mutatja, hogy a validáció működik:

```text
{
  "content": [],
  "error": {
    "code": "tool_not_found",
    "message": "Tool add2 not found."
  }
```

Próbálj meg egy `c` paramétert küldeni, amelyet az adatstruktúra elutasít, például így:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg c=2
```

Most "érvénytelen argumentumok" hibát kell látnod:

```text
{
  "content": [],
  "error": {
    "code": "invalid_arguments",
    "message": "Invalid arguments for tool add: [\n  {\n    \"code\": \"invalid_type\",\n    \"expected\": \"number\",\n    \"received\": \"undefined\",\n    \"path\": [\n      \"b\"\n    ],\n    \"message\": \"Required\"\n  }\n]"
  }
}
```
