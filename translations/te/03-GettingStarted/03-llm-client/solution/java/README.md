# క్యాల్క్యులేటర్ LLM క్లయింట్

GitHub Models ఇంటిగ్రేషన్‌తో MCP (మోడల్ కాంటెక్స్ట్ ప్రోటోకాల్) క్యాల్క్యులేటర్ సర్వీస్‌కు కనెక్ట్ కావడానికి LangChain4j ను ఎలా ఉపయోగించాలో చూపించే జావా అప్లికేషన్.

## ముందస్తు అవసరాలు

- జావా 21 లేదా అంతకంటే పైగా
- మావెన్ 3.6+ (లేదా చేర్చిన మావెన్ రాపర్ ఉపయోగించండి)
- GitHub Models కి యాక్సెస్ ఉన్న GitHub ఖాతా
- `http://localhost:8080` వద్ద నడుస్తున్న MCP క్యాల్క్యులేటర్ సర్వీస్

## GitHub టోకెన్ పొందడం

ఈ అప్లికేషన్ GitHub Models ఉపయోగిస్తుంది, దీనికి GitHub వ్యక్తిగత యాక్సెస్ టోకెన్ అవసరం. మీ టోకెన్ పొందడానికి ఈ దశలను అనుసరించండి:

### 1. GitHub Models యాక్సెస్ చేయండి
1. [GitHub Models](https://github.com/marketplace/models) కి వెళ్లండి
2. మీ GitHub ఖాతాతో సైన్ ఇన్ అవ్వండి
3. మీరు ఇప్పటికీ యాక్సెస్ పొందకపోతే GitHub Models కి యాక్సెస్ అభ్యర్థించండి

### 2. వ్యక్తిగత యాక్సెస్ టోకెన్ సృష్టించండి
1. [GitHub సెట్టింగ్స్ → డెవలపర్ సెట్టింగ్స్ → వ్యక్తిగత యాక్సెస్ టోకెన్లు → టోకెన్లు (క్లాసిక్)](https://github.com/settings/tokens) కి వెళ్లండి
2. "Generate new token" → "Generate new token (classic)" క్లిక్ చేయండి
3. మీ టోకెన్ కు వివరణాత్మక పేరు ఇవ్వండి (ఉదా: "MCP Calculator Client")
4. అవసరమైతే గడువు సెట్ చేయండి
5. క్రింది స్కోప్స్ ఎంచుకోండి:
   - `repo` (ప్రైవేట్ రిపాజిటరీలకు యాక్సెస్ ఉంటే)
   - `user:email`
6. "Generate token" క్లిక్ చేయండి
7. **ముఖ్యమైనది**: టోకెన్ ను వెంటనే కాపీ చేసుకోండి - మీరు మళ్లీ చూడలేరు!

### 3. ఎన్విరాన్‌మెంట్ వేరియబుల్ సెట్ చేయండి

#### విండోస్ (కమాండ్ ప్రాంప్ట్) లో:
```cmd
set GITHUB_TOKEN=your_github_token_here
```

#### విండోస్ (పవర్‌షెల్) లో:
```powershell
$env:GITHUB_TOKEN="your_github_token_here"
```

#### macOS/Linux లో:
```bash
export GITHUB_TOKEN=your_github_token_here
```

## సెటప్ మరియు ఇన్‌స్టాలేషన్

1. **ప్రాజెక్ట్ డైరెక్టరీని క్లోన్ చేయండి లేదా దానిలోకి వెళ్లండి**

2. **డిపెండెన్సీలను ఇన్‌స్టాల్ చేయండి**:
   ```cmd
   mvnw clean install
   ```
   లేదా మీరు గ్లోబల్‌గా మావెన్ ఇన్‌స్టాల్ చేసుకున్నట్లయితే:
   ```cmd
   mvn clean install
   ```

3. **ఎన్విరాన్‌మెంట్ వేరియబుల్ సెట్ చేయండి** (పై "GitHub టోకెన్ పొందడం" సెక్షన్ చూడండి)

4. **MCP క్యాల్క్యులేటర్ సర్వీస్ ప్రారంభించండి**:
   `http://localhost:8080/sse` వద్ద చాప్టర్ 1 లోని MCP క్యాల్క్యులేటర్ సర్వీస్ నడుస్తున్నదని నిర్ధారించుకోండి. క్లయింట్ ప్రారంభించే ముందు ఇది నడవాలి.

## అప్లికేషన్ నడిపించడం

```cmd
mvnw clean package
java -jar target\calculator-llm-client-0.0.1-SNAPSHOT.jar
```

## అప్లికేషన్ ఏమి చేస్తుంది

అప్లికేషన్ క్యాల్క్యులేటర్ సర్వీస్‌తో మూడు ప్రధాన ఇంటరాక్షన్లను చూపిస్తుంది:

1. **జోడింపు**: 24.5 మరియు 17.3 యొక్క మొత్తం లెక్కిస్తుంది
2. **వర్గమూలం**: 144 యొక్క వర్గమూలం లెక్కిస్తుంది
3. **సహాయం**: అందుబాటులో ఉన్న క్యాల్క్యులేటర్ ఫంక్షన్లను చూపిస్తుంది

## ఆశించిన అవుట్‌పుట్

సక్సెస్‌గా నడిచినప్పుడు, మీరు ఈ విధమైన అవుట్‌పుట్ చూడగలరు:

```
The sum of 24.5 and 17.3 is 41.8.
The square root of 144 is 12.
The calculator service provides the following functions: add, subtract, multiply, divide, sqrt, power...
```

## సమస్య పరిష్కారం

### సాధారణ సమస్యలు

1. **"GITHUB_TOKEN ఎన్విరాన్‌మెంట్ వేరియబుల్ సెట్ చేయబడలేదు"**
   - మీరు `GITHUB_TOKEN` ఎన్విరాన్‌మెంట్ వేరియబుల్ సెట్ చేశారా అని నిర్ధారించుకోండి
   - వేరియబుల్ సెట్ చేసిన తర్వాత టెర్మినల్/కమాండ్ ప్రాంప్ట్ రీస్టార్ట్ చేయండి

2. **"localhost:8080 కు కనెక్షన్ తిరస్కరించబడింది"**
   - MCP క్యాల్క్యులేటర్ సర్వీస్ 8080 పోర్ట్‌లో నడుస్తున్నదని నిర్ధారించుకోండి
   - మరొక సర్వీస్ 8080 పోర్ట్ ఉపయోగిస్తున్నదా అని తనిఖీ చేయండి

3. **"ఆథెంటికేషన్ విఫలమైంది"**
   - మీ GitHub టోకెన్ చెల్లుబాటు అయ్యేలా మరియు సరైన అనుమతులు ఉన్నాయా అని తనిఖీ చేయండి
   - GitHub Models కి మీకు యాక్సెస్ ఉందా అని చూడండి

4. **మావెన్ బిల్డ్ లో లోపాలు**
   - మీరు జావా 21 లేదా అంతకంటే పైగా వర్షన్ ఉపయోగిస్తున్నారా: `java -version`
   - బిల్డ్ క్లీనింగ్ ప్రయత్నించండి: `mvnw clean`

### డీబగ్గింగ్

డీబగ్ లాగింగ్ ఎనేబుల్ చేయడానికి, నడిపేటప్పుడు ఈ JVM ఆర్గ్యుమెంట్ జోడించండి:
```cmd
java -Dlogging.level.dev.langchain4j=DEBUG -jar target\calculator-llm-client-0.0.1-SNAPSHOT.jar
```

## కాన్ఫిగరేషన్

అప్లికేషన్ ఈ విధంగా కాన్ఫిగర్ చేయబడింది:
- GitHub Models ను `gpt-4.1-nano` మోడల్‌తో ఉపయోగించడం
- MCP సర్వీస్‌కు `http://localhost:8080/sse` వద్ద కనెక్ట్ కావడం
- అభ్యర్థనలకు 60 సెకన్ల టైమ్‌అవుట్ ఉపయోగించడం
- డీబగ్గింగ్ కోసం అభ్యర్థన/ప్రతిస్పందన లాగింగ్ ఎనేబుల్ చేయడం
