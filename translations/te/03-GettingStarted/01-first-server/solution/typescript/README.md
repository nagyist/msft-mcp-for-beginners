# ఈ నమూనాను నడపడం

మీకు `uv` ను ఇన్‌స్టాల్ చేయాలని సిఫార్సు చేయబడింది కానీ ఇది తప్పనిసరి కాదు, [సూచనలు](https://docs.astral.sh/uv/#highlights) చూడండి

## -1- డిపెండెన్సీలను ఇన్‌స్టాల్ చేయండి

```bash
npm install
```

## -3- నమూనాను నడపండి


```bash
npm run build
```

## -4- నమూనాను పరీక్షించండి

ఒక టెర్మినల్‌లో సర్వర్ నడుస్తున్నప్పుడు, మరో టెర్మినల్ తెరవండి మరియు క్రింది కమాండ్‌ను నడపండి:

```bash
npm run inspector
```

ఇది ఒక విజువల్ ఇంటర్‌ఫేస్‌తో కూడిన వెబ్ సర్వర్‌ను ప్రారంభించి, మీరు నమూనాను పరీక్షించడానికి అనుమతిస్తుంది.

సర్వర్ కనెక్ట్ అయిన తర్వాత:

- టూల్స్‌ను జాబితా చేయడానికి ప్రయత్నించండి మరియు `add` ను ఆర్గ్స్ 2 మరియు 4 తో నడపండి, ఫలితంలో 6 కనిపించాలి.
- రిసోర్సులు మరియు రిసోర్స్ టెంప్లేట్‌కు వెళ్లి "greeting" ను పిలవండి, ఒక పేరు టైప్ చేయండి మరియు మీరు అందించిన పేరుతో ఒక అభివాదం కనిపిస్తుంది.

### CLI మోడ్‌లో పరీక్షించడం

మీరు నడిపిన ఇన్స్పెక్టర్ వాస్తవానికి Node.js యాప్ మరియు `mcp dev` దాని చుట్టూ ఉన్న రాపర్.

మీరు దీన్ని నేరుగా CLI మోడ్‌లో క్రింది కమాండ్ నడపడం ద్వారా ప్రారంభించవచ్చు:

```bash
npx @modelcontextprotocol/inspector --cli node ./build/index.js --method tools/list
```

ఇది సర్వర్‌లో అందుబాటులో ఉన్న అన్ని టూల్స్‌ను జాబితా చేస్తుంది. మీరు క్రింది అవుట్పుట్ చూడాలి:

```text
{
  "tools": [
    {
      "name": "add",
      "description": "Add two numbers",
      "inputSchema": {
        "type": "object",
        "properties": {
          "a": {
            "title": "A",
            "type": "integer"
          },
          "b": {
            "title": "B",
            "type": "integer"
          }
        },
        "required": [
          "a",
          "b"
        ],
        "title": "addArguments"
      }
    }
  ]
}
```

ఒక టూల్‌ను పిలవడానికి టైప్ చేయండి:

```bash
nnpx @modelcontextprotocol/inspector --cli node ./build/index.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

మీరు క్రింది అవుట్పుట్ చూడాలి:

```text
{
  "content": [
    {
      "type": "text",
      "text": "3"
    }
  ],
  "isError": false
}
```

> [!TIP]
> ఇన్స్పెక్టర్‌ను బ్రౌజర్‌లో కంటే CLI మోడ్‌లో నడపడం సాధారణంగా చాలా వేగంగా ఉంటుంది.
> ఇన్స్పెక్టర్ గురించి మరింత చదవండి [ఇక్కడ](https://github.com/modelcontextprotocol/inspector).
