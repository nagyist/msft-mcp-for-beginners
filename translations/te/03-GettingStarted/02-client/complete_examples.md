# పూర్తి MCP క్లయింట్ ఉదాహరణలు

ఈ డైరెక్టరీలో వివిధ ప్రోగ్రామింగ్ భాషలలో పూర్తి, పనిచేసే MCP క్లయింట్ల ఉదాహరణలు ఉన్నాయి. ప్రతి క్లయింట్ ప్రధాన README.md ట్యుటోరియల్‌లో వివరించిన పూర్తి ఫంక్షనాలిటీని ప్రదర్శిస్తుంది.

## అందుబాటులో ఉన్న క్లయింట్లు

### 1. జావా క్లయింట్ (`client_example_java.java`)

- **ట్రాన్స్‌పోర్ట్**: SSE (సర్వర్-సెంట్ ఈవెంట్స్) HTTP పై
- **లక్ష్య సర్వర్**: `http://localhost:8080`
- **ఫీచర్లు**:
  - కనెక్షన్ స్థాపన మరియు పింగ్
  - టూల్ జాబితా
  - క్యాల్క్యులేటర్ ఆపరేషన్లు (జోడించు, తీసివేయి, గుణించు, భాగించు, సహాయం)
  - లోపాల నిర్వహణ మరియు ఫలితాల తీసుకోవడం

**రన్ చేయడానికి:**

```bash
# మీ MCP సర్వర్ localhost:8080 పై నడుస్తున్నదని నిర్ధారించుకోండి
javac client_example_java.java
java client_example_java
```

### 2. C# క్లయింట్ (`client_example_csharp.cs`)

- **ట్రాన్స్‌పోర్ట్**: Stdio (స్టాండర్డ్ ఇన్‌పుట్/అవుట్‌పుట్)
- **లక్ష్య సర్వర్**: dotnet run ద్వారా లోకల్ .NET MCP సర్వర్
- **ఫీచర్లు**:
  - stdio ట్రాన్స్‌పోర్ట్ ద్వారా ఆటోమేటిక్ సర్వర్ స్టార్టప్
  - టూల్ మరియు రిసోర్స్ జాబితా
  - క్యాల్క్యులేటర్ ఆపరేషన్లు
  - JSON ఫలితాల పార్సింగ్
  - సమగ్ర లోపాల నిర్వహణ

**రన్ చేయడానికి:**

```bash
dotnet run
```

### 3. టైప్‌స్క్రిప్ట్ క్లయింట్ (`client_example_typescript.ts`)

- **ట్రాన్స్‌పోర్ట్**: Stdio (స్టాండర్డ్ ఇన్‌పుట్/అవుట్‌పుట్)
- **లక్ష్య సర్వర్**: లోకల్ Node.js MCP సర్వర్
- **ఫీచర్లు**:
  - పూర్తి MCP ప్రోటోకాల్ మద్దతు
  - టూల్, రిసోర్స్, మరియు ప్రాంప్ట్ ఆపరేషన్లు
  - క్యాల్క్యులేటర్ ఆపరేషన్లు
  - రిసోర్స్ చదవడం మరియు ప్రాంప్ట్ అమలు
  - బలమైన లోపాల నిర్వహణ

**రన్ చేయడానికి:**

```bash
# మొదట టైప్‌స్క్రిప్ట్‌ను కంపైల్ చేయండి (అవసరమైతే)
npm run build

# ఆపై క్లయింట్‌ను నడపండి
npm run client
# లేదా
node client_example_typescript.js
```

### 4. పైథాన్ క్లయింట్ (`client_example_python.py`)

- **ట్రాన్స్‌పోర్ట్**: Stdio (స్టాండర్డ్ ఇన్‌పుట్/అవుట్‌పుట్)  
- **లక్ష్య సర్వర్**: లోకల్ Python MCP సర్వర్
- **ఫీచర్లు**:
  - ఆపరేషన్ల కోసం Async/await ప్యాటర్న్
  - టూల్ మరియు రిసోర్స్ కనుగొనడం
  - క్యాల్క్యులేటర్ ఆపరేషన్ల పరీక్ష
  - రిసోర్స్ కంటెంట్ చదవడం
  - క్లాస్-ఆధారిత సంస్థాపన

**రన్ చేయడానికి:**

```bash
python client_example_python.py
```

## అన్ని క్లయింట్లలో సాధారణ ఫీచర్లు

ప్రతి క్లయింట్ అమలు ఈ క్రింది వాటిని ప్రదర్శిస్తుంది:

1. **కనెక్షన్ నిర్వహణ**
   - MCP సర్వర్‌కు కనెక్షన్ స్థాపన
   - కనెక్షన్ లోపాలను నిర్వహించడం
   - సరైన క్లీనప్ మరియు వనరుల నిర్వహణ

2. **సర్వర్ కనుగొనడం**
   - అందుబాటులో ఉన్న టూల్స్ జాబితా
   - అందుబాటులో ఉన్న వనరులు జాబితా (మద్దతు ఉన్న చోట)
   - అందుబాటులో ఉన్న ప్రాంప్ట్‌లు జాబితా (మద్దతు ఉన్న చోట)

3. **టూల్ పిలుపు**
   - ప్రాథమిక క్యాల్క్యులేటర్ ఆపరేషన్లు (జోడించు, తీసివేయి, గుణించు, భాగించు)
   - సర్వర్ సమాచారం కోసం సహాయం కమాండ్
   - సరైన ఆర్గ్యుమెంట్ల పంపిణీ మరియు ఫలితాల నిర్వహణ

4. **లోపాల నిర్వహణ**
   - కనెక్షన్ లోపాలు
   - టూల్ అమలు లోపాలు
   - సున్నితమైన విఫలం మరియు వినియోగదారుని ఫీడ్‌బ్యాక్

5. **ఫలితాల ప్రాసెసింగ్**
   - ప్రతిస్పందనల నుండి టెక్స్ట్ కంటెంట్ తీసుకోవడం
   - చదవడానికి అనుకూలంగా అవుట్‌పుట్ ఫార్మాటింగ్
   - వివిధ ప్రతిస్పందన ఫార్మాట్లను నిర్వహించడం

## ముందస్తు అవసరాలు

ఈ క్లయింట్లను రన్ చేయడానికి ముందు, మీరు కలిగి ఉండాలి:

1. **సంబంధిత MCP సర్వర్ రన్ అవుతున్నది** (`../01-first-server/` నుండి)
2. **మీ ఎంచుకున్న భాషకు అవసరమైన డిపెండెన్సీలు ఇన్‌స్టాల్ చేయబడినవి**
3. **సరైన నెట్‌వర్క్ కనెక్టివిటీ** (HTTP ఆధారిత ట్రాన్స్‌పోర్ట్స్ కోసం)

## అమలుల మధ్య ముఖ్య తేడాలు

| భాష       | ట్రాన్స్‌పోర్ట్ | సర్వర్ స్టార్టప్ | అసింక్ మోడల్ | ముఖ్య లైబ్రరీలు       |
|------------|---------------|-----------------|---------------|-----------------------|
| జావా       | SSE/HTTP      | బాహ్య            | సింక్          | WebFlux, MCP SDK      |
| C#         | Stdio         | ఆటోమేటిక్       | అసింక్/అవైట్ | .NET MCP SDK          |
| టైప్‌స్క్రిప్ట్ | Stdio     | ఆటోమేటిక్       | అసింక్/అవైట్ | Node MCP SDK          |
| పైథాన్     | Stdio         | ఆటోమేటిక్       | AsyncIO       | Python MCP SDK        |
| రస్ట్       | Stdio         | ఆటోమేటిక్       | అసింక్/అవైట్ | Rust MCP SDK, Tokio   |

## తదుపరి దశలు

ఈ క్లయింట్ ఉదాహరణలను పరిశీలించిన తర్వాత:

1. **క్లయింట్లను మార్చండి** కొత్త ఫీచర్లు లేదా ఆపరేషన్లు జోడించడానికి
2. **మీ సొంత సర్వర్ సృష్టించండి** మరియు ఈ క్లయింట్లతో పరీక్షించండి
3. **వివిధ ట్రాన్స్‌పోర్ట్స్‌తో ప్రయోగాలు చేయండి** (SSE vs. Stdio)
4. **MCP ఫంక్షనాలిటీని సమ్మిళితం చేసే మరింత సంక్లిష్ట అప్లికేషన్ నిర్మించండి**

## సమస్య పరిష్కారం

### సాధారణ సమస్యలు

1. **కనెక్షన్ తిరస్కరించబడింది**: MCP సర్వర్ ఆశించిన పోర్ట్/పాత్‌లో రన్ అవుతుందో లేదో నిర్ధారించండి
2. **మాడ్యూల్ కనుగొనబడలేదు**: మీ భాషకు అవసరమైన MCP SDK ఇన్‌స్టాల్ చేయండి
3. **అనుమతి నిరాకరించబడింది**: stdio ట్రాన్స్‌పోర్ట్ కోసం ఫైల్ అనుమతులను తనిఖీ చేయండి
4. **టూల్ కనుగొనబడలేదు**: సర్వర్ ఆశించిన టూల్స్ అమలు చేస్తున్నదో నిర్ధారించండి

### డీబగ్ సూచనలు

1. **మీ MCP SDKలో విస్తృత లాగింగ్ ఎనేబుల్ చేయండి**
2. **సర్వర్ లాగ్‌లలో లోప సందేశాలను తనిఖీ చేయండి**
3. **క్లయింట్ మరియు సర్వర్ మధ్య టూల్ పేర్లు మరియు సంతకాలు సరిపోతున్నాయో నిర్ధారించండి**
4. **మొదట MCP ఇన్స్పెక్టర్‌తో పరీక్షించండి** సర్వర్ ఫంక్షనాలిటీని ధృవీకరించడానికి
