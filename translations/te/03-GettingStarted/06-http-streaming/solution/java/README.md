# క్యాల్క్యులేటర్ HTTP స్ట్రీమింగ్ డెమో

ఈ ప్రాజెక్ట్ Server-Sent Events (SSE) ఉపయోగించి HTTP స్ట్రీమింగ్‌ను Spring Boot WebFlux తో ప్రదర్శిస్తుంది. ఇది రెండు అప్లికేషన్లతో కూడి ఉంటుంది:

- **క్యాల్క్యులేటర్ సర్వర్**: గణనలను నిర్వహించే మరియు SSE ఉపయోగించి ఫలితాలను స్ట్రీమ్ చేసే రియాక్టివ్ వెబ్ సర్వీస్
- **క్యాల్క్యులేటర్ క్లయింట్**: స్ట్రీమింగ్ ఎండ్‌పాయింట్‌ను వినియోగించే క్లయింట్ అప్లికేషన్

## ముందస్తు అవసరాలు

- జావా 17 లేదా అంతకంటే పైగా
- మావెన్ 3.6 లేదా అంతకంటే పైగా

## ప్రాజెక్ట్ నిర్మాణం

```
java/
├── calculator-server/     # Spring Boot server with SSE endpoint
│   ├── src/main/java/com/example/calculatorserver/
│   │   ├── CalculatorServerApplication.java
│   │   └── CalculatorController.java
│   └── pom.xml
├── calculator-client/     # Spring Boot client application
│   ├── src/main/java/com/example/calculatorclient/
│   │   └── CalculatorClientApplication.java
│   └── pom.xml
└── README.md
```

## ఇది ఎలా పనిచేస్తుంది

1. **క్యాల్క్యులేటర్ సర్వర్** ఒక `/calculate` ఎండ్‌పాయింట్‌ను అందిస్తుంది, ఇది:
   - క్వెరీ పరామితులు స్వీకరిస్తుంది: `a` (సంఖ్య), `b` (సంఖ్య), `op` (ఆపరేషన్)
   - మద్దతు ఉన్న ఆపరేషన్లు: `add`, `sub`, `mul`, `div`
   - గణన ప్రగతి మరియు ఫలితంతో Server-Sent Events ను తిరిగి ఇస్తుంది

2. **క్యాల్క్యులేటర్ క్లయింట్** సర్వర్‌కు కనెక్ట్ అవుతుంది మరియు:
   - `7 * 5` గణన చేయడానికి అభ్యర్థన చేస్తుంది
   - స్ట్రీమింగ్ ప్రతిస్పందనను వినియోగిస్తుంది
   - ప్రతి ఈవెంట్‌ను కన్సోల్‌లో ప్రింట్ చేస్తుంది

## అప్లికేషన్లను నడపడం

### ఎంపిక 1: మావెన్ ఉపయోగించడం (సిఫార్సు చేయబడింది)

#### 1. క్యాల్క్యులేటర్ సర్వర్‌ను ప్రారంభించండి

టెర్మినల్ ఓపెన్ చేసి సర్వర్ డైరెక్టరీకి వెళ్లండి:

```bash
cd calculator-server
mvn clean package
mvn spring-boot:run
```

సర్వర్ `http://localhost:8080` వద్ద ప్రారంభమవుతుంది

మీకు ఇలాంటి అవుట్పుట్ కనిపించాలి:
```
Started CalculatorServerApplication in X.XXX seconds
Netty started on port 8080 (http)
```

#### 2. క్యాల్క్యులేటర్ క్లయింట్‌ను నడపండి

కొత్త టెర్మినల్ ఓపెన్ చేసి క్లయింట్ డైరెక్టరీకి వెళ్లండి:

```bash
cd calculator-client
mvn clean package
mvn spring-boot:run
```

క్లయింట్ సర్వర్‌కు కనెక్ట్ అవుతుంది, గణన చేస్తుంది, మరియు స్ట్రీమింగ్ ఫలితాలను ప్రదర్శిస్తుంది.

### ఎంపిక 2: జావా నేరుగా ఉపయోగించడం

#### 1. సర్వర్‌ను కంపైల్ చేసి నడపండి:

```bash
cd calculator-server
mvn clean package
java -jar target/calculator-server-0.0.1-SNAPSHOT.jar
```

#### 2. క్లయింట్‌ను కంపైల్ చేసి నడపండి:

```bash
cd calculator-client
mvn clean package
java -jar target/calculator-client-0.0.1-SNAPSHOT.jar
```

## సర్వర్‌ను మాన్యువల్‌గా పరీక్షించడం

మీరు వెబ్ బ్రౌజర్ లేదా కర్ల్ ఉపయోగించి సర్వర్‌ను పరీక్షించవచ్చు:

### వెబ్ బ్రౌజర్ ఉపయోగించడం:
విజిట్ చేయండి: `http://localhost:8080/calculate?a=10&b=5&op=add`

### కర్ల్ ఉపయోగించడం:
```bash
curl "http://localhost:8080/calculate?a=10&b=5&op=add" -H "Accept: text/event-stream"
```

## ఆశించిన అవుట్పుట్

క్లయింట్ నడిపినప్పుడు, మీరు ఈ విధమైన స్ట్రీమింగ్ అవుట్పుట్ చూడగలరు:

```
event:info
data:Calculating: 7.0 mul 5.0

event:result
data:35.0
```

## మద్దతు ఉన్న ఆపరేషన్లు

- `add` - జోడింపు (a + b)
- `sub` - తీసివేత (a - b)
- `mul` - గుణకం (a * b)
- `div` - భాగాకారం (a / b, b = 0 అయితే NaN తిరిగి ఇస్తుంది)

## API సూచిక

### GET /calculate

**పరామితులు:**
- `a` (అవసరం): మొదటి సంఖ్య (డబుల్)
- `b` (అవసరం): రెండవ సంఖ్య (డబుల్)
- `op` (అవసరం): ఆపరేషన్ (`add`, `sub`, `mul`, `div`)

**ప్రతిస్పందన:**
- కంటెంట్-టైప్: `text/event-stream`
- గణన ప్రగతి మరియు ఫలితంతో Server-Sent Events ను తిరిగి ఇస్తుంది

**ఉదాహరణ అభ్యర్థన:**
```
GET /calculate?a=7&b=5&op=mul HTTP/1.1
Host: localhost:8080
Accept: text/event-stream
```

**ఉదాహరణ ప్రతిస్పందన:**
```
event: info
data: Calculating: 7.0 mul 5.0

event: result
data: 35.0
```

## సమస్య పరిష్కారం

### సాధారణ సమస్యలు

1. **పోర్ట్ 8080 ఇప్పటికే ఉపయోగంలో ఉంది**
   - పోర్ట్ 8080 ఉపయోగిస్తున్న ఇతర అప్లికేషన్లను ఆపండి
   - లేదా `calculator-server/src/main/resources/application.yml` లో సర్వర్ పోర్ట్ మార్చండి

2. **కనెక్షన్ తిరస్కరించబడింది**
   - క్లయింట్ ప్రారంభించే ముందు సర్వర్ నడుస్తున్నదని నిర్ధారించుకోండి
   - సర్వర్ 8080 పోర్ట్‌లో విజయవంతంగా ప్రారంభమైందని తనిఖీ చేయండి

3. **పరామితి పేర్ల సమస్యలు**
   - ఈ ప్రాజెక్ట్ Maven కంపైలర్ కాన్ఫిగరేషన్‌లో `-parameters` ఫ్లాగ్‌ను కలిగి ఉంది
   - మీరు పరామితి బైండింగ్ సమస్యలను ఎదుర్కొంటే, ప్రాజెక్ట్ ఈ కాన్ఫిగరేషన్‌తో నిర్మించబడిందని నిర్ధారించుకోండి

### అప్లికేషన్లను ఆపడం

- ప్రతి అప్లికేషన్ నడుస్తున్న టెర్మినల్‌లో `Ctrl+C` నొక్కండి
- లేదా బ్యాక్‌గ్రౌండ్ ప్రాసెస్‌గా నడుస్తున్నట్లయితే `mvn spring-boot:stop` ఉపయోగించండి

## సాంకేతిక స్టాక్

- **Spring Boot 3.3.1** - అప్లికేషన్ ఫ్రేమ్‌వర్క్
- **Spring WebFlux** - రియాక్టివ్ వెబ్ ఫ్రేమ్‌వర్క్
- **Project Reactor** - రియాక్టివ్ స్ట్రీమ్స్ లైబ్రరీ
- **Netty** - నాన్-బ్లాకింగ్ I/O సర్వర్
- **Maven** - బిల్డ్ టూల్
- **Java 17+** - ప్రోగ్రామింగ్ భాష
