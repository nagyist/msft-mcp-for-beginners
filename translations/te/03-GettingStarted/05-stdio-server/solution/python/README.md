# MCP stdio సర్వర్ - Python పరిష్కారం

> **⚠️ ముఖ్యమైనది**: ఈ పరిష్కారం MCP స్పెసిఫికేషన్ 2025-06-18 ప్రకారం సిఫార్సు చేసిన **stdio ట్రాన్స్‌పోర్ట్** ఉపయోగించడానికి నవీకరించబడింది. అసలు SSE ట్రాన్స్‌పోర్ట్ డిప్రికేటెడ్ అయింది.

## అవలోకనం

ఈ Python పరిష్కారం ప్రస్తుత stdio ట్రాన్స్‌పోర్ట్ ఉపయోగించి MCP సర్వర్‌ను ఎలా నిర్మించాలో చూపిస్తుంది. stdio ట్రాన్స్‌పోర్ట్ సులభం, మరింత సురక్షితంగా ఉంటుంది మరియు డిప్రికేటెడ్ SSE పద్ధతికి కంటే మెరుగైన పనితీరును అందిస్తుంది.

## ముందస్తు అవసరాలు

- Python 3.8 లేదా తరువాతి సంస్కరణ
- ప్యాకేజ్ నిర్వహణ కోసం `uv` ఇన్‌స్టాల్ చేయాలని సిఫార్సు చేయబడింది, [సూచనలు](https://docs.astral.sh/uv/#highlights) చూడండి

## సెటప్ సూచనలు

### దశ 1: వర్చువల్ ఎన్విరాన్‌మెంట్ సృష్టించండి

```bash
python -m venv venv
```

### దశ 2: వర్చువల్ ఎన్విరాన్‌మెంట్‌ను యాక్టివేట్ చేయండి

**Windows:**
```bash
venv\Scripts\activate
```

**macOS/Linux:**
```bash
source venv/bin/activate
```

### దశ 3: డిపెండెన్సీలను ఇన్‌స్టాల్ చేయండి

```bash
pip install mcp
```

## సర్వర్ నడపడం

stdio సర్వర్ పాత SSE సర్వర్ కంటే వేరుగా నడుస్తుంది. వెబ్ సర్వర్ ప్రారంభించకుండా, ఇది stdin/stdout ద్వారా కమ్యూనికేట్ చేస్తుంది:

```bash
python server.py
```

**ముఖ్యమైనది**: సర్వర్ హ్యాంగ్ అయినట్లు కనిపిస్తుంది - ఇది సాధారణం! ఇది stdin నుండి JSON-RPC సందేశాలను ఎదురుచూస్తోంది.

## సర్వర్‌ను పరీక్షించడం

### పద్ధతి 1: MCP ఇన్స్పెక్టర్ ఉపయోగించడం (సిఫార్సు చేయబడింది)

```bash
npx @modelcontextprotocol/inspector python server.py
```

ఇది:
1. మీ సర్వర్‌ను సబ్‌ప్రాసెస్‌గా ప్రారంభిస్తుంది
2. పరీక్ష కోసం వెబ్ ఇంటర్‌ఫేస్‌ను తెరుస్తుంది
3. అన్ని సర్వర్ టూల్స్‌ను ఇంటరాక్టివ్‌గా పరీక్షించడానికి అనుమతిస్తుంది

### పద్ధతి 2: నేరుగా JSON-RPC పరీక్ష

మీరు JSON-RPC సందేశాలను నేరుగా పంపించి కూడా పరీక్షించవచ్చు:

1. సర్వర్‌ను ప్రారంభించండి: `python server.py`
2. JSON-RPC సందేశం పంపండి (ఉదాహరణ):

```json
{"jsonrpc": "2.0", "id": 1, "method": "tools/list"}
```

3. సర్వర్ అందుబాటులో ఉన్న టూల్స్‌తో స్పందిస్తుంది

### అందుబాటులో ఉన్న టూల్స్

సర్వర్ ఈ టూల్స్‌ను అందిస్తుంది:

- **add(a, b)**: రెండు సంఖ్యలను కలిపి జోడించండి
- **multiply(a, b)**: రెండు సంఖ్యలను గుణించండి  
- **get_greeting(name)**: వ్యక్తిగతీకరించిన అభివాదం సృష్టించండి
- **get_server_info()**: సర్వర్ గురించి సమాచారం పొందండి

### Claude Desktop తో పరీక్షించడం

ఈ సర్వర్‌ను Claude Desktop తో ఉపయోగించడానికి, మీ `claude_desktop_config.json` లో ఈ కాన్ఫిగరేషన్‌ను జోడించండి:

```json
{
  "mcpServers": {
    "example-stdio-server": {
      "command": "python",
      "args": ["path/to/server.py"]
    }
  }
}
```

## SSE నుండి ముఖ్యమైన తేడాలు

**stdio ట్రాన్స్‌పోర్ట్ (ప్రస్తుత):**
- ✅ సులభమైన సెటప్ - వెబ్ సర్వర్ అవసరం లేదు
- ✅ మెరుగైన సెక్యూరిటీ - HTTP ఎండ్పాయింట్లు లేవు
- ✅ సబ్‌ప్రాసెస్ ఆధారిత కమ్యూనికేషన్
- ✅ stdin/stdout పై JSON-RPC
- ✅ మెరుగైన పనితీరు

**SSE ట్రాన్స్‌పోర్ట్ (డిప్రికేటెడ్):**
- ❌ HTTP సర్వర్ సెటప్ అవసరం
- ❌ వెబ్ ఫ్రేమ్‌వర్క్ అవసరం (Starlette/FastAPI)
- ❌ క్లిష్టమైన రౌటింగ్ మరియు సెషన్ నిర్వహణ
- ❌ అదనపు సెక్యూరిటీ పరిగణనలు
- ❌ ఇప్పుడు MCP 2025-06-18 లో డిప్రికేటెడ్

## డీబగ్గింగ్ సూచనలు

- లాగింగ్ కోసం `stderr` ఉపయోగించండి (`stdout` కాదు)
- విజువల్ డీబగ్గింగ్ కోసం ఇన్స్పెక్టర్‌తో పరీక్షించండి
- అన్ని JSON సందేశాలు newline-డెలిమిటెడ్ ఉన్నాయో చూసుకోండి
- సర్వర్ ఎర్రర్ల లేకుండా ప్రారంభమవుతుందో నిర్ధారించుకోండి

ఈ పరిష్కారం ప్రస్తుత MCP స్పెసిఫికేషన్‌ను అనుసరిస్తుంది మరియు stdio ట్రాన్స్‌పోర్ట్ అమలుకు ఉత్తమ పద్ధతులను చూపిస్తుంది.
