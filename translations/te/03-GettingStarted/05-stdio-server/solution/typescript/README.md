# MCP stdio సర్వర్ - TypeScript పరిష్కారం

> **⚠️ ముఖ్యమైనది**: ఈ పరిష్కారం MCP స్పెసిఫికేషన్ 2025-06-18 ప్రకారం సిఫార్సు చేసిన **stdio ట్రాన్స్‌పోర్ట్** ఉపయోగించడానికి నవీకరించబడింది. అసలు SSE ట్రాన్స్‌పోర్ట్ డిప్రికేటెడ్ అయింది.

## అవలోకనం

ఈ TypeScript పరిష్కారం ప్రస్తుత stdio ట్రాన్స్‌పోర్ట్ ఉపయోగించి MCP సర్వర్‌ను ఎలా నిర్మించాలో చూపిస్తుంది. stdio ట్రాన్స్‌పోర్ట్ సులభం, మరింత సురక్షితమైనది మరియు డిప్రికేటెడ్ SSE పద్ధతికి కంటే మెరుగైన పనితీరును అందిస్తుంది.

## ముందస్తు అవసరాలు

- Node.js 18+ లేదా తరువాతి వెర్షన్
- npm లేదా yarn ప్యాకేజ్ మేనేజర్

## సెటప్ సూచనలు

### దశ 1: డిపెండెన్సీలను ఇన్‌స్టాల్ చేయండి

```bash
npm install
```

### దశ 2: ప్రాజెక్ట్‌ను బిల్డ్ చేయండి

```bash
npm run build
```

## సర్వర్ నడపడం

stdio సర్వర్ పాత SSE సర్వర్ కంటే వేరుగా నడుస్తుంది. వెబ్ సర్వర్ ప్రారంభించకుండా, ఇది stdin/stdout ద్వారా కమ్యూనికేట్ చేస్తుంది:

```bash
npm start
```

**ముఖ్యమైనది**: సర్వర్ హ్యాంగ్ అయినట్లు కనిపించవచ్చు - ఇది సాధారణం! ఇది stdin నుండి JSON-RPC సందేశాలను ఎదురుచూస్తోంది.

## సర్వర్‌ను పరీక్షించడం

### పద్ధతి 1: MCP ఇన్స్పెక్టర్ ఉపయోగించడం (సిఫార్సు చేయబడింది)

```bash
npm run inspector
```

ఇది:
1. మీ సర్వర్‌ను సబ్‌ప్రాసెస్‌గా ప్రారంభిస్తుంది
2. పరీక్ష కోసం వెబ్ ఇంటర్‌ఫేస్‌ను తెరుస్తుంది
3. అన్ని సర్వర్ టూల్స్‌ను ఇంటరాక్టివ్‌గా పరీక్షించడానికి అనుమతిస్తుంది

### పద్ధతి 2: డైరెక్ట్ కమాండ్ లైన్ పరీక్ష

ఇన్స్పెక్టర్‌ను నేరుగా ప్రారంభించి కూడా మీరు పరీక్షించవచ్చు:

```bash
npx @modelcontextprotocol/inspector node build/index.js
```

### అందుబాటులో ఉన్న టూల్స్

సర్వర్ ఈ టూల్స్‌ను అందిస్తుంది:

- **add(a, b)**: రెండు సంఖ్యలను కలపండి
- **multiply(a, b)**: రెండు సంఖ్యలను గుణించండి  
- **get_greeting(name)**: వ్యక్తిగతీకరించిన అభివాదం సృష్టించండి
- **get_server_info()**: సర్వర్ గురించి సమాచారం పొందండి

### Claude Desktop తో పరీక్షించడం

ఈ సర్వర్‌ను Claude Desktop తో ఉపయోగించడానికి, మీ `claude_desktop_config.json` లో ఈ కాన్ఫిగరేషన్‌ను జోడించండి:

```json
{
  "mcpServers": {
    "example-stdio-server": {
      "command": "node",
      "args": ["path/to/build/index.js"]
    }
  }
}
```

## SSE నుండి ముఖ్యమైన తేడాలు

**stdio ట్రాన్స్‌పోర్ట్ (ప్రస్తుత):**
- ✅ సులభమైన సెటప్ - HTTP సర్వర్ అవసరం లేదు
- ✅ మెరుగైన సెక్యూరిటీ - HTTP ఎండ్పాయింట్లు లేవు
- ✅ సబ్‌ప్రాసెస్ ఆధారిత కమ్యూనికేషన్
- ✅ stdin/stdout పై JSON-RPC
- ✅ మెరుగైన పనితీరు

**SSE ట్రాన్స్‌పోర్ట్ (డిప్రికేటెడ్):**
- ❌ Express సర్వర్ సెటప్ అవసరం
- ❌ క్లిష్టమైన రౌటింగ్ మరియు సెషన్ నిర్వహణ అవసరం
- ❌ ఎక్కువ డిపెండెన్సీలు (Express, HTTP హ్యాండ్లింగ్)
- ❌ అదనపు సెక్యూరిటీ పరిగణనలు
- ❌ ఇప్పుడు MCP 2025-06-18 లో డిప్రికేటెడ్

## అభివృద్ధి సూచనలు

- లాగింగ్ కోసం `console.error()` ఉపయోగించండి (`console.log()` కాదు ఎందుకంటే అది stdout కు రాస్తుంది)
- పరీక్షించడానికి ముందు `npm run build` తో బిల్డ్ చేయండి
- విజువల్ డీబగ్గింగ్ కోసం ఇన్స్పెక్టర్‌తో పరీక్షించండి
- అన్ని JSON సందేశాలు సరిగ్గా ఫార్మాట్ చేయబడ్డాయో నిర్ధారించండి
- సర్వర్ స్వయంచాలకంగా SIGINT/SIGTERM పై గ్రేస్‌ఫుల్ షట్‌డౌన్ నిర్వహిస్తుంది

ఈ పరిష్కారం ప్రస్తుత MCP స్పెసిఫికేషన్‌ను అనుసరిస్తుంది మరియు TypeScript ఉపయోగించి stdio ట్రాన్స్‌పోర్ట్ అమలుకు ఉత్తమ పద్ధతులను చూపిస్తుంది.
