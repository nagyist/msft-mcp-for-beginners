# MCP సర్వర్లను డిప్లాయ్ చేయడం

మీ MCP సర్వర్‌ను డిప్లాయ్ చేయడం ద్వారా ఇతరులు మీ స్థానిక వాతావరణం దాటి దాని టూల్స్ మరియు వనరులను యాక్సెస్ చేయగలుగుతారు. స్కేలబిలిటీ, నమ్మకదారితనం, మరియు నిర్వహణ సౌలభ్యం కోసం మీ అవసరాలపై ఆధారపడి అనేక డిప్లాయ్‌మెంట్ వ్యూహాలు ఉన్నాయి. క్రింద మీరు MCP సర్వర్లను స్థానికంగా, కంటైనర్లలో, మరియు క్లౌడ్‌లో డిప్లాయ్ చేయడానికి మార్గదర్శకాన్ని కనుగొంటారు.

## అవలోకనం

ఈ పాఠం మీ MCP సర్వర్ యాప్‌ను ఎలా డిప్లాయ్ చేయాలో కవర్ చేస్తుంది.

## నేర్చుకునే లక్ష్యాలు

ఈ పాఠం ముగిసిన తర్వాత, మీరు చేయగలుగుతారు:

- వివిధ డిప్లాయ్‌మెంట్ విధానాలను అంచనా వేయండి.
- మీ యాప్‌ను డిప్లాయ్ చేయండి.

## స్థానిక అభివృద్ధి మరియు డిప్లాయ్‌మెంట్

మీ సర్వర్ వినియోగదారుల యంత్రంలో నడుస్తూ వినియోగించబడాలని ఉద్దేశించబడితే, మీరు క్రింది దశలను అనుసరించవచ్చు:

1. **సర్వర్‌ను డౌన్లోడ్ చేయండి**. మీరు సర్వర్‌ను రాయకపోతే, ముందుగా దాన్ని మీ యంత్రానికి డౌన్లోడ్ చేసుకోండి.
1. **సర్వర్ ప్రాసెస్‌ను ప్రారంభించండి**: మీ MCP సర్వర్ అప్లికేషన్‌ను నడపండి

SSE కోసం (stdio రకం సర్వర్‌కు అవసరం లేదు)

1. **నెట్‌వర్కింగ్‌ను కాన్ఫిగర్ చేయండి**: సర్వర్ ఆశించిన పోర్ట్‌లో యాక్సెస్ చేయదగినదిగా ఉండాలి
1. **క్లయింట్లను కనెక్ట్ చేయండి**: `http://localhost:3000` వంటి స్థానిక కనెక్షన్ URLలను ఉపయోగించండి

## క్లౌడ్ డిప్లాయ్‌మెంట్

MCP సర్వర్లను వివిధ క్లౌడ్ ప్లాట్‌ఫారమ్‌లకు డిప్లాయ్ చేయవచ్చు:

- **సర్వర్‌లెస్ ఫంక్షన్స్**: తేలికపాటి MCP సర్వర్లను సర్వర్‌లెస్ ఫంక్షన్స్‌గా డిప్లాయ్ చేయండి
- **కంటైనర్ సర్వీసులు**: Azure Container Apps, AWS ECS, లేదా Google Cloud Run వంటి సర్వీసులను ఉపయోగించండి
- **కుబెర్నెటిస్**: MCP సర్వర్లను కుబెర్నెటిస్ క్లస్టర్లలో అధిక లభ్యత కోసం డిప్లాయ్ చేసి నిర్వహించండి

### ఉదాహరణ: Azure Container Apps

Azure Container Apps MCP సర్వర్ల డిప్లాయ్‌మెంట్‌ను మద్దతు ఇస్తుంది. ఇది ఇంకా అభివృద్ధిలో ఉంది మరియు ప్రస్తుతం SSE సర్వర్లను మద్దతు ఇస్తుంది.

ఇది ఎలా చేయాలో ఇక్కడ ఉంది:

1. ఒక రిపోను క్లోన్ చేయండి:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. దాన్ని స్థానికంగా నడపండి మరియు పరీక్షించండి:

  ```sh
  uv venv
  uv sync

  # లినక్స్/మ్యాక్OS
  export API_KEYS=<AN_API_KEY>
  # విండోస్
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. స్థానికంగా ప్రయత్నించడానికి, *.vscode* డైరెక్టరీలో *mcp.json* ఫైల్‌ను సృష్టించి క్రింది కంటెంట్‌ను జోడించండి:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  SSE సర్వర్ ప్రారంభమైన తర్వాత, మీరు JSON ఫైల్‌లో ప్లే ఐకాన్‌పై క్లిక్ చేయవచ్చు, ఇప్పుడు GitHub Copilot ద్వారా సర్వర్上的 టూల్స్‌ను గుర్తించబడినట్లు చూడగలుగుతారు, టూల్ ఐకాన్ చూడండి.

1. డిప్లాయ్ చేయడానికి, క్రింది కమాండ్‌ను నడపండి:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

ఇది మీకు ఉంది, స్థానికంగా డిప్లాయ్ చేయండి, ఈ దశల ద్వారా Azureకు డిప్లాయ్ చేయండి.
