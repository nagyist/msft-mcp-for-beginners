# MCP తో Azure Content Safety ను అమలు చేయడం

> **OWASP MCP ప్రమాదం పరిష్కరించబడింది**: [MCP06 - సందర్భపూర్వక పేలోడ్స్ ద్వారా ప్రాంప్ట్ ఇంజెక్షన్](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

ప్రాంప్ట్ ఇంజెక్షన్, టూల్ ఫాయిసనింగ్ మరియు ఇతర AI-స్పెసిఫిక్ సున్నితత్వాలపై MCP భద్రతను బలోపేతం చేయడానికి, Azure Content Safety ని సమకూర్చడం చాలా అవసరం. ఈ అమలుపై మార్గదర్శకం [MCP Security Summit Workshop (Sherpa)](https://azure-samples.github.io/sherpa/) క్యాంప్ 3: I/O భద్రతతో సమన్వయంగా ఉంటుంది.

## MCP సర్వర్‌తో సమకూర్చడం

మీ MCP సర్వర్‌తో Azure Content Safety ని సమకూర్చడానికి, మీ అభ్యర్థన ప్రాసెస్సింగ్ పైప్‌లైన్‌లో కంటెంట్ సేఫ్టీ ఫిల్టర్‌ను మిడిల్‌వేర్‌గా చేర్చండి:

1. సర్వర్ స్టార్ట్‌అప్ సమయంలో ఫిల్టర్‌ను ప్రారంభించండి
2. ప్రాసెస్ చేయడానికి ముందు అన్ని టూల్ అభ్యర్థనలను ధృవీకరించండి
3. క్లైంట్స్‌కు తిరిగి ఇవ్వడానికి ముందు అన్ని అవుట్గోయింగ్ ప్రతిస్పందనలను తనిఖీ చేయండి
4. సేఫ్టీ ఉల్లంఘనలపై లాగ్ చేసి అలర్ట్ చేయండి
5. కంటెంట్ సేఫ్టీ తనిఖీలు విఫలమైనప్పుడు సరైన లోప నిర్వహణను అమలు చేయండి

ఇది కింద వాటి నుండి బలమైన రక్షణను కల్పిస్తుంది:
- ప్రాంప్ట్ ఇంజెక్షన్ దాడులు
- టూల్ ఫాయిసనింగ్ ప్రయత్నాలు
- దుర్భావనీయ ఇన్‌పుట్ల ద్వారా డేటా ఎగ్మతింపు
- హానికరమైన కంటెంట్ ఉత్పత్తి

## Azure Content Safety సమకూర్చడానికి ఉత్తమ ఆచారాలు

1. **కస్టమ్ బ్లాక్‌లిస్టులు**: MCP ఇంజెక్షన్ నమూనాల కోసం ప్రత్యేకమైన కస్టమ్ బ్లాక్‌లిస్టులను రూపొందించండి
2. **గంభీరత సర్దుబాటు**: మీ ప్రత్యేక ఉపయోగం మరియు ప్రమాద సహనం ఆధారంగా గంభీరత మితులను సర్దుబాటు చేయండి
3. **సమగ్ర కవరేజ్**: అన్ని ఇన్‌పుట్లు మరియు అవుట్‌పుట్లకు కంటెంట్ సేఫ్టీ తనిఖీలు వర్తింపజేయండి
4. **పనితీరు ఆప్టిమైజేషన్**: పునరావృత కంటెంట్ సేఫ్టీ తనిఖీలకు క్యాషింగ్ అమలు చేయవచ్చు అని పరిగణించండి
5. **ష‌రతులుFallback మెకానిజం‌లు**: కంటెంట్ సేఫ్టీ సేవలు అందుబాటులో లేని సమయంలో స్పష్టమైన fallback ప్రవర్తనలను నిర్వచించండి
6. **వాడుకరి అభిప్రాయం**: సేఫ్టీ సమస్యల కారణంగా కంటెంట్ బ్లాక్ చేసినప్పుడు వినియోగదారులకు స్పష్టమైన అభిప్రాయాన్ని ఇవ్వండి
7. **నిరంతర మెరుగుదల**: ఎదుగుతున్న బ گرفته్ట్రేట్ల ఆధారంగా బ్లాక్‌లిస్టులు మరియు నమూనాలను నియమితంగా నవీకరణ చేయండి
