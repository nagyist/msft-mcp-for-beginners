# పాఠం: వెబ్ సెర్చ్ MCP సర్వర్ నిర్మాణం

ఈ అధ్యాయం బాహ్య APIs తో సమన్వయం చేసే, విభిన్న డేటా రకాలతో వ్యవహరించే, లోపాలను నిర్వహించే, మరియు అనేక టూల్స్‌ను సమన్వయించే వాస్తవ ప్రపంచ AI ఏజెంట్‌ను నిర్మించే విధానాన్ని చూపిస్తుంది—అన్నీ ప్రొడక్షన్-రెడీ ఫార్మాట్‌లో. మీరు చూడబోతున్నవి:

- **ప్రామాణీకరణ అవసరమయ్యే బాహ్య APIs తో సమన్వయం**
- **బహుళ ఎండ్పాయింట్ల నుండి విభిన్న డేటా రకాల నిర్వహణ**
- **దృఢమైన లోప నిర్వహణ మరియు లాగింగ్ వ్యూహాలు**
- **ఒకే సర్వర్‌లో బహుళ-టూల్ సమన్వయం**

చివరికి, మీరు అధునాతన AI మరియు LLM-చాలిత అనువర్తనాలకు అవసరమైన నమూనాలు మరియు ఉత్తమ పద్ధతులతో ప్రాక్టికల్ అనుభవం పొందుతారు.

## పరిచయం

ఈ పాఠంలో, మీరు SerpAPI ఉపయోగించి రియల్-టైమ్ వెబ్ డేటాతో LLM సామర్థ్యాలను విస్తరించే అధునాతన MCP సర్వర్ మరియు క్లయింట్‌ను ఎలా నిర్మించాలో నేర్చుకుంటారు. ఇది వెబ్ నుండి తాజా సమాచారాన్ని పొందగల డైనమిక్ AI ఏజెంట్లను అభివృద్ధి చేయడానికి కీలక నైపుణ్యం.

## నేర్చుకునే లక్ష్యాలు

ఈ పాఠం చివరికి, మీరు చేయగలుగుతారు:

- బాహ్య APIs (SerpAPI వంటి) ను సురక్షితంగా MCP సర్వర్‌లో సమన్వయించడం
- వెబ్, వార్తలు, ఉత్పత్తి శోధన, మరియు ప్రశ్నోత్తరాల కోసం బహుళ టూల్స్ అమలు చేయడం
- LLM వినియోగానికి నిర్మిత డేటాను పార్స్ చేసి ఫార్మాట్ చేయడం
- లోపాలను నిర్వహించడం మరియు API రేట్ పరిమితులను సమర్థవంతంగా నిర్వహించడం
- ఆటోమేటెడ్ మరియు ఇంటరాక్టివ్ MCP క్లయింట్లను నిర్మించి పరీక్షించడం

## వెబ్ సెర్చ్ MCP సర్వర్

ఈ విభాగం వెబ్ సెర్చ్ MCP సర్వర్ యొక్క నిర్మాణం మరియు లక్షణాలను పరిచయం చేస్తుంది. మీరు FastMCP మరియు SerpAPI కలిపి LLM సామర్థ్యాలను రియల్-టైమ్ వెబ్ డేటాతో ఎలా విస్తరించాలో చూడగలుగుతారు.

### అవలోకనం

ఈ అమలు నాలుగు టూల్స్‌ను కలిగి ఉంది, ఇవి MCP యొక్క విభిన్న, బాహ్య API ఆధారిత పనులను సురక్షితంగా మరియు సమర్థవంతంగా నిర్వహించే సామర్థ్యాన్ని ప్రదర్శిస్తాయి:

- **general_search**: విస్తృత వెబ్ ఫలితాల కోసం
- **news_search**: తాజా శీర్షికల కోసం
- **product_search**: ఈ-కామర్స్ డేటా కోసం
- **qna**: ప్రశ్న-సమాధాన స్నిపెట్ల కోసం

### లక్షణాలు
- **కోడ్ ఉదాహరణలు**: స్పష్టత కోసం కోడ్ పివాట్లను ఉపయోగించి Python (మరియు ఇతర భాషలకు సులభంగా విస్తరించదగిన) భాషా-ప్రత్యేక కోడ్ బ్లాక్స్‌ను కలిగి ఉంది

### Python

```python
# general_search టూల్ యొక్క ఉదాహరణ ఉపయోగం
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "open source LLMs"})
            print(result)
```

క్లయింట్‌ను నడిపించే ముందు, సర్వర్ ఏమి చేస్తుందో అర్థం చేసుకోవడం సహాయకరం. [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ఫైల్ MCP సర్వర్‌ను అమలు చేస్తుంది, ఇది SerpAPIతో సమన్వయం చేసి వెబ్, వార్తలు, ఉత్పత్తి శోధన, మరియు ప్రశ్నోత్తరాల కోసం టూల్స్‌ను అందిస్తుంది. ఇది రాబోయే అభ్యర్థనలను నిర్వహిస్తుంది, API కాల్స్‌ను నిర్వహిస్తుంది, ప్రతిస్పందనలను పార్స్ చేస్తుంది, మరియు నిర్మిత ఫలితాలను క్లయింట్‌కు తిరిగి ఇస్తుంది.

మీరు పూర్తి అమలును [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) లో సమీక్షించవచ్చు.

ఇక్కడ సర్వర్ ఒక టూల్‌ను ఎలా నిర్వచించి నమోదు చేస్తుందో సంక్షిప్త ఉదాహరణ ఉంది:

### Python సర్వర్

```python
# server.py (ఉద్భ్రంహణం)
from mcp.server import MCPServer, Tool

async def general_search(query: str):
    # ...అమలు...

server = MCPServer()
server.add_tool(Tool("general_search", general_search))

if __name__ == "__main__":
    server.run()
```

---

- **బాహ్య API సమన్వయం**: API కీలు మరియు బాహ్య అభ్యర్థనలను సురక్షితంగా నిర్వహించడం చూపిస్తుంది
- **నిర్మిత డేటా పార్సింగ్**: API ప్రతిస్పందనలను LLM-స్నేహపూర్వక ఫార్మాట్లుగా మార్చడం చూపిస్తుంది
- **లోప నిర్వహణ**: సరైన లాగింగ్‌తో దృఢమైన లోప నిర్వహణ
- **ఇంటరాక్టివ్ క్లయింట్**: ఆటోమేటెడ్ పరీక్షలు మరియు ఇంటరాక్టివ్ మోడ్ రెండింటినీ కలిగి ఉంటుంది
- **కాంటెక్స్ట్ నిర్వహణ**: MCP కాంటెక్స్ట్‌ను లాగింగ్ మరియు అభ్యర్థనలను ట్రాక్ చేయడానికి ఉపయోగిస్తుంది

## ముందస్తు అవసరాలు

మీరు ప్రారంభించే ముందు, మీ పరిసరాన్ని సక్రమంగా సెట్ చేసుకున్నారని నిర్ధారించుకోండి. ఇది అన్ని ఆధారాలను ఇన్‌స్టాల్ చేసి, మీ API కీలు సరిగా కాన్ఫిగర్ చేయబడినట్లు నిర్ధారిస్తుంది, తద్వారా అభివృద్ధి మరియు పరీక్ష సులభంగా జరుగుతుంది.

- Python 3.8 లేదా అంతకంటే పైగా
- SerpAPI API కీ ([SerpAPI](https://serpapi.com/) వద్ద సైన్ అప్ చేయండి - ఉచిత స్థాయి అందుబాటులో ఉంది)

## ఇన్‌స్టాలేషన్

ప్రారంభించడానికి, మీ పరిసరాన్ని సెట్ చేయడానికి ఈ దశలను అనుసరించండి:

1. uv (సిఫార్సు చేయబడింది) లేదా pip ఉపయోగించి ఆధారాలను ఇన్‌స్టాల్ చేయండి:

```bash
# uv ఉపయోగించడం (సిఫార్సు చేయబడింది)
uv pip install -r requirements.txt

# పిప్ ఉపయోగించడం
pip install -r requirements.txt
```

2. ప్రాజెక్ట్ రూట్‌లో `.env` ఫైల్ సృష్టించి మీ SerpAPI కీని చేర్చండి:

```
SERPAPI_KEY=your_serpapi_key_here
```

## ఉపయోగం

వెబ్ సెర్చ్ MCP సర్వర్ అనేది SerpAPIతో సమన్వయం చేసి వెబ్, వార్తలు, ఉత్పత్తి శోధన, మరియు ప్రశ్నోత్తరాల కోసం టూల్స్‌ను అందించే ప్రధాన భాగం. ఇది రాబోయే అభ్యర్థనలను నిర్వహిస్తుంది, API కాల్స్‌ను నిర్వహిస్తుంది, ప్రతిస్పందనలను పార్స్ చేస్తుంది, మరియు నిర్మిత ఫలితాలను క్లయింట్‌కు తిరిగి ఇస్తుంది.

మీరు పూర్తి అమలును [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) లో సమీక్షించవచ్చు.

### సర్వర్ నడపడం

MCP సర్వర్‌ను ప్రారంభించడానికి, ఈ కింది కమాండ్ ఉపయోగించండి:

```bash
python server.py
```

సర్వర్ stdio-ఆధారిత MCP సర్వర్‌గా నడుస్తుంది, దీనికి క్లయింట్ నేరుగా కనెక్ట్ కావచ్చు.

### క్లయింట్ మోడ్‌లు

క్లయింట్ (`client.py`) MCP సర్వర్‌తో ఇంటరాక్ట్ కావడానికి రెండు మోడ్‌లను మద్దతు ఇస్తుంది:

- **సాధారణ మోడ్**: అన్ని టూల్స్‌ను పరీక్షించే ఆటోమేటెడ్ టెస్టులను నడుపుతుంది మరియు వాటి ప్రతిస్పందనలను ధృవీకరిస్తుంది. ఇది సర్వర్ మరియు టూల్స్ సరిగా పనిచేస్తున్నాయో త్వరగా తనిఖీ చేయడానికి ఉపయోగపడుతుంది.
- **ఇంటరాక్టివ్ మోడ్**: మెనూ ఆధారిత ఇంటర్‌ఫేస్‌ను ప్రారంభిస్తుంది, ఇక్కడ మీరు మానవీయంగా టూల్స్‌ను ఎంచుకుని పిలవవచ్చు, కస్టమ్ ప్రశ్నలను నమోదు చేయవచ్చు, మరియు ఫలితాలను రియల్ టైమ్‌లో చూడవచ్చు. ఇది సర్వర్ సామర్థ్యాలను అన్వేషించడానికి మరియు వివిధ ఇన్‌పుట్లతో ప్రయోగాలు చేయడానికి అనుకూలం.

మీరు పూర్తి అమలును [`client.py`](../../../../05-AdvancedTopics/web-search-mcp/client.py) లో సమీక్షించవచ్చు.

### క్లయింట్ నడపడం

ఆటోమేటెడ్ టెస్టులను నడపడానికి (ఇది ఆటోమేటిక్‌గా సర్వర్‌ను ప్రారంభిస్తుంది):

```bash
python client.py
```

లేదా ఇంటరాక్టివ్ మోడ్‌లో నడపండి:

```bash
python client.py --interactive
```

### వేర్వేరు పద్ధతులతో పరీక్షించడం

మీ అవసరాలు మరియు వర్క్‌ఫ్లో ఆధారంగా సర్వర్ అందించే టూల్స్‌ను పరీక్షించడానికి మరియు ఇంటరాక్ట్ చేయడానికి అనేక మార్గాలు ఉన్నాయి.

#### MCP Python SDKతో కస్టమ్ టెస్ట్ స్క్రిప్టులు రాయడం
మీరు MCP Python SDK ఉపయోగించి మీ స్వంత టెస్ట్ స్క్రిప్టులను కూడా నిర్మించవచ్చు:

# [Python](../../../../05-AdvancedTopics/web-search-mcp)

```python
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def test_custom_query():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            # మీ అనుకూల పరామితులతో టూల్స్‌ను పిలవండి
            result = await session.call_tool("general_search", 
                                           arguments={"query": "your custom query"})
            # ఫలితాన్ని ప్రాసెస్ చేయండి
```

ఈ సందర్భంలో, "టెస్ట్ స్క్రిప్ట్" అంటే మీరు MCP సర్వర్ కోసం క్లయింట్‌గా పనిచేసే కస్టమ్ Python ప్రోగ్రామ్. ఇది ఫార్మల్ యూనిట్ టెస్ట్ కాకపోయినా, ఈ స్క్రిప్ట్ సర్వర్‌కు ప్రోగ్రామాటిక్‌గా కనెక్ట్ అయి, మీరు ఎంచుకున్న పారామితులతో ఏ టూల్‌ను అయినా పిలవడానికి, ఫలితాలను పరిశీలించడానికి అనుమతిస్తుంది. ఈ విధానం ఉపయోగపడుతుంది:

- టూల్ కాల్స్‌ను ప్రోటోటైప్ చేయడం మరియు ప్రయోగాలు చేయడం
- సర్వర్ వివిధ ఇన్‌పుట్లకు ఎలా స్పందిస్తుందో ధృవీకరించడం
- పునరావృత టూల్ పిలుపులను ఆటోమేటింగ్ చేయడం
- MCP సర్వర్‌పై మీ స్వంత వర్క్‌ఫ్లోలు లేదా సమన్వయాలను నిర్మించడం

మీరు టెస్ట్ స్క్రిప్టులను ఉపయోగించి కొత్త ప్రశ్నలను త్వరగా ప్రయత్నించవచ్చు, టూల్ ప్రవర్తనను డీబగ్ చేయవచ్చు, లేదా అధునాతన ఆటోమేషన్ కోసం ప్రారంభ బిందువుగా ఉపయోగించవచ్చు. క్రింద MCP Python SDK ఉపయోగించి అలాంటి స్క్రిప్ట్‌ను సృష్టించే ఉదాహరణ ఉంది:

## ఈ పాఠంలోని అధునాతన భావనలు

మీరు నిర్మించడం ప్రారంభించే ముందు, ఈ అధ్యాయం మొత్తం కనిపించే కొన్ని ముఖ్యమైన అధునాతన భావనలు ఇక్కడ ఉన్నాయి. వీటిని అర్థం చేసుకోవడం మీకు సహాయపడుతుంది, మీరు కొత్తవారైనా:

- **బహుళ-టూల్ సమన్వయం**: ఇది ఒకే MCP సర్వర్‌లో వెబ్ శోధన, వార్త శోధన, ఉత్పత్తి శోధన, మరియు ప్రశ్నోత్తరాలు వంటి అనేక టూల్స్ నడపడం. ఇది మీ సర్వర్‌ను ఒకే పనికి కాకుండా విభిన్న పనులను నిర్వహించగలిగేలా చేస్తుంది.
- **API రేట్ పరిమితి నిర్వహణ**: చాలా బాహ్య APIs (SerpAPI వంటి) మీరు ఒక నిర్దిష్ట సమయంలో చేయగల అభ్యర్థనల సంఖ్యను పరిమితం చేస్తాయి. మంచి కోడ్ ఈ పరిమితులను తనిఖీ చేసి, వాటిని సున్నితంగా నిర్వహిస్తుంది, తద్వారా మీరు పరిమితిని దాటినా మీ యాప్ క్రాష్ కాకుండా ఉంటుంది.
- **నిర్మిత డేటా పార్సింగ్**: API ప్రతిస్పందనలు తరచుగా సంక్లిష్టమైనవి మరియు లోపలికి ముడిపడినవి. ఈ భావన ఆ ప్రతిస్పందనలను శుభ్రంగా, సులభంగా ఉపయోగించదగిన ఫార్మాట్లుగా మార్చడం గురించి, ఇవి LLMs లేదా ఇతర ప్రోగ్రాములకు అనుకూలంగా ఉంటాయి.
- **లోప పునరుద్ధరణ**: కొన్ని సార్లు సమస్యలు వస్తాయి—నెట్‌వర్క్ విఫలమవడం లేదా API మీరు ఆశించినది ఇవ్వకపోవడం. లోప పునరుద్ధరణ అంటే మీ కోడ్ ఈ సమస్యలను నిర్వహించి, ఉపయోగకరమైన ఫీడ్బ్యాక్ ఇస్తుంది, క్రాష్ కాకుండా.
- **పారామితి ధృవీకరణ**: ఇది మీ టూల్స్‌కు అందే అన్ని ఇన్‌పుట్లను సరైనవి మరియు సురక్షితంగా ఉన్నాయో లేదో తనిఖీ చేయడం గురించి. ఇందులో డిఫాల్ట్ విలువలను సెట్ చేయడం మరియు రకాల సరైనతను నిర్ధారించడం ఉంటుంది, ఇది బగ్స్ మరియు గందరగోళాన్ని నివారిస్తుంది.

## సమస్య పరిష్కారం

వెబ్ సెర్చ్ MCP సర్వర్‌తో పని చేస్తూ మీరు సాధారణంగా ఎదుర్కొనే సమస్యలను గుర్తించి పరిష్కరించడానికి ఈ విభాగం సహాయపడుతుంది. మీరు లోపాలు లేదా అనూహ్య ప్రవర్తనను ఎదుర్కొంటే, ఈ సమస్య పరిష్కార సూచనలు మొదటిసారిగా చూడండి—ఇవి తరచుగా సమస్యలను త్వరగా పరిష్కరిస్తాయి.

### సాధారణ సమస్యలు

క్రింద కొన్ని సాధారణ సమస్యలు మరియు వాటి పరిష్కారాలు ఉన్నాయి:

1. **.env ఫైల్‌లో SERPAPI_KEY లేమి**
   - మీరు `SERPAPI_KEY environment variable not found` అనే లోపం చూస్తే, అంటే మీ అప్లికేషన్ SerpAPIకి అవసరమైన API కీని కనుగొనలేకపోతుంది. దీన్ని సరిచేయడానికి, ప్రాజెక్ట్ రూట్‌లో `.env` అనే ఫైల్ సృష్టించి, `SERPAPI_KEY=your_serpapi_key_here` అనే లైన్ చేర్చండి. ఇక్కడ `your_serpapi_key_here` స్థానంలో మీ SerpAPI వెబ్‌సైట్ నుండి పొందిన నిజమైన కీని ఉంచండి.
