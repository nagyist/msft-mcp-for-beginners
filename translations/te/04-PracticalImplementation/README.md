# პრაქტიკული განხორციელება

პრაქტიკული განხორციელება სწორედ ის ადგილია, სადაც Model Context Protocol (MCP)-ის ძალა محسوسდება. მიუხედავად იმისა, რომ MCP-ის თეორიისა და არქიტექტურის გაგება მნიშვნელოვანია, ნამდვილი ღირებულება მაშინ გამოიკვეთება, როცა ამ კონცეფციებს რეალურ პრობლემების გადაწყვეტისთვის გამოიყენებთ. ეს თავი ხსნის ცრურწმენებს საკონცეფციო ცოდნასა და პრაქტიკულ განვითარებას შორის და გიბიძგებთ MCP-ის საფუძველზე აპლიკაციების შექმნის პროცესში.

დაავად გაქვთ უძილესი მრჩევლები, AI-ის ინტეგრირება ბიზნეს სამუშაო პროცესებში თუ მონაცემთა დამუშავების სპეციალური ხელსაწყოების შექმნა, MCP იძლევა მოქნილ საფუძველს. მისი ენა-აზიატებული დიზაინი და ოფიციალური SDK-ები პოპულარულ პროგრამირების ენებზე ხელმისაწვდომია მრავალფეროვან დეველოპერებისთვის. ამ SDK-ების გამოყენებით სწრაფად შეგიძლიათ პროტოტიპის შექმნა, რედაქტირება და სხვადასხვა პლატფორმაზე, გარემოში მასშტაბირება.

შემდეგ ნაწილებში მოძებნით პრაქტიკულ მაგალითებს, ნიმუშ კოდებს და განთავსების სტრატეგიებს, რომლებიც აჩვენებს როგორ გამოიყენოთ MCP C#, Java Spring-ით, TypeScript-ით, JavaScript-ით და Python-ით. თქვენ ისწავლით ასევე როგორ დააბაგოთ და შეამოწმოთ MCP სერვერები, მართოთ API-ები და განთავსოთ გადაწყვეტილებები ღრუბელში Azure-ის გამოყენებით. ეს პრაქტიკული რესურსები დაგეხმარებათ სწავლის აჩქარებაში და ძლიერი, პროდუქტიული MCP აპლიკაციების დამზადებაში.

## მიმოხილვა

ეს გაკვეთილი მიმართულია MCP-ის პრაქტიკულ გამოყენებაზე მრავალ პროგრამირების ენებში. განვიხილავთ, როგორ გამოვიყენოთ MCP SDK-ები C#, Java Spring, TypeScript, JavaScript და Python-ში ძლიერი აპლიკაციების შესაქმნელად, MCP სერვერების დებაგირებისა და ტესტირებისთვის და ხელახლად გამოყენებად რესურსების, პრომპტების და ხელსაწყოების შექმნისთვის.

## მიზნები

ამ გაკვეთილის ბოლოს შეძლებთ:

- MCP-ის გადაწყვეტილებების განხორციელებას ოფიციალური SDK-ებით სხვადასხვა ენაზე
- MCP სერვერების სისტემატურ დებაგირებასა და ტესტირებას
- სერვერის ფუნქციების (რესურსები, პრომპტები და ხელსაწყოები) შექმნასა და გამოყენებას
- კომპლექსური ამოცანებისთვის ეფექტური MCP სამუშაო პროცესების დიზაინს
- MCP-ის შესრულების და საიმედოობის ოპტიმიზაციას

## ძირითადი სერვერის ფუნქციები

MCP სერვერები შეიძლება განხორციელდეს ქვემოთ ჩამოთვლილი ფუნქციების კომბინაციით:

### რესურსები

რესურსები უზრუნველყოფენ კონტექსტსა და მონაცემებს მომხმარებლის ან AI მოდელისთვის:

- დოკუმენტების რეპოზიტორიები
- ცოდნის ბაზები
- სტრუქტურირებული მონაცემთა წყაროები
- ფაილის სისტემები

### პრომპტები

პროპმტები არის მომხმარებლისთვის ცხრილში მოთავსებული შეტყობინებები და სამუშაო პროცესები:

- წინასწარ განსაზღვრული დიალოგის შაბლონები
- მართული ინტერაქციის მოდელები
- სპეციალური დიალოგის სტრუქტურები

### ხელსაწყოები

ხელისუფლება AI მოდელის კრიშნად ხელსაწყოებია:

- მონაცემთა დამუშავების კომპონენტები
- გარე API ინტეგრაციები
- გამოთვლითი შესაძლებლობები

## დისტანციური MCP სერვერის განთავსება Azure-ზე

ნახეთ შეგვიძლია თუ არა ზემოთ ნახსენები ნიმუშის განთავსება:

1. კლონი რეპოზიტორიიდან

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

1. დაარეგისტრირეთ `Microsoft.App` რესურსის მიმწოდებელი.

   - თუ იყენებთ Azure CLI-ს, ბრძანება: `az provider register --namespace Microsoft.App --wait`.
   - თუ იყენებთ Azure PowerShell-ს, ბრძანება: `Register-AzResourceProvider -ProviderNamespace Microsoft.App`. შემდეგ, გარკვეული დროის შემდეგ შეამოწმეთ `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` რეგისტრაციის დასრულებისთვის.

1. აწარმოეთ შემდეგი [azd](https://aka.ms/azd) ბრძანება API Management სერვისის, ფუნქციის აპლიკაციის (კოდით) და სხვა საჭირო Azure რესურსების დასამზადებლად

    ```shell
    azd up
    ```

    ეს ბრძანება ახორციელებს ყველა ღრუბელ რესურსის განთავსებას Azure-ზე

### ტესტირება MCP Inspector-ით

1. **ახალ ტერმინალში**, დააინსტალირეთ და ააშუქეთ MCP Inspector

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    მომხმარებას ეყოფა შემდეგი ინტერფეისი:

    ![Connect to Node inspector](../../../translated_images/te/connect.141db0b2bd05f096.webp)

1. CTRL+კლიკი მიანიჭეთ MCP Inspector ვებ აპლიკაციას URL-ზე (მაგ. [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources))
1. აირჩიეთ ტრანსპორტის ტიპი `SSE`
1. ჩასვით URL თქვენი მიმდინარე API Management SSE კავშირის ბოლო წერტილისთვის, რომელიც გამოჩნდება `azd up` შესრულების შემდეგ და დააჭირეთ **Connect**:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

1. **საჭრბელების სია**. დააჭირეთ ხელსაწყოს და შემდეგ დააჭირეთ **Run Tool**.

თუ ყველა ნაბიჯი წარიმართა წარმატებით, ახლა დაკავშირებული ხართ MCP სერვერზე და შეძლებული გაქვთ ხელსაწყოს გამოძახება.

## MCP სერვერები Azure-სთვის

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): ამ რეპოზიტორიების ნაკრები წარმოადგენს სწრაფი დაწყების შაბლონს, MCP სერვერების მშენებლობის და განთავსებისათვის Azure ფუნქციების გამოყენებით Python, C# .NET ან Node/TypeScript-ის გარშემო.

ეს ნიმუშები გვთავაზობენ სრულ გადაწყვეტას, რომლითაც დეველოპერებს შეეძლებათ:

- ლოკალურად აშენება და გაშვება: MCP სერვერის განვითარება და დებაგირება ლოკალურ მანქანაზე
- Azure-ზე განთავსება: მარტივად განთავსება ღრუბელში ერთი azd up ბრძანებით
- კლიენტებთან კავშირი: MCP სერვერთან მიმდინარე კავშირი სხვადასხვა კლიენტიდან, მათ შორის VS Code-ის Copilot აგენტის რეჟიმიდან და MCP Inspector-ით

### ძირითადი ფუნქციები

- უსაფრთხოება დიზაინიდან გამომდინარე: MCP სერვერი დაცულია გასაღებებისა და HTTPS-ის გამოყენებით
- ავტენტიფიკაციის ვარიანტები: მხარს უჭერს OAuth-ს, როგორც ჩაშენებული ავთენტიკაციით, ასევე API Management-ით
- ქსელის იზოლაცია: Azure Virtual Networks (VNET)-ით ქსელის იზოლაციის განხორციელება
- სერვისის არქიტექტურა: Azure Functions-ის გამოყენება მასშტაბირებადი, ღონისძიებათა დაფუძნებული შესრულებისთვის
- ლოკალური განვითარება: ყოვლისმომცველ მხარდაჭერა ადგილობრივი განვითარების და დებაგირებისთვის
- მარტივი განთავსება: გაუადვილებული პროცესები საწყისი Azure-ზე განთავსებისთვის

რეპოზიტორიაში შედის ყველა საჭირო კონფიგურაცია, წყარო კოდი და ინფრასტრუქტურული აღწერები MCP სერვერის პროდუქციისთვის მზად განთავსების დასაწყებად.

- [Azure Remote MCP Functions Python](https://github.com/Azure-Samples/remote-mcp-functions-python) - MCP-ის მაგალითი Azure Functions-ის გამოყენებით Python-ში

- [Azure Remote MCP Functions .NET](https://github.com/Azure-Samples/remote-mcp-functions-dotnet) - MCP-ის მაგალითი Azure Functions-ის გამოყენებით C# .NET-ში

- [Azure Remote MCP Functions Node/Typescript](https://github.com/Azure-Samples/remote-mcp-functions-typescript) - MCP-ის მაგალითი Azure Functions-ის გამოყენებით Node/TypeScript-ში.

## ძირითადი დასკვნები

- MCP SDK-ები უზრუნველყოფენ ენის სპეციფიკურ ხელსაწყოებს ძლიერი MCP გადაწყვეტილებების შესაქმნელად
- დებაგირების და ტესტირებისა კრიტიკული მნიშვნელობა აქვს MCP აპლიკაციების საიმედოობაში
- ხელახლად გამოყენებადი პრომპტების შაბლონები უზრუნველყოფს AI-ის კონსისტენტურ ინტერაქციებს
- კარგად შემუშავებული სამუშაო პროცესები მართავს რთულ ამოცანებს მრავალ ხელსაწყოს გამოყენებით
- MCP გადაწყვეტილებების განხორციელება მოითხოვს უსაფრთხოების, შესრულების და შეცდომების მართვის გათვალისწინებას

## სავარჯიშო

დიზაინერეთ პრაქტიკული MCP სამუშაო პროცესი, რომელიც მოგვარების საშუალებას იძლევა თქვენი სფეროს რეალურ პრობლემას:

1. განსაზღვრეთ 3-4 ხელსაწყო, რომლებიც გამოიყენება ამ პრობლემის გადასაჭრელად
2. შექმენით სამუშაო პროცესის დიაგრამა, რომელიც აჩვენებს ხელსაწყოების ინტერაქციას
3. შეასრულეთ ერთ-ერთი ხელსაწყოს მარტივი ვერსია თქვენი საყვარელი ენით
4. შექმენით პრომპტის შაბლონი, რომელიც დაეხმარება მოდელს ეფექტურად გამოიყენოს თქვენი ხელსაწყო
