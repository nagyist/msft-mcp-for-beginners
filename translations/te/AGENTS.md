# AGENTS.md

## ప్రాజెక్ట్ అవలోకనం

**MCP ఫర్ బిగినర్స్** అనేది మోడల్ కాంటెక్స్ట్ ప్రోటోకాల్ (MCP) నేర్చుకోవడానికి ఓపెన్-సోర్స్ విద్యా పాఠ్యక్రమం - ఇది AI మోడల్స్ మరియు క్లయింట్ అప్లికేషన్ల మధ్య పరస్పర చర్యలకు ఒక ప్రమాణీకృత ఫ్రేమ్‌వర్క్. ఈ రిపోజిటరీ వివిధ ప్రోగ్రామింగ్ భాషలలో హ్యాండ్స్-ఆన్ కోడ్ ఉదాహరణలతో సమగ్ర అభ్యాస సామగ్రిని అందిస్తుంది.

### ముఖ్య సాంకేతికతలు

- **ప్రోగ్రామింగ్ భాషలు**: C#, Java, JavaScript, TypeScript, Python, Rust
- **ఫ్రేమ్‌వర్క్స్ & SDKs**: 
  - MCP SDK (`@modelcontextprotocol/sdk`)
  - Spring Boot (Java)
  - FastMCP (Python)
  - LangChain4j (Java)
- **డేటాబేసులు**: pgvector ఎక్స్‌టెన్షన్‌తో PostgreSQL
- **క్లౌడ్ ప్లాట్‌ఫారమ్‌లు**: Azure (Container Apps, OpenAI, Content Safety, Application Insights)
- **బిల్డ్ టూల్స్**: npm, Maven, pip, Cargo
- **డాక్యుమెంటేషన్**: 48+ భాషలతో ఆటోమేటెడ్ బహుభాషా అనువాదంతో మార్క్డౌన్

### ఆర్కిటెక్చర్

- **11 కోర్ మాడ్యూల్స్ (00-11)**: ప్రాథమికాల నుండి అధునాతన అంశాల వరకు వరుసగా అభ్యాస మార్గం
- **హ్యాండ్స్-ఆన్ ల్యాబ్స్**: బహుభాషలలో పూర్తి పరిష్కార కోడ్‌తో ప్రాక్టికల్ వ్యాయామాలు
- **సాంపిల్ ప్రాజెక్టులు**: పనిచేసే MCP సర్వర్ మరియు క్లయింట్ అమలు
- **అనువాద వ్యవస్థ**: బహుభాషా మద్దతు కోసం ఆటోమేటెడ్ GitHub Actions వర్క్‌ఫ్లో
- **చిత్ర ఆస్తులు**: అనువాదిత వెర్షన్లతో కేంద్రీకృత చిత్రాల డైరెక్టరీ

## సెటప్ కమాండ్లు

ఇది డాక్యుమెంటేషన్-కేంద్రీకృత రిపోజిటరీ. ఎక్కువ సెటప్ వ్యక్తిగత సాంపిల్ ప్రాజెక్టులు మరియు ల్యాబ్స్‌లో జరుగుతుంది.

### రిపోజిటరీ సెటప్

```bash
# రిపోజిటరీని క్లోన్ చేయండి
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### సాంపిల్ ప్రాజెక్టులతో పని చేయడం

సాంపిల్ ప్రాజెక్టులు ఈ స్థానాల్లో ఉన్నాయి:
- `03-GettingStarted/samples/` - భాషా-ప్రత్యేక ఉదాహరణలు
- `03-GettingStarted/01-first-server/solution/` - మొదటి సర్వర్ అమలు
- `03-GettingStarted/02-client/solution/` - క్లయింట్ అమలు
- `11-MCPServerHandsOnLabs/` - సమగ్ర డేటాబేస్ ఇంటిగ్రేషన్ ల్యాబ్స్

ప్రతి సాంపిల్ ప్రాజెక్ట్‌కు స్వంత సెటప్ సూచనలు ఉన్నాయి:

#### TypeScript/JavaScript ప్రాజెక్టులు
```bash
cd <project-directory>
npm install
npm start
```

#### Python ప్రాజెక్టులు
```bash
cd <project-directory>
pip install -r requirements.txt
# లేదా
pip install -e .
python main.py
```

#### Java ప్రాజెక్టులు
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

## అభివృద్ధి వర్క్‌ఫ్లో

### డాక్యుమెంటేషన్ నిర్మాణం

- **మాడ్యూల్స్ 00-11**: వరుస క్రమంలో కోర్ పాఠ్యక్రమం
- **translations/**: భాషా-ప్రత్యేక వెర్షన్లు (ఆటో-జనరేటెడ్, నేరుగా సవరించవద్దు)
- **translated_images/**: స్థానికీకృత చిత్రం వెర్షన్లు (ఆటో-జనరేటెడ్)
- **images/**: మూల చిత్రం మరియు డయాగ్రామ్లు

### డాక్యుమెంటేషన్ మార్పులు చేయడం

1. రూట్ మాడ్యూల్ డైరెక్టరీలలో (00-11) ఇంగ్లీష్ మార్క్డౌన్ ఫైళ్లను మాత్రమే సవరించండి
2. అవసరమైతే `images/` డైరెక్టరీలో చిత్రాలను నవీకరించండి
3. co-op-translator GitHub Action ఆటోమేటిగ్గా అనువాదాలను సృష్టిస్తుంది
4. ప్రధాన బ్రాంచ్‌కు పుష్ చేసినప్పుడు అనువాదాలు పునఃసృష్టించబడతాయి

### అనువాదాలతో పని చేయడం

- **ఆటోమేటెడ్ అనువాదం**: GitHub Actions వర్క్‌ఫ్లో అన్ని అనువాదాలను నిర్వహిస్తుంది
- **`translations/` డైరెక్టరీలో ఫైళ్లను మానవీయంగా సవరించవద్దు**
- అనువాద మెటాడేటా ప్రతి అనువాద ఫైల్‌లో నిక్షిప్తం
- మద్దతు పొందిన భాషలు: అరబిక్, చైనీస్, ఫ్రెంచ్, జర్మన్, హిందీ, జపనీస్, కొరియన్, పోర్చుగీస్, రష్యన్, స్పానిష్ మరియు మరిన్ని 48+ భాషలు

## పరీక్షా సూచనలు

### డాక్యుమెంటేషన్ ధృవీకరణ

ఇది ప్రధానంగా డాక్యుమెంటేషన్ రిపోజిటరీ కావడంతో, పరీక్షా దృష్టి:

1. **లింక్ ధృవీకరణ**: అంతర్గత లింకులు సక్రమంగా పనిచేస్తున్నాయో చూసుకోండి

```bash
find . -name "*.md" -type f | xargs grep -n "\[.*\](../../.*)"
```

2. **కోడ్ సాంపిల్ ధృవీకరణ**: కోడ్ ఉదాహరణలు కంపైల్/రన్ అవుతాయో పరీక్షించండి

## బిల్డ్ మరియు డిప్లాయ్‌మెంట్

### డాక్యుమెంటేషన్ డిప్లాయ్‌మెంట్

రిపోజిటరీ డాక్యుమెంటేషన్ హోస్టింగ్ కోసం GitHub Pages లేదా సమానమైనది ఉపయోగిస్తుంది (అన్వయిస్తే). ప్రధాన బ్రాంచ్‌లో మార్పులు:

1. అనువాద వర్క్‌ఫ్లో (`.github/workflows/co-op-translator.yml`)
2. అన్ని ఇంగ్లీష్ మార్క్డౌన్ ఫైళ్ల ఆటోమేటెడ్ అనువాదం
3. అవసరమైతే చిత్రం స్థానికీకరణ

### బిల్డ్ ప్రాసెస్ అవసరం లేదు

ఈ రిపోజిటరీ ప్రధానంగా మార్క్డౌన్ డాక్యుమెంటేషన్ కలిగి ఉంటుంది. కోర్ పాఠ్యక్రమం కోసం కంపైల్ లేదా బిల్డ్ దశ అవసరం లేదు.

### సాంపిల్ ప్రాజెక్ట్ డిప్లాయ్‌మెంట్

వ్యక్తిగత సాంపిల్ ప్రాజెక్టులకు డిప్లాయ్‌మెంట్ సూచనలు ఉండవచ్చు:
- MCP సర్వర్ డిప్లాయ్‌మెంట్ మార్గదర్శకానికి `03-GettingStarted/09-deployment/` చూడండి
- Azure Container Apps డిప్లాయ్‌మెంట్ ఉదాహరణలు `11-MCPServerHandsOnLabs/` లో ఉన్నాయి

- `[Docs] వివరణ` సాధారణ డాక్యుమెంటేషన్ నవీకరణలకు

### ఏమి సహకరించాలి

- డాక్యుమెంటేషన్ లేదా కోడ్ సాంపిల్స్‌లో బగ్ ఫిక్సులు
- అదనపు భాషలలో కొత్త కోడ్ ఉదాహరణలు
- ఉన్న కంటెంట్‌కు స్పష్టతలు మరియు మెరుగుదలలు
- కొత్త కేసు అధ్యయనాలు లేదా ప్రాక్టికల్ ఉదాహరణలు
- అస్పష్టమైన లేదా తప్పు కంటెంట్‌కు ఇష్యూ రిపోర్టులు

### ఏమి చేయకూడదు

- `translations/` డైరెక్టరీలోని ఫైళ్లను నేరుగా సవరించవద్దు
- `translated_images/` డైరెక్టరీని సవరించవద్దు
- చర్చ లేకుండా పెద్ద బైనరీ ఫైళ్లను జోడించవద్దు
- అనువాద వర్క్‌ఫ్లో ఫైళ్లను సమన్వయం లేకుండా మార్చవద్దు

### సాధారణ సమస్యలు పరిష్కారం

**Q: నా PR అనువాద తనిఖీ విఫలమవుతోంది**
A: మీరు రూట్ మాడ్యూల్ డైరెక్టరీలలోని ఇంగ్లీష్ మార్క్డౌన్ ఫైళ్లను మాత్రమే సవరించారని నిర్ధారించుకోండి, అనువాద వెర్షన్లు కాదు.

**Q: నేను కొత్త భాషను ఎలా జోడించాలి?**
A: భాషా మద్దతు co-op-translator వర్క్‌ఫ్లో ద్వారా నిర్వహించబడుతుంది. కొత్త భాషలను చేర్చడానికి ఇష్యూ తెరవండి.

**Q: కోడ్ సాంపిల్స్ పనిచేయడం లేదు**
A: మీరు ప్రత్యేక సాంపిల్ READMEలోని సెటప్ సూచనలను అనుసరించారని నిర్ధారించుకోండి. అవసరమైన డిపెండెన్సీల సరైన వెర్షన్లు ఇన్‌స్టాల్ అయ్యాయో చూసుకోండి.

**Q: చిత్రాలు ప్రదర్శించబడడం లేదు**
A: చిత్రం మార్గాలు సాపేక్షంగా ఉండి ఫార్వర్డ్ స్లాష్‌లు ఉపయోగించబడుతున్నాయో నిర్ధారించుకోండి. చిత్రాలు `images/` డైరెక్టరీలో లేదా స్థానికీకృత వెర్షన్ల కోసం `translated_images/` లో ఉండాలి.
