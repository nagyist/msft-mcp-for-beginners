# Zagon vzorca

## Namestite odvisnosti

```sh
npm install
```

## Sestavite

```sh
npm run build
```

## Zaženite

```sh
npm start
```

Videti bi morali besedilo:

```text
Registering tools...
Starting server...
```

Odlično, to pomeni, da se pravilno zažene.

## Preizkusite strežnik

Preizkusite zmogljivosti z naslednjukom ukazom:

```sh
npx @modelcontextprotocol/inspector node build/app.js
```

To bi moralo zagnati spletni vmesnik orodja za pregledovanje.

Zaženite orodje:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

Videti bi morali odgovor, podoben temu:

```text
{
  "content": [
    {
      "type": "text",
      "text": "Tool add called with arguments: {\"a\":1,\"b\":2}, result: {\"content\":[{\"type\":\"text\",\"text\":\"3\"}]}"
    }
  ]
```

Poskusite preizkusiti orodje, ki ne obstaja, na primer "add2", z naslednjim ukazom:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add2 --tool-arg a=1 --tool-arg b=2
```

Zdaj bi morali videti to sporočilo, ki kaže, da vaša validacija deluje:

```text
{
  "content": [],
  "error": {
    "code": "tool_not_found",
    "message": "Tool add2 not found."
  }
```

Prav tako poskusite poslati parameter `c`, ki ga shema zavrne, kot sledi:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg c=2
```

Zdaj bi morali videti napako "neveljavni argumenti":

```text
{
  "content": [],
  "error": {
    "code": "invalid_arguments",
    "message": "Invalid arguments for tool add: [\n  {\n    \"code\": \"invalid_type\",\n    \"expected\": \"number\",\n    \"received\": \"undefined\",\n    \"path\": [\n      \"b\"\n    ],\n    \"message\": \"Required\"\n  }\n]"
  }
}
```
