# تنفيذ أمان المحتوى في Azure مع MCP

> **مخاطر OWASP MCP التي تم التعامل معها**: [MCP06 - حقن المطالبات عبر الحمولات السياقية](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

لتعزيز أمان MCP ضد حقن المطالبات، وتسميم الأدوات، وغيرها من نقاط الضعف الخاصة بالذكاء الصناعي، يُنصح بشدة بدمج أمان محتوى Azure. يتوافق هذا الدليل التنفيذي مع [ورشة عمل قمة أمان MCP (شيربا)](https://azure-samples.github.io/sherpa/) المخيم 3: أمان الإدخال/الإخراج.

## التكامل مع خادم MCP

لدمج أمان محتوى Azure مع خادم MCP الخاص بك، أضف مرشح أمان المحتوى كوسيط في خط معالجة الطلبات الخاص بك:

1. تهيئة المرشح عند بدء تشغيل الخادم
2. التحقق من صحة جميع طلبات الأدوات الواردة قبل المعالجة
3. فحص جميع الردود الصادرة قبل إرجاعها إلى العملاء
4. تسجيل التنبيهات عند انتهاكات الأمان
5. تنفيذ معالجة الأخطاء المناسبة لفشل فحوصات أمان المحتوى

يوفر هذا دفاعًا قويًا ضد:
- هجمات حقن المطالبات
- محاولات تسميم الأدوات
- تسرب البيانات عبر مدخلات خبيثة
- إنشاء محتوى ضار

## أفضل الممارسات لتكامل أمان محتوى Azure

1. **القوائم السوداء المخصصة**: إنشاء قوائم حظر مخصصة خصيصًا لأنماط حقن MCP
2. **ضبط الشدة**: ضبط عتبات الشدة بناءً على حالة الاستخدام الخاصة بك وتحملك للمخاطر
3. **التغطية الشاملة**: تطبيق فحوصات أمان المحتوى على جميع المدخلات والمخرجات
4. **تحسين الأداء**: النظر في تنفيذ التخزين المؤقت لفحوصات أمان المحتوى المتكررة
5. **آليات الاحتياط**: تحديد سلوكيات بديلة واضحة عندما تكون خدمات أمان المحتوى غير متاحة
6. **ردود فعل المستخدم**: تقديم ردود واضحة للمستخدمين عند حظر المحتوى لأسباب أمان
7. **التحسين المستمر**: تحديث القوائم والأنماط بانتظام بناءً على التهديدات الناشئة
