# Esegui esempio

## Installa le dipendenze

```sh
npm install
```

## Compilalo

```sh
npm run build
```

## Eseguilo

```sh
npm start
```

Dovresti vedere il testo:

```text
Registering tools...
Starting server...
```

Ottimo, significa che si sta avviando correttamente.

## Testa il server

Prova le funzionalità con il seguente comando:

```sh
npx @modelcontextprotocol/inspector node build/app.js
```

Questo dovrebbe avviare l'interfaccia web dello strumento di ispezione.

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/list
```

Esegui uno strumento:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

Dovresti vedere una risposta simile a:

```text
{
  "content": [
    {
      "type": "text",
      "text": "Tool add called with arguments: {\"a\":1,\"b\":2}, result: {\"content\":[{\"type\":\"text\",\"text\":\"3\"}]}"
    }
  ]
```

Prova a testare uno strumento che non esiste, come "add2", con questo comando:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add2 --tool-arg a=1 --tool-arg b=2
```

Ora dovresti vedere questo messaggio che mostra che la tua validazione funziona:

```text
{
  "content": [],
  "error": {
    "code": "tool_not_found",
    "message": "Tool add2 not found."
  }
```

Prova anche a inviare un parametro `c` che dovrebbe essere rifiutato dallo schema, come segue:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg c=2
```

Ora dovresti vedere l'errore "argomenti non validi":

```text
{
  "content": [],
  "error": {
    "code": "invalid_arguments",
    "message": "Invalid arguments for tool add: [\n  {\n    \"code\": \"invalid_type\",\n    \"expected\": \"number\",\n    \"received\": \"undefined\",\n    \"path\": [\n      \"b\"\n    ],\n    \"message\": \"Required\"\n  }\n]"
  }
```
