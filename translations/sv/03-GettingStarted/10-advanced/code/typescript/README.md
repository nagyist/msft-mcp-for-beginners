# Kör exempel

## Installera beroenden

```sh
npm install
```

## Bygg det

```sh
npm run build
```

## Kör det

```sh
npm start
```

Du bör se texten:

```text
Registering tools...
Starting server...
```

Bra, det betyder att det startar upp korrekt.

## Testa servern

Testa funktionerna med följande kommando:

```sh
npx @modelcontextprotocol/inspector node build/app.js
```

Detta bör starta webbgränssnittet för inspektionsverktyget.

Kör ett verktyg:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

Du bör se ett svar som liknar:

```text
{
  "content": [
    {
      "type": "text",
      "text": "Tool add called with arguments: {\"a\":1,\"b\":2}, result: {\"content\":[{\"type\":\"text\",\"text\":\"3\"}]}"
    }
  ]
```

Prova att testa ett verktyg som inte finns, som "add2", med detta kommando:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add2 --tool-arg a=1 --tool-arg b=2
```

Du bör nu se detta meddelande som visar att din validering fungerar:

```text
{
  "content": [],
  "error": {
    "code": "tool_not_found",
    "message": "Tool add2 not found."
  }
```

Prova också att skicka in en parameter `c` som bör avvisas av schemat, så här:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg c=2
```

Du bör nu se felet "ogiltiga argument":

```text
{
  "content": [],
  "error": {
    "code": "invalid_arguments",
    "message": "Invalid arguments for tool add: [\n  {\n    \"code\": \"invalid_type\",\n    \"expected\": \"number\",\n    \"received\": \"undefined\",\n    \"path\": [\n      \"b\"\n    ],\n    \"message\": \"Required\"\n  }\n]"
  }
}
```
