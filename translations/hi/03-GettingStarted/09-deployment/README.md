# MCP सर्वर डिप्लॉय करना

अपना MCP सर्वर डिप्लॉय करने से दूसरे लोग आपके स्थानीय वातावरण से परे इसके टूल्स और संसाधनों तक पहुँच सकते हैं। स्केलेबिलिटी, भरोसेमंदता और प्रबंधन की सरलता के लिए आपकी आवश्यकताओं के आधार पर कई डिप्लॉयमेंट रणनीतियाँ हैं। नीचे आपको MCP सर्वर को स्थानीय, कंटेनरों में, और क्लाउड में डिप्लॉय करने के लिए मार्गदर्शन मिलेगा।

## अवलोकन

यह पाठ यह बताता है कि आपका MCP सर्वर ऐप कैसे डिप्लॉय करें।

## सीखने के उद्देश्य

इस पाठ के अंत तक, आप सक्षम होंगे:

- विभिन्न डिप्लॉयमेंट दृष्टिकोणों का मूल्यांकन करें।
- अपना ऐप डिप्लॉय करें।

## स्थानीय विकास और डिप्लॉयमेंट

यदि आपका सर्वर उपयोगकर्ता के मशीन पर चलाने के लिए है, तो आप निम्नलिखित चरणों का पालन कर सकते हैं:

1. **सर्वर डाउनलोड करें**। यदि आपने सर्वर नहीं लिखा है, तो पहले इसे अपनी मशीन पर डाउनलोड करें।  
1. **सर्वर प्रक्रिया शुरू करें**: अपना MCP सर्वर ऐप्लिकेशन चलाएँ।

SSE के लिए (stdio प्रकार के सर्वर के लिए आवश्यक नहीं)

1. **नेटवर्किंग कॉन्फ़िगर करें**: सुनिश्चित करें कि सर्वर अपेक्षित पोर्ट पर पहुंच योग्य है।  
1. **क्लाइंट से कनेक्ट करें**: `http://localhost:3000` जैसे स्थानीय कनेक्शन URL का उपयोग करें।

## क्लाउड डिप्लॉयमेंट

MCP सर्वर विभिन्न क्लाउड प्लेटफॉर्मों पर डिप्लॉय किए जा सकते हैं:

- **सर्वरलेस फंक्शन्स**: हल्के MCP सर्वर को सर्वरलेस फंक्शन्स के रूप में डिप्लॉय करें।  
- **कंटेनर सर्विसेज**: Azure Container Apps, AWS ECS, या Google Cloud Run जैसी सेवाओं का उपयोग करें।  
- **कुबेरनेटिस**: उच्च उपलब्धता के लिए MCP सर्वरों को कुबेरनेटिस क्लस्टर में डिप्लॉय और प्रबंधित करें।

### उदाहरण: Azure Container Apps

Azure Container Apps MCP सर्वरों के डिप्लॉयमेंट का समर्थन करता है। यह अभी भी विकासाधीन है और वर्तमान में SSE सर्वरों का समर्थन करता है।

यहाँ आप इसे कैसे कर सकते हैं:

1. एक रिपोज़िटरी क्लोन करें:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. इसे स्थानीय रूप से चलाकर परीक्षण करें:

  ```sh
  uv venv
  uv sync

  # लिनक्स/मैकओएस
  export API_KEYS=<AN_API_KEY>
  # विंडोज
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. स्थानीय रूप से आज़माने के लिए, *.vscode* निर्देशिका में *mcp.json* फ़ाइल बनाएं और निम्नलिखित सामग्री जोड़ें:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  एक बार SSE सर्वर शुरू हो जाने के बाद, आप JSON फ़ाइल में प्ले आइकन पर क्लिक कर सकते हैं, अब आपको GitHub Copilot द्वारा सर्वर पर टूल्स चयनित होते दिखाई देंगे, टूल आइकन देखें।

1. डिप्लॉय करने के लिए, निम्नलिखित कमांड चलाएँ:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

तो आपने इसे लोकली डिप्लॉय किया, इन चरणों के माध्यम से इसे Azure पर डिप्लॉय करें।
