# MCP के साथ Azure Content Safety को लागू करना

> **OWASP MCP जोखिम संबोधित**: [MCP06 - संदर्भ payloads के माध्यम से प्रॉम्प्ट इंजेक्शन](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

प्रॉम्प्ट इंजेक्शन, टूल पॉइज़निंग, और अन्य AI-विशिष्ट कमजोरियों के खिलाफ MCP सुरक्षा को मजबूत करने के लिए, Azure Content Safety को एकीकृत करना अत्यंत अनुशंसित है। यह कार्यान्वयन गाइड [MCP सुरक्षा सम्मेलन कार्यशाला (Sherpa)](https://azure-samples.github.io/sherpa/) कैंप 3: I/O सुरक्षा के अनुरूप है।

## MCP सर्वर के साथ एकीकरण

अपने MCP सर्वर के साथ Azure Content Safety को एकीकृत करने के लिए, अपनी अनुरोध प्रसंस्करण पाइपलाइन में कंटेंट सेफ्टी फ़िल्टर को मिडलवेयर के रूप में जोड़ें:

1. सर्वर स्टार्टअप के दौरान फ़िल्टर को प्रारंभ करें
2. प्रसंस्करण से पहले सभी आने वाले टूल अनुरोधों को सत्यापित करें
3. क्लाइंट्स को लौटाने से पहले सभी आउटगोइंग प्रतिक्रियाओं की जांच करें
4. सुरक्षा उल्लंघनों पर लॉग और अलर्ट करें
5. विफल कंटेंट सेफ्टी जांचों के लिए उचित त्रुटि हैंडलिंग लागू करें

यह मजबूत रक्षा प्रदान करता है:
- प्रॉम्प्ट इंजेक्शन हमलों के खिलाफ
- टूल पॉइज़निंग प्रयासों के खिलाफ
- दुर्भावनापूर्ण इनपुट के माध्यम से डेटा चोरी से
- हानिकारक सामग्री के उत्पादन से

## Azure Content Safety एकीकरण के लिए सर्वोत्तम प्रथाएँ

1. **कस्टम ब्लॉकलिस्ट्स**: विशेष रूप से MCP इंजेक्शन पैटर्न के लिए कस्टम ब्लॉकलिस्ट बनाएं
2. **गंभीरता समायोजन**: आपकी विशिष्ट उपयोग केस और जोखिम सहिष्णुता के आधार पर गंभीरता सीमा समायोजित करें
3. **व्यापक कवरेज**: सभी इनपुट और आउटपुट पर कंटेंट सेफ्टी जांच लागू करें
4. **प्रदर्शन अनुकूलन**: बार-बार की जाने वाली कंटेंट सेफ्टी जांच के लिए कैशिंग विचार करें
5. **फ़ाल बैक मैकेनिज़्म**: जब कंटेंट सेफ्टी सेवाएँ अनुपलब्ध हों तब स्पष्ट फ़ाल बैक व्यवहार निर्दिष्ट करें
6. **उपयोगकर्ता प्रतिक्रिया**: जब सामग्री सुरक्षा कारणों से अवरुद्ध हो तो उपयोगकर्ताओं को स्पष्ट प्रतिक्रिया दें
7. **निरंतर सुधार**: उभरते खतरों के आधार पर नियमित रूप से ब्लॉकलिस्ट और पैटर्न अपडेट करें
