# လက်တွေ့အသုံးပြုမှု

[![How to Build, Test, and Deploy MCP Apps with Real Tools and Workflows](../../../translated_images/my/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(ဤသင်ခန်းစာ၏ ဗီဒီယိုကို ကြည့်ရန် ပုံကို နှိပ်ပါ)_

လက်တွေ့အသုံးပြုမှုမှာ Model Context Protocol (MCP) ၏အင်အားကို မြင်သာလာစေသည်။ MCP ၏နောက်ခံသီအိုရီနှင့် ဆောက်လုပ်ပုံကို နားလည်ခြင်းမှာ အရေးကြီးပေမယ့် စိတ်ကူးများကို အသုံးချ၍ အမှန်လွှတ်လပ်သော ပြဿနာများကို ဖြေရှင်းနိုင်သော ဖြေရှင်းချက်များကို တည်ဆောက်၊ စမ်းသပ်နှင့် ဖြန့်ချိတဲ့အခါ အမှန်တကယ်တန်ဖိုးရှိလာသည်။ ဤအခန်းသည် သဘောတရားအသိပညာနှင့် လက်တွေ့ဖွံ့ဖြိုးတိုးတက်မှုတို့အကြား ချိတ်ဆက်ရေးရာတွင် မျှော်မှန်းချက်များကို လမ်းညွှန်ပေးသည်။

သင်သည် အတုအသိဉာဏ်ပေးသူများကို ဖန်တီးခြင်း၊ စီးပွားရေးလုပ်ငန်းစဉ်များတွင် AI ပေါင်းစည်းခြင်း၊ ဒေတာကို ဖြေရှင်းရန် စိတ်ကြိုက်ကိရိယာများ တည်ဆောက်ခြင်းတို့ ကို လုပ်ဆောင်နေပါက MCP သည် ပိုမိုပြောင်းလဲနိုင်သော အခြေခံများကို ပံ့ပိုးပေးသည်။ ၎င်း၏ ဘာသာစကားအမျိုးမျိုးနှင့် ကိုက်ညီသော ဒီဇိုင်းနှင့် လူကြိုက်များသော ပရိုဂရမ်ဘာသာစကားများအတွက် တရားဝင် SDK များသည် ဖြေရှင်းနည်းက ထိရောက်စွာ အလုပ်လုပ်နိုင်ရန် အခွင့်အလမ်းများ ပေးသည်။ ၎င်း SDK များကို အသုံးပြု၍ များစွာသော ပလက်ဖောင်းများနှင့် ပတ်ဝန်းကျင်များတွင် မြန်မြန်ဆန်ဆန် နမူနာပြုခြင်း၊ ပြန်လည်ပြင်ဆင်ခြင်းနှင့် ဂဏန်းတိုးမြှင့်ခြင်းများ ပြုလုပ်နိုင်သည်။

နောက်ဆက်တွဲအပိုင်းများတွင် C#, Java with Spring, TypeScript, JavaScript, Python တို့တွင် MCP ကို ကျက်စား အောင် ပြုလုပ်နည်း များ၊ နမူနာကုတ်များ၊ ဖြန့်ချိမှုမူဝါဒ များ ရှိပြီး MCP စက်ဝိုင်းများကို မည်သို့ စမ်းသပ်၊ ဒီဘပ် အလုပ်လုပ်ကြောင်း၊ API များကို စီမံခန့်ခွဲခြင်းနှင့် Azure ကို အသုံးပြုပြီး ကောင်းမွန်စွာ ဖြန့်ချိ ရန် လမ်းညွှန်ချက်များပါရှိသည်။ ဤ လက်တွေ့ အရင်းအမြစ်များမှာ သင်၏ သင်ယူမှုကို မြန်ဆန်စေပြီး MCP အခြေပြု အသုံးပြုသူများကို ယုံကြည်စိတ်ချစွာ ဖန်တီးနိုင်ရန် အထောက်အကူပြုသည်။

## အကျဉ်းချုပ်

ဤသင်ခန်းစာသည် MCP ကို ပရိုဂရမ်ဘာသာစကားအမျိုးမျိုးတွင် လက်တွေ့အသုံးပြု နည်းများ အပေါ် ဂရုစိုက်ပြသသည်။ MCP SDK များကို C#, Java with Spring, TypeScript, JavaScript, Python တို့တွင် အသုံးပြု၍ သက်ဝင်လှုပ်ရှားသော အက်ပ်များ ဖန်တီးခြင်း၊ MCP စက်ဝိုင်းများကို ဒီဘပ်နှင့် စမ်းသပ်ခြင်း၊ အသုံးပြုနိုင်သော ဆာဗာအင်္ဂါရပ်များ၊ prompt များနှင့် ကိရိယာများ ဖန်တီးပုံကို စူးစမ်းလေ့လာပါမည်။

## သင်ယူရမည့် ရည်မှန်းချက်များ

ဤသင်ခန်းစာ ပြီးဆုံးချိန်တွင် သင်သည် -

- ပရိုဂရမ်ဘာသာစကားအမျိုးမျိုးတွင် တရားဝင် SDK များကို အသုံးပြုပြီး MCP ဖြေရှင်းနည်းများ တည်ဆောက်နိုင်မည်။
- MCP ဆာဗာများကို စနစ်တကျ ဒီဘပ်နှင့် စမ်းသပ်နိုင်မည်။
- ဆာဗာဖန်တီးမှု (Resources, Prompts, Tools) များ အသုံးပြုနိုင်မည်။
- ခက်ခဲသော လုပ်ငန်းစဉ်များအတွက် MCP အလုပ်စဉ်များကို အထိရောက်ဆုံး ဒီဇိုင်းဆွဲနိုင်မည်။
- ပြုလုပ်ထားသော MCP ဖြေရှင်းနည်းများကို စွမ်းဆောင်ရည်နှင့် ယုံကြည်စိတ်ချ ရှိမှုအတွက် အထိရောက်ဆုံး ထိန်းသိမ်းနိုင်မည်။

## အဓိကဆာဗာ အင်္ဂါရပ်များ

MCP ဆာဗာများတွင် အောက်ပါ အင်္ဂါရပ်များတွင် မည်သည့်ပေါင်းစပ်မှုမဆို လုပ်ဆောင်နိုင်သည် -

### Resources

အသုံးပြုသူ သို့မဟုတ် AI မော်ဒယ် အသုံးပြုရန် context နှင့် ဒေတာများကို ပေးသည်။

- စာရွက်စာတမ်း စုစည်းခန်းများ
- ဗဟုသုတအခြေခံများ
- ဖွဲ့စည်းထားသော ဒေတာ အရင်းအမြစ်များ
- ဖိုင်စနစ်များ

### Prompts

အသုံးပြုသူများအတွက် ပုံစံပြသမှု့ မက်ဆေ့ချ်များနှင့်လုပ်ဆောင်ချက်ခြင်းများ -

- ကြိုတင်သတ်မှတ်ထားသော မေးမြန်းမှု ပုံစံများ
- လမ်းညွှန်ပြသမှု ပုံစံများ
- အထူးပြု အပြောအဆို ဖွဲ့စည်းမှုများ

### Tools

AI မော်ဒယ် အကောင်အထည်ဖော်ရန် အသုံးပြု ကိရိယာများ -

- ဒေတာကို ဆက်လက်စီမံရန် စက်တွေ့ရေးမှုများ
- ပြင်ပ API ပေါင်းစည်းခြင်းများ
- ကွန်ပျူတာလုပ်ဆောင်ချက်များ
- ရှာဖွေမှု လုပ်ဆောင်ချက်များ

## နမူနာ အကောင်အထည်ဖော်မှု - C# Implementation

တရားဝင် C# SDK repository တွင် MCP ၏ မတူညီသော အစိတ်အပိုင်းများကို ပြသသည့် နမူနာများ ပါဝင်သည် -

- **အခြေခံ MCP Client** - MCP client ဖန်တီးနည်းနှင့် ကိရိယာများ ခေါ်ယူနည်း ဥပမာ လွယ်ကူသော ပုံစံ
- **အခြေခံ MCP Server** - အခြေခံ ကိရိယာ မှတ်ပုံတင်မှုနှင့် အနည်းငယ်ဆာဗာ ဖန်တီးမှု
- **တိုးတက်သော MCP Server** - အပြည့်အစုံရှိ ကိရိယာ မှတ်ပုံတင်ခြင်း၊ အတည်ပြုခြင်းနှင့် အမှားကိုင်တွယ်ခြင်းများ ပါဝင်သော ဆာဗာ
- **ASP.NET ပေါင်းစည်းမှု** - ASP.NET Core နှင့် ပေါင်းစည်းခြင်း အကြောင်း ဥပမာများ
- **ကိရိယာ အသုံးပြုမှု နည်းပညာ များ** - အတိုးအကျယ်အလိုက် ကိရိယာများ အသုံးပြုခြင်း နမူနာ အမျိုးမျိုး

MCP C# SDK သည် မကြာမီ ကြိုတင်ပြသနေပြီး API များ လည်း ပြောင်းလဲနိုင်သည်။ SDK တိုးတက်မှုနှင့်အမျှ ဘလော့ခ်ကို ဆက်လက်ပြင်ဆင်သွားမည်။

### အဓိက အင်္ဂါရပ်များ

- [C# MCP Nuget ModelContextProtocol](https://www.nuget.org/packages/ModelContextProtocol)
- သင့်ရဲ့ [ပထမဆုံး MCP ဆာဗာ ဖန်တီးခြင်း](https://devblogs.microsoft.com/dotnet/build-a-model-context-protocol-mcp-server-in-csharp/)။

အပြည့်အစုံ C# implementation နမူနာများအတွက် [တရားဝင် C# SDK နမူနာ repository](https://github.com/modelcontextprotocol/csharp-sdk) ကို ကြည့်ရှုပါ။

## နမူနာ အကောင်အထည်ဖော်မှု - Java with Spring Implementation

Java with Spring SDK သည် စက်ရုပ် MCP အကောင်အထည်ဖော်မှု အတွက် အင်တပ်ပါက အင်္ဂါရပ် နှင့် ပံ့ပိုးမှုများစွာ ပါဝင်သည်။

### အဓိက အင်္ဂါရပ်များ

- Spring Framework ပေါင်းစည်းမှု
- သေချာသော Type Safety
- Reactive programming ပံ့ပိုးမှု
- အပြည့်အစုံ error ကိုင်တွယ်မှု

Java with Spring implementation အပြည့်အစုံအတွက် samples ဖိုလ်ဒါရှိ [Java with Spring sample](samples/java/containerapp/README.md) ကို ကြည့်ပါ။

- Express နှင့် အခြား Frameworks တွင် လွယ်ကူစွာ ပေါင်းစည်းနိုင်မှု
- Streaming အတွက် WebSocket ပံ့ပိုးမှု

JavaScript implementation အပြည့်အစုံ နမူနာအတွက် samples ဖိုလ်ဒါရှိ [JavaScript sample](samples/javascript/README.md) ကို ကြည့်ပါ။

### အဓိက အင်္ဂါရပ်များ

- FastAPI ပေါင်းစည်းမှု
- ရိုးရှင်းသော ကိရိယာ မှတ်ပုံတင်ခြင်း
- လူကြိုက်များသော ML စာကြည့်တိုက်များနှင့် တွဲဖက် ပေါင်းစည်းမှု

Python implementation အပြည့်အစုံ နမူနာအတွက် samples ဖိုလ်ဒါရှိ [Python sample](samples/python/README.md) ကို ကြည့်ပါ။

## API စီမံခန့်ခွဲခြင်း

Azure API Management သည် MCP ဆာဗာများကို ဘယ်လိုလုံခြုံစွာ ထိန်းသိမ်းရမည်ဆိုသည်ကို ဖြေရှင်းပေးနိုင်သော ကောင်းမွန်သော ရွေးချယ်မှုတစ်ခုဖြစ်သည်။ MCP ဆာဗာ များရှေ့တွင် Azure API Management များကို တပ်ဆင်ထားခြင်းဖြင့် -

- စည်းကမ်းဖြင့် အသုံးချသူများကို ကန့်သတ်ခြင်း (rate limiting)
- token များ စီမံခြင်း
- လုပ်ဆောင်မှု များ စစ်ဆေးခြင်း (monitoring)
- load balancing
- လုံခြုံရေး

စသည့် အင်္ဂါရပ်များကို ကိုင်တွယ်ပေးနိုင်သည်။

## Remote MCP Server ကို Azure တွင် ဖြန့်ချိခြင်း

ယခင်တွင် ဥပမာပြထားသည့် ပရိုဂရမ်ကို တိုက်ရိုက်ဖြန့်ချိနိုင်မည် ဟု ကြည့်ကြပါစို့။

1. Repository ကို Clone လုပ်ပါ

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

2. `Microsoft.App` resource provider ကို မှတ်ပုံတင်ပါ။

   - Azure CLI သုံးပါက `az provider register --namespace Microsoft.App --wait` ကို လုပ်ဆောင်ပါ။
   - Azure PowerShell သုံးပါက `Register-AzResourceProvider -ProviderNamespace Microsoft.App` ကို လုပ်ဆောင်ပြီးနောက် `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` ကို တစ်ခါတစ်ရံ စစ်ဆေးပါ။

3. [azd](https://aka.ms/azd) ကွန်မန်းကို အသုံးပြု၍ API Management service, function app(ကုတ်နှင့်အတူ) နှင့် Azure resource အားလုံးကို provision လုပ်ပါ။

    ```shell
    azd up
    ```

    ဤကွန်မန်းသည် Azure ပလက်ဖောင်းပေါ်တွင် cloud resource များအားလုံးကို ဖြန့်ချိမည် ဖြစ်သည်။

### MCP Inspector ဖြင့် ဆာဗာ စမ်းသပ်ခြင်း

1. **အသစ်သော terminal မျက်နှာပြင်** ဖြင့် MCP Inspector ကို ထည့်သွင်းပြီး လည်ပတ်ပါ။

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    အောက်ပါအတိုင်း UI တစ်ခုကို မြင်ရပါမည် -

    ![Connect to Node inspector](../../../translated_images/my/connect.141db0b2bd05f096.webp)

2. MCP Inspector web app URL (ဥပမာ [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources)) ကို CTRL ဖြင့် နှိပ်၍ ဖွင့်ပါ။
3. Transport type ကို `SSE` သို့ ပြောင်းပါ။
4. အဲ့ဒီအချိန် `azd up` ထွက်လာသော အတည်ပြု API Management SSE endpoint URL ကို ထည့်၍ **Connect** နိုင်ပါပြီ။

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

5. **List Tools** ကို နှိပ်ပါ။ ကိရိယာတစ်ခုကို ရွေးပြီး **Run Tool** ကို နှိပ်ပါ။

အဆင့်များအားလုံး ပြီးစီးပါက MCP ဆာဗာနှင့် ချိတ်ဆက်ပြီး ကိရိယာကို ခေါ်ယူနိုင်ပြီဖြစ်ပါသည်။

## Azure အတွက် MCP ဆာဗာများ

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet):  သည် Azure Functions နှင့် Python, C# .NET, Node/TypeScript ကို အသုံးပြု၍ စိတ်ကြိုက် remote MCP server များကို အလျင်မြန်စွာ ဖန်တီး၊ ဖြန့်ချိရ အတွက် template များကို ထောက်ပံ့သည်။

ဤ နမူနာများသည် Developer များကို အောက်ပါအရာများ ပြုလုပ်ရန် အတွက် အကောင်းဆုံးဖြေရှင်းချက်တစ်ခု အဖြစ် တင်ပြသည်။

- ဒေသတွင်း ဖန်တီးပြီး စမ်းသပ်ရန် MCP ဆာဗာပြုလုပ်ခြင်း
- azd up ကွန်မန်းတစ်ခုဖြင့် Cloud သို့ ဖြန့်ချိရာမှာ လွယ်ကူခြင်း
- VS Code Copilot agent mode နှင့် MCP Inspector tools များကဲ့သို့ ကုဒ်ဖိုင်များမှ ဆာဗာသို့ ချိတ်ဆက်နိုင်ခြင်း

### အဓိက အင်္ဂါရပ်များ

- ဒီဇိုင်းဖြင့် လုံခြုံရေး: MCP ဆာဗာကို key နှင့် HTTPS ဖြင့် လုံခြုံစွာ ထိန်းသိမ်းထားသည်
- အတည်ပြုမှု ရွေးချယ်စရာများ: Built-in authentication သို့မဟုတ် API Management အသုံးပြုပြီး OAuth ပံ့ပိုးမှု
- ကွန်ရက် သီးခြားမှု: Azure Virtual Networks (VNET) ဖြင့် ကွန်ရက် သီးခြားမှု ပံ့ပိုးခြင်း
- Serverless architecture: Azure Functions ကို အသုံးပြုသော scalable၊ အဖြစ်အပျက်အခြေချရှိသော လည်ပတ်မှု
- ဒေသတွင်း ဖွံ့ဖြိုးမှု: ဒေသတွင်း လုပ်ငန်းဖွံ့ဖြိုးမှုနှင့် ဒီဘပ် ထောက်ပံ့မှု
- လွယ်ကူသော ဖြန့်ချိမှု: Azure သို့ ချောမွေ့သော ဖြန့်ချိမှု လုပ်ငန်းစဉ်

Repositories တွင် လိုအပ်သော ပုံစံဖိုင်များ၊ ကုတ်အရင်းအမြစ်များနှင့် အင်ဖရာစက်ချက်များ ပါရှိပြီး အမြန်ဆုံး ပြုလုပ်နိုင်သော MCP ဆာဗာ implementation များကို စတင်နိုင်ပါသည်။

- [Azure Remote MCP Functions Python](https://github.com/Azure-Samples/remote-mcp-functions-python) - Azure Functions နှင့် Python အသုံးပြု MCP implementation နမူနာ
- [Azure Remote MCP Functions .NET](https://github.com/Azure-Samples/remote-mcp-functions-dotnet) - Azure Functions နှင့် C# .NET အသုံးပြု MCP implementation နမူနာ
- [Azure Remote MCP Functions Node/Typescript](https://github.com/Azure-Samples/remote-mcp-functions-typescript) - Azure Functions နှင့် Node/TypeScript အသုံးပြု MCP implementation နမူနာ

## အဓိက အချက်များ

- MCP SDK များသည် တစ်ခုချင်းသော ဘာသာစကားအတွက် သက်ဆိုင်ရာ ကိရိယာများ ပံ့ပိုးပေးသည်။
- ဒီဘပ်နှင့် စမ်းသပ်ခြင်းလုပ်ငန်းစဉ်သည် ယုံကြည်စိတ်ချရသော MCP ပရိုဂရမ်များအတွက် အရေးကြီးသည်။
- ပြန်လည်အသုံးပြုနိုင်သော prompt များသည် AI နှင့် ကောင်းမွန်သည့် ပါ၀င်ဆက်ဆံမှုကို အထောက်အကူဖြစ်စေသည်။
- ကောင်းစွာ စီမံထားသော အလုပ်စဉ်များက ခက်ခဲသော လုပ်ငန်းများကို များစွာသော ကိရိယာများဖြင့် ရှင်းလင်းတက်နိုင်သည်။
- MCP ဖြေရှင်းနည်းတည်ဆောက်မှု၌ လုံခြုံရေး၊ စွမ်းဆောင်ရည်နှင့် အမှားကိုင်တွယ်မှုကို စဉ်းစားရမည်။

## လေ့ကျင့်ခန်း

မိမိလုပ်ကိုင်ရာ ဒေသအတွင်း မှန်ကန်သော ပြဿနာတစ်ခုကို ဖြေရှင်းနိုင်သော MCP workflow တည်ဆောက်ပါ -

1. ဤပြဿနာကို ဖြေရှင်းရန် ဘာသာရပ်ကြိုက် ၃-၄ ချက်ကို သတ်မှတ်ပါ။
2. ကိရိယာများ များကို ပြသသော workflow diagram တစ်ခုကို ဖန်တီးပါ။
3. သင်နှစ်သက်သော ဘာသာစကားဖြင့် ကိရိယာတစ်ခု၏ အခြေခံ ဗားရှင်းကို အကောင်အထည်ဖော်ပါ။
4. မော်ဒယ်သည် သင်၏ ကိရိယာကို ထိရောက်စွာ အသုံးပြုနိုင်ရန် prompt template တစ်ခု ဖန်တီးပါ။
