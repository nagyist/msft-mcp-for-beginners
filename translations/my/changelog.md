# ပြောင်းလဲမှုမှတ်တမ်း: MCP for Beginners သင်ရိုးညွှန်းတမ်း

ဤစာတမ်းသည် Model Context Protocol (MCP) for Beginners သင်ရိုးညွှန်းတမ်းတွင် ပြုလုပ်ခဲ့သော အရေးကြီးပြောင်းလဲမှုများအား အမှတ်တရ အဖြစ် မှတ်တမ်းတင်ထားသည်။ ပြောင်းလဲမှုများကို အချိန်စဉ်ကျရောက်အပြီးမှ စီထားသည် (နောက်ဆုံးပြောင်းလဲမှုများ ပထမဆံုး)။

## ၂၀၂၆ ဇန်နဝါရီ ၂၈ ရက်

### MCP Specification 2025-11-25 ကိုလိုက်နာမှုပြန်လည်သုံးသပ်ခြင်း

#### အဓိက အယူအဆ တိုးတက်မှု (01-CoreConcepts/)
- **Client Primitive အသစ် - Roots**: Roots client primitive အကြောင်းအရာ အသေးစိတ် ထည့်သွင်း; ဆာဗာများသည် ဖိုင်စနစ်နယ်မြေများနှင့် ခွင့်ပြုချက်များကို နားလည်နိုင်ရန်
- **ကိရိယာ အမှတ်အသား အတိပျူစီ**: tool ဖော်ပြချက်များ (readOnlyHint, destructiveHint) ကို ထည့်သွင်းကူညီရန်
- **Sampling တွင် ကိရိယာခေါ်ဆိုမှု**: Sampling စာရွက်စာတမ်းတွင် tools နှင့် toolChoice ပရမားတာများ ကို ဦးစားပေး ထည့်သွင်း
- **URL အခြေခံ mode ဖော်ပြချက်**: ဆာဗာမှ တစ်ဆင့် web အပြင်ဘက်လုပ်ငန်းဆောင်တာများ ဖော်ပြချက် ထည့်သွင်း
- **Tasks (စမ်းသပ်မှု)**: ဝန်ဆောင်မှု တာဝန်ထမ်းဆောင်မှု အာမခံချက်များနှင့် ရလဒ် deferred ပြန်ထုတ်ခြင်း အတွက် experimental Tasks  အပိုင်း အသစ် ထည့်သွင်း
- **ရန်ပုံငှားများ ထောက်ပံ့မှု**: ကိရိယာများ၊ အရင်းအမြစ်များ၊ သုံးရေးသားမှုကမ်းလှမ်းချက်များတွင် icon များ ထည့်သွင်းနိုင်မည်

#### စာရွက်စာတမ်း အပ်ဒိတ်
- **README.md**: MCP Specification 2025-11-25 ဗားရှင်း နှင့် ရက်စွဲ-အပေါ် အခြေခံ ဗားရှင်း အကြောင်း ဖော်ပြချက် ထည့်
- **study_guide.md**: Core Concepts အပိုင်းတွင် Tasks နှင့် Tool Annotations ပါဝင်မှု ရေးသားပြီး curriculum map ကို အပ်ဒိတ်လုပ်; စာရွက်စာတမ်း ရက်စွဲ ပြန်စစ်

#### Specification လိုက်နာမှု အတည်ပြုခြင်း
- **Protocol ဗားရှင်း**: အသစ်မှတ်တမ်းများအား MCP Specification 2025-11-25 နှင့် ကိုက်ညီကြောင်း သက်သေ
- **ဖွဲ့စည်းပုံ ထဲတွဲ**: နှစ်စိတ် ဖွဲ့စည်းပုံ (Data Layer + Transport Layer) လူသိများမှု အတည်ပြု
- **Primitives စာရွက်စာတမ်းများ**: ဆာဗာ primitives (Resources, Prompts, Tools) နှင့် client primitives (Sampling, Elicitation, Logging, Roots) ကို အတည်ပြု
- **သယ်ယူပို့ဆောင်မှု မက်ကာနစ်များ**: STDIO နှင့် Streamable HTTP ပို့ဆောင်မှု စာရွက်စာတမ်းမှန်ကန်မှု စစ်ဆေး
- **လုံခြုံရေး လမ်းညွှန်ချက်များ**: လက်ရှိ MCP လုံခြုံရေးဆိုင်းငံ့ ချုပ်တားချက်များနှင့် ကိုက်ညီမှု အတည်ပြု

#### အဓိက MCP 2025-11-25 Features ဖော်ပြချက်များ
- **OpenID Connect အတည်ပြုစနစ်**: OIDC အောက် auth ဆာဗာ ရှာဖွေမှု
- **OAuth Client ID Metadata စာရွက်စာတမ်းများ**: client စာရင်းသွင်းမှု စနစ် အကြံပြုချက်
- **JSON Schema 2020-12**: MCP schema အဓိက ဘာသာစကား
- **SDK Tiering System**: SDK လုပ်ဆောင်ချက်နှင့် ပြုပြင်ထိန်းသိမ်းမှု လိုအပ်ချက်များ အတည်ပြု
- **အုပ်ချုပ်ရေး ဖွဲ့စည်းပုံ**: MCP အုပ်ချုပ်ရေး၌ အလုပ်အဖွဲ့များနှင့် စိတ်ပါဝင်စားသူ အုပ်စုများ ဖော်ပြထားခြင်း

### လုံခြုံရေး စာရွက်စာတမ်း အပ်ဒိတ် - MCP Specification 2025-11-25

#### MCP လုံခြုံရေး အကောင်းဆုံးကျင့်သုံးနည်း (02-Security/mcp-best-practices.md) - Specification ဗားရှင်း အပ်ဒိတ်
- **Protocol ဗားရှင်း အပ်ဒိတ်**: နောက်ဆုံး MCP Specification 2025-11-25 (၂၀၂၅ နိုဝင်ဘာ ၂၅ ထုတ်) ကို ကိုးကား
  - ၂၀၂၅-၀၆-၁၈ မှ ၂၀၂၅-၁၁-၂၅ သို့ specification ဗားရှင်း ကိုးကားချက်အားလုံး ပြောင်းလဲ
  - စာရွက်စာတမ်းရက်စွဲကို ၂၀၂၅ မှ ၈ ရက် ၁၈ မှ ၁၂ ရက် ၁၈ သို့ ပြောင်းလဲ
  - specification URL များသည် လက်ရှိစာရွက်စာတမ်းသို့ ချိန်ညှိထားမှုကို စစ်ဆေး
- **အကြောင်းအရာအတည်ပြုချက်**: လုံခြုံရေးအကောင်းဆုံးကျင့်သုံးမှုများအား နောက်ဆုံးစံနှုန်းအတိုင်း စစ်ဆေး
  - **Microsoft Security ဆောင်ရွက်ချက်များ**: Prompt Shields (ယခင် “Jailbreak risk detection”), Azure Content Safety, Microsoft Entra ID နှင့် Azure Key Vault ဆိုင်ရာ အဆိုအရ စစ်ဆေးပြီး
  - **OAuth 2.1 လုံခြုံရေး**: နောက်ဆုံး OAuth  လုံခြုံမှု အကောင်းဆုံးကျင့်သုံးမှုများနှင့် ကိုက်ညီမှု အတည်ပြု
  - **OWASP စံနှုန်းများ**: LLMs အတွက် OWASP Top 10 ကို လက်ရှိဖြစ်မှုဖြစ်ကြောင်း သက်သေပြ
  - **Azure ဝန်ဆောင်မှုများ**: Microsoft Azure လက်ရှိ စာရွက်စာတမ်း လင့်ခ်များနှင့် အကောင်းဆုံးကျင့်သုံးမှုများ
- **စံညွှန်းလိုက်နာမှု**: ကိုးကားထားသော လုံခြုံရေးစံနှုန်းများ အားလုံးသည် လက်ရှိဖြစ်မှုဖြစ်ကြောင်း သက်သေပြ
  - NIST AI Risk Management Framework
  - ISO 27001:2022
  - OAuth 2.1 အကောင်းဆုံးလုံခြုံရေးကျင့်သုံးမှု
  - Azure လုံခြုံမှုနှင့် အသိအမှတ်ပြု ဖွဲ့စည်းပုံများ
- **အကောင်အထည်ဖော် အရင်းအမြစ်များ**: အကောင်အထည်ဖော်လမ်းညွှန်များနှင့် အရင်းအမြစ်များ စစ်ဆေးပြီး
  - Azure API Management အတည်ပြုမူ ပုံစံများ
  - Microsoft Entra ID ပေါင်းစပ်ခြင်း လမ်းညွှန်များ
  - Azure Key Vault သတ္တိမရှိသော စီမံခန့်ခွဲမှု
  - DevSecOps pipeline များနှင့် စောင့်ကြပ်မှု ဖြေရှင်းမှု

### စာရွက်စာတမ်း အရည်အသွေး အာမခံချက်
- **Specification ကိုက်ညီမှု**: MCP လုံခြုံရေး လိုအပ်ချက် (MUST / MUST NOT) အားလုံးသည် နောက်ဆုံး specification နှင့် ကိုက်ညီမှုရှိစွာ သေချာစစ်ဆေး
- **အရင်းအမြစ် အသစ်ဆုံး**: Microsoft documentation, လုံခြုံရေးစံနှုန်းများ၊ အကောင်အထည်ဖော် လမ်းညွှန်များ အားလုံးသည် အားထားစိတ်ချရမှု နှင့် လက်ရှိ ဖြစ်တည်မှုရှိ
- **အကောင်းဆုံးကျင့်သုံးမှု ပါဝင်မှု**: အတည်ပြု၊ ခွင့်ပြုမှု၊ AI ပေါ်လာသော မတရားမှုများ၊ ပံ့ပိုးကွင်းလုံခြုံမှု၊ စက်မှုလုပ်ငန်းပုံစံများ အားလုံး ပါဝင်မှု စုံရိပ်သည်

## ၂၀၂၅ တောက်တိုက် ၆ ရက်

### Getting Started အပိုင်း တိုးချဲ့မှု – မြင့်မားသော ဆာဗာ အသုံးပြုမှု နှင့် ရိုးရှင်း အတည်ပြုကိရိယာ

#### မြင့်မားသော ဆာဗာ အသုံးပြုမှု (03-GettingStarted/10-advanced)
- **အခန်းသစ် ထည့်သွင်း**: MCP ဆာဗာ အသုံးပြုမှု မြင့်မားသော အကြောင်းအရာ လမ်းညွှန် ရေးသားခြင်း၊ ရိုးရာအဆင့်နှင့် အောက်ခံဆာဗာ ဖွဲ့စည်းပုံများ တိုင်းပြခြင်း
  - **ရိုးရာ နှင့် အောက်ခံ ဆာဗာ**: Python နှင့် TypeScript အသုံးပြုပြီး  နှိုင်းယှဉ် နမူနာများ ဖော်ပြ
  - **Handler-အပေါ် မူတည်ချက် ဒီဇိုင်း**: scalable၊ အလွယ်တကူ ပြင်ဆင်နိုင်စေရန် ကိရိယာ၊ အရင်းအမြစ်၊ prompt များ ဆောင်ရွက်မှု အကြောင်းရှင်းလင်း
  - **လက်တွေ့ pattern များ**: မြင့်မားသော feature နှင့် ဖွဲ့စည်းပုံများတွင် အောက်ခံဆာဗာများ အသုံးဖို့ အသုံးဝင်သော နမူနာများ

#### ရိုးရှင်းအတည်ပြုကိရိယာ (03-GettingStarted/11-simple-auth)
- **အခန်းသစ် ထည့်သွင်း**: MCP ဆာဗာများတွင် ရိုးရှင်းသော အတည်ပြုကိရိယာ ဖော်ဆောင်နည်း လမ်းညွှန်သစ်
  - **Auth အယူအဆ**: အတည်ပြုခြင်းနှင့် ခွင့်ပြုခြင်း နှိယာမများ ရှင်းလင်းခြင်း နှင့် လက်မှတ်ကိုင်တွယ်မှု
  - **အခြေခံ Auth အကောင်အထည်ဖော်မှု**: Python (Starlette) နှင့် TypeScript (Express) ထည့်သွင်းဖြင့် middleware အပြင် advantage pattern များ
  - **မြင့်မားသော လုံခြုံရေးသို့ တိုးတက်ရန် လမ်းညွှန်ချက်**: ရိုးရှင်း Auth မှ OAuth 2.1 နှင့် RBAC သို့ မှီတက်ရန် လမ်းညွှန်ချက် နှင့် မြင့်မားသော လုံခြုံရေး module များ ကိုညွှန်ပြ

ဤထည့်သွင်းမှုများသည် MCP ဆာဗာများအား ပိုမိုကြံ့ခိုင်၊ လုံခြုံသော နှင့် ပြင်ဆင်နိုင်မှု ကောင်းမွန်သော အကောင်အထည်ဖော်ခြင်းနှင့် အခြေခံအယူအဆများကို မြင့်မားသော ထုတ်လုပ်မှု pattern များနှင့် တက်လှမ်းရန် လက်တွေ့ လမ်းညွှန်ချက်များကို ပံ့ပိုးပေးသည်။

  - **တိုးတက်သော Features များ**: Row Level Security (RLS), ဆင်တမေ့တစ်ရှာ, များစွာအသုံးပြုသူဒေတာ လက်ဝယ်ခံခြင်း, ဗက်တာ embedding များ, တိုက်ရိုက် စောင့်ကြည့်ခြင်း

### တိုးတက်သော ခေါင်းစဉ်များ တိုးချဲ့မှု - စိတ်ကြိုက် ပို့ဆောင်မှုများနှင့် Context အင်ဂျင်နီယာရေး

#### MCP စိတ်ကြိုက် ပို့ဆောင်မှုများ (05-AdvancedTopics/mcp-transport/) - အဆင့်မြင့် အကောင်အထည်ဖော်လမ်းညွှန် အသစ်
- **README.md**: စိတ်ကြိုက် MCP ပို့ဆောင်မှု ထိန်းချုပ်မှု စနစ် အကောင်အထည်ဖော်လမ်းညွှန် အပြည့်အစုံ
  - **Azure Event Grid ပို့ဆောင်မှု**: ဆာဗာလစ်လက်လှမ်းမီဖြစ်သော အမှတ်အသားအခြေပြု ပို့ဆောင်မှု အကောင်အထည်ဖော်မှု
    - C#, TypeScript, Python နမူနာများနှင့် Azure Functions ပေါင်းစည်းမှု
    - MCP ဖြန့်ဝေရေး ချဲ့ထွင်မှု အတွက် ပွင့်လင်းသော ပျံနှံ့စေသော event-driven ပုံစံများ
    - Webhook လက်ခံသူများနှင့် push-based မက်ဆေ့ခ်ျ က xử lýမှုများ
  - **Azure Event Hubs ပို့ဆောင်မှု**: အမြင့်သံချိုသံသာ streaming ပို့ဆောင်မှုအကောင်အထည်ဖော်မှု
    - အချိန်နုတ်နည်း လျော့နည်းမှုနှင့် တိုက်ရိုက် ဖြန့်ဝေရေး ဓါတ်ဆောင်နိုင်စွမ်း
    - Partitioning မဟာဗျူဟာများနှင့် checkpoint စီမံခန့်ခွဲမှု
    - မက်ဆေ့ခ်ျ အစုံကောက်ယူခြင်းနှင့် ယာယီတိုးတက်မှု
  - **စီးပွားရေး ပေါင်းစပ်မှု ပုံစံများ**: ထုတ်လုပ်မှုအဆင်သင့် စက်ဆင်တင် အထူးအစီအစဉ်များ
    - Azure Functions အမျိုးမျိုးအား ဖြန့်ဝေသော MCP သုံးဆာဗာ လည်ပတ်မှု
    - ပို့ဆောင်မှုမျိုးစုံပေါင်းစည်းထားသော ပုံစံများ
    - မက်ဆေ့ခ်ျ တည်ငြိမ်မှု၊ ယုံကြည်စိတ်ချမှုနှင့် အမှားကြောင်း စီမံခန့်ခွဲမှု မဟာဗျူဟာများ
  - **လုံခြုံရေးနှင့် ကြည့်ရှုမှု**: Azure Key Vault ပေါင်းစည်းမှု နှင့် ကြည်လင်မြင်သာရေး ပုံစံများ
    - Managed identity မှတ်ပုံတင်မှုနှင့် အနည်းဆုံးခွင့်ပြုချက်များ
    - Application Insights telemetry နှင့် ချဲ့ထွင်နိုင်သော ဖော်ပြချက်များ
    - Circuit breakers နှင့် ချို့ယွင်းချက်ခံနိုင်ခြင်းပုံစံများ
  - **စမ်းသပ်မှု စနစ်များ**: စိတ်ကြိုက်ပို့ဆောင်မှုများအတွက် စမ်းသပ်မှု မဟာဗျူဟာ အပြည့်အစုံ
    - Unit test များအတွက် test doubles နှင့် mocking frameworks အသုံးပြုမှု
    - Azure Test Containers ဖြင့် integration စမ်းသပ်မှု
    - စွမ်းဆောင်ရည်နှင့်အလေးပေးလုပ်ဆောင်မှု စမ်းသပ်ချက်များ

#### Context အင်ဂျင်နီယာရေး (05-AdvancedTopics/mcp-contextengineering/) - စတင်ပေါ်ထွန်းလာသော AI နယ်ပယ်
- **README.md**: Context engineering ကို အသစ်စတင် ပေါ်ထွန်းလာသောနယ်ပယ်အဖြစ် အပြည့်အဝ ရှာဖွေခွင့်ပြုချက်
  - **အဓိကအခြေခံ နည်းလမ်းများ**: context ပြန်ဝေစုခြင်း, ဆုံးဖြတ်ချက် ပြုလုပ်မှု အသိပညာ, context window စီမံခန့်ခွဲမှု
  - **MCP Protocol နှင့် ကိုက်ညီမှု**: MCP ဒီဇိုင်းသည် context engineering ရဲ့ စိန်ခေါ်မှုများကို မည်သို့ ဖြေရှင်းသည်
    - context window ကန့်သတ်ချက်များနှင့် တိုးတက်စေ၍ တင်ပို့မှု မဟာဗျူဟာများ
    - သက်ဆိုင်မှု သတ်မှတ်ခြင်းနှင့် dynamic context ရယူခြင်း
    - Multi-modal context ကိုင်တွယ်မှု နှင့် လုံခြုံရေးစနစ်များ
  - **အကောင်အထည်ဖော် နည်းလမ်းများ**: တစ်ခုတည်း ချောထားသည့် architecture နှင့် များစွာ agent architecture များ
    - context ကို ဖြတ်တောက်ခြင်းနှင့် ဦးစားပေးစနစ်များ
    - တိုးတက်စေသော context တင်ပို့မှုနှင့် ဖိအားနည်းစနစ်
    - အလွှာခွဲ context နည်းလမ်းများနှင့် အဆင်သင့်ရှာဖွေမှုတိုးတက်မှု
  - **တိုင်းတာမှုပုံစံ**: context ထိရောက်မှု အကဲဖြတ်ရန် စတင်မိတ်ဆက်သောတိုင်းတာမှုများ
    - input ထိရောက်မှု, စွမ်းဆောင်ရည်, အရည်အသွေးနှင့် အသုံးပြုသူ အတွေ့အကြုံ စဉ်းစားချက်များ
    - experimental နည်းလမ်းများနှင့် context တိုးတက်မှုပေးနိုင်ရေး
    - မအောင်မြင်မှုများအား ချုပ်ခြား စနစ်တကျ တိုးတက်စေရေး နည်းလမ်းများ

## ၂၀၂၅ ခုနှစ် စက်တင်ဘာလ ၁၈ ရက်

### စာရွက်စာတမ်း အပြည့်အစုံ ပြုပြင်မှု - MCP 2025-06-18 စံသတ်မှတ်ချက်

#### MCP လုံခြုံရေး အကောင်းဆုံးလမ်းညွှန်ချက်များ (02-Security/) - အပြည့်အဝ ပြုပြင်မွမ်းမံမှု
- **MCP-SECURITY-BEST-PRACTICES-2025.md**: MCP Specification 2025-06-18 ကိုက်ညီစွာ ပြန်ရေးသားခြင်း
  - **လိုအပ်ချက် တင်းကျပ်မှု**: တရားဝင် မူဝါဒများအရ MUST/MUST NOT လိုအပ်ချက်များ ကြည်လင်သောအမှတ်အသားများဖြင့် ထည့်သွင်းထားသည်
  - **လုံခြုံရေး အခြေခံ လေ့လာချက် ၁၂ ချက်**: ၁၅ ချက်စာရင်းမှ ခွဲခြား၍ လုံခြုံရေးနယ်ပယ်အလိုက် ပြင်ဆင်တည်းဖြတ်ထားသည်
    - Token security နှင့် အတည်ပြုမှု (external identity provider ပေါင်းစည်းမှု)
    - Session management နှင့် ပို့ဆောင်မှု လုံခြုံရေး (cryptographic လိုအပ်ချက်များ)
    - AI-specific အန္တရာယ်ကာကွယ်မှု (Microsoft Prompt Shields ပေါင်းစည်းမှု)
    - Access control နှင့် ခွင့်ပြုချက်များ (principle of least privilege)
    - အကြောင်းအရာလုံခြုံရေးနှင့် ကြည့်ရှုစောင့်ကြည့်ရေး (Azure Content Safety ပေါင်းစည်းမှု)
    - ကုန်ကြမ်းမှထုတ်လုပ်မှု လုံခြုံရေး (component စစ်ဆေးများ)
    - OAuth လုံခြုံရေးနှင့် Confused Deputy ကာကွယ်ခြင်း (PKCE အကောင်အထည်ဖော်မှု)
    - ဖြစ်ရပ် တုံ့ပြန်မှုနှင့် ပြန်လည် နိုင်ရေး (အလိုအလျောက် စနစ်များ)
    - ဥပဒေအရလိုက်နာမှုနှင့် အုပ်ချုပ်မှု (regulatory alignment)
    - တိုးတက်သော လုံခြုံရေး ထိန်းချုပ်မှုများ (zero trust architecture)
    - Microsoft လုံခြုံရေး ပတ်ဝန်းကျင် ပေါင်းစည်းမှု (Prompt Shields, Azure Content Safety, Entra ID, GitHub Advanced Security)
    - ဆက်တိုက် တိုးတက်လာသည့် လုံခြုံရေး နည်းလမ်းများ
  - **Microsoft လုံခြုံရေး အစီအစဉ်များ**: Prompt Shields, Azure Content Safety, Entra ID, GitHub Advanced Security အတွက် ပေါင်းစည်းမှု လမ်းညွှန်ချက်များ တိုးမြှင့်ခြင်း
  - **အကောင်အထည်ဖော် အရင်းမြစ်များ**: တရားဝင် MCP စာရွက်စာတမ်းများ၊ Microsoft လုံခြုံရေး ဖြေရှင်းချက်များ၊ လုံခြုံရေး စံနှုန်းများ၊ အသုံးပြုလမ်းညွှန်များ ဘာသာဖြင့် ဖွဲ့ကာ စုစည်းထားသည်

#### တိုးတက်သော လုံခြုံရေး ထိန်းချုပ်မှုများ (02-Security/) - စီးပွားရေးအဆင့် အကောင်အထည်ဖော်မှု
- **MCP-SECURITY-CONTROLS-2025.md**: စီးပွားရေးအဆင့် လုံခြုံရေး ပြင်ဆင်မှု လက်တွေ့နည်းလမ်းများဖြင့် ပြန်ရေးသားပြီး ဖြန့်ချိခြင်း
  - **လုံခြုံရေး နယ်ပယ် ၉ ချက် အပြည့်အစုံ**: အခြေခံထိန်းချုပ်မှုမှ စီးပွားရေး ညာကျောမှုများအထိ တိုးချဲ့ထားသည်
    - အတည်ပြုမှုနှင့် ခွင့်ပြုမှု တိုးတက်မြှင့်တင်မှု (Microsoft Entra ID ပေါင်းစည်းမှု)
    - Token security နှင့် anti-passthrough ထိန်းချုပ်မှုများ (အမှတ်အသား စစ်ဆေးမှု)
    - Session security ထိန်းချုပ်မှုများ (hijacking ကာကွယ်မှု)
    - AI-specific လုံခြုံရေး ထိန်းချုပ်မှုများ (prompt injectionနှင့် tool poisoning ကာကွယ်မှု)
    - Confused Deputy လှုံ့ဆော်မှု ကာကွယ်မှု (OAuth proxy security)
    - ကိရိယာ လည်ပတ်မှု လုံခြုံရေး (sandboxing နှင့် isolation)
    - ကုန်ကြမ်းမှထုတ်လုပ်မှု လုံခြုံရေး ထိန်းချုပ်မှု (dependency စစ်ဆေးမှု)
    - ကြည့်ရှုထောက်လှမ်းမှုနှင့် ဖော်ထုတ်မှု ထိန်းချုပ်မှုများ (SIEM ပေါင်းစည်းမှု)
    - ဖြစ်ရပ်တုံ့ပြန်မှုနှင့် ပြန်လည်ထူထောင်မှု (အလိုအလျောက် စနစ်များ)
  - **နမူနာများ**: YAML configuration ကုဒ်ပိုဒ်များနှင့် နမူနာ ကုဒ်များ ချဲ့ထွင်ပေးထားသည်
  - **Microsoft ဖြေရှင်းချက်ပေါင်းစည်းမှု**: Azure security ဝန်ဆောင်မှုများ၊ GitHub Advanced Security၊ စီးပွားရေး အသိမှတ်ပြု အထောက်အကူများကို ကွပ်ကဲဖော်ပြထားသည်

#### အဆင့်မြှင့် ခေါင်းစဉ် လုံခြုံရေး (05-AdvancedTopics/mcp-security/) - ထုတ်လုပ်မှုဆိုင်ရာ လက်တွေ့အကောင်အထည်ဖော်မှု
- **README.md**: စီးပွားရေးလုံခြုံရေး အကောင်အထည်ဖော်ချက် အပြည့်အစုံဖြင့် ပြန်ရေးသားခြင်း
  - **လက်ရှိ စံသတ်မှတ်ချက်နှင့် ကိုက်ညီမှု**: MCP Specification 2025-06-18 အတိုင်း လုံခြုံရေး လိုအပ်ချက် အပြည့်အစုံပြောင်းလဲထားသည်
  - **အတည်ပြုမှု တိုးတက်မှု**: Microsoft Entra ID ပေါင်းစည်းမှု, .NET နှင့် Java Spring Security နမူနာအစုံတို့ဖြင့် ဖော်ပြထားသည်
  - **AI လုံခြုံရေး ပေါင်းစည်းမှု**: Microsoft Prompt Shields နှင့် Azure Content Safety ကို Python နမူနာများနှင့် ပေါင်းစည်းခြင်း
  - **တိုးတက်သော အန္တရာယ် ကာကွယ်မှု**: 
    - Confused Deputy ကာကွယ်မှု (PKCE နှင့် user consent စစ်ဆေးမှုပါရှိ)
    - Token Passthrough ကာကွယ်မှု (audience စစ်ဆေးမှု၊ လုံခြုံသော token စီမံခန့်ခွဲမှု)
    - Session Hijacking ကာကွယ်မှု (cryptographic binding နှင့် သုံးစွဲသူ ခံစားချက် စစ်ဆေးမှုများ)
  - **စီးပွားရေး လုံခြုံရေး ပေါင်းစည်းမှု**: Azure Application Insights စောင့်ကြည့်မှု၊ ဘေးအန္တရာယ် ရှာဖွေရေး pipeline များနှင့် ကုန်ကြမ်းမှထုတ်လုပ်မှုလုံခြုံရေး
  - **အကောင်အထည်ဖော် မှတ်တမ်းစာရင်း**: မပြုလုပ်မိသော လိုအပ်ချက်နှင့် အကြံပြုချက်များ ပိုင်းခြား အသေးစိတ် ဖော်ပြထားကာ Microsoft လုံခြုံရေး ပတ်ဝန်းကျင် အကျိုးကျေးဇူးများ တင်ပြထားသည်

### စာရွက်စာတမ်း အရည်အသွေး နှင့် စံချိန်ညီမှု
- **စံသတ်မှတ်ချက် ရည်ညွှန်းချက်များ**: MCP Specification 2025-06-18 အားလုံးတွင် ကိုးကားထားသည်
- **Microsoft လုံခြုံရေး ပတ်ဝန်းကျင် ပေါင်းစည်းမှု**: လုံခြုံရေး စာရွက်စာတမ်းများတွင် တိုးတက်စွာလမ်းညွှန်ချက်များ ပေါင်းထည့်ထားသည်
- **လက်တွေ့ ကိုးကားချက်များ**: .NET, Java, Python နမူနာကုဒ်များ နှင့် စီးပွားရေး နည်းပညာ ပုံစံများ ထည့်သွင်းထားသည်
- **အရင်းအမြစ် စုစည်းမှု**: တရားဝင် စာရွက်စာတမ်းများ၊ လုံခြုံရေး စံနှုန်းများနှင့် အသုံးပြု လမ်းညွှန် စာရင်းများ စနစ်တကျ အစုလိုက် စုပေါင်းထားသည်
- **မြင်သာသော အမှတ်အသားများ**: အရေးကြီး လိုအပ်ချက်များနှင့် အကြံပြုချက်များကို ဖော်ပြ သတ်မှတ်ချက်များ ပါရှိသည်


#### အဓိက အယူအဆများ (01-CoreConcepts/) - အပြည့်အစုံ ပြန်လည်ချိန်ညှိခြင်း
- **Protocol ဗားရှင်း အသစ်ထည့်သွင်းမှု**: MCP Specification 2025-06-18 ကိုအထောက်အကူပြု၍ (YYYY-MM-DD ပုံစံ) ဗားရှင်းသတ်မှတ်ချက်အသစ်ထည့်သွင်းခြင်း
- **အင်ဂျင်နီယာ ဖော်ပြချက် တိုးတက်မှု**: Hosts, Clients, Servers အကြောင်း ဖော်ပြချက်များကို လောလောဆယ် MCP ဖွံ့ဖြိုးတိုးတက်နေဆဲ ပုံစံနှင့် ကိုက်ညီအောင် ပြင်ဆင်ထားသည်
  - Hosts တွင် အခု AI အပလီကေးရှင်းများအဖြစ် သတ်မှတ်ထားပြီး MCP client များအတွက် connection များကို ကိုယ့်ကိုယ် coordinate လုပ်နေသည်။
  - Clients များကို protocol connectors အဖြစ် ဖော်ပြထားပြီး server နှင့် တစ်ဦးချင်းဆက်သွယ်မှုများကို ထိန်းသိမ်းထားသည်။
  - Servers များကို ဒေသဆိုင်ရာ နှင့် remote တစ်ခုထက် မတူညီသော deployment အခြေအနေများဖြင့် တိုးတက်ပြောင်းလဲထားသည်။
- **Primitive Restructuring**: server နှင့် client primitives များကို ပြည့်ဝစွာ ပြန်လည်ဖွဲ့စည်းခြင်း
  - Server Primitives: Resources (ဒေတာရင်းမြစ်များ), Prompts (စံနမူနာများ), Tools (အလုပ်လုပ်နိုင်သော function များ) နဲ့ အသေးစိတ်ရှင်းပြချက်များနှင့် ဥပမာများပါဝင်သည်။
  - Client Primitives: Sampling (LLM ပြည့်စုံမှုများ), Elicitation (အသုံးပြုသူထည့်သွင်းမှု), Logging (debugging/monitoring)
  - လက်ရှိ ရှာဖွေရေး (`*/list`), ပြန်ယူရေး (`*/get`), နှင့် အကောင်အထည်ဖော်ရေး (`*/call`) method များကို အပ်ဒိတ်လုပ်ထားသည်။
- **Protocol Architecture**: အဆင့်နှစ်ခု လွှာထားသော အဆောက်အအုံ မော်ဒယ်ကို မိတ်ဆက်
  - Data Layer: JSON-RPC 2.0 အခြေခံပြီး အသက်တာစစ်ပြီး primitives များ ပါဝင်သည်။
  - Transport Layer: STDIO (ဒေသဆိုင်ရာ) နှင့် Streamable HTTP ကို SSE (remote) ပို့ဆောင်မှု သယံဇာတများဖြင့် အသုံးပြုသည်။
- **Security Framework**: ဖုံးကွယ်မှု ပြည့်ဝပြီး အသုံးပြုသူ အတည်ပြုခြင်း၊ ဒေတာဖြင့်ပိုင်ဆိုင်မှု ကာကွယ်မှု၊ ကိရိယာများ အကောင်အထည်ဖော်ရာတွင် လုံခြုံမှုနှင့် ပို့ဆောင်မှု အဆင့်လုံခြုံမှု စသည့် လုံခြုံရေး အခြေခံသဘောတရားများ အပါအဝင်။
- **Communication Patterns**: protocol ဆက်သွယ်မှု မက်ဆေ့ခ်ျများ အား initialization, ရှာဖွေမှု, အကောင်အထည်ဖော်မှု နှင့် အသိပေးမှု လုပ်ငန်းစဉ်များ ပြသခြင်းဖြင့် အပ်ဒိတ်လုပ်ထားသည်။
- **Code Examples**: လက်ရှိ MCP SDK မော်ဒယ်များကို ဖြည့်စွက်ဖော်ပြထားသည့် ဘာသာစကားစုံ (.NET, Java, Python, JavaScript) မှ ဥပမာများ ပြန်လည်စစ်ဆေးတင်ပြထားသည်။

#### Security (02-Security/) - လုံခြုံရေး ပြောင်းလဲမှု စုံလင်စွာ ပြုလုပ်မှု  
- **စံချိန်စံညွှန်း မျှတမှု**: MCP Specification 2025-06-18 လုံခြုံရေး လိုအပ်ချက်များနှင့် ပြည့်စုံစွာ ကိုက်ညီမှု
- **အတည်ပြုမှု တိုးတက်မှု**: စိတ်ကြိုက် OAuth server များမှ ပြည်တွင်း အရာရှိမှတ်ပုံတင်ထောက်ပံ့သူ (Microsoft Entra ID) သို့ delegated authentication အပြောင်းအလဲ အရေးအသား
- **AI အထူး ခြိမ်းခြောက်မှု ခွဲခြမ်းစိတ်ဖြာမှု**: နောက်ဆုံးပေါ် AI ထိခိုက်မှု လမ်းကြောင်းများ စနစ်တကျ ချီးမြှင့်ထားခြင်း
  - ဒါရိုက် prompt injection ကို အကြံပြုမှုဖြင့် စက်တင်နမူနာများ ပါဝင်သည်။
  - ကိရိယာ ပျက်စီးမှု သေသေချာချာနဲ့ "rug pull" ထိခိုက်မှု နမူနာများ
  - Context window ပျက်စီးမှုနှင့် မော်ဒယ်မှားယွင်းမှုတိုက်ခိုက်မှုများ
- **Microsoft AI လုံခြုံရေး ဖြေရှင်းမှုများ**: Microsoft လုံခြုံရေးပတ်ဝန်းကျင် အပြည့်အစုံစောင့်ရှောက်မှု
  - AI Prompt Shields အသုံးပြု advanced detection, spotlighting နှင့် delimiter နည်းစနစ်များ
  - Azure Content Safety ပေါင်းစည်းမှု မော်ဒယ်များ
  - GitHub Advanced Security ကို supply chain ကာကွယ်ရေးအတွက် အသုံးပြုခြင်း
- **တိုးတက်သော ခြိမ်းခြောက်မှု ကာကွယ်မှု**: အသေးစိတ် လုံခြုံရေးထိန်းချုပ်မှုများ
  - MCP-specific attack scenario များနှင့် cryptographic session ID လိုအပ်ချက်များ ပါဝင်သော session hijacking ကာကွယ်မှု
  - MCP proxy များရှိငြင်းပယ်သူ confusion ပဋိပက္ခများအတွက် အသုံးပြုသူ သဘောတူညီချက် ရယူမှု
  - token passthrough ခြိမ်းခြောက်မှု အတွက် လိုအပ်သော စစ်ဆေးခြင်း ထိန်းချုပ်မှုများ
- **Supply Chain လုံခြုံရေး**: ဘုံ AI supply chain အတွင်း foundation models, embeddings services, context providers နှင့် သုံးသပ်သူ၏ အသုံးပြု APIs အပြည့်အစုံ စိတ်ဝင်စားစရာတွဲဖက်မှု
- **Foundation Security**: ကုမ္ပဏီ၏ လုံခြုံရေးပုံစံများဖြင့် တိုးတက်ကောင်းမွန်စွာ ပေါင်းစည်းမှု၊ zero trust architecture နှင့် Microsoft လုံခြုံရေးပတ်ဝန်းကျင် တိုးတက်မှု
- **အရင်းအမြစ် စုစည်းမှု**: အမျိုးအစားအလိုက် အဖွဲ့စည်းထားသော လက်ရှိ အရင်းအမြစ် လင့်ခ်များ (အစိုးရစာရွက်စာတမ်းများ၊ စံချိန်စံညွှန်းများ၊ သုတေသန၊ Microsoft ဖြေရှင်းချက်များ၊ ပြုလုပ်နည်းလမ်းညွှန်များ)

### စာရွက်စာတမ်း အရည်အသွေး တိုးတက်မှုများ
