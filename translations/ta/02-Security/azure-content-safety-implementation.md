# MCP உடன் Azure உள்ளடக்கம் பாதுகாப்பை செயல்படுத்துதல்

> **OWASP MCP அபாயம் முகாமைப்படுத்தப்பட்டது**: [MCP06 - சூழலியல் ஓரமான உள்ளீடுகள் மூலம் ப்ராம்ட் செருகல்](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

ப்ராம்ட் செருகல், கருவி மாசுபாடு மற்றும் பிற AI-கேடுஅழுத்தங்கள் என்பவர்களுக்கு எதிராக MCP பாதுகாப்பை வலுப்படுத்த, Azure Content Safety ஐ இணைப்பது மிகுந்த பரிந்துரைக்கப்படுகிறது. இந்த செயலாக்க வழிகாட்டி [MCP பாதுகாப்பு சாமிட் பணிமனை (Sherpa)](https://azure-samples.github.io/sherpa/) கேمپ 3: I/O பாதுகாப்புடன் ஒத்துழைகிறது.

## MCP சர்வருடன் ஒருங்கிணைப்பு

Azure Content Safety ஐ உங்கள் MCP சர்வருடன் ஒருங்கிணைக்க, உள்ளடக்கம் பாதுகாப்பு வடிகட்டியை உங்கள் கோரிக்கை செயலாக்க குழாயில் மிடில்வேராகச் சேர்க்கவும்:

1. சர்வர் துவக்கத்தில் வடிகட்டியை ஆரம்பிக்கவும்  
2. செயலாக்கத்திற்கு முன் வரும் எல்லா கருவி கோரிக்கைகளையும் सत्यापிக்கவும்  
3. வாடிக்கையாளர்களுக்கு அனுப்பும் அடுத்த அனைத்துப் பதில்களையும் சரிபார்க்கவும்  
4. பாதுகாப்பு மீறல்களைக் பதிவு செய்து எச்சரிக்கை விடுக்கவும்  
5. தவறான உள்ளடக்கம் பாதுகாப்பு சோதனைகளுக்கு மேற்கொண்டு பொருத்தமான பிழை கையாளுதலை செயல்படுத்தவும்  

இது கீழ்க்கண்டவற்றுக்கு வலுவான பாதுகாப்பை வழங்குகிறது:  
- ப்ராம்ட் செருகல் தாக்குதல்கள்  
- கருவி மாசுபாடு முயற்சிகள்  
- தீங்கு விளைவிக்கும் உள்ளீடுகள் மூலம் தரவு திருட்டு  
- தீங்கு விளைவிக்கும் உள்ளடக்கம் உருவாக்கம்  

## Azure Content Safety ஒருங்கிணைப்புக்கான சிறந்த நடைமுறைகள்

1. **தனிப்பயன் தடுப்பு பட்டியல்கள்**: MCP செருகல் வடிவங்களுக்கு தனிப்பட்ட தடுப்பு பட்டியல்களை உருவாக்கவும்  
2. **தீவிரம் கவனிப்பு**: உங்கள் தனிப்பட்ட பயன்பாடு மற்றும் அபாய நிலைக்கேற்ப தீவிரம் எல்லைகளை சீரமைக்கவும்  
3. **பரந்த ஊடுருவல்**: அனைத்து உள்ளீடுகளுக்கும் மற்றும் வெளியீடுகளுக்கும் உள்ளடக்கம் பாதுகாப்பு சோதனைகளை அமைக்கவும்  
4. **செயல்திறன் மேம்பாடு**: மறுமுறை உள்ளடக்கம் பாதுகாப்பு சோதனைகளுக்கு கேச்சிங் அமுல்படுத்த பரிந்துரைக்கப்படுகிறது  
5. **மீளும் செயல்பாடுகள்**: உள்ளடக்கம் பாதுகாப்பு சேவைகள் கிடைக்காதபோது தெளிவான மீளுதலை நடத்தை வரையறுக்கவும்  
6. **பயனர் கருத்து**: பாதுகாப்புக் காரணங்களால் உள்ளடக்கம் தடுப்பதற்கான நேரடி மற்றும் தெளிவான கருத்தை பயனர்களுக்கு வழங்கவும்  
7. **தொடர்ச்சி மேம்பாடு**: புதிய அபாயங்களின் அடிப்படையில் தடுப்பு பட்டியல்கள் மற்றும் வடிவங்களை சீர்திருத்தம் செய்யவும்
