# MCP சர்வர்களை நடத்தியல்

உங்கள் MCP சர்வரை நடத்தியால், மற்றவர்கள் அதன் கருவிகள் மற்றும் வளங்களை உங்கள் உள்ளூர் சூழலைவிட அதிகமாக அணுக முடியும். பருமானத்திற்கான தேவைகள், நம்பகத்தன்மை மற்றும் நிர்வாக எளிமை ஆகியவற்றின் அடிப்படையில் பல நடத்தியல் முயற்சிகள் உள்ளன. கீழே, MCP சர்வர்களை உள்ளூர், கண்டெயினர்கள் மற்றும் கிளவுட் ஆகியவற்றில் இயக்குவதற்கான வழிகாட்டுதல் காணலாம்.

## அறிமுகம்

இந்த பாடம் உங்கள் MCP சர்வர் செயலியை எப்படி நடத்தியது என்பதை கவர் செய்யும்.

## கற்றல் நோக்கியங்கள்

இந்த பாடம் முடியும் போது, நீங்கள்:

- பல்வேறு நடத்தியல் அணுகுமுறைகளை மதிப்பிட முடியும்.
- உங்கள் செயலியை நடத்தியல் செய்ய முடியும்.

## உள்ளூர் வளர்ச்சி மற்றும் நடத்தியல்

உங்கள் சர்வர் பயனரின் இயந்திரத்தில் இயக்கப்பட வேண்டும் என்றால், கீழ்காணும் படிகளை பின்பற்றலாம்:

1. **சர்வரைக் க 다운로드 செய்க**. நீங்கள் சர்வரை எழுதவில்லை என்றால், முதலில் உங்கள் இயந்திரத்தில் அதனை பதிவிறக்கம் செய்யவும்.
1. **சர்வர் செயலியை தொடங்குக**: உங்கள் MCP சர்வர் செயலியை இயக்கவும்.

SSE க்காக (stdio வகை சர்வருக்கு தேவையில்லை)

1. **நெட்வொர்க் அமைக்கவும்**: சர்வர் எதிர்பார்க்கப்படும் போர்டில் அணுகக்கூடியதாக இருப்பதை உறுதிசெய்க.
1. **கிளையெண்டுகளை இணைக்கவும்**: `http://localhost:3000` போன்ற உள்ளூர் இணைப்பு URL-களை பயன்படுத்தவும்.

## கிளவுட் நடத்தியல்

MCP சர்வர்கள் பல கிளவுட் தளங்களில் நடத்தியல் செய்யப்படலாம்:

- **சர்வர்லெஸ் செயல்பாடுகள்**: எளிய MCP சர்வர்களை சர்வர்லெஸ் செயல்பாடுகளாக நடத்தியல் செய்யவும்.
- **கண்டெயினர் சேவைகள்**: Azure Container Apps, AWS ECS, அல்லது Google Cloud Run போன்ற சேவைகளை பயன்படுத்தவும்.
- **Kubernetes**: MCP சர்வர்களை உயர் கிடைக்கும் தன்மைக்கான Kubernetes கிளஸ்டர்களில் இயக்கவும் மற்றும் நிர்வகிக்கவும்.

### எடுத்துக்காட்டு: Azure Container Apps

Azure Container Apps MCP சர்வர்களை நடத்தியலை ஆதரிக்கிறது. இது இன்னும் வளர்ச்சியில் உள்ளது மற்றும் தற்போது SSE சர்வர்களுக்கு ஆதரவாக உள்ளது.

இதனை நீங்கள் எப்படி செய்யலாம்:

1. ஒரு ரெப்போவை கிளோன் செய்யவும்:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. உள்ளூரில் இயக்கி முயற்சி செய்யவும்:

  ```sh
  uv venv
  uv sync

  # லினக்ஸ்/மக்OS
  export API_KEYS=<AN_API_KEY>
  # விண்டோஸ்
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. உள்ளூரில் முயற்சி செய்ய, *.vscode* கோப்பிரிவில் *mcp.json* கோப்பை உருவாக்கி கீழ்காணும் உள்ளடக்கத்தைச் சேர்க்கவும்:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  SSE சர்வர் தொடங்கப்பட்டதும், JSON கோப்பில் play ஐகானை கிளிக் செய்யலாம், இப்போது சர்வரில் கருவிகள் GitHub Copilot மூலம் பிடிக்கப்படுவதை பார்க்க முடியும், கருவி ஐகானைப் பார்க்கவும்.

1. நடத்தியல் செய்ய, கீழ்காணும் கட்டளையை இயக்கவும்:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

இதுதான், அதை உள்ளூரில் நடத்தியல் செய்க, Azure வழியாகவும் இத்தகைய படிகள் மூலம் நடத்தியல் செய்க.
