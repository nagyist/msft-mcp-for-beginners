## শুরু করা  

[![আপনার প্রথম MCP সার্ভার তৈরি করুন](../../../translated_images/bn/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(এই পাঠের ভিডিও দেখতে উপরের ছবিটি ক্লিক করুন)_

এই বিভাগে বেশ কয়েকটি পাঠ রয়েছে:

- **1 আপনার প্রথম সার্ভার**, এই প্রথম পাঠে আপনি শিখবেন কীভাবে আপনার প্রথম সার্ভার তৈরি করবেন এবং ইনস্পেক্টর টুল দিয়ে এটি পরীক্ষা করবেন, যা আপনার সার্ভার টেস্ট এবং ডিবাগ করার একটি মূল্যবান উপায়, [পাঠে যান](01-first-server/README.md)

- **2 ক্লায়েন্ট**, এই পাঠে আপনি শিখবেন কীভাবে একটি ক্লায়েন্ট লিখবেন যা আপনার সার্ভারের সাথে সংযোগ করতে পারে, [পাঠে যান](02-client/README.md)

- **3 LLM সহ ক্লায়েন্ট**, ক্লায়েন্ট লেখার একটি আরও উন্নত উপায় হল এতে একটি LLM যোগ করা যাতে এটি আপনার সার্ভারের সাথে "আলোচনা" করতে পারে কী করতে হবে, [পাঠে যান](03-llm-client/README.md)

- **4 Visual Studio Code-এ GitHub Copilot Agent মোড ব্যবহার করে সার্ভার চালনা**। এখানে, আমরা Visual Studio Code এর ভিতর থেকে আমাদের MCP সার্ভার চালানোর বিষয়ে জানব, [পাঠে যান](04-vscode/README.md)

- **5 stdio ট্রান্সপোর্ট সার্ভার** stdio ট্রান্সপোর্ট হল স্থানীয় MCP সার্ভার থেকে ক্লায়েন্ট যোগাযোগের জন্য সুপারিশকৃত স্ট্যান্ডার্ড, যা সাবপ্রসেস ভিত্তিক নিরাপদ যোগাযোগ প্রদান করে প্রক্রিয়ার পৃথকীকরণ সহ [পাঠে যান](05-stdio-server/README.md)

- **6 MCP দিয়ে HTTP স্ট্রিমিং (স্ট্রিমেবল HTTP)**। আধুনিক HTTP স্ট্রিমিং ট্রান্সপোর্ট সম্পর্কে জানুন (দূরবর্তী MCP সার্ভারের জন্য সুপারিশকৃত পদ্ধতি [MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http) অনুযায়ী), অগ্রগতি বিজ্ঞপ্তিগুলি এবং কীভাবে স্কেলেবল, রিয়েল-টাইম MCP সার্ভার এবং ক্লায়েন্ট তৈরি করবেন Streamable HTTP ব্যবহার করে। [পাঠে যান](06-http-streaming/README.md)

- **7 VSCode-এর জন্য AI Toolkit ব্যবহার** MCP ক্লায়েন্ট এবং সার্ভার টেস্ট ও ব্যবহার করার জন্য [পাঠে যান](07-aitk/README.md)

- **8 পরীক্ষা**। এখানে আমরা বিশেষভাবে ফোকাস করব কীভাবে বিভিন্ন উপায়ে আমাদের সার্ভার এবং ক্লায়েন্ট পরীক্ষা করা যায়, [পাঠে যান](08-testing/README.md)

- **9 ডিপ্লয়মেন্ট**। এই অধ্যায়ে আপনার MCP সমাধান ডিপ্লয়মেন্টের বিভিন্ন পদ্ধতি দেখানো হবে, [পাঠে যান](09-deployment/README.md)

- **10 উন্নত সার্ভার ব্যবহার**। এই অধ্যায়টি উন্নত সার্ভার ব্যবহারের বিষয়টি কভার করে, [পাঠে যান](./10-advanced/README.md)

- **11 অথেন্টিকেশন**। এই অধ্যায়টি সহজ অথেন্টিকেশন যোগ করার বিষয় নিয়ে আলোচনা করে, বেসিক অথ থেকে JWT এবং RBAC ব্যবহার করা পর্যন্ত। আপনাকে এখানে শুরু করার পরামর্শ দেওয়া হয়, তারপর অধ্যায় 5-এর উন্নত বিষয়গুলি দেখুন এবং অধ্যায় 2-তে প্রস্তাবিত অতিরিক্ত সুরক্ষা শক্তিশালীকরণ করুন, [পাঠে যান](./11-simple-auth/README.md)

- **12 MCP হোস্টস**। জনপ্রিয় MCP হোস্ট ক্লায়েন্ট যেমন Claude Desktop, Cursor, Cline, এবং Windsurf কনফিগার এবং ব্যবহার করুন। ট্রান্সপোর্ট ধরন এবং সমস্যার সমাধান শিখুন, [পাঠে যান](./12-mcp-hosts/README.md)

- **13 MCP ইনস্পেক্টর**। MCP সার্ভারগুলি পারস্পরিকভাবে ডিবাগ এবং পরীক্ষা করার জন্য MCP ইনস্পেক্টর টুল ব্যবহার করুন। সমস্যার সমাধান, টুল, রিসোর্স এবং প্রোটোকল মেসেজ বুঝুন, [পাঠে যান](./13-mcp-inspector/README.md)

Model Context Protocol (MCP) হল একটি খোলা প্রোটোকল যা অ্যাপ্লিকেশনগুলি কিভাবে LLM-কে প্রসঙ্গ প্রদান করে তা স্ট্যান্ডার্ড করে তোলে। MCP কে ভাবুন AI অ্যাপ্লিকেশনের জন্য একটি USB-C পোর্ট হিসাবে - এটি AI মডেলকে বিভিন্ন ডেটা সোর্স এবং টুলের সাথে সংযুক্ত করার একটি স্ট্যান্ডার্ডকৃত উপায় প্রদান করে।

## শেখার উদ্দেশ্য

এই পাঠের শেষে আপনি পারবেন:

- C#, Java, Python, TypeScript এবং JavaScript এ MCP জন্য ডেভেলপমেন্ট পরিবেশ সেটআপ করতে
- কাস্টম ফিচার (রিসোর্স, প্রম্পট, টুলস) সহ সাধারণ MCP সার্ভার তৈরি ও ডিপ্লয় করতে
- MCP সার্ভারের সাথে সংযোগকারী হোস্ট অ্যাপ্লিকেশন তৈরি করতে
- MCP ইম্প্লিমেন্টেশন টেস্ট ও ডিবাগ করতে
- সাধারণ সেটআপ চ্যালেঞ্জ ও তার সমাধান বুঝতে
- জনপ্রিয় LLM সার্ভিসগুলোর সাথে MCP ইম্প্লিমেন্টেশন সংযোগ করতে

## আপনার MCP পরিবেশ সেটআপ করা

MCP নিয়ে কাজ শুরু করার আগে, আপনার ডেভেলপমেন্ট পরিবেশ প্রস্তুত করা এবং মৌলিক ওয়ার্কফ্লো বুঝা জরুরি। এই বিভাগ আপনার হাতে MCP এর সঙ্গে মসৃণ শুরু নিশ্চিত করার জন্য প্রাথমিক সেটআপ ধাপগুলি নির্দেশনা দেবে।

### পূর্বশর্ত

MCP ডেভেলপমেন্টে ঝাঁপ দেওয়ার আগে নিশ্চিত করুন আপনার কাছে রয়েছে:

- **ডেভেলপমেন্ট পরিবেশ**: আপনার নির্বাচিত ভাষার (C#, Java, Python, TypeScript, অথবা JavaScript)
- **IDE/এডিটর**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm অথবা যেকোন আধুনিক কোড এডিটর
- **প্যাকেজ ম্যানেজার**: NuGet, Maven/Gradle, pip অথবা npm/yarn
- **API কী**: যেকোন AI সার্ভিসের জন্য যেগুলো আপনি আপনার হোস্ট অ্যাপে ব্যবহার করবেন


### অফিসিয়াল SDK

আসন্ন অধ্যায়গুলিতে আপনি Python, TypeScript, Java এবং .NET ব্যবহার করে নির্মিত সমাধান দেখবেন। এখানে সমস্ত অফিসিয়ালি সমর্থিত SDK রয়েছে।

MCP বহু ভাষার জন্য অফিসিয়াল SDK প্রদান করে ([MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/) অনুসারে):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - Microsoft এর সাথে সহযোগিতায় রক্ষণাবেক্ষণ
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI এর সাথে সহযোগিতায় রক্ষণাবেক্ষণ
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - অফিসিয়াল TypeScript ইম্প্লিমেন্টেশন
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - অফিসিয়াল Python ইম্প্লিমেন্টেশন (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - অফিসিয়াল Kotlin ইম্প্লিমেন্টেশন
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI এর সাথে সহযোগিতায় রক্ষণাবেক্ষণ
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - অফিসিয়াল Rust ইম্প্লিমেন্টেশন
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - অফিসিয়াল Go ইম্প্লিমেন্টেশন

## মূল বিষয়সমূহ

- ভাষা-নির্দিষ্ট SDK ব্যবহার করে MCP ডেভেলপমেন্ট পরিবেশ সেটআপ করা সহজ
- MCP সার্ভার তৈরি মানে স্পষ্ট স্কিমা সহ টুল তৈরি ও নিবন্ধন করা
- MCP ক্লায়েন্ট সার্ভার ও মডেলের সাথে সংযোগ স্থাপন করে বর্ধিত ক্ষমতা গ্রহণ করে
- পরীক্ষণ ও ডিবাগিং MCP ইম্প্লিমেন্টেশনের জন্য অপরিহার্য
- ডিপ্লয়মেন্ট বিকল্প স্থানীয় ডেভেলপমেন্ট থেকে ক্লাউড-ভিত্তিক সমাধান পর্যন্ত বিস্তৃত
