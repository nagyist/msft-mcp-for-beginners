# Ανάπτυξη Διακομιστών MCP

Η ανάπτυξη του διακομιστή MCP σας επιτρέπει σε άλλους να έχουν πρόσβαση στα εργαλεία και τους πόρους του πέρα από το τοπικό σας περιβάλλον. Υπάρχουν διάφορες στρατηγικές ανάπτυξης που πρέπει να ληφθούν υπόψη, ανάλογα με τις απαιτήσεις σας για κλιμάκωση, αξιοπιστία και ευκολία διαχείρισης. Παρακάτω θα βρείτε οδηγίες για την ανάπτυξη διακομιστών MCP τοπικά, σε κοντέινερ και στο cloud.

## Επισκόπηση

Αυτό το μάθημα καλύπτει πώς να αναπτύξετε την εφαρμογή MCP Server σας.

## Στόχοι Μάθησης

Στο τέλος αυτού του μαθήματος, θα μπορείτε να:

- Αξιολογείτε διάφορες προσεγγίσεις ανάπτυξης.
- Αναπτύσσετε την εφαρμογή σας.

## Τοπική ανάπτυξη και ανάπτυξη

Εάν ο διακομιστής σας προορίζεται να χρησιμοποιηθεί εκτελώντας τον στον υπολογιστή του χρήστη, μπορείτε να ακολουθήσετε τα παρακάτω βήματα:

1. **Κατεβάστε τον διακομιστή**. Εάν δεν γράψατε τον διακομιστή, πρώτα κατεβάστε τον στον υπολογιστή σας. 
1. **Ξεκινήστε τη διαδικασία διακομιστή**: Εκτελέστε την εφαρμογή διακομιστή MCP σας.

Για SSE (όχι απαραίτητο για διακομιστές τύπου stdio)

1. **Προσαρμόστε το δίκτυο**: Βεβαιωθείτε ότι ο διακομιστής είναι προσβάσιμος στην αναμενόμενη θύρα.
1. **Συνδέστε πελάτες**: Χρησιμοποιήστε τοπικούς συνδέσμους URL όπως `http://localhost:3000`

## Ανάπτυξη στο Cloud

Οι διακομιστές MCP μπορούν να αναπτυχθούν σε διάφορες πλατφόρμες cloud:

- **Serverless Functions**: Αναπτύξτε ελαφρούς διακομιστές MCP ως serverless functions.
- **Υπηρεσίες Κοντέινερ**: Χρησιμοποιήστε υπηρεσίες όπως Azure Container Apps, AWS ECS, ή Google Cloud Run.
- **Kubernetes**: Αναπτύξτε και διαχειριστείτε διακομιστές MCP σε clusters Kubernetes για υψηλή διαθεσιμότητα.

### Παράδειγμα: Azure Container Apps

Τα Azure Container Apps υποστηρίζουν την ανάπτυξη MCP Servers. Είναι ακόμα σε εξέλιξη και υποστηρίζει προς το παρόν διακομιστές SSE.

Να πώς μπορείτε να προχωρήσετε:

1. Κλωνοποιήστε ένα αποθετήριο:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. Εκτελέστε το τοπικά για δοκιμή:

  ```sh
  uv venv
  uv sync

  # linux/macOS
  export API_KEYS=<AN_API_KEY>
  # windows
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. Για να το δοκιμάσετε τοπικά, δημιουργήστε ένα αρχείο *mcp.json* σε έναν κατάλογο *.vscode* και προσθέστε το παρακάτω περιεχόμενο:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  Μόλις ξεκινήσει ο SSE διακομιστής, μπορείτε να κάνετε κλικ στο εικονίδιο αναπαραγωγής στο αρχείο JSON, τώρα θα πρέπει να δείτε τα εργαλεία στον διακομιστή να αναγνωρίζονται από το GitHub Copilot, δείτε το εικονίδιο Εργαλείου.

1. Για ανάπτυξη, εκτελέστε την ακόλουθη εντολή:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

Ορίστε λοιπόν, αναπτύξτε το τοπικά ή στην Azure με αυτά τα βήματα.
