# Beispiel ausführen

## Abhängigkeiten installieren

```sh
npm install
```

## Erstellen

```sh
npm run build
```

## Ausführen

```sh
npm start
```

Du solltest den folgenden Text sehen:

```text
Registering tools...
Starting server...
```

Super, das bedeutet, dass es korrekt gestartet wurde.

## Server testen

Teste die Funktionen mit dem folgenden Befehl:

```sh
npx @modelcontextprotocol/inspector node build/app.js
```

Dies sollte die Weboberfläche des Inspektor-Tools starten.

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/list
```

Ein Tool ausführen:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

Du solltest eine Antwort ähnlich der folgenden sehen:

```text
{
  "content": [
    {
      "type": "text",
      "text": "Tool add called with arguments: {\"a\":1,\"b\":2}, result: {\"content\":[{\"type\":\"text\",\"text\":\"3\"}]}"
    }
  ]
```

Teste ein Tool, das nicht existiert, wie "add2", mit diesem Befehl:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add2 --tool-arg a=1 --tool-arg b=2
```

Du solltest jetzt diese Nachricht sehen, die zeigt, dass deine Validierung funktioniert:

```text
{
  "content": [],
  "error": {
    "code": "tool_not_found",
    "message": "Tool add2 not found."
  }
```

Versuche auch, einen Parameter `c` zu senden, der vom Schema abgelehnt werden sollte, wie folgt:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg c=2
```

Du solltest jetzt den Fehler "ungültige Argumente" sehen:

```text
{
  "content": [],
  "error": {
    "code": "invalid_arguments",
    "message": "Invalid arguments for tool add: [\n  {\n    \"code\": \"invalid_type\",\n    \"expected\": \"number\",\n    \"received\": \"undefined\",\n    \"path\": [\n      \"b\"\n    ],\n    \"message\": \"Required\"\n  }\n]"
  }
}
```
