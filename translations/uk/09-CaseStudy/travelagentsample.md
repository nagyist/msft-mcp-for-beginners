# Case Study: Azure AI Travel Agents – Референсна реалізація

## Огляд

[Azure AI Travel Agents](https://github.com/Azure-Samples/azure-ai-travel-agents) — це комплексне референсне рішення, розроблене Microsoft, яке демонструє, як створити багатоярусний застосунок для планування подорожей на базі штучного інтелекту з використанням Model Context Protocol (MCP), Azure OpenAI та Azure AI Search. Цей проєкт ілюструє найкращі практики оркестрації кількох агентів ШІ, інтеграції корпоративних даних і забезпечення безпечної, розширюваної платформи для реальних сценаріїв.

## Основні функції
- **Оркестрація багатьох агентів:** Використовує MCP для координації спеціалізованих агентів (наприклад, агентів по авіаперельотах, готелях і маршрутах), які співпрацюють для виконання складних завдань планування подорожей.
- **Інтеграція корпоративних даних:** Підключається до Azure AI Search та інших корпоративних джерел даних для надання актуальної та релевантної інформації для рекомендацій по подорожах.
- **Безпечна та масштабована архітектура:** Використовує Azure сервіси для аутентифікації, авторизації та масштабованого розгортання з дотриманням найкращих практик безпеки підприємства.
- **Розширювані інструменти:** Реалізує багаторазові MCP інструменти та шаблони запитів, що дозволяє швидко адаптуватися до нових сфер або бізнес-вимог.
- **Користувацький досвід:** Надає розмовний інтерфейс для взаємодії користувачів з агентами подорожей на базі Azure OpenAI та MCP.

## Архітектура
![Architecture](https://raw.githubusercontent.com/Azure-Samples/azure-ai-travel-agents/main/docs/ai-travel-agents-architecture-diagram.png)

### Опис діаграми архітектури

Рішення Azure AI Travel Agents спроектоване для модульності, масштабованості та безпечної інтеграції кількох агентів ШІ та корпоративних джерел даних. Основні компоненти та потік даних описані нижче:

- **Користувацький інтерфейс:** Користувачі взаємодіють із системою через розмовний інтерфейс (наприклад, веб-чат або бот у Teams), який відправляє запити користувача і отримує рекомендації щодо подорожей.
- **MCP Сервер:** Виконує роль центрального оркестратора, приймає вхідні дані користувача, керує контекстом і координує дії спеціалізованих агентів (наприклад, FlightAgent, HotelAgent, ItineraryAgent) через Model Context Protocol.
- **Агенти ШІ:** Кожен агент відповідає за певну доменну область (авіаперельоти, готелі, маршрути) і реалізований як інструмент MCP. Агенти використовують шаблони запитів та логику для обробки запитів і формування відповідей.
- **Azure OpenAI Service:** Забезпечує передове розуміння природної мови та генерацію тексту, дозволяючи агентам інтерпретувати наміри користувача і формувати розмовні відповіді.
- **Azure AI Search та корпоративні дані:** Агенти виконують запити до Azure AI Search і інших корпоративних джерел, щоб отримати актуальну інформацію про рейси, готелі та варіанти подорожей.
- **Аутентифікація та безпека:** Інтегрується з Microsoft Entra ID для безпечної аутентифікації та застосовує принцип найменших привілеїв для доступу до усіх ресурсів.
- **Розгортання:** Розроблено для розгортання в Azure Container Apps, що забезпечує масштабованість, моніторинг і ефективність експлуатації.

Ця архітектура забезпечує безперебійну оркестрацію кількох агентів ШІ, безпечну інтеграцію з корпоративними даними та надійну, розширювану платформу для створення галузево-специфічних рішень на базі ШІ.

## Покрокове пояснення діаграми архітектури
Уявіть, що ви плануєте велику подорож і маєте команду експертних помічників, які допомагають з кожною деталлю. Система Azure AI Travel Agents працює подібним чином, використовуючи різні частини (як учасників команди), кожен із яких виконує спеціалізовану роль. Ось як це усе працює разом:

### Користувацький інтерфейс (UI):
Уявіть це як ресепшн вашого агента з подорожей. Тут ви (користувач) ставите питання або робите запити, наприклад, "Знайди мені рейс до Парижа." Це може бути вікно чату на сайті або в додатку для повідомлень.

### MCP Сервер (Координатор):
MCP Сервер — це як менеджер, який слухає ваш запит на ресепшн і вирішує, який спеціаліст повинен обробити кожну частину. Він веде облік діалогу і стежить, щоб усе працювало гладко.

### Агенти ШІ (Спеціалісти-помічники):
Кожен агент — експерт у своїй області: один знає все про рейси, інший — про готелі, а ще один — про планування маршруту. Коли ви робите запит, MCP Сервер направляє ваш запит відповідному агенту(агентам). Ці агенти використовують знання та інструменти, щоб знайти найкращі варіанти для вас.

### Azure OpenAI Service (Мовний експерт):
Це як мовний експерт, який розуміє точно, що ви питаєте, незалежно від того, як ви це сформулювали. Він допомагає агентам розуміти ваші запити і відповідати природньою, розмовною мовою.

### Azure AI Search та корпоративні дані (Інформаційна бібліотека):
Уявіть велику, актуальну бібліотеку з усією новою інформацією про подорожі — розклад рейсів, наявність готелів і багато іншого. Агенти шукають у цій бібліотеці, щоб надати вам найточніші відповіді.

### Аутентифікація та безпека (Охоронець):
Як охоронець перевіряє, хто може входити у певні зони, ця частина системи впевнюється, що доступ до конфіденційної інформації мають лише уповноважені користувачі та агенти. Ваша інформація зберігається у безпеці та конфіденційності.

### Розгортання в Azure Container Apps (Будівля):
Усі ці помічники та інструменти працюють разом у безпечній, масштабованій "будівлі" (хмарі). Це означає, що система може обслуговувати багато користувачів одночасно і завжди готова до роботи, коли ви її потребуєте.

## Як це працює разом:

Ви починаєте, ставлячи запит на ресепшн (UI).
Менеджер (MCP Сервер) визначає, який спеціаліст (агент) допоможе.
Спеціаліст використовує мовного експерта (OpenAI) для розуміння вашого запиту та бібліотеку (AI Search) для пошуку найкращої відповіді.
Охоронець (Аутентифікація) пильно стежить, щоб усе було безпечно.
Все це відбувається всередині надійної, масштабованої будівлі (Azure Container Apps), тому ваш досвід залишається плавним і захищеним.
Ця спільна робота дозволяє системі швидко і безпечно допомагати вам планувати подорож, як команда експертних агентів з подорожей, що працюють у сучасному офісі!

## Технічна реалізація
- **MCP Сервер:** Хостить основну логіку оркестрації, надає інструменти агентів та керує контекстом для багатокрокових робочих процесів планування подорожей.
- **Агенти:** Кожен агент (наприклад, FlightAgent, HotelAgent) реалізований як інструмент MCP із власними шаблонами запитів і логікою.
- **Інтеграція з Azure:** Використовує Azure OpenAI для розуміння природної мови та Azure AI Search для отримання даних.
- **Безпека:** Інтегрується з Microsoft Entra ID для аутентифікації та застосовує політику найменших привілеїв до всіх ресурсів.
- **Розгортання:** Підтримує розгортання в Azure Container Apps для масштабованості та ефективності експлуатації.

## Результати та вплив
- Демонструє, як MCP можна використовувати для оркестрації кількох агентів ШІ в реальному виробничому сценарії.
- Прискорює розробку рішення, надаючи повторно використовувані шаблони для координації агентів, інтеграції даних і безпечного розгортання.
- Служить шаблоном для створення галузево-специфічних застосунків на базі ШІ з використанням MCP і сервісів Azure.
