# ایم سی پی سرورز کو تعینات کرنا

اپنا ایم سی پی سرور تعینات کرنے سے دوسرے اس کے ٹولز اور وسائل تک آپ کے مقامی ماحول سے باہر بھی رسائی حاصل کر سکتے ہیں۔ آپ کی ضروریات کے مطابق پیمائش، اعتبار، اور آسانی انتظام کے لیے مختلف تعیناتی حکمت عملیاں ہیں۔ نیچے آپ کو ایم سی پی سرورز کو مقامی، کنٹینرز میں، اور کلاؤڈ پر تعینات کرنے کے لیے رہنمائی ملے گی۔

## جائزہ

یہ سبق آپ کے ایم سی پی سرور ایپ کو تعینات کرنے کا طریقہ کار سکھاتا ہے۔

## تعلیمی مقاصد

اس سبق کے اختتام تک، آپ قادر ہوں گے:

- مختلف تعیناتی طریقوں کا جائزہ لیں۔
- اپنی ایپ کو تعینات کریں۔

## مقامی ترقی اور تعینات

اگر آپ کا سرور صارف کے کمپیوٹر پر چل کر استعمال ہونے والا ہے، تو آپ درج ذیل اقدامات پر عمل کر سکتے ہیں:

1. **سرور ڈاؤن لوڈ کریں**۔ اگر آپ نے سرور نہیں لکھا ہے، تو پہلے اسے اپنی مشین پر ڈاؤن لوڈ کریں۔  
1. **سرور عمل شروع کریں**: اپنی ایم سی پی سرور ایپلیکیشن چلائیں  

ایس ایس ای کے لیے (stdio قسم کے سرور کے لیے ضرورت نہیں)

1. **نیٹ ورکنگ ترتیب دیں**: یقینی بنائیں کہ سرور متوقع پورٹ پر قابل رسائی ہے  
1. **کلائنٹس کو جوڑیں**: مقامی کنیکشن URL استعمال کریں جیسے `http://localhost:3000`

## کلاؤڈ میں تعیناتی

ایم سی پی سرورز مختلف کلاؤڈ پلیٹ فارمز پر تعینات کیے جا سکتے ہیں:

- **سرو لیس فنکشنز**: ایم سی پی سرورز کو بطور سرو لیس فنکشنز تعینات کریں  
- **کنٹینر سروسز**: ازور کنٹینر ایپس، AWS ECS، یا گوگل کلاؤڈ رن جیسی سروسز استعمال کریں  
- **کوبرنیٹس**: ایم سی پی سرورز کو کوبرنیٹس کلسٹروں میں تعینات اور انتظام کریں تاکہ اعلی دستیابی حاصل ہو

### مثال: ازور کنٹینر ایپس

ازور کنٹینر ایپس ایم سی پی سرورز کی تعیناتی کی حمایت کرتی ہیں۔ یہ ابھی ترقی کے مراحل میں ہے اور اس میں فی الحال ایس ایس ای سرورز کی حمایت ہے۔

یہاں آپ کیسے کر سکتے ہیں:

1. ایک ریپوزیٹری کلون کریں:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. اسے مقامی طور پر چلا کر جانچ کریں:

  ```sh
  uv venv
  uv sync

  # لینکس/میک او ایس
  export API_KEYS=<AN_API_KEY>
  # ونڈوز
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. مقامی جانچ کے لیے، *.vscode* ڈائریکٹری میں ایک *mcp.json* فائل بنائیں اور درج ذیل مواد شامل کریں:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  جب ایس ایس ای سرور شروع ہو جائے، تو آپ JSON فائل میں پلے آئیکن پر کلک کر سکتے ہیں، اب آپ گیٹ ہب کوپائلٹ کے ذریعے سرور پر موجود ٹولز دیکھ سکیں گے، ٹول آئیکن دیکھیں۔  

1. تعیناتی کے لیے مندرجہ ذیل کمانڈ چلائیں:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

بس یہی ہے، اسے مقامی طور پر تعینات کریں، ازور پر درج بالا مراحل کے ذریعے تعینات کریں۔
