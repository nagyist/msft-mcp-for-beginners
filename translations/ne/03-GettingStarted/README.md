## सुरु गर्ने तरिका  

[![तपाईंको पहिलो MCP सर्भर बनाउनुहोस्](../../../translated_images/ne/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(यस पाठको भिडियो हेर्न माथिको तस्बिरमा क्लिक गर्नुहोस्)_

यस खण्डमा धेरै पाठहरू समावेश छन्:

- **1 तपाईंको पहिलो सर्भर**, यस पहिलो पाठमा तपाईँले आफ्नो पहिलो सर्भर कसरी बनाउन र निरीक्षक उपकरणमार्फत परीक्षण गर्ने सिक्नुहुनेछ, यो तपाईंको सर्भर परीक्षण र डिबग गर्नको लागि उपयोगी उपाय हो, [पाठमा जानुहोस्](01-first-server/README.md)

- **2 क्लाइन्ट**, यस पाठमा तपाईँले आफ्नो सर्भरसँग जडान हुने क्लाइन्ट कसरी लेख्ने सिक्नुहुनेछ, [पाठमा जानुहोस्](02-client/README.md)

- **3 LLM सहितको क्लाइन्ट**, क्लाइन्ट लेख्ने अझ राम्रो तरिका भनेको यसमा LLM थप्नु हो जसले सर्भरसँग "वार्ता" गर्न सक्छ कि के गर्ने, [पाठमा जानुहोस्](03-llm-client/README.md)

- **4 Visual Studio Code मा GitHub Copilot एजेन्ट मोडमा सर्भर उपभोग गर्ने**। यहाँ, हामी Visual Studio Code भित्रबाट हाम्रो MCP सर्भर सञ्चालन गर्ने बारे हेर्दैछौं, [पाठमा जानुहोस्](04-vscode/README.md)

- **5 stdio ट्रान्सपोर्ट सर्भर** stdio ट्रान्सपोर्ट स्थानीय MCP सर्भर-देखि-क्लाइन्ट संचारको लागि सिफारिस गरिएको मानक हो, जसले सुरक्षित subprocess आधारित संचार र प्रक्रिया अलगाव प्रदान गर्दछ [पाठमा जानुहोस्](05-stdio-server/README.md)

- **6 MCP सँग HTTP स्ट्रिमिङ (Streamable HTTP)**। आधुनिक HTTP स्ट्रिमिङ ट्रान्सपोर्ट बारे सिक्नुहोस् (दूर MCP सर्भरहरूको लागि सिफारिस गरिएको तरिका [MCP विशिष्टता 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http) अनुसार), प्रगति सूचनाहरू, र कसरी स्केलेबल, रियल-टाइम MCP सर्भर र क्लाइन्टहरू Streamable HTTP प्रयोग गरी कार्यान्वयन गर्ने। [पाठमा जानुहोस्](06-http-streaming/README.md)

- **7 VSCode को लागि AI टूलकिट प्रयोग गरी** तपाईँका MCP क्लाइन्ट र सर्भरहरू उपभोग गर्न र परीक्षण गर्न [पाठमा जानुहोस्](07-aitk/README.md)

- **8 परीक्षण**। यहाँ हामी विशेष गरी हाम्रो सर्भर र क्लाइन्टलाई विभिन्न तरिकाले कसरी परीक्षण गर्नेमा ध्यान दिनेछौं, [पाठमा जानुहोस्](08-testing/README.md)

- **9 डिप्लोइमेन्ट**। यस अध्यायमा तपाईँका MCP समाधानहरू डिप्लोइ गर्ने विभिन्न तरिका हेर्नेछौं, [पाठमा जानुहोस्](09-deployment/README.md)

- **10 उन्नत सर्भर प्रयोग**। यो अध्यायले उन्नत सर्भर प्रयोग समेट्छ, [पाठमा जानुहोस्](./10-advanced/README.md)

- **11 प्रामाणिकरण (Auth)**। यस अध्यायले सरल प्रामाणिकरण थप्ने तरिका समेट्छ, Basic Auth देखि JWT र RBAC सम्म। तपाईंलाई यहाँबाट सुरु गरी अध्याय 5 का उन्नत विषयहरू र अध्याय 2 मा सिफारिस गरिएको अतिरिक्त सुरक्षा कडा गर्ने कार्यहरू हेर्न प्रोत्साहित गरिन्छ, [पाठमा जानुहोस्](./11-simple-auth/README.md)

- **12 MCP होस्टहरू**। लोकप्रिय MCP होस्ट क्लाइन्टहरू जस्तै Claude Desktop, Cursor, Cline, र Windsurf कन्फिगर र प्रयोग गर्नुहोस्। ट्रान्सपोर्ट प्रकारहरू र समस्याहरू समाधान गर्ने तरिका सिक्नुहोस्, [पाठमा जानुहोस्](./12-mcp-hosts/README.md)

- **13 MCP निरीक्षक**। MCP निरीक्षक उपकरण प्रयोग गरी तपाईँका MCP सर्भरहरू अन्तरक्रियात्मक रूपमा डिबग र परीक्षण गर्नुहोस्। उपकरणहरू, स्रोतहरू, र प्रोटोकल सन्देशहरू समाधान गर्न सिक्नुहोस्, [पाठमा जानुहोस्](./13-mcp-inspector/README.md)

Model Context Protocol (MCP) एउटा खुला प्रोटोकल हो जसले LLMs लाई सन्दर्भ प्रदान गर्ने तरिका मानकीकृत गर्छ। MCP लाई AI अनुप्रयोगहरूको लागि USB-C पोर्ट जस्तो सोच्नुहोस् - यसले AI मोडेलहरूलाई विभिन्न डाटास्रोत र उपकरणहरूमा जडान गर्ने मानकीकृत तरिका प्रदान गर्दछ।

## सिकाइ लक्ष्यहरू

यस पाठको अन्त्यसम्म, तपाईं सक्षम हुनुहुनेछ:

- C#, Java, Python, TypeScript, र JavaScript मा MCP को लागि विकास वातावरण सेटअप गर्न
- कस्टम सुविधाहरू (स्रोतहरू, प्रॉम्प्टहरू, र उपकरणहरू) सहित आधारभूत MCP सर्भरहरू निर्माण र डिप्लोइ गर्न
- MCP सर्भरहरूसँग जडान हुने होस्ट अनुप्रयोगहरू सिर्जना गर्न
- MCP कार्यान्वयनहरू परीक्षण र डिबग गर्न
- सामान्य सेटअप चुनौतीहरू र तिनीहरूको समाधान बुझ्न
- तपाईंका MCP कार्यान्वयनहरू लोकप्रिय LLM सेवाहरूसँग जडान गर्न

## तपाईँको MCP वातावरण सेटअप गर्नुहोस्

MCPसँग काम सुरु गर्नु अघि, विकास वातावरण तयार पार्नु र आधारभूत कार्यप्रवाह बुझ्नु महत्वपूर्ण छ। यो खण्डले प्रारम्भिक सेटअप चरणहरूमा तपाईंको मार्गनिर्देशन गर्नेछ जसले MCP सँग सहज सुरूवात सुनिश्चित गर्छ।

### आवश्यकताहरू

MCP विकासमा लाग्नुअघि, यकिन गर्नुहोस् कि तपाईंसँग:

- **विकास वातावरण**: छनोट गरिएको भाषामा (C#, Java, Python, TypeScript, वा JavaScript)
- **IDE/संपादक**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm, वा कुनै आधुनिक कोड संपादक
- **प्याकेज प्रबन्धकहरू**: NuGet, Maven/Gradle, pip, वा npm/yarn
- **API कुञ्जीहरू**: तपाइँले होस्ट अनुप्रयोगहरूमा प्रयोग गर्ने कुनै पनि AI सेवाहरूको लागि

### आधिकारिक SDK हरू

अगामी अध्यायहरूमा तपाईंले Python, TypeScript, Java र .NET प्रयोग गरी निर्मित समाधानहरू देख्नु हुनेछ। यी सबै आधिकारिक रूपमा समर्थित SDK हरू हुन्।

MCP ले धेरै भाषाहरूका लागि आधिकारिक SDK प्रदान गर्दछ ([MCP विशिष्टता 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/) संग मेल खाने):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - Microsoft सँगको सहकार्यमा मर्मत गरिएको
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI सँगको सहकार्यमा मर्मत गरिएको
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - आधिकारिक TypeScript कार्यान्वयन
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - आधिकारिक Python कार्यान्वयन (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - आधिकारिक Kotlin कार्यान्वयन
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI सँगको सहकार्यमा मर्मत गरिएको
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - आधिकारिक Rust कार्यान्वयन
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - आधिकारिक Go कार्यान्वयन

## मुख्य सिकाइहरू

- भाषाविशेष SDK हरूको साथ MCP विकास वातावरण सेटअप गर्न सजिलो छ
- MCP सर्भरहरू निर्माण गर्न स्पष्ट स्किमाहरू सहित उपकरणहरू सिर्जना र दर्ता गर्नुपर्छ
- MCP क्लाइन्टहरूले सर्भर र मोडेलहरूसँग जडान गरी विस्तारित क्षमताहरू उपयोग गर्छन्
- विश्वसनीय MCP कार्यान्वयनका लागि परीक्षण र डिबग आवश्यक छ
- डिप्लोइ विकल्पहरू स्थानीय विकासदेखि क्लाउड आधारित समाधानसम्म फैलिएका छन्
