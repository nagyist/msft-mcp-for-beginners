# MCP सर्भरहरू तैनाथ गर्ने

तपाईंको MCP सर्भर तैनाथ गर्दा अरूले यसको उपकरण र स्रोतहरू तपाईंको स्थानीय वातावरण भन्दा बाहिर पहुँच गर्न सक्छन्। तपाईंको मापनक्षमता, विश्वसनीयता, र व्यवस्थापन सजिलोपनको आवश्यकताहरू अनुसार विचार गर्नुपर्ने विभिन्न तैनाथ रणनीतिहरू छन्। तल तपाईंले स्थानीय रूपमा, कन्टेनरहरूमा, र क्लाउडमा MCP सर्भरहरू तैनाथ गर्ने दिशानिर्देश पाउनुहुनेछ।

## अवलोकन

यस पाठले तपाईंको MCP सर्भर एप्लिकेसन कसरी तैनाथ गर्ने भनी सिकाउँछ।

## सिकाइ उद्देश्यहरू

यस पाठको अन्त्यसम्म, तपाईं सक्षम हुनेछ:

- विभिन्न तैनाथ गर्ने तरिका मूल्याङ्कन गर्न।
- तपाईंको एप्लिकेसन तैनाथ गर्न।

## स्थानीय विकास र तैनाथीकरण

यदि तपाईंको सर्भर प्रयोगकर्ताहरूको मेसिनमा चल्ने गरी उपभोग गरिनु पर्ने हो भने, तपाईंले निम्न चरणहरू अनुसरण गर्न सक्नुहुन्छ:

1. **सर्भर डाउनलोड गर्नुहोस्**। यदि तपाईंले सर्भर लेख्नुभएको छैन भने, पहिले यसलाई तपाईंको मेसिनमा डाउनलोड गर्नुहोस्।
1. **सर्भर प्रक्रिया सुरु गर्नुहोस्**: तपाईंको MCP सर्भर एप्लिकेसन चलाउनुहोस्।

SSE का लागि (stdio प्रकार सर्भरका लागि आवश्यक छैन)

1. **नेटवर्किङ कन्फिगर गर्नुहोस्**: सुनिश्चित गर्नुहोस् कि सर्भर अपेक्षित पोर्टमा पहुँचयोग्य छ।
1. **क्लाइन्टहरू जडान गर्नुहोस्**: `http://localhost:3000` जस्ता स्थानीय कनेक्शन URL हरू प्रयोग गर्नुहोस्।

## क्लाउड तैनाथीकरण

MCP सर्भरहरू विभिन्न क्लाउड प्लेटफर्महरूमा तैनाथ गर्न सकिन्छ:

- **सर्भररहित कार्यहरू**: MCP सर्भरहरूलाई सर्भररहित कार्यहरू रूपमा तैनाथ गर्नुहोस्।
- **कन्टेनर सेवाहरू**: Azure Container Apps, AWS ECS, वा Google Cloud Run जस्ता सेवाहरू प्रयोग गर्नुहोस्।
- **कुबेरनेट्स**: उच्च उपलब्धताका लागि MCP सर्भरहरू कुबेरनेट्स क्लस्टरहरूमा तैनाथ र व्यवस्थापन गर्नुहोस्।

### उदाहरण: Azure Container Apps

Azure Container Apps ले MCP सर्भरहरूको तैनाथीकरण समर्थन गर्दछ। यो अझै प्रगतिमा छ र हाल SSE सर्भरहरूलाई समर्थन गर्दछ।

यसरी अघि बढ्न सकिन्छ:

1. एउटा रिपोजिटरी क्लोन गर्नुहोस्:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. स्थानीय रूपमा परीक्षण गर्न त्यसलाई चलाउनुहोस्:

  ```sh
  uv venv
  uv sync

  # लिनक्स/म्याकओएस
  export API_KEYS=<AN_API_KEY>
  # विन्डोज्
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. स्थानीय रूपमा प्रयास गर्नको लागि, *.vscode* निर्देशिकामा *mcp.json* फाइल बनाएर तलको सामग्री थप्नुहोस्:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  एक पटक SSE सर्भर सुरु भएपछि, JSON फाइलमा प्ले आइकनमा क्लिक गर्न सक्नुहुन्छ, अब सर्भरमा रहेका उपकरणहरूले GitHub Copilot द्वारा समातिएको देखिनेछ, उपकरण आइकन हेर्नुहोस्।

1. तैनाथीकरण गर्न, तलको आदेश चलाउनुहोस्:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

यसो गर्दा, तपाईंले यसलाई स्थानीय रूपमा तैनाथ गर्नुभयो, यी चरणहरू मार्फत Azure मा तैनाथ गर्नुभयो।
