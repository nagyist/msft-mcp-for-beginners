# MCP ਨਾਲ Azure ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਲਾਗੂ ਕਰਨਾ

> **OWASP MCP ਜੋ ਖਤਰਾ ਪਤਾ ਲਗਾਇਆ**: [MCP06 - ਸੰਦਰਭੀ ਪੇਲੋਡਾਂ ਰਾਹੀਂ ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ, ਟੂਲ ਜਹਿਰੀਲਾ ਕਰਨ ਅਤੇ ਹੋਰ AI-ਵਿਸ਼ੇਸ਼ ਖਾਮੀਆਂ ਦੇ ਖ਼ਿਲਾਫ MCP ਸੁਰੱਖਿਆ ਮਜ਼ਬੂਤ ਕਰਨ ਲਈ, Azure ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਨੂੰ ਇਕੱਠਾ ਕਰਨਾ ਬਹੁਤ ਸਿਫ਼ਾਰਸ਼ੀ ਹੈ। ਇਹ ਲਾਗੂ ਕਰਨ ਦੀ ਗਾਈਡ [MCP ਸੁਰੱਖਿਆ ਸਮੀਟ ਵਰਕਸ਼ਾਪ (ਸ਼ੇਰਪਾ)](https://azure-samples.github.io/sherpa/) ਕੈਂਪ 3: I/O ਸੁਰੱਖਿਆ ਨਾਲ ਮੇਲ ਖਾਂਦੀ ਹੈ।

## MCP ਸਰਵਰ ਨਾਲ ਇੰਟੀਗ੍ਰੇਸ਼ਨ

ਆਪਣੇ MCP ਸਰਵਰ ਨਾਲ Azure ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਇੰਟੀਗ੍ਰੇਟ ਕਰਨ ਲਈ, ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਫਿਲਟਰ ਨੂੰ ਆਪਣੇ ਬੇਨਤੀ ਪ੍ਰਕਿਰਿਆ ਪਾਇਪਲਾਈਨ ਵਿੱਚ ਮਿੱਡਲਵੇਅਰ ਵਜੋਂ ਸ਼ਾਮਲ ਕਰੋ:

1. ਸਰਵਰ ਸ਼ੁਰੂਆਤ ਦੌਰਾਨ ਫਿਲਟਰ ਨੂੰ ਸ਼ੁਰੂ ਕਰੋ
2. ਪ੍ਰਕਿਰਿਆ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ ਸਾਰੇ ਆਉਣ ਵਾਲੇ ਟੂਲ ਬੇਨਤੀਆਂ ਦੀ ਜਾਂਚ ਕਰੋ
3. ਗਾਹਕਾਂ ਨੂੰ ਵਾਪਸ ਭੇਜਣ ਤੋਂ ਪਹਿਲਾਂ ਸਾਰੀਆਂ ਬਾਹਰ ਜਾ ਰਹੀਆਂ ਪ੍ਰਤੀਕਾਰ ਜਾਂਚੋ
4. ਸੁਰੱਖਿਆ ਉਲੰਘਣਾਂ 'ਤੇ ਲਾਗ ਅਤੇ ਅਲਰਟ ਕਰੋ
5. ਅਸਫਲ ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਜਾਂਚਾਂ ਲਈ ਉਚਿਤ ਗਲਤੀ ਸੰਭਾਲ ਲਾਗੂ ਕਰੋ

ਇਹ ਮਜ਼ਬੂਤ ਰੱਖਿਆ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ:
- ਪ੍ਰਾਂਪਟ ਇੰਜੈਕਸ਼ਨ ਹਮਲੇ
- ਟੂਲ ਜਹਿਰੀਲਾ ਕਰਨ ਦੇ ਯਤਨ
- ਬੁਰੇ ਇਨਪੁੱਟ ਰਾਹੀਂ ਡੇਟਾ ਚੋਰੀ
- ਹਰਾਨਕਾਰਕ ਸਮੱਗਰੀ ਦਾ ਉਤਪਾਦਨ

## Azure ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਇੰਟੀਗ੍ਰੇਸ਼ਨ ਲਈ ਸਰਵੋਤਮ ਅਭਿਆਸ

1. **ਕਸਟਮ ਬਲੌਕਲਿਸਟ**: ਖਾਸ MCP ਇੰਜੈਕਸ਼ਨ ਪੈਟਰਨ ਲਈ ਕਸਟਮ ਬਲੌਕਲਿਸਟ ਬਣਾਓ
2. **ਗੰਭੀਰਤਾ ਟਿਊਨਿੰਗ**: ਆਪਣੀ ਵਿਸ਼ੇਸ਼ ਵਰਤੋਂ ਅਤੇ ਜੋਖਮ ਸਹਿਣਸ਼ੀਲਤਾ ਦੇ ਅਧਾਰ 'ਤੇ ਗੰਭੀਰਤਾ ਸੀਮਾਵਾਂ ਸੈੱਟ ਕਰੋ
3. **ਵਿਆਪਕ ਕਵਰੇਜ**: ਸਾਰੀਆਂ ਇਨਪੁੱਟ ਅਤੇ ਆਉਟਪੁੱਟ 'ਤੇ ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਦੀ ਜਾਂਚ ਲਾਗੂ ਕਰੋ
4. **ਕਾਰਗੁਜ਼ਾਰੀ ਲਈ ਸੁਧਾਰ**: ਦੁਹਰਾਈ ਜਾਣ ਵਾਲੀ ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਜਾਂਚਾਂ ਲਈ ਕੈਸ਼ਿੰਗ ਲਾਗੂ ਕਰਨ ਬਾਰੇ ਸੋਚੋ
5. **ਬੈਕਅਪ ਮਕੈਨੀਜ਼ਮ**: ਜਦੋਂ ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਸੇਵਾਵਾਂ ਉਪਲੱਬਧ ਨਾ ਹੋਣ, ਤਾਂ ਸਾਫ਼ ਸਪਸ਼ਟ ਬੈਕਅਪ ਵਿਹਾਰ ਤਿਆਰ ਕਰੋ
6. **ਉਪਭੋਗਤਾ ਪ੍ਰਤੀਕ੍ਰਿਆ**: ਜਦੋਂ ਸਮੱਗਰੀ ਸੁਰੱਖਿਆ ਸੰਬੰਧੀ ਕਾਰਨਾਂ ਕਰਕੇ ਰੋਕੀ ਜਾਂਦੀ ਹੈ, ਉਪਭੋਗਤਾਵਾਂ ਨੂੰ ਸਪਸ਼ਟ ਫੀਡਬੈਕ ਦਿਓ
7. **ਲਗਾਤਾਰ ਸੁਧਾਰ**: ਨਵੇਂ ਖਤਰਨਾਕ ਰੂਪ ਦੇ ਆਧਾਰ 'ਤੇ ਬਲੌਕਲਿਸਟ ਅਤੇ ਪੈਟਰਨ ਨੂੰ ਨਿਯਮਤ ਰੂਪ ਵਿੱਚ ਅੱਪਡੇਟ ਕਰੋ
