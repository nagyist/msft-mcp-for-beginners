# Разгръщане на MCP сървъри

Разгръщането на вашия MCP сървър позволява на други да имат достъп до инструментите и ресурсите му извън локалната ви среда. Съществуват няколко стратегии за разгръщане, които да вземете под внимание, в зависимост от вашите изисквания за мащабируемост, надеждност и лесна поддръжка. По-долу ще намерите насоки за разгръщане на MCP сървъри локално, в контейнери и в облака.

## Преглед

Този урок обхваща как да разгръщате вашето MCP Server приложение.

## Учебни цели

След края на този урок, ще можете да:

- Оцените различни подходи за разгръщане.
- Разгърнете вашето приложение.

## Локална разработка и разгръщане

Ако вашият сървър е предназначен да се използва, работейки на машината на потребителите, можете да следвате следните стъпки:

1. **Изтеглете сървъра**. Ако не сте писали сървъра, първо го изтеглете на своята машина.
1. **Стартирайте процеса на сървъра**: Стартирайте вашето MCP сървър приложение.

За SSE (не е необходимо за stdio тип сървър):

1. **Конфигурирайте мрежата**: Уверете се, че сървърът е достъпен на очаквания порт.
1. **Свържете клиенти**: Използвайте локални връзки като `http://localhost:3000`.

## Разгръщане в облака

MCP сървъри могат да бъдат разгърнати на различни облачни платформи:

- **Serverless функции**: Разгръщайте леки MCP сървъри като безсървърни функции.
- **Контейнерни услуги**: Използвайте услуги като Azure Container Apps, AWS ECS или Google Cloud Run.
- **Kubernetes**: Разгръщайте и управлявайте MCP сървъри в Kubernetes клъстери за висока наличност.

### Пример: Azure Container Apps

Azure Container Apps поддържа разгръщането на MCP сървъри. Това все още е в процес на разработка и в момента поддържа SSE сървъри.

Ето как можете да го направите:

1. Клонирайте репо:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. Стартирайте го локално, за да направите тест:

  ```sh
  uv venv
  uv sync

  # linux/macOS
  export API_KEYS=<AN_API_KEY>
  # windows
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. За да го опитате локално, създайте файл *mcp.json* в директория *.vscode* и добавете следното съдържание:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  След като SSE сървърът е стартиран, можете да кликнете върху иконата за пускане в JSON файла, трябва вече да виждате инструментите на сървъра, които се разпознават от GitHub Copilot, вижте иконата на инструмент.

1. За да разположите, изпълнете следната команда:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

Ето, готово, разгърнете го локално или го разположете в Azure чрез тези стъпки.
