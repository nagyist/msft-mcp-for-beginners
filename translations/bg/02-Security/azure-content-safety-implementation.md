# Изпълнение на Azure Content Safety с MCP

> **OWASP MCP адресиран риск**: [MCP06 - Вмъкване на подканващ текст чрез контекстуални полезни товари](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

За засилване на сигурността на MCP срещу вмъкване на подканващ текст, отравяне на инструменти и други специфични уязвимости, свързани с изкуствения интелект, се препоръчва интеграция с Azure Content Safety. Това ръководство за изпълнение е в съответствие с [MCP Security Summit Workshop (Sherpa)](https://azure-samples.github.io/sherpa/) лагера 3: I/O сигурност.

## Интеграция с MCP сървър

За да интегрирате Azure Content Safety със своя MCP сървър, добавете филтъра за безопасност на съдържанието като междинен софтуер в конвейера за обработка на заявки:

1. Инициализирайте филтъра при стартиране на сървъра
2. Валидация на всички входящи заявки за инструменти преди обработка
3. Проверкa на всички изходящи отговори преди връщането им към клиентите
4. Водене на регистрация и подаване на сигнали при нарушения на безопасността
5. Изпълнение на подходяща обработка на грешки при неуспешни проверки за безопасност на съдържанието

Това осигурява здрава защита срещу:
- Атаки с вмъкване на подканващ текст
- Опити за отравяне на инструменти
- Изтичане на данни чрез зловредни входове
- Генериране на вредно съдържание

## Най-добри практики за интеграция на Azure Content Safety

1. **Персонализирани блок-листи**: Създайте персонализирани блок-листи специално за MCP модели на вмъкване
2. **Настройка на тежестта**: Регулирайте праговете за тежест според конкретния случай и толерантността към риск
3. **Цялостно покритие**: Приложете проверки за безопасност на съдържанието за всички входове и изходи
4. **Оптимизация на производителността**: Помислете за използване на кеширане за повторни проверки на безопасността на съдържанието
5. **Механизми на резервно действие**: Определете ясни резервни поведения, когато услугите за безопасност на съдържанието са недостъпни
6. **Обратна връзка към потребителите**: Осигурете ясна обратна връзка за потребителите, когато съдържанието е блокирано поради опасения за безопасността
7. **Непрекъснато подобряване**: Редовно актуализирайте блок-листите и моделите въз основа на нововъзникващи заплахи
