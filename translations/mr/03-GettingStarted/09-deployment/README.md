# MCP सर्व्हर तैनात करणे

आपला MCP सर्व्हर तैनात केल्यामुळे इतर लोक त्याच्या साधनांचा आणि स्त्रोतांचा आपल्या स्थानिक पर्यावरणाबाहेर प्रवेश करू शकतात. आपल्या प्रमाणात वाढ, विश्वसनीयता, आणि व्यवस्थापनाच्या सुलभीकरणासाठी आवश्यकतेनुसार तैनातीच्या विविध धोरणांचा विचार केला जाऊ शकतो. खाली आपल्याला MCP सर्व्हर स्थानिक पातळीवर, कंटेनरमध्ये, आणि क्लाऊडवर तैनात करण्यासाठी मार्गदर्शन मिळेल.

## आढावा

हा धडा कसा MCP सर्व्हर अ‍ॅप तैनात करायचा यावर केंद्रित आहे.

## शिकण्याची उद्दिष्टे

या धड्याच्या शेवटी, आपण करू शकाल:

- विविध तैनातीच्या दृष्टिकोनांचे मूल्यांकन करणे.
- आपला अ‍ॅप तैनात करणे.

## स्थानिक विकास आणि तैनाती

जर आपला सर्व्हर वापरकर्त्यांच्या मशीनवर चालवल्या जाणाऱ्या सार्व्हर म्हणून वापरायचा असेल, तर आपण खालील पद्धती अनुसरू शकता:

1. **सर्व्हर डाउनलोड करा**. आपण सर्व्हर लिहिला नसेल तर प्रथम तो आपल्या मशीनवर डाउनलोड करा.  
1. **सर्व्हर प्रक्रिया सुरू करा**: आपला MCP सर्व्हर अ‍ॅप्लिकेशन चालवा

SSE (stdio प्रकारच्या सर्व्हरसाठी आवश्यक नाही)

1. **नेटवर्किंग कॉन्फिगर करा**: निश्चित करा की सर्व्हर अपेक्षित पोर्टवर प्रवेशयोग्य आहे  
1. **क्लायंट कनेक्ट करा**: `http://localhost:3000` सारख्या स्थानिक कनेक्शन URL वापरा

## क्लाऊड तैनाती

MCP सर्व्हर अनेक क्लाऊड प्लॅटफॉर्मवर तैनात केले जाऊ शकतात:

- **सर्व्हरलेस फंक्शन्स**: हलके MCP सर्व्हर सर्व्हरलेस फंक्शन्स म्हणून तैनात करा  
- **कंटेनर सेवा**: Azure Container Apps, AWS ECS, किंवा Google Cloud Run सारख्या सेवांचा वापर करा  
- **कुबेरनेट्स**: उच्च उपलब्धतेसाठी कुबेरनेट्स क्लस्टरमध्ये MCP सर्व्हर तैनात करा आणि व्यवस्थापित करा

### उदाहरण: Azure Container Apps

Azure Container Apps MCP सर्व्हरच्या तैनातीला समर्थन देते. हा अजूनही प्रगतीच्या टप्यात आहे आणि सध्या SSE सर्व्हरला समर्थन करते.

हे कसे करायचे ते इथे पाहू:

1. एका रेपोचे क्लोन करा:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. गोष्टी तपासण्यासाठी स्थानिकरित्या चालवा:

  ```sh
  uv venv
  uv sync

  # लिनक्स/मॅकOS
  export API_KEYS=<AN_API_KEY>
  # विंडोज
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. स्थानिकरित्या प्रयत्न करण्यासाठी, *.vscode* निर्देशिकेत *mcp.json* ही फाईल तयार करा आणि खालील सामग्री जोडा:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  SSE सर्व्हर सुरू झाल्यावर, आपण JSON फाईलमधील प्ले आयकॉनवर क्लिक करू शकता, आता GitHub Copilot द्वारे सर्व्हरवरील साधने ओळखली जातील, टूल आयकॉन पहा.

1. तैनात करण्यासाठी खालील आदेश चालवा:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

बरं आहे, स्थानिकरित्या तैनात करा, किंवा Azure वर या पायऱ्यांद्वारे तैनात करा.
