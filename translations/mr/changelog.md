## जानेवारी २८, २०२६

### MCP स्पेसिफिकेशन २०२५-११-२५ अनुपालन पुनरावलोकन

#### मूलभूत संकल्पना सुधारणा (01-CoreConcepts/)
- **नवीन क्लायंट प्रिमिटिव्ह - Roots**: Roots क्लायंट प्रिमिटिव्हवर सविस्तर दस्तऐवज जोडला, ज्यामुळे सर्व्हरना फाइलसिस्टम सीमारेषा आणि प्रवेश परवानग्या समजतात
- **टूल अ‍ॅनोटेशन्स**: टूल व्यवहारात्मक अ‍ॅनोटेशन्स (`readOnlyHint`, `destructiveHint`) संदर्भातील दस्तऐवज वाढवले, ज्यामुळे टूल अंमलबजावणी निर्णय सुधारतात
- **सॅम्पलिंगमध्ये टूल कॉलिंग**: सॅम्पलिंग दस्तऐवजात `tools` आणि `toolChoice` पॅरामीटर्स जोडले, जे सॅम्पलिंग विनंत्यांमध्ये मॉडल-चालित टूल कॉलना सक्षम करतात
- **URL मोड उद्घाटन**: सर्व्हर-प्रेरित बाह्य वेब संवादासाठी URL-आधारित उद्घाटनावर दस्तऐवज जोडले
- **टास्क (प्रायोगिक)**: टास्क वैशिष्ट्यावर प्रायोगिक विभाग जोडला, जो टिकाऊ अंमलबजावणी आवरणे आणि विलंबित निकाल पुनर्प्राप्तीसाठी आहे
- **आयकॉन समर्थन**: टूल्स, स्त्रोत, स्त्रोत टेम्प्लेट्स आणि प्रॉम्प्ट्समध्ये आता अतिरिक्त मेटाडेटा म्हणून आयकॉनस समाविष्ट करू शकतात

#### दस्तऐवज अद्यतन
- **README.md**: MCP स्पेसिफिकेशन २०२५-११-२५ आवृत्ती संदर्भ आणि तारखेवर आधारित आवृत्तीकरणाचे स्पष्टीकरण जोडले
- **study_guide.md**: मूलभूत संकल्पना विभागात टास्क आणि टूल अ‍ॅनोटेशन्स समाविष्ट करिकुलम नकाशा अद्यतनित; दस्तऐवज_TIMESTAMP अद्यतनित

#### स्पेसिफिकेशन अनुपालन पडताळणी
- **प्रोटोकॉल आवृत्ती**: सर्व दस्तऐवज MCP स्पेसिफिकेशन २०२५-११-२५ ची सध्याची आवृत्ती दाखवतात याची खात्री केली
- **आर्किटेक्चर संरेखन**: दोन-स्तरीय आर्किटेक्चर (डेटा लेयर + ट्रान्सपोर्ट लेयर) दस्तऐवज अचूकता पुष्टी केली
- **प्रिमिटिव्ह दस्तऐवज**: सर्व्हरसाठी प्रिमिटिव्हस (Resources, Prompts, Tools) आणि क्लायंटसाठी प्रिमिटिव्हस (Sampling, Elicitation, Logging, Roots) पडताळले
- **ट्रान्सपोर्ट यंत्रणा**: STDIO आणि स्ट्रीम करण्यायोग्य HTTP ट्रान्सपोर्ट दस्तऐवज अचूकता पडताळली
- **सुरक्षा मार्गदर्शन**: सध्याच्या MCP सुरक्षा सर्वोत्तम पद्धती दस्तऐवजांसह संरेखित असल्याची पुष्टी केली

#### मुख्य MCP २०२५-११-२५ वैशिष्ट्ये दस्तऐवज
- **OpenID Connect डिस्कव्हरी**: OIDC द्वारे प्रमाणीकरण सर्व्हर शोध
- **OAuth क्लायंट आयडी मेटाडेटा दस्तऐवज**: शिफारस केलेली क्लायंट नोंदणी पद्धत
- **JSON स्कीमा २०२०-१२**: MCP स्कीमा व्याख्या साठी डीफॉल्ट डायलेक्ट
- **SDK टियरिंग सिस्टम**: SDK वैशिष्ट्य समर्थन आणि देखभाल यासाठी औपचारिक आवश्यकता
- **शासन रचना**: MCP शासनातील कार्य गट आणि हित गटांची औपचारिक रचना

#### प्रगत सर्व्हर वापर (03-GettingStarted/10-advanced)
- **नवीन अध्याय जोडला**: प्रगत MCP सर्व्हर वापराबाबत सविस्तर मार्गदर्शक, नियमित आणि लो-लेव्हल सर्व्हर आर्किटेक्चर समाविष्ट
  - **नियमित विरुद्ध लो-लेव्हल सर्व्हर**: दोन्ही पद्धतींसाठी Python आणि TypeScript मध्ये तपशीलवार तुलना आणि कोड उदाहरणे
  - **हँडलर-आधारित डिझाइन**: स्केलेबल, लवचीक सर्व्हर अंमलबजावणीसाठी टूल/स्त्रोत/प्रॉम्प्ट व्यवस्थापन स्पष्टीकरण
  - **व्यावहारिक पॅटर्न्स**: अशा वास्तव परिस्थिति जिथे लो-लेव्हल सर्व्हर पॅटर्न्स प्रगत वैशिष्ट्ये आणि आर्किटेक्चर साठी उपयुक्त

#### सोपी प्रमाणीकरण (03-GettingStarted/11-simple-auth)
- **नवीन अध्याय जोडला**: MCP सर्व्हरमध्ये सोप्या प्रमाणीकरणासाठी चरण-दर-चरण मार्गदर्शक
  - **ऑथ संकल्पना**: प्रमाणीकरण विरुद्ध प्राधिकरण यांचे स्पष्ट स्पष्टीकरण, आणि क्रेडेन्शियल हाताळणी
  - **बेसिक ऑथ अंमलबजावणी**: Python (Starlette) आणि TypeScript (Express) मध्ये मिडलवेअर-आधारित प्रमाणीकरण पॅटर्न्स, कोड उदाहरणांसह
  - **प्रगत सुरक्षेकडे प्रगती**: सोप्या ऑथ पासून OAuth 2.1 आणि RBAC कडे कसे पुढे जायचे यासाठी मार्गदर्शन, प्रगत सुरक्षा मॉड्यूल संदर्भांसह

ही भर घालणे अधिक मजबूत, सुरक्षित, आणि लवचीक MCP सर्व्हर अंमलबजावण्या तयार करण्यासाठी व्यावहारिक, हाताळणी मार्गदर्शन पुरवते, मूलभूत संकल्पनाकडे प्रगत उत्पादन नमुन्यांचे पूल बांधताना.

  - सर्व्हर प्रिमिटिव्ह्स: संसाधने (डेटा स्त्रोत), प्रम्प्ट्स (साचे), साधने (अंमलात येणारे फंक्शन्स) तपशीलवार स्पष्टीकरणे आणि उदाहरणांसह
  - क्लायंट प्रिमिटिव्ह्स: सॅम्पलिंग (LLM पूर्णता), प्रश्न विचारणे (वापरकर्ता इनपुट), लॉगिंग (डीबगिंग/मॉनिटरिंग)
  - सध्याच्या शोध (`*/list`), पुनर्प्राप्ति (`*/get`), आणि अंमलबजावणी (`*/call`) पद्धतींनी अद्ययावत
- **प्रोटोकॉल आर्किटेक्चर**: दोन स्तरांची आर्किटेक्चर मॉडेल प्रस्तावित
  - डेटा स्तर: JSON-RPC 2.0 पाया जीवनचक्र व्यवस्थापन आणि प्रिमिटिव्ह्ससह
  - ट्रान्सपोर्ट स्तर: STDIO (स्थानिक) आणि Streamable HTTP सह SSE (दूरस्थ) ट्रान्सपोर्ट यंत्रणा
- **सुरक्षा चौकट**: व्यापक सुरक्षा तत्त्वांमध्ये स्पष्ट वापरकर्ता संमती, डेटा गोपनीयता संरक्षण, साधन अंमलबजावणी सुरक्षा, आणि ट्रान्सपोर्ट स्तर सुरक्षा समाविष्ट
- **संचार पॅटर्न्स**: प्रारंभिकरण, शोध, अंमलबजावणी, आणि सूचना प्रवाह दर्शविण्यासाठी प्रोटोकॉल संदेश अद्ययावत
- **कोड उदाहरणे**: बहुभाषी उदाहरणे (.NET, Java, Python, JavaScript) सध्याच्या MCP SDK पॅटर्न्सवर आधारित ताजी
#### सुरक्षा (02-Security/) - व्यापक सुरक्षा पुनर्रचना  
- **मानके सुसंगतता**: MCP तपशील 2025-06-18 सुरक्षा आवश्यकता पूर्ण सुसंगतता
- **प्रमाणीकरण विकास**: कस्टम OAuth सर्व्हरपासून बाह्य ओळख प्रदाता प्रतिनिधिमत्त्व (Microsoft Entra ID) पर्यंत विकास दस्तऐवजीकरण
- **AI-विशिष्ट धोक्यांचे विश्लेषण**: आधुनिक AI हल्ला मार्गांचे विस्तृत कव्हरेज
  - प्रत्यक्ष उदाहरणांसह तपशीलवार प्रम्प्ट इंजेक्शन हल्ले
  - साधन विषबाधा यंत्रणा आणि "रग पुल" हल्ला पॅटर्न्स
  - संदर्भ विंडो विषबाधा आणि मॉडेल गोंधळ हल्ले
- **Microsoft AI सुरक्षा उपाय**: Microsoft सुरक्षा वातावरणाचे सखोल कव्हरेज
  - प्रगत शोध, स्पॉटलाइटिंग, आणि डेलिमीटर तंत्रांसह AI प्रम्प्ट शील्ड्स
  - Azure कंटेंट सेफ्टी एकत्रिकरण पॅटर्न्स
  - GitHub Advanced Security पुरवठा साखळी संरक्षणासाठी
- **प्रगत धोका प्रतिबंध**: तपशीलवार सुरक्षा नियंत्रणांसाठी
  - MCP-विशिष्ट हल्ला परिस्थितींसह सत्र हिजॅकिंग आणि क्रिप्टोग्राफिक सत्र ID आवश्यकताएं
  - MCP प्रॉक्सी परिस्थितींमध्ये गोंधळलेला डिप्युटी अडचणी स्पष्ट संमतीसह
  - टोकन पासथ्रू दुर्बलतांसाठी अनिवार्य पडताळणी नियंत्रण
- **पुरवठा साखळी सुरक्षा**: फाउंडेशन मॉडेल्स, एम्बेडिंग सेवा, संदर्भ प्रदाते, आणि तृतीय-पक्ष API समावेश करून AI पुरवठा साखळीचे विस्तृत आच्छादन
- **फाउंडेशन सुरक्षा**: एंटरप्राइझ सुरक्षा पॅटर्न्ससह समाकलन सुधारित, ज्यात झीरो ट्रस्ट आर्किटेक्चर आणि Microsoft सुरक्षा वातावरण समाविष्ट
- **संसाधन संघटन**: प्रकारानुसार व्यापक संसाधन लिंक्सचे वर्गीकरण (अधिकृत दस्तऐवज, मानके, संशोधन, Microsoft उपाय, अंमलबजावणी मार्गदर्शक)
### दस्तऐवज गुणवत्ता सुधारणा
