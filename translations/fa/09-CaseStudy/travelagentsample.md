# مطالعه موردی: عامل‌های مسافرتی هوش مصنوعی آزور – پیاده‌سازی مرجع

## مروری کلی

[عامل‌های مسافرتی هوش مصنوعی آزور](https://github.com/Azure-Samples/azure-ai-travel-agents) یک راه‌حل مرجع جامع توسعه یافته توسط مایکروسافت است که نشان می‌دهد چگونه می‌توان یک برنامه برنامه‌ریزی سفر چندعامله و مبتنی بر هوش مصنوعی را با استفاده از پروتکل زمینه مدل (MCP)، Azure OpenAI و Azure AI Search ساخت. این پروژه بهترین شیوه‌ها برای هماهنگی بین چند عامل هوش مصنوعی، یکپارچه‌سازی داده‌های سازمانی و ارائه یک پلتفرم امن و قابل گسترش برای سناریوهای دنیای واقعی را به نمایش می‌گذارد.

## ویژگی‌های کلیدی
- **هماهنگی چندعامله:** استفاده از MCP برای هماهنگی عوامل تخصصی (مانند عوامل پرواز، هتل و برنامه سفر) که برای انجام وظایف پیچیده برنامه‌ریزی سفر با هم همکاری می‌کنند.
- **یکپارچه‌سازی داده‌های سازمانی:** اتصال به Azure AI Search و منابع داده سازمانی دیگر برای ارائه اطلاعات به‌روز و مرتبط در توصیه‌های سفر.
- **معماری امن و مقیاس‌پذیر:** بهره‌گیری از خدمات آزور برای احراز هویت، مجوزدهی و استقرار مقیاس‌پذیر، مطابق با بهترین شیوه‌های امنیت سازمانی.
- **ابزارهای قابل گسترش:** پیاده‌سازی ابزارها و قالب‌های درخواست MCP قابل استفاده مجدد، که امکان سازگاری سریع با حوزه‌ها یا نیازهای کسب‌وکار جدید را فراهم می‌کند.
- **تجربه کاربری:** ارائه یک رابط مکالمه‌ای برای تعامل کاربران با عوامل مسافرتی، که توسط Azure OpenAI و MCP قدرت گرفته است.

## معماری
![Architecture](https://raw.githubusercontent.com/Azure-Samples/azure-ai-travel-agents/main/docs/ai-travel-agents-architecture-diagram.png)

### توضیح نمودار معماری

راه‌حل عامل‌های مسافرتی هوش مصنوعی آزور برای مدولار بودن، مقیاس‌پذیری و یکپارچه‌سازی امن چندین عامل هوش مصنوعی و منابع داده سازمانی طراحی شده است. اجزاء اصلی و جریان داده به شرح زیر هستند:

- **رابط کاربری:** کاربران از طریق رابط مکالمه‌ای (مانند گفت‌وگوی وب یا بات تیمز) با سیستم تعامل دارند، پرسش‌های خود را ارسال می‌کنند و توصیه‌های سفر را دریافت می‌کنند.
- **سرور MCP:** به عنوان هماهنگ‌کننده مرکزی عمل می‌کند، ورودی کاربر را دریافت می‌کند، زمینه را مدیریت می‌کند و اقدامات عوامل تخصصی (مانند FlightAgent، HotelAgent، ItineraryAgent) را از طریق پروتکل زمینه مدل هماهنگ می‌نماید.
- **عوامل هوش مصنوعی:** هر عامل مسئول یک حوزه خاص (پروازها، هتل‌ها، برنامه سفر) است و به عنوان یک ابزار MCP پیاده‌سازی شده است. عوامل از قالب‌های درخواست و منطق برای پردازش درخواست‌ها و تولید پاسخ‌ها استفاده می‌کنند.
- **خدمت Azure OpenAI:** درک و تولید زبان طبیعی پیشرفته را فراهم می‌کند و به عوامل امکان می‌دهد مقصود کاربر را تفسیر کرده و پاسخ‌های مکالمه‌ای تولید کنند.
- **Azure AI Search و داده‌های سازمانی:** عوامل برای دریافت اطلاعات به‌روز درباره پروازها، هتل‌ها و گزینه‌های سفر، از Azure AI Search و منابع داده سازمانی دیگر پرس‌وجو می‌کنند.
- **احراز هویت و امنیت:** با مایکروسافت Entra ID برای احراز هویت امن ادغام شده و کنترل‌های دسترسی با کمترین امتیاز را به تمام منابع اعمال می‌کند.
- **استقرار:** برای استقرار در برنامه‌های کانتینر آزور طراحی شده است، که مقیاس‌پذیری، نظارت و کارایی عملیاتی را تضمین می‌کند.

این معماری امکان هماهنگی بی‌وقفه بین چندین عامل هوش مصنوعی، یکپارچه‌سازی امن با داده‌های سازمانی و یک پلتفرم مقاوم و قابل گسترش برای ساخت راه‌حل‌های هوش مصنوعی خاص حوزه را فراهم می‌کند.

## شرح مرحله به مرحله نمودار معماری
تصور کنید در حال برنامه‌ریزی یک سفر بزرگ هستید و تیمی از دستیاران متخصص در هر جزئیات به شما کمک می‌کنند. سیستم عامل‌های مسافرتی هوش مصنوعی آزور به همین شکل کار می‌کند، با استفاده از بخش‌های مختلف (مانند اعضای تیم) که هرکدام وظیفه ویژه‌ای دارند. در اینجا نحوه هماهنگی همه چیز آمده است:

### رابط کاربری (UI):
این مثل میز پذیرش نماینده مسافرت شما است. جایی که شما (کاربر) سوال می‌پرسید یا درخواست می‌دهید، مانند «برايم یک پرواز به پاریس پیدا کن». این می‌تواند یک پنجره چت در وب‌سایت یا یک اپ پیام‌رسان باشد.

### سرور MCP (هماهنگ‌کننده):
سرور MCP مانند مدیری است که درخواست شما را در میز پذیرش می‌شنود و تصمیم می‌گیرد کدام متخصص باید هر بخش را مدیریت کند. این سرور مکالمه شما را پیگیری می‌کند و اطمینان حاصل می‌کند همه چیز به خوبی پیش می‌رود.

### عوامل هوش مصنوعی (دستیاران متخصص):
هر عامل متخصص در یک حوزه خاص است — یکی همه چیز را درباره پروازها می‌داند، دیگری درباره هتل‌ها و دیگری درباره برنامه‌ریزی سفر شما. وقتی درخواست سفر می‌دهید، سرور MCP درخواست شما را به عامل یا عوامل مناسب ارسال می‌کند. این عوامل از دانش و ابزارهای خود برای یافتن بهترین گزینه‌ها استفاده می‌کنند.

### خدمت Azure OpenAI (کارشناس زبان):
این مثل داشتن یک کارشناس زبان است که دقیقاً می‌داند شما چه می‌گویید، بدون توجه به نحوه بیان شما. این کارشناس به عوامل کمک می‌کند درخواست‌های شما را درک کرده و پاسخ‌های طبیعی و مکالمه‌ای تولید کنند.

### Azure AI Search و داده‌های سازمانی (کتابخانه اطلاعاتی):
تصور کنید یک کتابخانه عظیم و به‌روز دارید که همه اطلاعات سفر — از برنامه‌های پرواز گرفته تا وضعیت هتل‌ها و موارد دیگر — را دارد. عوامل این کتابخانه را برای یافتن دقیق‌ترین پاسخ‌ها جست‌وجو می‌کنند.

### احراز هویت و امنیت (نگهبان امنیت):
مثل نگهبانی که بررسی می‌کند چه کسانی اجازه ورود به مناطق خاص را دارند، این بخش اطمینان می‌دهد که فقط افراد و عوامل مجاز به اطلاعات حساس دسترسی پیدا کنند. این بخش داده‌های شما را امن و خصوصی نگه می‌دارد.

### استقرار روی برنامه‌های کانتینر آزور (ساختمان):
تمام این دستیاران و ابزارها در یک ساختمان امن و مقیاس‌پذیر (ابر) کار می‌کنند. این یعنی سیستم می‌تواند همزمان کاربران زیادی را مدیریت کند و همیشه هنگام نیاز در دسترس باشد.

## چطور همه این‌ها با هم کار می‌کنند:

شما با طرح سوالی در میز پذیرش (UI) شروع می‌کنید.
مدیر (سرور MCP) مشخص می‌کند کدام متخصص (عامل) باید به شما کمک کند.
متخصص با کمک کارشناس زبان (OpenAI) درخواست شما را درک کرده و با کتابخانه اطلاعات (AI Search) بهترین پاسخ را پیدا می‌کند.
نگهبان امنیت (احراز هویت) اطمینان می‌دهد همه چیز امن است.
همه اینها داخل ساختمانی قابل اعتماد و مقیاس‌پذیر (Azure Container Apps) اتفاق می‌افتد، تا تجربه شما روان و ایمن باشد.
این همکاری تیمی به سیستم اجازه می‌دهد سریع و امن به شما در برنامه‌ریزی سفر کمک کند، درست مثل تیمی از نمایندگان مسافرتی متخصص که در یک دفتر مدرن با هم کار می‌کنند!

## پیاده‌سازی فنی
- **سرور MCP:** میزبان منطق هماهنگ‌سازی اصلی، ارائه ابزارهای عامل و مدیریت زمینه برای گردش‌کارهای چندمرحله‌ای برنامه‌ریزی سفر.
- **عوامل:** هر عامل (مانند FlightAgent، HotelAgent) به عنوان یک ابزار MCP با قالب‌های درخواست و منطق خاص خود پیاده‌سازی شده است.
- **یکپارچه‌سازی آزور:** از Azure OpenAI برای درک زبان طبیعی و از Azure AI Search برای بازیابی داده‌ها استفاده می‌کند.
- **امنیت:** با Microsoft Entra ID برای احراز هویت یکپارچه شده و کنترل‌های دسترسی حداقلی به همه منابع اعمال می‌کند.
- **استقرار:** از استقرار روی برنامه‌های کانتینر آزور برای مقیاس‌پذیری و کارایی عملیاتی پشتیبانی می‌کند.

## نتایج و تاثیر
- نشان می‌دهد چگونه MCP می‌تواند برای هماهنگی چندین عامل هوش مصنوعی در یک سناریوی عملی و آماده تولید استفاده شود.
- توسعه راه‌حل را با ارائه الگوهای قابل استفاده مجدد برای هماهنگی عامل‌ها، یکپارچه‌سازی داده و استقرار امن سرعت می‌بخشد.
- به عنوان نقشه راهی برای ساخت برنامه‌های خاص حوزه و مبتنی بر هوش مصنوعی با استفاده از MCP و خدمات آزور عمل می‌کند.
