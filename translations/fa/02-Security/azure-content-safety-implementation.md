# پیاده‌سازی ایمنی محتوا Azure با MCP

> **ریسک OWASP MCP که برطرف شده است**: [MCP06 - تزریق پرامپت از طریق بارهای زمینه‌ای](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

برای تقویت امنیت MCP در برابر تزریق پرامپت، مسمومیت ابزار و سایر آسیب‌پذیری‌های خاص هوش مصنوعی، توصیه می‌شود Azure Content Safety را ادغام کنید. این راهنمای پیاده‌سازی با [کارگاه اجلاس امنیتی MCP (شرپا)](https://azure-samples.github.io/sherpa/) کمپ ۳: امنیت ورودی/خروجی هم‌راستا است.

## ادغام با سرور MCP

برای ادغام Azure Content Safety با سرور MCP خود، فیلتر ایمنی محتوا را به عنوان میان‌افزار در خط لوله پردازش درخواست‌های خود اضافه کنید:

۱. فیلتر را در زمان راه‌اندازی سرور مقداردهی اولیه کنید  
۲. همه درخواست‌های ابزار ورودی را قبل از پردازش اعتبارسنجی کنید  
۳. همه پاسخ‌های خروجی را قبل از بازگرداندن به مشتریان بررسی کنید  
۴. روی نقض‌های ایمنی ثبت و هشدار دهید  
۵. مدیریت خطای مناسب برای بررسی‌های ناموفق ایمنی محتوا پیاده‌سازی کنید  

این یک دفاع قوی در برابر موارد زیر فراهم می‌کند:  
- حملات تزریق پرامپت  
- تلاش‌های مسمومیت ابزار  
- استخراج داده از طریق ورودی‌های مخرب  
- تولید محتواهای مضر  

## بهترین شیوه‌ها برای ادغام Azure Content Safety

۱. **فهرست‌های مسدودسازی سفارشی**: فهرست‌های مسدودسازی سفارشی مخصوص الگوهای تزریق MCP ایجاد کنید  
۲. **تنظیم شدت**: آستانه‌های شدت را بر اساس مورد استفاده خاص و میزان تحمل ریسک خود تنظیم کنید  
۳. **پوشش جامع**: بررسی‌های ایمنی محتوا را روی همه ورودی‌ها و خروجی‌ها اعمال کنید  
۴. **بهینه‌سازی عملکرد**: به پیاده‌سازی کشینگ برای بررسی‌های ایمنی محتوا که تکراری هستند، فکر کنید  
۵. **مکانیزم‌های پشتیبان**: رفتارهای پشتیبان واضح داشته باشید زمانی که سرویس‌های ایمنی محتوا در دسترس نیستند  
۶. **بازخورد به کاربر**: بازخورد واضح به کاربران ارائه دهید زمانی که محتوا به دلیل نگرانی‌های ایمنی مسدود می‌شود  
۷. **بهبود مستمر**: به طور منظم فهرست‌ها و الگوها را براساس تهدیدات نوظهور به‌روزرسانی کنید
