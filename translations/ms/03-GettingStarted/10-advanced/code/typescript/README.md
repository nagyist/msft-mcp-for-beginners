# Jalankan sampel

## Pasang kebergantungan

```sh
npm install
```

## Bina ia

```sh
npm run build
```

## Jalankan ia

```sh
npm start
```

Anda sepatutnya melihat teks:

```text
Registering tools...
Starting server...
```

Hebat, itu bermakna ia bermula dengan betul.

## Uji pelayan

Uji keupayaan dengan arahan berikut:

```sh
npx @modelcontextprotocol/inspector node build/app.js
```

Ini sepatutnya membuka antara muka web alat pemeriksa.

Jalankan alat:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

Anda sepatutnya melihat respons yang serupa dengan:

```text
{
  "content": [
    {
      "type": "text",
      "text": "Tool add called with arguments: {\"a\":1,\"b\":2}, result: {\"content\":[{\"type\":\"text\",\"text\":\"3\"}]}"
    }
  ]
```

Cuba uji alat yang tidak wujud seperti "add2" dengan arahan ini:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add2 --tool-arg a=1 --tool-arg b=2
```

Anda kini sepatutnya melihat mesej ini menunjukkan bahawa pengesahan anda berfungsi:

```text
{
  "content": [],
  "error": {
    "code": "tool_not_found",
    "message": "Tool add2 not found."
  }
```

Juga cuba hantar parameter `c` yang sepatutnya ditolak oleh skema seperti berikut:

```sh
npx @modelcontextprotocol/inspector --cli node ./build/app.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg c=2
```

Anda kini sepatutnya melihat ralat "argumen tidak sah":

```text
{
  "content": [],
  "error": {
    "code": "invalid_arguments",
    "message": "Invalid arguments for tool add: [\n  {\n    \"code\": \"invalid_type\",\n    \"expected\": \"number\",\n    \"received\": \"undefined\",\n    \"path\": [\n      \"b\"\n    ],\n    \"message\": \"Required\"\n  }\n]"
  }
}
```
