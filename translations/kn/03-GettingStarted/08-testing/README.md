## ಪರೀಕ್ಷೆ ಮತ್ತು ಡಿಬಗಿಂಗ್

ನೀವು ನಿಮ್ಮ MCP ಸರ್ವರ್ ಅನ್ನು ಪರೀಕ್ಷಿಸುವ ಮೊದಲು, ಲಭ್ಯವಿರುವ ಸಾಧನಗಳು ಮತ್ತು ಡಿಬಗಿಂಗ್‌ಗಾಗಿ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಮುಖ್ಯ. ಪರಿಣಾಮಕಾರಿ ಪರೀಕ್ಷೆ ನಿಮ್ಮ ಸರ್ವರ್ ನಿರೀಕ್ಷಿತವಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುವುದನ್ನು ಖಚಿತಪಡಿಸುತ್ತದೆ ಮತ್ತು ಸಮಸ್ಯೆಗಳನ್ನು ತ್ವರಿತವಾಗಿ ಗುರುತಿಸಿ ಪರಿಹರಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ಕೆಳಗಿನ ವಿಭಾಗವು ನಿಮ್ಮ MCP ಅನುಷ್ಠಾನವನ್ನು ಮಾನ್ಯಗೊಳಿಸಲು ಶಿಫಾರಸು ಮಾಡಲಾದ ವಿಧಾನಗಳನ್ನು ವಿವರಿಸುತ್ತದೆ.

## ಅವಲೋಕನ

ಈ ಪಾಠವು ಸರಿಯಾದ ಪರೀಕ್ಷಾ ವಿಧಾನವನ್ನು ಆಯ್ಕೆಮಾಡುವುದು ಮತ್ತು ಅತ್ಯಂತ ಪರಿಣಾಮಕಾರಿ ಪರೀಕ್ಷಾ ಸಾಧನವನ್ನು ಹೇಗೆ ಬಳಸುವುದು ಎಂಬುದನ್ನು ಒಳಗೊಂಡಿದೆ.

## ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳು

ಈ ಪಾಠದ ಅಂತ್ಯಕ್ಕೆ, ನೀವು ಈ ಕೆಳಗಿನವುಗಳನ್ನು ಮಾಡಲು ಸಾಧ್ಯವಾಗುತ್ತದೆ:

- ಪರೀಕ್ಷೆಗಾಗಿ ವಿವಿಧ ವಿಧಾನಗಳನ್ನು ವರ್ಣಿಸುವುದು.
- ನಿಮ್ಮ ಕೋಡ್ ಅನ್ನು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ಪರೀಕ್ಷಿಸಲು ವಿಭಿನ್ನ ಸಾಧನಗಳನ್ನು ಬಳಸುವುದು.

## MCP ಸರ್ವರ್‌ಗಳನ್ನು ಪರೀಕ್ಷಿಸುವುದು

MCP ನಿಮ್ಮ ಸರ್ವರ್‌ಗಳನ್ನು ಪರೀಕ್ಷಿಸಲು ಮತ್ತು ಡಿಬಗ್ ಮಾಡಲು ಸಹಾಯ ಮಾಡುವ ಸಾಧನಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ:

- **MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್**: CLI ಸಾಧನವಾಗಿ ಮತ್ತು ದೃಶ್ಯ ಸಾಧನವಾಗಿ ಎರಡೂ ರೀತಿ ಚಾಲನೆ ಮಾಡಬಹುದಾದ ಕಮಾಂಡ್ ಲೈನ್ ಸಾಧನ.
- **ಮ್ಯಾನುಯಲ್ ಪರೀಕ್ಷೆ**: ನೀವು curl ನಂತಹ ಸಾಧನವನ್ನು ಬಳಸಿಕೊಂಡು ವೆಬ್ ವಿನಂತಿಗಳನ್ನು ನಡೆಸಬಹುದು, ಆದರೆ ಯಾವುದೇ HTTP ಚಾಲನೆ ಮಾಡಬಲ್ಲ ಸಾಧನ ಬಳಸಬಹುದು.
- **ಯೂನಿಟ್ ಪರೀಕ್ಷೆ**: ಸರ್ವರ್ ಮತ್ತು ಕ್ಲೈಂಟ್ ಎರಡರ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಪರೀಕ್ಷಿಸಲು ನಿಮ್ಮ ಇಷ್ಟದ ಪರೀಕ್ಷಾ ಫ್ರೇಮ್ವರ್ಕ್ ಅನ್ನು ಬಳಸಬಹುದು.

### MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಬಳಕೆ

ನಾವು ಈ ಸಾಧನದ ಬಳಕೆಯನ್ನು ಹಿಂದಿನ ಪಾಠಗಳಲ್ಲಿ ವಿವರಿಸಿದ್ದೇವೆ ಆದರೆ ಇದನ್ನು ಸ್ವಲ್ಪ ಉನ್ನತ ಮಟ್ಟದಲ್ಲಿ ಚರ್ಚಿಸೋಣ. ಇದು Node.js ನಲ್ಲಿ ನಿರ್ಮಿತ ಸಾಧನ ಮತ್ತು ನೀವು `npx` ಎಕ್ಸಿಕ್ಯೂಟಬಲ್ ಅನ್ನು ಕರೆದು ಬಳಸಬಹುದು, ಇದು ತಾತ್ಕಾಲಿಕವಾಗಿ ಸಾಧನವನ್ನು ಡೌನ್‌ಲೋಡ್ ಮಾಡಿ ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡುತ್ತದೆ ಮತ್ತು ನಿಮ್ಮ ವಿನಂತಿ ಪೂರ್ಣಗೊಂಡ ನಂತರ ಸ್ವತಃ ಸ್ವಚ್ಛಗೊಳಿಸುತ್ತದೆ.

[MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್](https://github.com/modelcontextprotocol/inspector) ನಿಮಗೆ ಸಹಾಯ ಮಾಡುತ್ತದೆ:

- **ಸರ್ವರ್ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಕಂಡುಹಿಡಿಯುವುದು**: ಲಭ್ಯವಿರುವ ಸಂಪನ್ಮೂಲಗಳು, ಸಾಧನಗಳು ಮತ್ತು ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಪತ್ತೆಹಚ್ಚುವುದು
- **ಸಾಧನ ಕಾರ್ಯಾಚರಣೆಯನ್ನು ಪರೀಕ್ಷಿಸುವುದು**: ವಿಭಿನ್ನ ಪರಿಮಾಣಗಳನ್ನು ಪ್ರಯತ್ನಿಸಿ ನೈಜ ಸಮಯದಲ್ಲಿ ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ನೋಡುವುದು
- **ಸರ್ವರ್ ಮೆಟಾಡೇಟಾವನ್ನು ವೀಕ್ಷಿಸುವುದು**: ಸರ್ವರ್ ಮಾಹಿತಿ, ಸ್ಕೀಮಾಗಳು ಮತ್ತು ಸಂರಚನೆಗಳನ್ನು ಪರಿಶೀಲಿಸುವುದು

ಸಾಧನದ ಸಾಮಾನ್ಯ ಚಾಲನೆ ಹೀಗಿರಬಹುದು:

```bash
npx @modelcontextprotocol/inspector node build/index.js
```

ಮೇಲಿನ ಕಮಾಂಡ್ ಒಂದು MCP ಮತ್ತು ಅದರ ದೃಶ್ಯ ಇಂಟರ್ಫೇಸ್ ಅನ್ನು ಪ್ರಾರಂಭಿಸಿ ನಿಮ್ಮ ಬ್ರೌಸರ್‌ನಲ್ಲಿ ಸ್ಥಳೀಯ ವೆಬ್ ಇಂಟರ್ಫೇಸ್ ಅನ್ನು ಪ್ರಾರಂಭಿಸುತ್ತದೆ. ನೀವು ನೋಂದಾಯಿತ MCP ಸರ್ವರ್‌ಗಳು, ಅವುಗಳ ಲಭ್ಯವಿರುವ ಸಾಧನಗಳು, ಸಂಪನ್ಮೂಲಗಳು ಮತ್ತು ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ಪ್ರದರ್ಶಿಸುವ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್ ಅನ್ನು ನೋಡಬಹುದು. ಈ ಇಂಟರ್ಫೇಸ್ ನಿಮಗೆ ಸಾಧನ ಕಾರ್ಯಾಚರಣೆಯನ್ನು ಸಂವಹನಾತ್ಮಕವಾಗಿ ಪರೀಕ್ಷಿಸಲು, ಸರ್ವರ್ ಮೆಟಾಡೇಟಾವನ್ನು ಪರಿಶೀಲಿಸಲು ಮತ್ತು ನೈಜ ಸಮಯದ ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ವೀಕ್ಷಿಸಲು ಅನುಕೂಲ ಮಾಡಿಕೊಡುತ್ತದೆ, ಇದರಿಂದ ನಿಮ್ಮ MCP ಸರ್ವರ್ ಅನುಷ್ಠಾನಗಳನ್ನು ಮಾನ್ಯಗೊಳಿಸುವುದು ಮತ್ತು ಡಿಬಗ್ ಮಾಡುವುದು ಸುಲಭವಾಗುತ್ತದೆ.

ಇದು ಹೀಗಿರಬಹುದು: ![Inspector](../../../../translated_images/kn/connect.141db0b2bd05f096.webp)

ನೀವು ಈ ಸಾಧನವನ್ನು CLI ಮೋಡ್‌ನಲ್ಲಿ ಕೂಡ ಚಾಲನೆ ಮಾಡಬಹುದು, ಆ ಸಂದರ್ಭದಲ್ಲಿ ನೀವು `--cli` ಗುಣಲಕ್ಷಣವನ್ನು ಸೇರಿಸುತ್ತೀರಿ. ಸರ್ವರ್‌ನ ಎಲ್ಲಾ ಸಾಧನಗಳನ್ನು ಪಟ್ಟಿ ಮಾಡುವ "CLI" ಮೋಡ್‌ನಲ್ಲಿ ಸಾಧನವನ್ನು ಚಾಲನೆ ಮಾಡುವ ಉದಾಹರಣೆ ಇಲ್ಲಿದೆ:

```sh
npx @modelcontextprotocol/inspector --cli node build/index.js --method tools/list
```

### ಮ್ಯಾನುಯಲ್ ಪರೀಕ್ಷೆ

ಸರ್ವರ್ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಪರೀಕ್ಷಿಸಲು ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಸಾಧನವನ್ನು ಚಾಲನೆ ಮಾಡುವುದರ ಹೊರತಾಗಿ, ಇನ್ನೊಂದು ಸಮಾನ ವಿಧಾನವೆಂದರೆ HTTP ಬಳಸಬಲ್ಲ ಕ್ಲೈಂಟ್ ಅನ್ನು ಚಾಲನೆ ಮಾಡುವುದು, ಉದಾಹರಣೆಗೆ curl.

curl ಬಳಸಿ, ನೀವು MCP ಸರ್ವರ್‌ಗಳನ್ನು ನೇರವಾಗಿ HTTP ವಿನಂತಿಗಳ ಮೂಲಕ ಪರೀಕ್ಷಿಸಬಹುದು:

```bash
# ಉದಾಹರಣೆ: ಪರೀಕ್ಷಾ ಸರ್ವರ್ ಮೆಟಾಡೇಟಾ
curl http://localhost:3000/v1/metadata

# ಉದಾಹರಣೆ: ಒಂದು ಸಾಧನವನ್ನು ಕಾರ್ಯಗತಗೊಳಿಸಿ
curl -X POST http://localhost:3000/v1/tools/execute \
  -H "Content-Type: application/json" \
  -d '{"name": "calculator", "parameters": {"expression": "2+2"}}'
```

ಮೇಲಿನ curl ಬಳಕೆಯಿಂದ ನೀವು ನೋಡಬಹುದು, ನೀವು ಸಾಧನದ ಹೆಸರು ಮತ್ತು ಅದರ ಪರಿಮಾಣಗಳನ್ನು ಒಳಗೊಂಡ ಪೇಲೋಡ್ ಬಳಸಿ ಸಾಧನವನ್ನು ಕರೆಸಲು POST ವಿನಂತಿಯನ್ನು ಬಳಸುತ್ತೀರಿ. ನಿಮಗೆ ಸೂಕ್ತವಾದ ವಿಧಾನವನ್ನು ಬಳಸಿಕೊಳ್ಳಿ. ಸಾಮಾನ್ಯವಾಗಿ CLI ಸಾಧನಗಳು ವೇಗವಾಗಿ ಬಳಸಲು ಅನುಕೂಲಕರವಾಗಿದ್ದು, ಸ್ಕ್ರಿಪ್ಟ್ ಮಾಡಲು ಸಹಾಯಕವಾಗುತ್ತವೆ, ಇದು CI/CD ಪರಿಸರದಲ್ಲಿ ಉಪಯುಕ್ತವಾಗಬಹುದು.

### ಯೂನಿಟ್ ಪರೀಕ್ಷೆ

ನಿಮ್ಮ ಸಾಧನಗಳು ಮತ್ತು ಸಂಪನ್ಮೂಲಗಳು ನಿರೀಕ್ಷಿತವಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಲು ಯೂನಿಟ್ ಪರೀಕ್ಷೆಗಳನ್ನು ರಚಿಸಿ. ಇಲ್ಲಿದೆ ಕೆಲವು ಉದಾಹರಣೆಯ ಪರೀಕ್ಷಾ ಕೋಡ್.

```python
import pytest

from mcp.server.fastmcp import FastMCP
from mcp.shared.memory import (
    create_connected_server_and_client_session as create_session,
)

# ಅಸಿಂಕ್ರೋನಸ್ ಪರೀಕ್ಷೆಗಳಿಗಾಗಿ ಸಂಪೂರ್ಣ ಮೋಡ್ಯೂಲ್ ಅನ್ನು ಗುರುತಿಸಿ
pytestmark = pytest.mark.anyio


async def test_list_tools_cursor_parameter():
    """Test that the cursor parameter is accepted for list_tools.

    Note: FastMCP doesn't currently implement pagination, so this test
    only verifies that the cursor parameter is accepted by the client.
    """

 server = FastMCP("test")

    # ಕೆಲವು ಪರೀಕ್ಷಾ ಉಪಕರಣಗಳನ್ನು ರಚಿಸಿ
    @server.tool(name="test_tool_1")
    async def test_tool_1() -> str:
        """First test tool"""
        return "Result 1"

    @server.tool(name="test_tool_2")
    async def test_tool_2() -> str:
        """Second test tool"""
        return "Result 2"

    async with create_session(server._mcp_server) as client_session:
        # ಕರ್ಸರ್ ಪ್ಯಾರಾಮೀಟರ್ ಇಲ್ಲದೆ ಪರೀಕ್ಷೆ (ಬಿಟ್ಟಿದೆ)
        result1 = await client_session.list_tools()
        assert len(result1.tools) == 2

        # cursor=None ಜೊತೆಗೆ ಪರೀಕ್ಷೆ
        result2 = await client_session.list_tools(cursor=None)
        assert len(result2.tools) == 2

        # ಸ್ಟ್ರಿಂಗ್ ಆಗಿರುವ ಕರ್ಸರ್ ಜೊತೆಗೆ ಪರೀಕ್ಷೆ
        result3 = await client_session.list_tools(cursor="some_cursor_value")
        assert len(result3.tools) == 2

        # ಖಾಲಿ ಸ್ಟ್ರಿಂಗ್ ಕರ್ಸರ್ ಜೊತೆಗೆ ಪರೀಕ್ಷೆ
        result4 = await client_session.list_tools(cursor="")
        assert len(result4.tools) == 2
    
```

ಮೇಲಿನ ಕೋಡ್ ಈ ಕೆಳಗಿನವುಗಳನ್ನು ಮಾಡುತ್ತದೆ:

- pytest ಫ್ರೇಮ್ವರ್ಕ್ ಅನ್ನು ಬಳಸುತ್ತದೆ, ಇದು ನಿಮಗೆ ಫಂಕ್ಷನ್‌ಗಳಾಗಿ ಪರೀಕ್ಷೆಗಳನ್ನು ರಚಿಸಲು ಮತ್ತು assert ಹೇಳಿಕೆಗಳನ್ನು ಬಳಸಲು ಅನುಮತಿಸುತ್ತದೆ.
- ಎರಡು ವಿಭಿನ್ನ ಸಾಧನಗಳೊಂದಿಗೆ MCP ಸರ್ವರ್ ಅನ್ನು ರಚಿಸುತ್ತದೆ.
- ಕೆಲವು ಶರತ್ತುಗಳು ಪೂರೈಸಲ್ಪಟ್ಟಿರುವುದನ್ನು ಪರಿಶೀಲಿಸಲು `assert` ಹೇಳಿಕೆಯನ್ನು ಬಳಸುತ್ತದೆ.

[ಪೂರ್ಣ ಫೈಲ್ ಇಲ್ಲಿ ನೋಡಿ](https://github.com/modelcontextprotocol/python-sdk/blob/main/tests/client/test_list_methods_cursor.py)

ಮೇಲಿನ ಫೈಲ್ ಆಧರಿಸಿ, ನೀವು ನಿಮ್ಮ ಸ್ವಂತ ಸರ್ವರ್ ಅನ್ನು ಪರೀಕ್ಷಿಸಿ ಸಾಮರ್ಥ್ಯಗಳು ಸರಿಯಾಗಿ ರಚಿಸಲ್ಪಟ್ಟಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಬಹುದು.

ಎಲ್ಲಾ ಪ್ರಮುಖ SDK ಗಳಿಗೂ ಸಮಾನ ಪರೀಕ್ಷಾ ವಿಭಾಗಗಳಿವೆ, ಆದ್ದರಿಂದ ನೀವು ನಿಮ್ಮ ಆಯ್ಕೆಮಾಡಿದ ರನ್‌ಟೈಮ್‌ಗೆ ಹೊಂದಿಕೊಳ್ಳಬಹುದು.
