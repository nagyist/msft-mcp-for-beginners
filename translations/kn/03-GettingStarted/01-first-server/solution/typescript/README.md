# ಈ ಮಾದರಿಯನ್ನು ಚಾಲನೆ ಮಾಡುವುದು

ನೀವು `uv` ಅನ್ನು ಸ್ಥಾಪಿಸಲು ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ ಆದರೆ ಅದು ಅಗತ್ಯವಿಲ್ಲ, [ಸೂಚನೆಗಳು](https://docs.astral.sh/uv/#highlights) ನೋಡಿ

## -1- ಅವಲಂಬನೆಗಳನ್ನು ಸ್ಥಾಪಿಸಿ

```bash
npm install
```

## -3- ಮಾದರಿಯನ್ನು ಚಾಲನೆ ಮಾಡಿ


```bash
npm run build
```

## -4- ಮಾದರಿಯನ್ನು ಪರೀಕ್ಷಿಸಿ

ಒಂದು ಟರ್ಮಿನಲ್‌ನಲ್ಲಿ ಸರ್ವರ್ ಚಾಲನೆಯಲ್ಲಿದ್ದಾಗ, ಮತ್ತೊಂದು ಟರ್ಮಿನಲ್ ತೆರೆಯಿರಿ ಮತ್ತು ಕೆಳಗಿನ ಆಜ್ಞೆಯನ್ನು ಚಾಲನೆ ಮಾಡಿ:

```bash
npm run inspector
```

ಇದು ಮಾದರಿಯನ್ನು ಪರೀಕ್ಷಿಸಲು ದೃಶ್ಯಾತ್ಮಕ ಇಂಟರ್ಫೇಸ್ ಹೊಂದಿರುವ ವೆಬ್ ಸರ್ವರ್ ಅನ್ನು ಪ್ರಾರಂಭಿಸಬೇಕು.

ಸರ್ವರ್ ಸಂಪರ್ಕಗೊಂಡ ನಂತರ: 

- ಉಪಕರಣಗಳನ್ನು ಪಟ್ಟಿ ಮಾಡಿ ಮತ್ತು `add` ಅನ್ನು 2 ಮತ್ತು 4 ಆರ್ಗ್ಯುಮೆಂಟ್‌ಗಳೊಂದಿಗೆ ಚಾಲನೆ ಮಾಡಿ, ಫಲಿತಾಂಶದಲ್ಲಿ 6 ಕಾಣಿಸಬೇಕು.
- ಸಂಪನ್ಮೂಲಗಳು ಮತ್ತು ಸಂಪನ್ಮೂಲ ಟೆಂಪ್ಲೇಟ್ಗೆ ಹೋಗಿ "greeting" ಅನ್ನು ಕರೆ ಮಾಡಿ, ಒಂದು ಹೆಸರು ಟೈಪ್ ಮಾಡಿ ಮತ್ತು ನೀವು ನೀಡಿದ ಹೆಸರಿನೊಂದಿಗೆ ಒಂದು ಸ್ವಾಗತ ಸಂದೇಶವನ್ನು ಕಾಣಬೇಕು.

### CLI ಮೋಡ್‌ನಲ್ಲಿ ಪರೀಕ್ಷೆ

ನೀವು ಚಾಲನೆ ಮಾಡಿದ ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ನಿಜವಾಗಿಯೂ Node.js ಅಪ್ಲಿಕೇಶನ್ ಆಗಿದ್ದು `mcp dev` ಅದರ ಸುತ್ತಲೂ ಇರುವ ರಾಪರ್ ಆಗಿದೆ.

ನೀವು ಕೆಳಗಿನ ಆಜ್ಞೆಯನ್ನು ಚಾಲನೆ ಮಾಡುವ ಮೂಲಕ ಅದನ್ನು ನೇರವಾಗಿ CLI ಮೋಡ್‌ನಲ್ಲಿ ಪ್ರಾರಂಭಿಸಬಹುದು:

```bash
npx @modelcontextprotocol/inspector --cli node ./build/index.js --method tools/list
```

ಇದು ಸರ್ವರ್‌ನಲ್ಲಿ ಲಭ್ಯವಿರುವ ಎಲ್ಲಾ ಉಪಕರಣಗಳನ್ನು ಪಟ್ಟಿ ಮಾಡುತ್ತದೆ. ನೀವು ಕೆಳಗಿನ ಔಟ್‌ಪುಟ್ ಅನ್ನು ಕಾಣಬೇಕು:

```text
{
  "tools": [
    {
      "name": "add",
      "description": "Add two numbers",
      "inputSchema": {
        "type": "object",
        "properties": {
          "a": {
            "title": "A",
            "type": "integer"
          },
          "b": {
            "title": "B",
            "type": "integer"
          }
        },
        "required": [
          "a",
          "b"
        ],
        "title": "addArguments"
      }
    }
  ]
}
```

ಒಂದು ಉಪಕರಣವನ್ನು ಕರೆ ಮಾಡಲು ಟೈಪ್ ಮಾಡಿ:

```bash
nnpx @modelcontextprotocol/inspector --cli node ./build/index.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

ನೀವು ಕೆಳಗಿನ ಔಟ್‌ಪುಟ್ ಅನ್ನು ಕಾಣಬೇಕು:

```text
{
  "content": [
    {
      "type": "text",
      "text": "3"
    }
  ],
  "isError": false
}
```

> [!TIP]
> ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಅನ್ನು ಬ್ರೌಸರ್‌ನಲ್ಲಿ ಚಾಲನೆ ಮಾಡುವುದಕ್ಕಿಂತ CLI ಮೋಡ್‌ನಲ್ಲಿ ಚಾಲನೆ ಮಾಡುವುದು ಸಾಮಾನ್ಯವಾಗಿ ಬಹಳ ವೇಗವಾಗಿದೆ.
> ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಬಗ್ಗೆ ಹೆಚ್ಚಿನ ಮಾಹಿತಿಗಾಗಿ [ಇಲ್ಲಿ](https://github.com/modelcontextprotocol/inspector) ಓದಿ.
