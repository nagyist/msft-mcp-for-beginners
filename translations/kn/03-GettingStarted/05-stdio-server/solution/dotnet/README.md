# MCP stdio ಸರ್ವರ್ - .NET ಪರಿಹಾರ

> **⚠️ ಪ್ರಮುಖ**: ಈ ಪರಿಹಾರವನ್ನು MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-06-18 ನಿಂದ ಶಿಫಾರಸು ಮಾಡಲಾದ **stdio ಸಾರಿಗೆ** ಬಳಕೆಗೆ ನವೀಕರಿಸಲಾಗಿದೆ. ಮೂಲ SSE ಸಾರಿಗೆ ನಿಷೇಧಿಸಲಾಗಿದೆ.

## ಅವಲೋಕನ

ಈ .NET ಪರಿಹಾರವು ಪ್ರಸ್ತುತ stdio ಸಾರಿಗೆಯನ್ನು ಬಳಸಿ MCP ಸರ್ವರ್ ಅನ್ನು ಹೇಗೆ ನಿರ್ಮಿಸುವುದೆಂದು ತೋರಿಸುತ್ತದೆ. stdio ಸಾರಿಗೆ ಸರಳ, ಹೆಚ್ಚು ಸುರಕ್ಷಿತ ಮತ್ತು ನಿಷೇಧಿತ SSE ವಿಧಾನಕ್ಕಿಂತ ಉತ್ತಮ ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ.

## ಪೂರ್ವಾಪೇಕ್ಷಿತಗಳು

- .NET 9.0 SDK ಅಥವಾ ನಂತರದ ಆವೃತ್ತಿ
- .NET ಡಿಪೆಂಡೆನ್ಸಿ ಇಂಜೆಕ್ಷನ್ ಬಗ್ಗೆ ಮೂಲಭೂತ ತಿಳಿವು

## ಸೆಟ್‌ಅಪ್ ಸೂಚನೆಗಳು

### ಹಂತ 1: ಅವಲಂಬನೆಗಳನ್ನು ಮರುಸ್ಥಾಪಿಸಿ

```bash
dotnet restore
```

### ಹಂತ 2: ಪ್ರಾಜೆಕ್ಟ್ ಅನ್ನು ನಿರ್ಮಿಸಿ

```bash
dotnet build
```

## ಸರ್ವರ್ ಚಾಲನೆ

stdio ಸರ್ವರ್ ಹಳೆಯ HTTP ಆಧಾರಿತ ಸರ್ವರ್‌ನಿಂದ ವಿಭಿನ್ನವಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ. ವೆಬ್ ಸರ್ವರ್ ಪ್ರಾರಂಭಿಸುವ ಬದಲು, ಇದು stdin/stdout ಮೂಲಕ ಸಂವಹನ ಮಾಡುತ್ತದೆ:

```bash
dotnet run
```

**ಪ್ರಮುಖ**: ಸರ್ವರ್ ಹ್ಯಾಂಗ್ ಆಗಿರುವಂತೆ ಕಾಣಬಹುದು - ಇದು ಸಾಮಾನ್ಯ! ಇದು stdin ನಿಂದ JSON-RPC ಸಂದೇಶಗಳನ್ನು ಕಾಯುತ್ತಿದೆ.

## ಸರ್ವರ್ ಪರೀಕ್ಷೆ

### ವಿಧಾನ 1: MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಬಳಸಿ (ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ)

```bash
npx @modelcontextprotocol/inspector dotnet run
```

ಇದು:
1. ನಿಮ್ಮ ಸರ್ವರ್ ಅನ್ನು ಉಪಪ್ರಕ್ರಿಯೆಯಾಗಿ ಪ್ರಾರಂಭಿಸುತ್ತದೆ
2. ಪರೀಕ್ಷೆಗಾಗಿ ವೆಬ್ ಇಂಟರ್ಫೇಸ್ ತೆರೆಯುತ್ತದೆ
3. ಎಲ್ಲಾ ಸರ್ವರ್ ಉಪಕರಣಗಳನ್ನು ಸಂವಹನಾತ್ಮಕವಾಗಿ ಪರೀಕ್ಷಿಸಲು ಅನುಮತಿಸುತ್ತದೆ

### ವಿಧಾನ 2: ನೇರ ಕಮಾಂಡ್ ಲೈನ್ ಪರೀಕ್ಷೆ

ನೀವು ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಅನ್ನು ನೇರವಾಗಿ ಪ್ರಾರಂಭಿಸಿ ಪರೀಕ್ಷೆ ಮಾಡಬಹುದು:

```bash
npx @modelcontextprotocol/inspector dotnet run --project .
```

### ಲಭ್ಯವಿರುವ ಉಪಕರಣಗಳು

ಸರ್ವರ್ ಈ ಉಪಕರಣಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ:

- **AddNumbers(a, b)**: ಎರಡು ಸಂಖ್ಯೆಗಳ ಮೊತ್ತವನ್ನು ಸೇರಿಸಿ
- **MultiplyNumbers(a, b)**: ಎರಡು ಸಂಖ್ಯೆಗಳ ಗುಣಾಕಾರ ಮಾಡಿ  
- **GetGreeting(name)**: ವೈಯಕ್ತಿಕೃತ ಸ್ವಾಗತ ಸಂದೇಶವನ್ನು ರಚಿಸಿ
- **GetServerInfo()**: ಸರ್ವರ್ ಬಗ್ಗೆ ಮಾಹಿತಿ ಪಡೆಯಿರಿ

### Claude ಡೆಸ್ಕ್‌ಟಾಪ್ ಬಳಕೆ

ಈ ಸರ್ವರ್ ಅನ್ನು Claude ಡೆಸ್ಕ್‌ಟಾಪ್ ಜೊತೆಗೆ ಬಳಸಲು, ನಿಮ್ಮ `claude_desktop_config.json` ಗೆ ಈ ಸಂರಚನೆಯನ್ನು ಸೇರಿಸಿ:

```json
{
  "mcpServers": {
    "example-stdio-server": {
      "command": "dotnet",
      "args": ["run", "--project", "path/to/server.csproj"]
    }
  }
}
```

```

├── Program.cs           # Main server setup and configuration
├── Tools.cs            # Tool implementations
├── server.csproj       # Project file with dependencies
├── server.sln         # Solution file
├── Properties/         # Project properties
└── README.md          # This file
```

## HTTP/SSE ನಿಂದ ಪ್ರಮುಖ ಭೇದಗಳು

**stdio ಸಾರಿಗೆ (ಪ್ರಸ್ತುತ):**
- ✅ ಸರಳ ಸೆಟ್‌ಅಪ್ - ವೆಬ್ ಸರ್ವರ್ ಅಗತ್ಯವಿಲ್ಲ
- ✅ ಉತ್ತಮ ಸುರಕ್ಷತೆ - HTTP ಎಂಡ್ಪಾಯಿಂಟ್‌ಗಳಿಲ್ಲ
- ✅ `WebApplication.CreateBuilder()` ಬದಲು `Host.CreateApplicationBuilder()` ಬಳಕೆ
- ✅ `WithHttpTransport()` ಬದಲು `WithStdioTransport()` ಬಳಕೆ
- ✅ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ ಬದಲು ಕಾನ್ಸೋಲ್ ಅಪ್ಲಿಕೇಶನ್
- ✅ ಉತ್ತಮ ಕಾರ್ಯಕ್ಷಮತೆ

**HTTP/SSE ಸಾರಿಗೆ (ನಿಷೇಧಿತ):**
- ❌ ASP.NET ಕೋರ್ ವೆಬ್ ಸರ್ವರ್ ಅಗತ್ಯವಿತ್ತು
- ❌ `app.MapMcp()` ರೌಟಿಂಗ್ ಸೆಟ್‌ಅಪ್ ಅಗತ್ಯವಿತ್ತು
- ❌ ಹೆಚ್ಚು ಸಂಕೀರ್ಣ ಸಂರಚನೆ ಮತ್ತು ಅವಲಂಬನೆಗಳು
- ❌ ಹೆಚ್ಚುವರಿ ಸುರಕ್ಷತಾ ಪರಿಗಣನೆಗಳು
- ❌ ಈಗ MCP 2025-06-18 ನಲ್ಲಿ ನಿಷೇಧಿಸಲಾಗಿದೆ

## ಅಭಿವೃದ್ಧಿ ವೈಶಿಷ್ಟ್ಯಗಳು

- **ಡಿಪೆಂಡೆನ್ಸಿ ಇಂಜೆಕ್ಷನ್**: ಸೇವೆಗಳು ಮತ್ತು ಲಾಗಿಂಗ್‌ಗೆ ಪೂರ್ಣ DI ಬೆಂಬಲ
- **ಸಂರಚಿತ ಲಾಗಿಂಗ್**: `ILogger<T>` ಬಳಸಿ stderr ಗೆ ಸರಿಯಾದ ಲಾಗಿಂಗ್
- **ಉಪಕರಣ ಗುಣಲಕ್ಷಣಗಳು**: `[McpServerTool]` ಗುಣಲಕ್ಷಣಗಳ ಮೂಲಕ ಸ್ವಚ್ಛ ಉಪಕರಣ ವ್ಯಾಖ್ಯಾನ
- **ಅಸಿಂಕ್ರೋನಸ್ ಬೆಂಬಲ**: ಎಲ್ಲಾ ಉಪಕರಣಗಳು ಅಸಿಂಕ್ರೋನಸ್ ಕಾರ್ಯಾಚರಣೆಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತವೆ
- **ದೋಷ ನಿರ್ವಹಣೆ**: ಸೌಮ್ಯ ದೋಷ ನಿರ್ವಹಣೆ ಮತ್ತು ಲಾಗಿಂಗ್

## ಅಭಿವೃದ್ಧಿ ಸಲಹೆಗಳು

- ಲಾಗಿಂಗ್‌ಗೆ `ILogger` ಬಳಸಿ (ನೇರವಾಗಿ stdout ಗೆ ಬರೆಯಬೇಡಿ)
- ಪರೀಕ್ಷೆ ಮಾಡುವ ಮೊದಲು `dotnet build` ಬಳಸಿ ನಿರ್ಮಿಸಿ
- ದೃಶ್ಯ ಡಿಬಗ್ಗಿಂಗ್‌ಗೆ ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಬಳಸಿ ಪರೀಕ್ಷೆ ಮಾಡಿ
- ಎಲ್ಲಾ ಲಾಗಿಂಗ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ stderr ಗೆ ಹೋಗುತ್ತದೆ
- ಸರ್ವರ್ ಸೌಮ್ಯ ಶಟ್‌ಡೌನ್ ಸಿಗ್ನಲ್‌ಗಳನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ

ಈ ಪರಿಹಾರವು ಪ್ರಸ್ತುತ MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಅನುಸರಿಸುತ್ತದೆ ಮತ್ತು .NET ಬಳಸಿ stdio ಸಾರಿಗೆ ಅನುಷ್ಠಾನಕ್ಕೆ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳನ್ನು ತೋರಿಸುತ್ತದೆ.
