# MCP stdio ಸರ್ವರ್ - ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್ ಪರಿಹಾರ

> **⚠️ ಪ್ರಮುಖ**: ಈ ಪರಿಹಾರವನ್ನು MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-06-18 ನಿಂದ ಶಿಫಾರಸು ಮಾಡಲಾದ **stdio ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್** ಬಳಕೆಗಾಗಿ ನವೀಕರಿಸಲಾಗಿದೆ. ಮೂಲ SSE ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ನಿಷೇಧಿಸಲಾಗಿದೆ.

## ಅವಲೋಕನ

ಈ ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್ ಪರಿಹಾರವು ಪ್ರಸ್ತುತ stdio ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಬಳಸಿ MCP ಸರ್ವರ್ ಅನ್ನು ಹೇಗೆ ನಿರ್ಮಿಸುವುದನ್ನು ತೋರಿಸುತ್ತದೆ. stdio ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಸರಳ, ಹೆಚ್ಚು ಸುರಕ್ಷಿತ ಮತ್ತು ನಿಷೇಧಿತ SSE ವಿಧಾನಕ್ಕಿಂತ ಉತ್ತಮ ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ.

## ಪೂರ್ವಾಪೇಕ್ಷೆಗಳು

- Node.js 18+ ಅಥವಾ ನಂತರದ ಆವೃತ್ತಿ
- npm ಅಥವಾ yarn ಪ್ಯಾಕೇಜ್ ಮ್ಯಾನೇಜರ್

## ಸೆಟ್‌ಅಪ್ ಸೂಚನೆಗಳು

### ಹಂತ 1: ಅವಲಂಬನೆಗಳನ್ನು ಸ್ಥಾಪಿಸಿ

```bash
npm install
```

### ಹಂತ 2: ಪ್ರಾಜೆಕ್ಟ್ ಅನ್ನು ನಿರ್ಮಿಸಿ

```bash
npm run build
```

## ಸರ್ವರ್ ಚಾಲನೆ

stdio ಸರ್ವರ್ ಹಳೆಯ SSE ಸರ್ವರ್‌ನಿಂದ ವಿಭಿನ್ನವಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ. ವೆಬ್ ಸರ್ವರ್ ಪ್ರಾರಂಭಿಸುವ ಬದಲು, ಇದು stdin/stdout ಮೂಲಕ ಸಂವಹನ ಮಾಡುತ್ತದೆ:

```bash
npm start
```

**ಪ್ರಮುಖ**: ಸರ್ವರ್ ಹ್ಯಾಂಗ್ ಆಗಿರುವಂತೆ ಕಾಣಬಹುದು - ಇದು ಸಾಮಾನ್ಯ! ಇದು stdin ನಿಂದ JSON-RPC ಸಂದೇಶಗಳನ್ನು ಕಾಯುತ್ತಿದೆ.

## ಸರ್ವರ್ ಪರೀಕ್ಷೆ

### ವಿಧಾನ 1: MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಬಳಸಿ (ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ)

```bash
npm run inspector
```

ಇದು:
1. ನಿಮ್ಮ ಸರ್ವರ್ ಅನ್ನು ಉಪಪ್ರಕ್ರಿಯೆಯಾಗಿ ಪ್ರಾರಂಭಿಸುತ್ತದೆ
2. ಪರೀಕ್ಷೆಗಾಗಿ ವೆಬ್ ಇಂಟರ್ಫೇಸ್ ತೆರೆಯುತ್ತದೆ
3. ಎಲ್ಲಾ ಸರ್ವರ್ ಉಪಕರಣಗಳನ್ನು ಸಂವಹನಾತ್ಮಕವಾಗಿ ಪರೀಕ್ಷಿಸಲು ಅನುಮತಿಸುತ್ತದೆ

### ವಿಧಾನ 2: ನೇರ ಕಮಾಂಡ್ ಲೈನ್ ಪರೀಕ್ಷೆ

ನೀವು ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಅನ್ನು ನೇರವಾಗಿ ಪ್ರಾರಂಭಿಸಿ ಪರೀಕ್ಷೆ ಮಾಡಬಹುದು:

```bash
npx @modelcontextprotocol/inspector node build/index.js
```

### ಲಭ್ಯವಿರುವ ಉಪಕರಣಗಳು

ಸರ್ವರ್ ಈ ಉಪಕರಣಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ:

- **add(a, b)**: ಎರಡು ಸಂಖ್ಯೆಗಳ ಮೊತ್ತವನ್ನು ಸೇರಿಸಿ
- **multiply(a, b)**: ಎರಡು ಸಂಖ್ಯೆಗಳ ಗುಣಾಕಾರ ಮಾಡಿ  
- **get_greeting(name)**: ವೈಯಕ್ತಿಕೃತ ಸ್ವಾಗತ ಸಂದೇಶವನ್ನು ರಚಿಸಿ
- **get_server_info()**: ಸರ್ವರ್ ಬಗ್ಗೆ ಮಾಹಿತಿ ಪಡೆಯಿರಿ

### Claude ಡೆಸ್ಕ್‌ಟಾಪ್ ಬಳಕೆ

ಈ ಸರ್ವರ್ ಅನ್ನು Claude ಡೆಸ್ಕ್‌ಟಾಪ್ ಜೊತೆಗೆ ಬಳಸಲು, ನಿಮ್ಮ `claude_desktop_config.json` ಗೆ ಈ ಸಂರಚನೆಯನ್ನು ಸೇರಿಸಿ:

```json
{
  "mcpServers": {
    "example-stdio-server": {
      "command": "node",
      "args": ["path/to/build/index.js"]
    }
  }
}
```

## SSE ನಿಂದ ಪ್ರಮುಖ ಭೇದಗಳು

**stdio ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ (ಪ್ರಸ್ತುತ):**
- ✅ ಸರಳ ಸೆಟ್‌ಅಪ್ - HTTP ಸರ್ವರ್ ಅಗತ್ಯವಿಲ್ಲ
- ✅ ಉತ್ತಮ ಸುರಕ್ಷತೆ - HTTP ಎಂಡ್‌ಪಾಯಿಂಟ್‌ಗಳಿಲ್ಲ
- ✅ ಉಪಪ್ರಕ್ರಿಯೆ ಆಧಾರಿತ ಸಂವಹನ
- ✅ stdin/stdout ಮೂಲಕ JSON-RPC
- ✅ ಉತ್ತಮ ಕಾರ್ಯಕ್ಷಮತೆ

**SSE ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ (ನಿಷೇಧಿತ):**
- ❌ Express ಸರ್ವರ್ ಸೆಟ್‌ಅಪ್ ಅಗತ್ಯವಿತ್ತು
- ❌ ಸಂಕೀರ್ಣ ರೌಟಿಂಗ್ ಮತ್ತು ಸೆಷನ್ ನಿರ್ವಹಣೆ ಬೇಕಾಗಿತ್ತು
- ❌ ಹೆಚ್ಚಿನ ಅವಲಂಬನೆಗಳು (Express, HTTP ಹ್ಯಾಂಡ್ಲಿಂಗ್)
- ❌ ಹೆಚ್ಚುವರಿ ಸುರಕ್ಷತಾ ಪರಿಗಣನೆಗಳು
- ❌ ಈಗ MCP 2025-06-18 ನಲ್ಲಿ ನಿಷೇಧಿಸಲಾಗಿದೆ

## ಅಭಿವೃದ್ಧಿ ಸಲಹೆಗಳು

- ಲಾಗಿಂಗ್‌ಗೆ `console.error()` ಬಳಸಿ (`console.log()` ಬಳಸಬೇಡಿ ಏಕೆಂದರೆ ಅದು stdout ಗೆ ಬರೆಯುತ್ತದೆ)
- ಪರೀಕ್ಷೆ ಮಾಡುವ ಮೊದಲು `npm run build` ಮೂಲಕ ನಿರ್ಮಿಸಿ
- ದೃಶ್ಯ ಡಿಬಗ್ಗಿಂಗ್‌ಗೆ ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಬಳಸಿ
- ಎಲ್ಲಾ JSON ಸಂದೇಶಗಳು ಸರಿಯಾಗಿ ಫಾರ್ಮ್ಯಾಟ್ ಆಗಿರಬೇಕು
- ಸರ್ವರ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ SIGINT/SIGTERM ನಲ್ಲಿ ಸೌಮ್ಯ ಶಟ್ಡೌನ್ ನಿಭಾಯಿಸುತ್ತದೆ

ಈ ಪರಿಹಾರವು ಪ್ರಸ್ತುತ MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಅನುಸರಿಸುತ್ತದೆ ಮತ್ತು ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್ ಬಳಸಿ stdio ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಅನುಷ್ಠಾನಕ್ಕೆ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳನ್ನು ತೋರಿಸುತ್ತದೆ.
