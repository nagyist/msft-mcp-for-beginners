# MCP stdio ಸರ್ವರ್ - ಪೈಥಾನ್ ಪರಿಹಾರ

> **⚠️ ಪ್ರಮುಖ**: ಈ ಪರಿಹಾರವನ್ನು MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-06-18 ನಿಂದ ಶಿಫಾರಸು ಮಾಡಲಾದ **stdio ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್** ಬಳಕೆಗಾಗಿ ನವೀಕರಿಸಲಾಗಿದೆ. ಮೂಲ SSE ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ನಿಷೇಧಿಸಲಾಗಿದೆ.

## ಅವಲೋಕನ

ಈ ಪೈಥಾನ್ ಪರಿಹಾರವು ಪ್ರಸ್ತುತ stdio ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಬಳಸಿ MCP ಸರ್ವರ್ ಅನ್ನು ಹೇಗೆ ನಿರ್ಮಿಸುವುದನ್ನು ತೋರಿಸುತ್ತದೆ. stdio ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಸರಳ, ಹೆಚ್ಚು ಸುರಕ್ಷಿತ ಮತ್ತು ನಿಷೇಧಿತ SSE ವಿಧಾನಕ್ಕಿಂತ ಉತ್ತಮ ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ.

## ಪೂರ್ವಾಪೇಕ್ಷಿತಗಳು

- ಪೈಥಾನ್ 3.8 ಅಥವಾ ನಂತರದ ಆವೃತ್ತಿ
- ಪ್ಯಾಕೇಜ್ ನಿರ್ವಹಣೆಗೆ `uv` ಅನ್ನು ಸ್ಥಾಪಿಸಲು ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ, [ಸೂಚನೆಗಳು](https://docs.astral.sh/uv/#highlights) ನೋಡಿ

## ಸೆಟ್‌ಅಪ್ ಸೂಚನೆಗಳು

### ಹಂತ 1: ವರ್ಚುವಲ್ ಪರಿಸರವನ್ನು ರಚಿಸಿ

```bash
python -m venv venv
```

### ಹಂತ 2: ವರ್ಚುವಲ್ ಪರಿಸರವನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ

**ವಿಂಡೋಸ್:**
```bash
venv\Scripts\activate
```

**ಮ್ಯಾಕ್‌ಒಎಸ್/ಲಿನಕ್ಸ್ನಲ್ಲಿ:**
```bash
source venv/bin/activate
```

### ಹಂತ 3: ಅವಲಂಬನೆಗಳನ್ನು ಸ್ಥಾಪಿಸಿ

```bash
pip install mcp
```

## ಸರ್ವರ್ ಚಾಲನೆ

stdio ಸರ್ವರ್ ಹಳೆಯ SSE ಸರ್ವರ್‌ನಿಂದ ವಿಭಿನ್ನವಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ. ವೆಬ್ ಸರ್ವರ್ ಪ್ರಾರಂಭಿಸುವ ಬದಲು, ಇದು stdin/stdout ಮೂಲಕ ಸಂವಹನ ಮಾಡುತ್ತದೆ:

```bash
python server.py
```

**ಪ್ರಮುಖ**: ಸರ್ವರ್ ಹ್ಯಾಂಗ್ ಆಗಿರುವಂತೆ ಕಾಣಬಹುದು - ಇದು ಸಾಮಾನ್ಯ! ಇದು stdin ನಿಂದ JSON-RPC ಸಂದೇಶಗಳನ್ನು ಕಾಯುತ್ತಿದೆ.

## ಸರ್ವರ್ ಪರೀಕ್ಷೆ

### ವಿಧಾನ 1: MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಬಳಸಿ (ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ)

```bash
npx @modelcontextprotocol/inspector python server.py
```

ಇದು:
1. ನಿಮ್ಮ ಸರ್ವರ್ ಅನ್ನು ಉಪಪ್ರಕ್ರಿಯೆಯಾಗಿ ಪ್ರಾರಂಭಿಸುತ್ತದೆ
2. ಪರೀಕ್ಷೆಗಾಗಿ ವೆಬ್ ಇಂಟರ್ಫೇಸ್ ತೆರೆಯುತ್ತದೆ
3. ಎಲ್ಲಾ ಸರ್ವರ್ ಉಪಕರಣಗಳನ್ನು ಸಂವಹನಾತ್ಮಕವಾಗಿ ಪರೀಕ್ಷಿಸಲು ಅನುಮತಿಸುತ್ತದೆ

### ವಿಧಾನ 2: ನೇರ JSON-RPC ಪರೀಕ್ಷೆ

ನೀವು ನೇರವಾಗಿ JSON-RPC ಸಂದೇಶಗಳನ್ನು ಕಳುಹಿಸುವ ಮೂಲಕ ಸಹ ಪರೀಕ್ಷೆ ಮಾಡಬಹುದು:

1. ಸರ್ವರ್ ಪ್ರಾರಂಭಿಸಿ: `python server.py`
2. JSON-RPC ಸಂದೇಶ ಕಳುಹಿಸಿ (ಉದಾಹರಣೆ):

```json
{"jsonrpc": "2.0", "id": 1, "method": "tools/list"}
```

3. ಸರ್ವರ್ ಲಭ್ಯವಿರುವ ಉಪಕರಣಗಳೊಂದಿಗೆ ಪ್ರತಿಕ್ರಿಯಿಸುತ್ತದೆ

### ಲಭ್ಯವಿರುವ ಉಪಕರಣಗಳು

ಸರ್ವರ್ ಈ ಉಪಕರಣಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ:

- **add(a, b)**: ಎರಡು ಸಂಖ್ಯೆಗಳ ಮೊತ್ತವನ್ನು ಸೇರಿಸಿ
- **multiply(a, b)**: ಎರಡು ಸಂಖ್ಯೆಗಳ ಗುಣಾಕಾರ ಮಾಡಿ  
- **get_greeting(name)**: ವೈಯಕ್ತಿಕೃತ ಸ್ವಾಗತ ಸಂದೇಶವನ್ನು ರಚಿಸಿ
- **get_server_info()**: ಸರ್ವರ್ ಬಗ್ಗೆ ಮಾಹಿತಿ ಪಡೆಯಿರಿ

### ಕ್ಲೋಡ್ ಡೆಸ್ಕ್‌ಟಾಪ್ ಬಳಸಿ ಪರೀಕ್ಷೆ

ಈ ಸರ್ವರ್ ಅನ್ನು ಕ್ಲೋಡ್ ಡೆಸ್ಕ್‌ಟಾಪ್ ಜೊತೆಗೆ ಬಳಸಲು, ನಿಮ್ಮ `claude_desktop_config.json` ಗೆ ಈ ಸಂರಚನೆಯನ್ನು ಸೇರಿಸಿ:

```json
{
  "mcpServers": {
    "example-stdio-server": {
      "command": "python",
      "args": ["path/to/server.py"]
    }
  }
}
```

## SSE ನಿಂದ ಪ್ರಮುಖ ಭೇದಗಳು

**stdio ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ (ಪ್ರಸ್ತುತ):**
- ✅ ಸರಳ ಸೆಟ್‌ಅಪ್ - ವೆಬ್ ಸರ್ವರ್ ಅಗತ್ಯವಿಲ್ಲ
- ✅ ಉತ್ತಮ ಸುರಕ್ಷತೆ - HTTP ಎಂಡ್ಪಾಯಿಂಟ್‌ಗಳಿಲ್ಲ
- ✅ ಉಪಪ್ರಕ್ರಿಯೆ ಆಧಾರಿತ ಸಂವಹನ
- ✅ stdin/stdout ಮೂಲಕ JSON-RPC
- ✅ ಉತ್ತಮ ಕಾರ್ಯಕ್ಷಮತೆ

**SSE ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ (ನಿಷೇಧಿತ):**
- ❌ HTTP ಸರ್ವರ್ ಸೆಟ್‌ಅಪ್ ಅಗತ್ಯವಿತ್ತು
- ❌ ವೆಬ್ ಫ್ರೇಮ್ವರ್ಕ್ (Starlette/FastAPI) ಬೇಕಾಗಿತ್ತು
- ❌ ಹೆಚ್ಚು ಸಂಕೀರ್ಣ ಮಾರ್ಗನಿರ್ದೇಶನ ಮತ್ತು ಸೆಷನ್ ನಿರ್ವಹಣೆ
- ❌ ಹೆಚ್ಚುವರಿ ಸುರಕ್ಷತಾ ಪರಿಗಣನೆಗಳು
- ❌ ಈಗ MCP 2025-06-18 ನಲ್ಲಿ ನಿಷೇಧಿಸಲಾಗಿದೆ

## ಡಿಬಗ್ ಸಲಹೆಗಳು

- ಲಾಗಿಂಗ್‌ಗೆ `stderr` ಬಳಸಿ (`stdout` ಅಲ್ಲ)
- ದೃಶ್ಯ ಡಿಬಗ್‌ಗಾಗಿ ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಬಳಸಿ ಪರೀಕ್ಷೆ ಮಾಡಿ
- ಎಲ್ಲಾ JSON ಸಂದೇಶಗಳು ಹೊಸ ಸಾಲಿನಲ್ಲಿ ಮುಗಿಯುವಂತೆ ಇರಲಿ
- ಸರ್ವರ್ ದೋಷರಹಿತವಾಗಿ ಪ್ರಾರಂಭವಾಗುತ್ತಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ

ಈ ಪರಿಹಾರವು ಪ್ರಸ್ತುತ MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಅನುಸರಿಸುತ್ತದೆ ಮತ್ತು stdio ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಅನುಷ್ಠಾನಕ್ಕೆ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳನ್ನು ತೋರಿಸುತ್ತದೆ.
