# ಕ್ಯಾಲ್ಕುಲೇಟರ್ HTTP ಸ್ಟ್ರೀಮಿಂಗ್ ಡೆಮೊ

ಈ ಪ್ರಾಜೆಕ್ಟ್ ಸರ್ವರ್-ಸೆಂಟ್ ಇವೆಂಟ್ಸ್ (SSE) ಬಳಸಿ HTTP ಸ್ಟ್ರೀಮಿಂಗ್ ಅನ್ನು Spring Boot WebFlux ಮೂಲಕ ಪ್ರದರ್ಶಿಸುತ್ತದೆ. ಇದು ಎರಡು ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಒಳಗೊಂಡಿದೆ:

- **ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಸರ್ವರ್**: ಲೆಕ್ಕಾಚಾರಗಳನ್ನು ನಿರ್ವಹಿಸುವ ಮತ್ತು SSE ಬಳಸಿ ಫಲಿತಾಂಶಗಳನ್ನು ಸ್ಟ್ರೀಮ್ ಮಾಡುವ ಪ್ರತಿಕ್ರಿಯಾತ್ಮಕ ವೆಬ್ ಸೇವೆ
- **ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಕ್ಲೈಂಟ್**: ಸ್ಟ್ರೀಮಿಂಗ್ ಎಂಡ್ಪಾಯಿಂಟ್ ಅನ್ನು ಉಪಯೋಗಿಸುವ ಕ್ಲೈಂಟ್ ಅಪ್ಲಿಕೇಶನ್

## ಪೂರ್ವಾಪೇಕ್ಷಿತಗಳು

- ಜಾವಾ 17 ಅಥವಾ ಹೆಚ್ಚಿನ ಆವೃತ್ತಿ
- ಮೇವೆನ್ 3.6 ಅಥವಾ ಹೆಚ್ಚಿನ ಆವೃತ್ತಿ

## ಪ್ರಾಜೆಕ್ಟ್ ರಚನೆ

```
java/
├── calculator-server/     # Spring Boot server with SSE endpoint
│   ├── src/main/java/com/example/calculatorserver/
│   │   ├── CalculatorServerApplication.java
│   │   └── CalculatorController.java
│   └── pom.xml
├── calculator-client/     # Spring Boot client application
│   ├── src/main/java/com/example/calculatorclient/
│   │   └── CalculatorClientApplication.java
│   └── pom.xml
└── README.md
```

## ಇದು ಹೇಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ

1. **ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಸರ್ವರ್** `/calculate` ಎಂಡ್ಪಾಯಿಂಟ್ ಅನ್ನು ಒದಗಿಸುತ್ತದೆ, ಇದು:
   - ಕ್ವೇರಿ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳನ್ನು ಸ್ವೀಕರಿಸುತ್ತದೆ: `a` (ಸಂಖ್ಯೆ), `b` (ಸಂಖ್ಯೆ), `op` (ಆಪರೇಶನ್)
   - ಬೆಂಬಲಿತ ಆಪರೇಶನ್‌ಗಳು: `add`, `sub`, `mul`, `div`
   - ಲೆಕ್ಕಾಚಾರದ ಪ್ರಗತಿ ಮತ್ತು ಫಲಿತಾಂಶದೊಂದಿಗೆ ಸರ್ವರ್-ಸೆಂಟ್ ಇವೆಂಟ್ಸ್ ಅನ್ನು ಹಿಂತಿರುಗಿಸುತ್ತದೆ

2. **ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಕ್ಲೈಂಟ್** ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕ ಹೊಂದಿ:
   - `7 * 5` ಲೆಕ್ಕಾಚಾರ ಮಾಡಲು ವಿನಂತಿ ಮಾಡುತ್ತದೆ
   - ಸ್ಟ್ರೀಮಿಂಗ್ ಪ್ರತಿಕ್ರಿಯೆಯನ್ನು ಉಪಯೋಗಿಸುತ್ತದೆ
   - ಪ್ರತಿ ಇವೆಂಟ್ ಅನ್ನು ಕಾನ್ಸೋಲ್‌ನಲ್ಲಿ ಮುದ್ರಿಸುತ್ತದೆ

## ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಚಾಲನೆ ಮಾಡುವುದು

### ಆಯ್ಕೆ 1: ಮೇವೆನ್ ಬಳಸಿ (ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ)

#### 1. ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಸರ್ವರ್ ಅನ್ನು ಪ್ರಾರಂಭಿಸಿ

ಟರ್ಮಿನಲ್ ತೆರೆಯಿರಿ ಮತ್ತು ಸರ್ವರ್ ಡೈರೆಕ್ಟರಿಯ ಕಡೆಗೆ ಹೋಗಿ:

```bash
cd calculator-server
mvn clean package
mvn spring-boot:run
```

ಸರ್ವರ್ `http://localhost:8080` ನಲ್ಲಿ ಪ್ರಾರಂಭವಾಗುತ್ತದೆ

ನೀವು ಈ ರೀತಿಯ ಔಟ್‌ಪುಟ್ ನೋಡಬಹುದು:
```

Netty started on port 8080 (http)
```

#### 2. ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಕ್ಲೈಂಟ್ ಅನ್ನು ಚಾಲನೆ ಮಾಡಿ

ಹೊಸ ಟರ್ಮಿನಲ್ ತೆರೆಯಿರಿ ಮತ್ತು ಕ್ಲೈಂಟ್ ಡೈರೆಕ್ಟರಿಯ ಕಡೆಗೆ ಹೋಗಿ:

```bash
cd calculator-client
mvn clean package
mvn spring-boot:run
```

ಕ್ಲೈಂಟ್ ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕ ಹೊಂದಿ ಲೆಕ್ಕಾಚಾರವನ್ನು ಮಾಡುತ್ತದೆ ಮತ್ತು ಸ್ಟ್ರೀಮಿಂಗ್ ಫಲಿತಾಂಶಗಳನ್ನು ಪ್ರದರ್ಶಿಸುತ್ತದೆ.

## ಸರ್ವರ್ ಅನ್ನು ಕೈಯಿಂದ ಪರೀಕ್ಷಿಸುವುದು

ನೀವು ವೆಬ್ ಬ್ರೌಸರ್ ಅಥವಾ ಕರ್ಳ್ ಬಳಸಿ ಸರ್ವರ್ ಅನ್ನು ಪರೀಕ್ಷಿಸಬಹುದು:

### ವೆಬ್ ಬ್ರೌಸರ್ ಬಳಸಿ:
ಭೇಟಿ ನೀಡಿ: `http://localhost:8080/calculate?a=10&b=5&op=add`

### ಕರ್ಳ್ ಬಳಸಿ:
```bash
curl "http://localhost:8080/calculate?a=10&b=5&op=add" -H "Accept: text/event-stream"
```

## ನಿರೀಕ್ಷಿತ ಔಟ್‌ಪುಟ್

ಕ್ಲೈಂಟ್ ಚಾಲನೆ ಮಾಡಿದಾಗ, ನೀವು ಈ ರೀತಿಯ ಸ್ಟ್ರೀಮಿಂಗ್ ಔಟ್‌ಪುಟ್ ನೋಡಬಹುದು:

```
event:info
data:Calculating: 7.0 mul 5.0

event:result
data:35.0
```

## ಬೆಂಬಲಿತ ಆಪರೇಶನ್‌ಗಳು

- `add` - ಸೇರಿಸುವಿಕೆ (a + b)
- `sub` - ಕಡಿತ (a - b)
- `mul` - ಗುಣಾಕಾರ (a * b)
- `div` - ಭಾಗಾಕಾರ (a / b, b = 0 ಆಗಿದ್ದರೆ NaN ಹಿಂತಿರುಗಿಸುತ್ತದೆ)

## API ರೆಫರೆನ್ಸ್

### GET /calculate

**ಪ್ಯಾರಾಮೀಟರ್‌ಗಳು:**
- `a` (ಅಗತ್ಯ): ಮೊದಲ ಸಂಖ್ಯೆ (ಡಬಲ್)
- `b` (ಅಗತ್ಯ): ಎರಡನೇ ಸಂಖ್ಯೆ (ಡಬಲ್)
- `op` (ಅಗತ್ಯ): ಆಪರೇಶನ್ (`add`, `sub`, `mul`, `div`)

**ಪ್ರತಿಕ್ರಿಯೆ:**
- ಕಂಟೆಂಟ್-ಟೈಪ್: `text/event-stream`
- ಲೆಕ್ಕಾಚಾರದ ಪ್ರಗತಿ ಮತ್ತು ಫಲಿತಾಂಶದೊಂದಿಗೆ ಸರ್ವರ್-ಸೆಂಟ್ ಇವೆಂಟ್ಸ್ ಅನ್ನು ಹಿಂತಿರುಗಿಸುತ್ತದೆ

**ಉದಾಹರಣೆಯ ವಿನಂತಿ:**
```
GET /calculate?a=7&b=5&op=mul HTTP/1.1
Host: localhost:8080
Accept: text/event-stream
```

**ಉದಾಹರಣೆಯ ಪ್ರತಿಕ್ರಿಯೆ:**
```
event: info
data: Calculating: 7.0 mul 5.0

event: result
data: 35.0
```

## ಸಮಸ್ಯೆ ಪರಿಹಾರ

### ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳು

1. **ಪೋರ್ಟ್ 8080 ಈಗಾಗಲೇ ಬಳಕೆಯಲ್ಲಿದೆ**
   - ಪೋರ್ಟ್ 8080 ಬಳಸುತ್ತಿರುವ ಇತರ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿಲ್ಲಿಸಿ
   - ಅಥವಾ `calculator-server/src/main/resources/application.yml` ನಲ್ಲಿ ಸರ್ವರ್ ಪೋರ್ಟ್ ಬದಲಾಯಿಸಿ

2. **ಸಂಪರ್ಕ ನಿರಾಕರಿಸಲಾಗಿದೆ**
   - ಕ್ಲೈಂಟ್ ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು ಸರ್ವರ್ ಚಾಲನೆಗೊಂಡಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
   - ಸರ್ವರ್ ಪೋರ್ಟ್ 8080 ನಲ್ಲಿ ಯಶಸ್ವಿಯಾಗಿ ಪ್ರಾರಂಭವಾಗಿದೆ ಎಂದು ಪರಿಶೀಲಿಸಿ

3. **ಪ್ಯಾರಾಮೀಟರ್ ಹೆಸರು ಸಮಸ್ಯೆಗಳು**
   - ಈ ಪ್ರಾಜೆಕ್ಟ್‌ನಲ್ಲಿ ಮೇವೆನ್ ಕಂಪೈಲರ್ ಕಾನ್ಫಿಗರೇಶನ್ `-parameters` ಫ್ಲ್ಯಾಗ್ ಜೊತೆಗೆ ಸೇರಿಸಲಾಗಿದೆ
   - ಪ್ಯಾರಾಮೀಟರ್ ಬಾಂಧನ ಸಮಸ್ಯೆಗಳು ಎದುರಾದರೆ, ಈ ಕಾನ್ಫಿಗರೇಶನ್ ಬಳಸಿ ಪ್ರಾಜೆಕ್ಟ್ ನಿರ್ಮಿಸಲಾಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ

### ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿಲ್ಲಿಸುವುದು

- ಪ್ರತಿ ಅಪ್ಲಿಕೇಶನ್ ಚಾಲನೆಗೊಂಡಿರುವ ಟರ್ಮಿನಲ್‌ನಲ್ಲಿ `Ctrl+C` ಒತ್ತಿ
- ಅಥವಾ ಬ್ಯಾಕ್ಗ್ರೌಂಡ್ ಪ್ರಕ್ರಿಯೆಯಾಗಿ ಚಾಲನೆಗೊಂಡಿದ್ದರೆ `mvn spring-boot:stop` ಬಳಸಿ
