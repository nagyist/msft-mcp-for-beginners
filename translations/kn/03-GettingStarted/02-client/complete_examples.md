# ಸಂಪೂರ್ಣ MCP ಕ್ಲೈಂಟ್ ಉದಾಹರಣೆಗಳು

ಈ ಡೈರೆಕ್ಟರಿ ವಿವಿಧ ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳಲ್ಲಿ MCP ಕ್ಲೈಂಟ್‌ಗಳ ಸಂಪೂರ್ಣ, ಕಾರ್ಯನಿರ್ವಹಿಸುವ ಉದಾಹರಣೆಗಳನ್ನು ಹೊಂದಿದೆ. ಪ್ರತಿ ಕ್ಲೈಂಟ್ ಮುಖ್ಯ README.md ಟ್ಯುಟೋರಿಯಲ್‌ನಲ್ಲಿ ವಿವರಿಸಲಾದ ಸಂಪೂರ್ಣ ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ಪ್ರದರ್ಶಿಸುತ್ತದೆ.

## ಲಭ್ಯವಿರುವ ಕ್ಲೈಂಟ್‌ಗಳು

### 1. ಜಾವಾ ಕ್ಲೈಂಟ್ (`client_example_java.java`)

- **ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್**: SSE (ಸರ್ವರ್-ಸೆಂಟ್ ಇವೆಂಟ್ಸ್) HTTP ಮೂಲಕ
- **ಲಕ್ಷ್ಯ ಸರ್ವರ್**: `http://localhost:8080`
- **ವೈಶಿಷ್ಟ್ಯಗಳು**:
  - ಸಂಪರ್ಕ ಸ್ಥಾಪನೆ ಮತ್ತು ಪಿಂಗ್
  - ಟೂಲ್ ಪಟ್ಟಿ
  - ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಕಾರ್ಯಾಚರಣೆಗಳು (ಸೇರಿಸು, ಕಡಿಮೆಮಾಡು, ಗುಣಿಸು, ಭಾಗಿಸು, ಸಹಾಯ)
  - ದೋಷ ನಿರ್ವಹಣೆ ಮತ್ತು ಫಲಿತಾಂಶ ಹೊರತೆಗೆಯುವುದು

**ನಡೆಸಲು:**

```bash
# ನಿಮ್ಮ MCP ಸರ್ವರ್ localhost:8080 ನಲ್ಲಿ ಚಾಲನೆಯಲ್ಲಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
javac client_example_java.java
java client_example_java
```

### 2. C# ಕ್ಲೈಂಟ್ (`client_example_csharp.cs`)

- **ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್**: ಸ್ಟ್ಡಿಯೋ (ಸ್ಟ್ಯಾಂಡರ್ಡ್ ಇನ್‌ಪುಟ್/ಔಟ್‌ಪುಟ್)
- **ಲಕ್ಷ್ಯ ಸರ್ವರ್**: ಡಾಟ್‌ನೆಟ್ ರನ್ ಮೂಲಕ ಸ್ಥಳೀಯ .NET MCP ಸರ್ವರ್
- **ವೈಶಿಷ್ಟ್ಯಗಳು**:
  - ಸ್ಟ್ಡಿಯೋ ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಮೂಲಕ ಸ್ವಯಂಚಾಲಿತ ಸರ್ವರ್ ಪ್ರಾರಂಭ
  - ಟೂಲ್ ಮತ್ತು ಸಂಪನ್ಮೂಲ ಪಟ್ಟಿ
  - ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಕಾರ್ಯಾಚರಣೆಗಳು
  - JSON ಫಲಿತಾಂಶ ಪಾರ್ಸಿಂಗ್
  - ಸಮಗ್ರ ದೋಷ ನಿರ್ವಹಣೆ

**ನಡೆಸಲು:**

```bash
dotnet run
```

### 3. ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್ ಕ್ಲೈಂಟ್ (`client_example_typescript.ts`)

- **ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್**: ಸ್ಟ್ಡಿಯೋ (ಸ್ಟ್ಯಾಂಡರ್ಡ್ ಇನ್‌ಪುಟ್/ಔಟ್‌ಪುಟ್)
- **ಲಕ್ಷ್ಯ ಸರ್ವರ್**: ಸ್ಥಳೀಯ Node.js MCP ಸರ್ವರ್
- **ವೈಶಿಷ್ಟ್ಯಗಳು**:
  - ಸಂಪೂರ್ಣ MCP ಪ್ರೋಟೋಕಾಲ್ ಬೆಂಬಲ
  - ಟೂಲ್, ಸಂಪನ್ಮೂಲ ಮತ್ತು ಪ್ರಾಂಪ್ಟ್ ಕಾರ್ಯಾಚರಣೆಗಳು
  - ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಕಾರ್ಯಾಚರಣೆಗಳು
  - ಸಂಪನ್ಮೂಲ ಓದುವುದು ಮತ್ತು ಪ್ರಾಂಪ್ಟ್ ಕಾರ್ಯಗತಗೊಳಿಸುವಿಕೆ
  - ಬಲವಾದ ದೋಷ ನಿರ್ವಹಣೆ

**ನಡೆಸಲು:**

```bash
# ಮೊದಲು ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್ ಅನ್ನು ಸಂಯೋಜಿಸಿ (ಅಗತ್ಯವಿದ್ದರೆ)
npm run build

# ನಂತರ ಕ್ಲೈಂಟ್ ಅನ್ನು ಚಾಲನೆ ಮಾಡಿ
npm run client
# ಅಥವಾ
node client_example_typescript.js
```

### 4. ಪೈಥಾನ್ ಕ್ಲೈಂಟ್ (`client_example_python.py`)

- **ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್**: ಸ್ಟ್ಡಿಯೋ (ಸ್ಟ್ಯಾಂಡರ್ಡ್ ಇನ್‌ಪುಟ್/ಔಟ್‌ಪುಟ್)  
- **ಲಕ್ಷ್ಯ ಸರ್ವರ್**: ಸ್ಥಳೀಯ ಪೈಥಾನ್ MCP ಸರ್ವರ್
- **ವೈಶಿಷ್ಟ್ಯಗಳು**:
  - ಕಾರ್ಯಾಚರಣೆಗಳಿಗೆ ಅಸಿಂಕ್/ಅವೇಟ್ ಮಾದರಿ
  - ಟೂಲ್ ಮತ್ತು ಸಂಪನ್ಮೂಲ ಅನ್ವೇಷಣೆ
  - ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಕಾರ್ಯಾಚರಣೆಗಳ ಪರೀಕ್ಷೆ
  - ಸಂಪನ್ಮೂಲ ವಿಷಯ ಓದುವುದು
  - ವರ್ಗ ಆಧಾರಿತ ಸಂಘಟನೆ

**ನಡೆಸಲು:**

```bash
python client_example_python.py
```

## ಎಲ್ಲಾ ಕ್ಲೈಂಟ್‌ಗಳಲ್ಲಿ ಸಾಮಾನ್ಯ ವೈಶಿಷ್ಟ್ಯಗಳು

ಪ್ರತಿ ಕ್ಲೈಂಟ್ ಅನುಷ್ಠಾನವು ತೋರಿಸುತ್ತದೆ:

1. **ಸಂಪರ್ಕ ನಿರ್ವಹಣೆ**
   - MCP ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕ ಸ್ಥಾಪನೆ
   - ಸಂಪರ್ಕ ದೋಷಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು
   - ಸರಿಯಾದ ಕ್ಲೀನಪ್ ಮತ್ತು ಸಂಪನ್ಮೂಲ ನಿರ್ವಹಣೆ

2. **ಸರ್ವರ್ ಅನ್ವೇಷಣೆ**
   - ಲಭ್ಯವಿರುವ ಟೂಲ್‌ಗಳ ಪಟ್ಟಿ
   - ಲಭ್ಯವಿರುವ ಸಂಪನ್ಮೂಲಗಳ ಪಟ್ಟಿ (ಬೆಂಬಲಿತಿದ್ದಲ್ಲಿ)
   - ಲಭ್ಯವಿರುವ ಪ್ರಾಂಪ್ಟ್‌ಗಳ ಪಟ್ಟಿ (ಬೆಂಬಲಿತಿದ್ದಲ್ಲಿ)

3. **ಟೂಲ್ ಕರೆಮಾಡುವುದು**
   - ಮೂಲ ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಕಾರ್ಯಾಚರಣೆಗಳು (ಸೇರಿಸು, ಕಡಿಮೆಮಾಡು, ಗುಣಿಸು, ಭಾಗಿಸು)
   - ಸರ್ವರ್ ಮಾಹಿತಿಗಾಗಿ ಸಹಾಯ ಕಮಾಂಡ್
   - ಸರಿಯಾದ ಆರ್ಗ್ಯುಮೆಂಟ್ ಪಾಸ್ ಮಾಡುವುದು ಮತ್ತು ಫಲಿತಾಂಶ ನಿರ್ವಹಣೆ

4. **ದೋಷ ನಿರ್ವಹಣೆ**
   - ಸಂಪರ್ಕ ದೋಷಗಳು
   - ಟೂಲ್ ಕಾರ್ಯಾಚರಣೆ ದೋಷಗಳು
   - ಸೌಮ್ಯ ವಿಫಲತೆ ಮತ್ತು ಬಳಕೆದಾರ ಪ್ರತಿಕ್ರಿಯೆ

5. **ಫಲಿತಾಂಶ ಪ್ರಕ್ರಿಯೆ**
   - ಪ್ರತಿಕ್ರಿಯೆಗಳಿಂದ ಪಠ್ಯ ವಿಷಯ ಹೊರತೆಗೆಯುವುದು
   - ಓದಲು ಸುಲಭವಾಗುವಂತೆ ಔಟ್‌ಪುಟ್ ಫಾರ್ಮ್ಯಾಟ್ ಮಾಡುವುದು
   - ವಿಭಿನ್ನ ಪ್ರತಿಕ್ರಿಯೆ ಫಾರ್ಮ್ಯಾಟ್‌ಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು

## ಪೂರ್ವಾಪೇಕ್ಷೆಗಳು

ಈ ಕ್ಲೈಂಟ್‌ಗಳನ್ನು ಚಾಲನೆ ಮಾಡುವ ಮೊದಲು, ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ:

1. **ಸಂಬಂಧಿತ MCP ಸರ್ವರ್ ಚಾಲನೆಯಲ್ಲಿದೆ** (`../01-first-server/` ನಿಂದ)
2. **ನಿಮ್ಮ ಆಯ್ದ ಭಾಷೆಗೆ ಅಗತ್ಯವಿರುವ ಅವಲಂಬನೆಗಳು ಸ್ಥಾಪಿಸಲಾಗಿದೆ**
3. **ಸರಿಯಾದ ನೆಟ್‌ವರ್ಕ್ ಸಂಪರ್ಕ** (HTTP ಆಧಾರಿತ ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್‌ಗಳಿಗೆ)

## ಅನುಷ್ಠಾನಗಳ ನಡುವಿನ ಪ್ರಮುಖ ವ್ಯತ್ಯಾಸಗಳು

| ಭಾಷೆ       | ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ | ಸರ್ವರ್ ಪ್ರಾರಂಭ | ಅಸಿಂಕ್ ಮಾದರಿ | ಪ್ರಮುಖ ಗ್ರಂಥಾಲಯಗಳು       |
|------------|----------------|----------------|--------------|--------------------------|
| ಜಾವಾ       | SSE/HTTP       | ಬಾಹ್ಯ          | ಸಿಂಕ್        | WebFlux, MCP SDK         |
| C#         | ಸ್ಟ್ಡಿಯೋ       | ಸ್ವಯಂಚಾಲಿತ    | ಅಸಿಂಕ್/ಅವೇಟ್ | .NET MCP SDK             |
| ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್ | ಸ್ಟ್ಡಿಯೋ       | ಸ್ವಯಂಚಾಲಿತ    | ಅಸಿಂಕ್/ಅವೇಟ್ | Node MCP SDK             |
| ಪೈಥಾನ್     | ಸ್ಟ್ಡಿಯೋ       | ಸ್ವಯಂಚಾಲಿತ    | ಅಸಿಂಕ್‌ಐಒ  | Python MCP SDK           |
| ರಸ್ಟ್       | ಸ್ಟ್ಡಿಯೋ       | ಸ್ವಯಂಚಾಲಿತ    | ಅಸಿಂಕ್/ಅವೇಟ್ | Rust MCP SDK, Tokio      |

## ಮುಂದಿನ ಹಂತಗಳು

ಈ ಕ್ಲೈಂಟ್ ಉದಾಹರಣೆಗಳನ್ನು ಅನ್ವೇಷಿಸಿದ ನಂತರ:

1. **ಕ್ಲೈಂಟ್‌ಗಳನ್ನು ಬದಲಾಯಿಸಿ** ಹೊಸ ವೈಶಿಷ್ಟ್ಯಗಳು ಅಥವಾ ಕಾರ್ಯಾಚರಣೆಗಳನ್ನು ಸೇರಿಸಲು
2. **ನಿಮ್ಮದೇ ಸರ್ವರ್ ರಚಿಸಿ** ಮತ್ತು ಈ ಕ್ಲೈಂಟ್‌ಗಳೊಂದಿಗೆ ಪರೀಕ್ಷಿಸಿ
3. **ವಿಭಿನ್ನ ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್‌ಗಳೊಂದಿಗೆ ಪ್ರಯೋಗ ಮಾಡಿ** (SSE ವಿರುದ್ಧ ಸ್ಟ್ಡಿಯೋ)
4. **MCP ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ಸಂಯೋಜಿಸುವ** ಹೆಚ್ಚು ಸಂಕೀರ್ಣ ಅಪ್ಲಿಕೇಶನ್ ನಿರ್ಮಿಸಿ

## ಸಮಸ್ಯೆ ಪರಿಹಾರ

### ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳು

1. **ಸಂಪರ್ಕ ನಿರಾಕರಿಸಲಾಗಿದೆ**: MCP ಸರ್ವರ್ ನಿರೀಕ್ಷಿತ ಪೋರ್ಟ್/ಪಥದಲ್ಲಿ ಚಾಲನೆಯಲ್ಲಿದೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
2. **ಮಾಡ್ಯೂಲ್ ಕಂಡುಬಂದಿಲ್ಲ**: ನಿಮ್ಮ ಭಾಷೆಗೆ ಅಗತ್ಯವಿರುವ MCP SDK ಅನ್ನು ಸ್ಥಾಪಿಸಿ
3. **ಅನುಮತಿ ನಿರಾಕರಿಸಲಾಗಿದೆ**: ಸ್ಟ್ಡಿಯೋ ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್‌ಗೆ ಫೈಲ್ ಅನುಮತಿಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
4. **ಟೂಲ್ ಕಂಡುಬಂದಿಲ್ಲ**: ಸರ್ವರ್ ನಿರೀಕ್ಷಿತ ಟೂಲ್‌ಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುತ್ತಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ

### ಡಿಬಗ್ ಸಲಹೆಗಳು

1. **ನಿಮ್ಮ MCP SDK ನಲ್ಲಿ ವಿವರವಾದ ಲಾಗಿಂಗ್ ಸಕ್ರಿಯಗೊಳಿಸಿ**
2. **ದೋಷ ಸಂದೇಶಗಳಿಗಾಗಿ ಸರ್ವರ್ ಲಾಗ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಿ**
3. **ಕ್ಲೈಂಟ್ ಮತ್ತು ಸರ್ವರ್ ನಡುವೆ ಟೂಲ್ ಹೆಸರುಗಳು ಮತ್ತು ಸಹಿ ಹೊಂದಿಕೆಯಾಗಿವೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ**
4. **ಮೊದಲು MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಬಳಸಿ ಸರ್ವರ್ ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ಪರಿಶೀಲಿಸಿ**
