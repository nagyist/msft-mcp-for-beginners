## ಪ್ರಾರಂಭಿಸೋಣ  

[![ನೀವು ಮೊತ್ತಮೊದಲ MCP ಸರ್ವರ್ ನಿರ್ಮಿಸಿ](../../../translated_images/kn/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(ಈ ಪಾಠದ ವೀಡಿಯೋವನ್ನು ನೋಡುವುದಕ್ಕಾಗಿ ಮೇಲಿನ ಚಿತ್ರವನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ)_

ಈ ವಿಭಾಗದಲ್ಲಿ ಹಲವಾರು ಪಾಠಗಳಿವೆ:

- **1 ನಿಮ್ಮ ಮೊಟ್ಟ ಮೊದಲ ಸರ್ವರ್**, ಈ ಮೊದಲ ಪಾಠದಲ್ಲಿ, ನೀವು ನಿಮ್ಮ ಮೊತ್ತಮೊದಲ ಸರ್ವರ್ ಅನ್ನು ಹೇಗೆ ರಚಿಸುವುದು ಹಾಗೂ ಇನ್ಸ್ಪೆಕ್ಟರ್ ಸಾಧನದಿಂದ ಅದನ್ನು ಪರಿಶೀಲಿಸುವುದು ಎನ್ನುತ್ತಿವೆ, ಇದು ನಿಮ್ಮ ಸರ್ವರ್ ಅನ್ನು ಪರೀಕ್ಷಿಸಿ ಡಿಬಗ್ ಮಾಡಲು ಉತ್ತಮ ವಿಧಾನ, [ಪಾಠಕ್ಕೆ](01-first-server/README.md)

- **2 ಕ್ಲೈಂಟ್**, ಈ ಪಾಠದಲ್ಲಿ, ನೀವು ನಿಮ್ಮ ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕದೊಡನೆ ಸಂವಹನ ಮಾಡಬಹುದಾದ ಕ್ಲೈಂಟ್ ಹೇಗೆ ಬರೆಯುವುದು ಎಂಬುದನ್ನು ಕಲಿಯುತ್ತೀರಿ, [ಪಾಠಕ್ಕೆ](02-client/README.md)

- **3 ಎಲ್‌ಎಲ್‌ಎಂ ಹೊಂದಿದ ಕ್ಲೈಂಟ್**, ಒಂದು ಇನ್ನಷ್ಟು ಉತ್ತಮ ವಿಧಾನ ಎಲೆಗಂಟಿಂಗ್ ಸಹಿತ ಕ್ಲೈಂಟ್ ಬರೆಯುವುದು, ಅದು ನಿಮ್ಮ ಸರ್ವರ್ ಜೊತೆ "ಸಂಬಂಧಿಸಬಲ್ಲದು" ಎಂದು ಕಲಿಯಿರಿ, [ಪಾಠಕ್ಕೆ](03-llm-client/README.md)

- **4 ಸರ್ವರ್ GitHub Copilot ಏಜೆಂಟ್ ಮೋಡ್ ಬಳಸಿ Visual Studio Codeನಲ್ಲಿ**. ಇಲ್ಲಿ, ನಾವು ನಮ್ಮ MCP ಸರ್ವರ್ ಅನ್ನು Visual Studio Code ಒಳಗಿಂದ ಹಾದು ಮುಟ್ಟುತ್ತಿರುವುದನ್ನು ನೋಡುತ್ತೇವೆ, [ಪಾಠಕ್ಕೆ](04-vscode/README.md)

- **5 stdio ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಸರ್ವರ್** stdio ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ MCP ಸರ್ವರ್-ಕ್ಲೈಂಟ್ ಸಂವಹನಕ್ಕೆ ಶಿಫಾರಸುಮಾಡಿದ ಮಾನದಂಡವಾಗಿದೆ, ಸುರಕ್ಷಿತ ಉಪಪ್ರಕ್ರಿಯೆ ಆಧಾರಿತ ಸಂವಹನವನ್ನು ಒದಗಿಸುತ್ತದೆ ಮತ್ತು ಪ್ರಕ್ರಿಯೆ ಬೇರ್ಪಾಡನ್ನು ಒಳಗೊಂಡಿದೆ [ಪಾಠಕ್ಕೆ](05-stdio-server/README.md)

- **6 MCP ಮೂಲಕ HTTP ಸ್ಟ್ರೀಮಿಂಗ್ (ಸ್ಟ್ರೀಮಬಲ್ HTTP)**. ಆಧುನಿಕ HTTP ಸ್ಟ್ರೀಮಿಂಗ್ ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಬಗ್ಗೆ ಕಲಿಯಿರಿ (ದೂರಸ್ಥ MCP ಸರ್ವರ್‌ಗಳಿಗೆ ಶಿಫಾರಸುಮಾಡಲಾದ ವಿಧಾನ [MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http) ಪ್ರಕಾರ), ಪ್ರಗತಿ ನೋಟೀಫಿಕೇಶನ್, ಮತ್ತು ಸ್ಟ್ರೀಮಬಲ್ HTTP ಅನ್ನು ಬಳಸಿಕೊಂಡು ಪರಿಣಾಮಕಾರಿಯಾಗಿ, ಯಥಾರ್ಥ ಸಮಯ MCP ಸರ್ವರ್ ಮತ್ತು ಕ್ಲೈಂಟ್‌ಗಳನ್ನು ಹೇಗೆ ರೂಪಿಸುವುದು ಎಂಬುದನ್ನು ಕಲಿಯಿರಿ. [ಪಾಠಕ್ಕೆ](06-http-streaming/README.md)

- **7 VSCodeಗಾಗಿ AI ಟೂಲ್ಕಿಟ್ ಬಳಸಿ** ನಿಮ್ಮ MCP ಕ್ಲೈಂಟ್ ಮತ್ತು ಸರ್ವರ್‌ಗಳನ್ನು ತಂದೆಯಿಂದ ನಿರೀಕ್ಷಿಸಿ ಮತ್ತು ಪರೀಕ್ಷಿಸಿ [ಪಾಠಕ್ಕೆ](07-aitk/README.md)

- **8 ಪರೀಕ್ಷಣೆ**. ಇಲ್ಲಿ ನಾವು ತಮ್ಮದೇ ಆದ ಸರ್ವರ್ ಮತ್ತು ಕ್ಲೈಂಟ್ ಅನ್ನು ವಿಭಿನ್ನ ರೀತಿಯಲ್ಲಿ ಏನೇನಾಗಿ ಪರೀಕ್ಷಿಸಬಹುದು ಎಂಬುದರ ಮೇಲೆ ಒತ್ತು ಕೊಡುವೆವು, [ಪಾಠಕ್ಕೆ](08-testing/README.md)

- **9 ನಿಯೋಜನೆ**. ಇವತ್ತು ನೀವು MCP ಪರಿಹಾರಗಳನ್ನು ವಿಭಿನ್ನ ರೀತಿಯಲ್ಲಿ ನಿಯೋಜಿಸುವ ಮಾರ್ಗಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುತ್ತೀರಿ, [ಪಾಠಕ್ಕೆ](09-deployment/README.md)

- **10 ಸುಧಾರಿತ ಸರ್ವರ್ ಬಳಕೆ**. ಈ ಅಧ್ಯಾಯವು ಅತೀ ಸುಧಾರಿತ ಸರ್ವರ್ ಬಳಕೆಯನ್ನು ವಿಸ್ತರಿಸುತ್ತದೆ, [ಪಾಠಕ್ಕೆ](./10-advanced/README.md)

- **11 ಪ್ರಾಮಾಣೀಕರಣ**. ಈ ಅಧ್ಯಾಯವು ಸರಳ ಪ್ರಾಮಾಣೀಕರಣವನ್ನು ಸೇರಿಸುವ ವಿಧಾನಗಳನ್ನು ಒಳಗೊಂಡಿದೆ, ಹಂತದ ಪ್ರಾಮಾಣೀಕರಣದಿಂದ JWT ಮತ್ತು RBAC ಬಳಕೆವರೆಗೆ. ನಿಮ್ಮನ್ನು ಇಲ್ಲಿ ಪ್ರಾರಂಭಿಸಲು ಮತ್ತು ನಂತರ ಅಧ್ಯಾಯ 5 ರಲ್ಲಿ ಸುಧಾರಿತ ವಿಷಯಗಳನ್ನು ಪರಿಶೀಲಿಸಲು ಪ್ರೇರೇಪಿಸಲಾಗುತ್ತದೆ ಮತ್ತು ಅಧ್ಯಾಯ 2 ರ ಶಿಫಾರಸುಗಳ ಮೂಲಕ ಹೆಚ್ಚುವರಿ ಸುರಕ್ಷತೆ ಹೆಚ್ಚಿಸುವುದನ್ನು ಮಾಡಿ, [ಪಾಠಕ್ಕೆ](./11-simple-auth/README.md)

- **12 MCP ಹೋಸ್ಟ್‌ಗಳು**. ಕ್ಲೌಡ್ ಡೆಸ್ಕ್‌ಟಾಪ್, ಕರಸರ್, ಕ್ಲೈನ್ ಮತ್ತು ವಿಂಡ್ಸರ್ಫ್ ಸೇರಿದಂತೆ ಜನಪ್ರಿಯ MCP ಹೋಸ್ಟ್ ಕ್ಲೈಂಟ್‌ಗಳನ್ನು ಗುಡ್ಡಿಸಿ ಮತ್ತು ಬಳಸಿ. ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಪ್ರಕಾರಗಳು ಮತ್ತು ಸಮಸ್ಯೆ ಪರಿಹಾರ ಕಲಿಯಿರಿ, [ಪಾಠಕ್ಕೆ](./12-mcp-hosts/README.md)

- **13 MCP ಇನ್ಸ್ಪೆಕ್ಟರ್**. MCP ಇನ್ಸ್ಪೆಕ್ಟರ್ ಸಾಧನ ಬಳಸಿ ನಿಮ್ಮ MCP ಸರ್ವರ್‌ಗಳನ್ನು ಪರಸ್ಪರ ಸಂವಾದಾತ್ಮಕವಾಗಿ ಡಿಬಗ್ ಮತ್ತು ಪರೀಕ್ಷಿಸಿ. ಸಾಧನಗಳು, ಸಂಪನ್ಮೂಲಗಳು ಮತ್ತು ಪ್ರೋಟೋಕಾಲ್ ಸಂದೇಶಗಳನ್ನು ಹೇಗೆ ಪರಿಹರಿಸುವುದನ್ನು ಕಲಿಯಿರಿ, [ಪಾಠಕ್ಕೆ](./13-mcp-inspector/README.md)

ಮಾಡೆಲ್ ಕಾಂಟೆಕ್ಸ್ಟ್ ಪ್ರೋಟೋಕಾಲ್ (MCP) ಒಂದು ಮುಕ್ತ ಪ್ರೋಟೋಕಾಲ್ ಆಗಿದ್ದು, ಅನ್ವಯಿಕೆಗಳು LLMಗಳಿಗೆ ಪ್ರಾಸಂಗಿಕತೆ ಒದಗಿಸುವ ಮಾರ್ದರಿಕ್ ಮಾಡುತ್ತದೆ. MCP ಅನ್ನು AI ಅನ್ವಯಿಕೆಗಳಿಗೆ USB-C ಪೋರ್ಟ್ ಎಂದು חשבו - ಇದು ಆಧಾರಿತ ಶ್ರೇಣಿಗೆ AI ಮಾದರಿಗಳನ್ನು ವಿಭಿನ್ನ ಡೇಟಾ ಮೂಲಗಳ ಮತ್ತು ಸಾಧನಗಳಿಗೆ ಸಂಪರ್ಕಿಸಲು ಮಾನದಂಡದ ಮಾರ್ಗ ನೀಡುತ್ತದೆ.

## ಅಧ್ಯಯನ ಗುರಿಗಳು

ಈ ಪಾಠದ ಅಂತ್ಯಕ್ಕೆ, ನೀವು ಇವುಗಳನ್ನು ಮಾಡಬಹುದು:

- C#, ಜಾವಾ, ಪೈಥಾನ್, ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್ ಮತ್ತು ಜಾವಾಸ್ಕ್ರಿಪ್ಟ್‌ನಲ್ಲಿನ MCP ಅಭಿವೃದ್ಧಿ ವಾತಾವರಣಗಳನ್ನು ಸೆಟ್ ಮಾಡುವುದು
- ಕಸ್ಟಮ್ ವೈಶಿಷ್ಟ್ಯಗಳೊಂದಿಗೆ MCP ಸರ್ವರ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಿ ಮತ್ತು ನಿಯೋಜಿಸುವುದು (ಸಂಪನ್ಮೂಲಗಳು, ಪ್ರಾಂಪ್ಟ್‌ಗಳು ಮತ್ತು ಸಾಧನಗಳು)
- MCP ಸರ್ವರ್‌ಗಳಿಗೆ ಸಂಪರ್ಕ ಸ್ಥಾಪಿಸುವ ಹೋಸ್ಟ್ ಅನ್ವಯಿಕೆಗಳನ್ನು ರಚಿಸುವುದು
- MCP ಅನುಷ್ಠಾನಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ ಡಿಬಗ್ ಮಾಡುವುದು
- ಸಾಮಾನ್ಯ ಸೆಟ್ ಅಪ್ ಸವಾಲುಗಳನ್ನು ಮತ್ತು ಅವುಗಳ ಪರಿಹಾರಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು
- ನಿಮ್ಮ MCP ಅನುಷ್ಠಾನಗಳನ್ನು ಪ್ರಸಿದ್ಧ LLM ಸೇವೆಗಳಿಗೆ ಸಂಪರ್ಕಿಸುವುದು

## ನಿಮ್ಮ MCP ವಾತಾವರಣವನ್ನು ಸೆಟ್ ಮಾಡುವುದು

MCP ಜೊತೆಗೆ ಕೆಲಸ ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು, ನಿಮ್ಮ ಅಭಿವೃದ್ಧಿ ವಾತಾವರಣವನ್ನು ಸಿದ್ದಪಡಿಸುವುದು ಮತ್ತು ಮೂಲ ಕೆಲಸದಪ್ರವಾಹವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಮುಖ್ಯ. ಈ ವಿಭಾಗವು MCP ಅನ್ನು ಸುಗಮವಾಗಿ ಪ್ರಾರಂಭಿಸಲು ಪ್ರಾಥಮಿಕ ಸೆಟ್ ಅಪ್ ಹಂತಗಳನ್ನು ಮಾರ್ಗದರ್ಶನ ಮಾಡುತ್ತದೆ.

### ಪೂರ್ವಶರತ್ತುಗಳು

MCP ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ ಕೈ ಬಿಡುವ ಮೊದಲು, ನೀವು ಇವರನ್ನು ಹೊಂದಿರುವುದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ:

- **ಅಭಿವೃದ್ಧಿ ವಾತಾವರಣ**: ನಿಮ್ಮ ಆರಿಸಿದ ಭಾಷೆಗೆ (C#, ಜಾವಾ, ಪೈಥಾನ್, ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್, ಅಥವಾ ಜಾವಾಸ್ಕ್ರಿಪ್ಟ್)
- **IDE/ಎಡಿಟರ್**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm, ಅಥವಾ ಯಾವುದೇ ಆಧುನಿಕ ಕೋಡ್ ಎಡಿಟರ್
- **ಪ್ಯಾಕೇಜ್ ಮ್ಯಾನೇಜರ್‌ಗಳು**: NuGet, Maven/Gradle, pip, ಅಥವಾ npm/yarn
- **API ಕೀಲಿಗಳು**: ನಿಮ್ಮ ಹೋಸ್ಟ್ ಅನ್ವಯಿಕೆಗಳಲ್ಲಿ ಉಪಯೋಗಿಸುವ ಯಾವುದಾದರೂ AI ಸೇವೆಗಳಿಗೆ

### ಅಧಿಕೃತ SDKಗಳು

ಮುಂದಿನ ಅಧ್ಯಾಯಗಳಲ್ಲಿ ನೀವು ಪೈಥಾನ್, ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್, ಜಾವಾ ಮತ್ತು .NET ಬಳಸಿ ನಿರ್ಮಿಸಿದ ಪರಿಹಾರಗಳನ್ನು ನೋಡುತ್ತೀರಿ. ಇಲ್ಲಿ ಎಲ್ಲಾ ಅಧಿಕೃತವಾಗಿ ಬೆಂಬಲಿತ SDKಗಳು ಇವೆ.

MCP ವಿವಿಧ ಭಾಷೆಗಳಿಗೆ ಅಧಿಕೃತ SDKಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ ([MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/) ಪ್ರಕಾರ):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - ಮೈಕ್ರೋಸಾಫ್ಟ್ ಅವರ ಸಹಯೋಗದಲ್ಲಿ ನಿರ್ವಹಿಸಲಾಗುತ್ತಿದೆ
- [ಜಾವಾ SDK](https://github.com/modelcontextprotocol/java-sdk) - ಸ್ಪ್ರಿಂಗ್ AI ಸಹಯೋಗದಲ್ಲಿ ನಿರ್ವಹಿಸಲಾಗುತ್ತಿದೆ
- [ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್ SDK](https://github.com/modelcontextprotocol/typescript-sdk) - ಅಧಿಕೃತ ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್ ಅನುಷ್ಠಾನ
- [ಪೈಥಾನ್ SDK](https://github.com/modelcontextprotocol/python-sdk) - ಅಧಿಕೃತ ಪೈಥಾನ್ ಅನುಷ್ಠಾನ (FastMCP)
- [ಕೋಟ್ಲಿನ್ SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - ಅಧಿಕೃತ ಕೋಟ್ಲಿನ್ ಅನುಷ್ಠಾನ
- [ಸ್ವಿಫ್ಟ್ SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI ಸಹಯೋಗದಲ್ಲಿ ನಿರ್ವಹಿಸಲಾಗುತ್ತಿದೆ
- [ರಸ್ಟ್ SDK](https://github.com/modelcontextprotocol/rust-sdk) - ಅಧಿಕೃತ ರಸ್ಟ್ ಅನುಷ್ಠಾನ
- [ಗೋ SDK](https://github.com/modelcontextprotocol/go-sdk) - ಅಧಿಕೃತ ಗೋ ಅನುಷ್ಠಾನ

## ಮುಖ್ಯ ಅಂಕೆಗಳು

- ಭಾಷಾ-ವಿಶಿಷ್ಟ SDKಗಳೊಂದಿಗೆ MCP ಅಭಿವೃದ್ಧಿ ವಾತಾವರಣವನ್ನು ಸ್ಥಾಪಿಸುವುದು ಸುಲಭ
- MCP ಸರ್ವರ್‌ಗಳನ್ನು ನಿರ್ಮಿಸುವುದು ಸ್ಪಷ್ಟ ಯೋಜನೆಗಳೊಂದಿಗೆ ಸಾಧನಗಳನ್ನು ರಚಿಸುವ ಹಾಗೂ ನೋಂದಾಯಿಸುವುದನ್ನು ಒಳಗೊಂಡಿದೆ
- MCP ಕ್ಲೈಂಟ್‌ಗಳು ಸರ್ವರ್ ಮತ್ತು ಮಾದರಿಗಳೊಂದಿಗೆ ಸಂಪರ್ಕ ಸಾಧಿಸಿ ವಿಸ್ತೃತ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಬಳಕೆಮಾಡುತ್ತವೆ
- ಪರೀಕ್ಷಣೆ ಮತ್ತು ಡಿಬಗ್ ಮಾಡುವುದರಿಂದ MCP ಅನುಷ್ಠಾನಗಳು ನಂಬಿಕಪಾತ್ರವಾಗುತ್ತವೆ
- ನಿಯೋಜನೆ ಆಯ್ಕೆಗಳು ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿಯಿಂದ ಮೇಘಾಧಾರಿತ ಪರಿಹಾರಗಳ ವರೆಗೂ ವ್ಯಾಪಿಸಿದೆ
