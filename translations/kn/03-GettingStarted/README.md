## ಪ್ರಾರಂಭಿಸುವುದು  

[![ನಿಮ್ಮ ಮೊದಲ MCP ಸರ್ವರ್ ನಿರ್ಮಿಸಿ](../../../translated_images/04.0ea920069efd979a0b2dad51e72c1df7ead9c57b3305796068a6cee1f0dd6674.kn.png)](https://youtu.be/sNDZO9N4m9Y)

_(ಈ ಪಾಠದ ವೀಡಿಯೋವನ್ನು ನೋಡಲು ಮೇಲಿನ ಚಿತ್ರವನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ)_

ಈ ವಿಭಾಗದಲ್ಲಿ ಹಲವಾರು ಪಾಠಗಳಿವೆ:

- **1 ನಿಮ್ಮ ಮೊದಲ ಸರ್ವರ್**, ಈ ಮೊದಲ ಪಾಠದಲ್ಲಿ, ನೀವು ನಿಮ್ಮ ಮೊದಲ ಸರ್ವರ್ ಅನ್ನು ಹೇಗೆ ರಚಿಸುವುದು ಮತ್ತು ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಟೂಲ್‌ನೊಂದಿಗೆ ಅದನ್ನು ಪರಿಶೀಲಿಸುವುದು, ನಿಮ್ಮ ಸರ್ವರ್ ಅನ್ನು ಪರೀಕ್ಷಿಸಲು ಮತ್ತು ಡಿಬಗ್ ಮಾಡಲು ಮೌಲ್ಯಯುತವಾದ ವಿಧಾನವನ್ನು ಕಲಿಯುತ್ತೀರಿ, [ಪಾಠಕ್ಕೆ](01-first-server/README.md)

- **2 ಕ್ಲೈಂಟ್**, ಈ ಪಾಠದಲ್ಲಿ, ನೀವು ನಿಮ್ಮ ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕಿಸಬಹುದಾದ ಕ್ಲೈಂಟ್ ಅನ್ನು ಹೇಗೆ ಬರೆಯುವುದು ಎಂದು ಕಲಿಯುತ್ತೀರಿ, [ಪಾಠಕ್ಕೆ](02-client/README.md)

- **3 LLM ಹೊಂದಿರುವ ಕ್ಲೈಂಟ್**, ಕ್ಲೈಂಟ್ ಬರೆಯುವ ಇನ್ನೂ ಉತ್ತಮ ವಿಧಾನವೆಂದರೆ ಅದಕ್ಕೆ LLM ಅನ್ನು ಸೇರಿಸುವುದು, ಇದರಿಂದ ಅದು ನಿಮ್ಮ ಸರ್ವರ್ ಜೊತೆ "ಚರ್ಚೆ" ಮಾಡಬಹುದು ಏನು ಮಾಡಬೇಕೆಂದು, [ಪಾಠಕ್ಕೆ](03-llm-client/README.md)

- **4 Visual Studio Code ನಲ್ಲಿ GitHub Copilot ಏಜೆಂಟ್ ಮೋಡ್ ಬಳಸಿ ಸರ್ವರ್ ಬಳಕೆ**. ಇಲ್ಲಿ, ನಾವು Visual Studio Code ಒಳಗಿಂದ ನಮ್ಮ MCP ಸರ್ವರ್ ಅನ್ನು ಚಾಲನೆ ಮಾಡುವುದನ್ನು ನೋಡುತ್ತಿದ್ದೇವೆ, [ಪಾಠಕ್ಕೆ](04-vscode/README.md)

- **5 stdio ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಸರ್ವರ್** stdio ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಪ್ರಸ್ತುತ ನಿರ್ದಿಷ್ಟೀಕರಣದಲ್ಲಿ MCP ಸರ್ವರ್-ಕ್ಲೈಂಟ್ ಸಂವಹನಕ್ಕೆ ಶಿಫಾರಸು ಮಾಡಲಾದ ಮಾನದಂಡವಾಗಿದೆ, ಸುರಕ್ಷಿತ ಉಪಪ್ರಕ್ರಿಯೆ ಆಧಾರಿತ ಸಂವಹನವನ್ನು ಒದಗಿಸುತ್ತದೆ [ಪಾಠಕ್ಕೆ](05-stdio-server/README.md)

- **6 MCP ಜೊತೆಗೆ HTTP ಸ್ಟ್ರೀಮಿಂಗ್ (ಸ್ಟ್ರೀಮಬಲ್ HTTP)**. ಆಧುನಿಕ HTTP ಸ್ಟ್ರೀಮಿಂಗ್, ಪ್ರಗತಿ ಸೂಚನೆಗಳು ಮತ್ತು ಸ್ಟ್ರೀಮಬಲ್ HTTP ಬಳಸಿ ವ್ಯಾಪಕ, ರಿಯಲ್-ಟೈಮ್ MCP ಸರ್ವರ್‌ಗಳು ಮತ್ತು ಕ್ಲೈಂಟ್‌ಗಳನ್ನು ಹೇಗೆ ಅನುಷ್ಠಾನಗೊಳಿಸುವುದು ಎಂದು ಕಲಿಯಿರಿ. [ಪಾಠಕ್ಕೆ](06-http-streaming/README.md)

- **7 VSCode ಗಾಗಿ AI ಟೂಲ್‌ಕಿಟ್ ಬಳಕೆ** ನಿಮ್ಮ MCP ಕ್ಲೈಂಟ್‌ಗಳು ಮತ್ತು ಸರ್ವರ್‌ಗಳನ್ನು ಉಪಯೋಗಿಸಿ ಪರೀಕ್ಷಿಸಲು [ಪಾಠಕ್ಕೆ](07-aitk/README.md)

- **8 ಪರೀಕ್ಷೆ**. ಇಲ್ಲಿ ನಾವು ವಿಶೇಷವಾಗಿ ನಮ್ಮ ಸರ್ವರ್ ಮತ್ತು ಕ್ಲೈಂಟ್ ಅನ್ನು ವಿಭಿನ್ನ ರೀತಿಗಳಲ್ಲಿ ಹೇಗೆ ಪರೀಕ್ಷಿಸಬಹುದು ಎಂಬುದರ ಮೇಲೆ ಗಮನಹರಿಸುತ್ತೇವೆ, [ಪಾಠಕ್ಕೆ](08-testing/README.md)

- **9 ನಿಯೋಜನೆ**. ಈ ಅಧ್ಯಾಯದಲ್ಲಿ ನಿಮ್ಮ MCP ಪರಿಹಾರಗಳನ್ನು ನಿಯೋಜಿಸುವ ವಿಭಿನ್ನ ವಿಧಾನಗಳನ್ನು ನೋಡಲಾಗುತ್ತದೆ, [ಪಾಠಕ್ಕೆ](09-deployment/README.md)

- **10 ಉನ್ನತ ಸರ್ವರ್ ಬಳಕೆ**. ಈ ಅಧ್ಯಾಯವು ಉನ್ನತ ಸರ್ವರ್ ಬಳಕೆಯನ್ನು ಒಳಗೊಂಡಿದೆ, [ಪಾಠಕ್ಕೆ](./10-advanced/README.md)

- **11 ಪ್ರಾಮಾಣೀಕರಣ**. ಈ ಅಧ್ಯಾಯವು ಸರಳ ಪ್ರಾಮಾಣೀಕರಣವನ್ನು ಸೇರಿಸುವುದನ್ನು ಒಳಗೊಂಡಿದೆ, ಮೂಲಭೂತ ಪ್ರಾಮಾಣೀಕರಣದಿಂದ JWT ಮತ್ತು RBAC ಬಳಕೆವರೆಗೆ. ನೀವು ಇಲ್ಲಿ ಪ್ರಾರಂಭಿಸಲು ಪ್ರೋತ್ಸಾಹಿತರಾಗಿದ್ದೀರಿ ಮತ್ತು ನಂತರ ಅಧ್ಯಾಯ 5 ರ ಉನ್ನತ ವಿಷಯಗಳನ್ನು ನೋಡಿ, ಅಧ್ಯಾಯ 2 ರ ಶಿಫಾರಸುಗಳ ಮೂಲಕ ಹೆಚ್ಚುವರಿ ಭದ್ರತಾ ಬಲಪಡಿಸುವಿಕೆಗಳನ್ನು ಮಾಡಬಹುದು, [ಪಾಠಕ್ಕೆ](./11-simple-auth/README.md)

ಮಾದರಿ ಸನ್ನಿವೇಶ ಪ್ರೋಟೋಕಾಲ್ (MCP) ಒಂದು ತೆರೆಯಲಾದ ಪ್ರೋಟೋಕಾಲ್ ಆಗಿದ್ದು, ಅಪ್ಲಿಕೇಶನ್‌ಗಳು LLMಗಳಿಗೆ ಸನ್ನಿವೇಶವನ್ನು ಒದಗಿಸುವ ವಿಧಾನವನ್ನು ಮಾನದಂಡಗೊಳಿಸುತ್ತದೆ. MCP ಅನ್ನು AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ USB-C ಪೋರ್ಟ್ ಎಂದು ಭಾವಿಸಿ - ಇದು AI ಮಾದರಿಗಳನ್ನು ವಿಭಿನ್ನ ಡೇಟಾ ಮೂಲಗಳು ಮತ್ತು ಉಪಕರಣಗಳಿಗೆ ಸಂಪರ್ಕಿಸುವ ಮಾನದಂಡಿತ ವಿಧಾನವನ್ನು ಒದಗಿಸುತ್ತದೆ.

## ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳು

ಈ ಪಾಠದ ಕೊನೆಯಲ್ಲಿ, ನೀವು ಸಾಧ್ಯವಾಗುವುದು:

- C#, ಜಾವಾ, ಪೈಥಾನ್, ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್ ಮತ್ತು ಜಾವಾಸ್ಕ್ರಿಪ್ಟ್‌ನಲ್ಲಿ MCP ಅಭಿವೃದ್ಧಿ ಪರಿಸರಗಳನ್ನು ಸ್ಥಾಪಿಸುವುದು
- ಕಸ್ಟಮ್ ವೈಶಿಷ್ಟ್ಯಗಳೊಂದಿಗೆ ಮೂಲ MCP ಸರ್ವರ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಿ ಮತ್ತು ನಿಯೋಜಿಸುವುದು (ಸಂಪನ್ಮೂಲಗಳು, ಪ್ರಾಂಪ್ಟ್‌ಗಳು ಮತ್ತು ಉಪಕರಣಗಳು)
- MCP ಸರ್ವರ್‌ಗಳಿಗೆ ಸಂಪರ್ಕಿಸುವ ಹೋಸ್ಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ರಚಿಸುವುದು
- MCP ಅನುಷ್ಠಾನಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ ಮತ್ತು ಡಿಬಗ್ ಮಾಡುವುದು
- ಸಾಮಾನ್ಯ ಸ್ಥಾಪನೆ ಸವಾಲುಗಳು ಮತ್ತು ಅವುಗಳ ಪರಿಹಾರಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು
- ನಿಮ್ಮ MCP ಅನುಷ್ಠಾನಗಳನ್ನು ಜನಪ್ರಿಯ LLM ಸೇವೆಗಳಿಗೆ ಸಂಪರ್ಕಿಸುವುದು

## ನಿಮ್ಮ MCP ಪರಿಸರವನ್ನು ಸ್ಥಾಪಿಸುವುದು

MCP ಜೊತೆಗೆ ಕೆಲಸ ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು, ನಿಮ್ಮ ಅಭಿವೃದ್ಧಿ ಪರಿಸರವನ್ನು ಸಿದ್ಧಪಡಿಸುವುದು ಮತ್ತು ಮೂಲ ಕಾರ್ಯಪ್ರವಾಹವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಮುಖ್ಯ. ಈ ವಿಭಾಗವು MCP ಜೊತೆಗೆ ಸುಗಮ ಪ್ರಾರಂಭಕ್ಕಾಗಿ ಪ್ರಾಥಮಿಕ ಸ್ಥಾಪನೆ ಹಂತಗಳನ್ನು ಮಾರ್ಗದರ್ಶನ ಮಾಡುತ್ತದೆ.

### ಪೂರ್ವಾಪೇಕ್ಷಿತಗಳು

MCP ಅಭಿವೃದ್ಧಿಯಲ್ಲಿ ಮುಳುಗುವ ಮೊದಲು, ನೀವು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ:

- **ಅಭಿವೃದ್ಧಿ ಪರಿಸರ**: ನಿಮ್ಮ ಆಯ್ದ ಭಾಷೆಗೆ (C#, ಜಾವಾ, ಪೈಥಾನ್, ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್ ಅಥವಾ ಜಾವಾಸ್ಕ್ರಿಪ್ಟ್)
- **IDE/ಎಡಿಟರ್**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm ಅಥವಾ ಯಾವುದೇ ಆಧುನಿಕ ಕೋಡ್ ಎಡಿಟರ್
- **ಪ್ಯಾಕೇಜ್ ಮ್ಯಾನೇಜರ್‌ಗಳು**: NuGet, Maven/Gradle, pip, ಅಥವಾ npm/yarn
- **API ಕೀಗಳು**: ನಿಮ್ಮ ಹೋಸ್ಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಲ್ಲಿ ಬಳಸಲು ಯೋಜಿಸಿರುವ ಯಾವುದೇ AI ಸೇವೆಗಳಿಗಾಗಿ

### ಅಧಿಕೃತ SDKಗಳು

ಮುಂದಿನ ಅಧ್ಯಾಯಗಳಲ್ಲಿ ನೀವು ಪೈಥಾನ್, ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್, ಜಾವಾ ಮತ್ತು .NET ಬಳಸಿ ನಿರ್ಮಿಸಲಾದ ಪರಿಹಾರಗಳನ್ನು ನೋಡುತ್ತೀರಿ. ಇಲ್ಲಿ ಎಲ್ಲಾ ಅಧಿಕೃತವಾಗಿ ಬೆಂಬಲಿತ SDKಗಳು ಇವೆ.

MCP ಬಹುಭಾಷೆಗಳಿಗಾಗಿ ಅಧಿಕೃತ SDKಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ:
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - ಮೈಕ್ರೋಸಾಫ್ಟ್ ಜೊತೆಗೆ ಸಹಯೋಗದಲ್ಲಿ ನಿರ್ವಹಿಸಲಾಗಿದೆ
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI ಜೊತೆಗೆ ಸಹಯೋಗದಲ್ಲಿ ನಿರ್ವಹಿಸಲಾಗಿದೆ
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - ಅಧಿಕೃತ ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್ ಅನುಷ್ಠಾನ
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - ಅಧಿಕೃತ ಪೈಥಾನ್ ಅನುಷ್ಠಾನ
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - ಅಧಿಕೃತ ಕೊಟ್ಲಿನ್ ಅನುಷ್ಠಾನ
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI ಜೊತೆಗೆ ಸಹಯೋಗದಲ್ಲಿ ನಿರ್ವಹಿಸಲಾಗಿದೆ
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - ಅಧಿಕೃತ ರಸ್ಟ್ ಅನುಷ್ಠಾನ

## ಪ್ರಮುಖ ಅಂಶಗಳು

- MCP ಅಭಿವೃದ್ಧಿ ಪರಿಸರವನ್ನು ಭಾಷಾ-ನಿರ್ದಿಷ್ಟ SDKಗಳೊಂದಿಗೆ ಸ್ಥಾಪಿಸುವುದು ಸರಳ
- MCP ಸರ್ವರ್‌ಗಳನ್ನು ನಿರ್ಮಿಸುವುದು ಸ್ಪಷ್ಟ ಸ್ಕೀಮಾ ಹೊಂದಿರುವ ಉಪಕರಣಗಳನ್ನು ರಚಿಸುವುದು ಮತ್ತು ನೋಂದಾಯಿಸುವುದನ್ನು ಒಳಗೊಂಡಿದೆ
- MCP ಕ್ಲೈಂಟ್‌ಗಳು ಸರ್ವರ್‌ಗಳು ಮತ್ತು ಮಾದರಿಗಳೊಂದಿಗೆ ಸಂಪರ್ಕಿಸಿ ವಿಸ್ತೃತ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಉಪಯೋಗಿಸುತ್ತವೆ
- ಪರೀಕ್ಷೆ ಮತ್ತು ಡಿಬಗ್ MCP ಅನುಷ್ಠಾನಗಳಿಗೆ ಅವಶ್ಯಕ
- ನಿಯೋಜನೆ ಆಯ್ಕೆಗಳು ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿಯಿಂದ ಕ್ಲೌಡ್ ಆಧಾರಿತ ಪರಿಹಾರಗಳವರೆಗೆ ವ್ಯಾಪಿಸಿದೆ
