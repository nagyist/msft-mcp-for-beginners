# MCP ಸರ್ವರ್‌ಗಳನ್ನು ನಿಯೋಜಿಸುವುದು

ನಿಮ್ಮ MCP ಸರ್ವರ್ ಅನ್ನು ನಿಯೋಜಿಸುವ ಮೂಲಕ, ಅದರ ಸಾಧನಗಳು ಮತ್ತು ಸಂಪನ್ಮೂಲಗಳನ್ನು ಬೇರೆ ಜನರು ನಿಮ್ಮ ಸ್ಥಳೀಯ ಪರಿಸರದ ಹೊರಗೆ ಬಳಸಬಹುದು. ಪ್ರಮಾಣವನ್ನು ಹೊಂದಿಸುವಿಕೆ, ನಂಬಕತೆ, ಮತ್ತು ನಿರ್ವಹಣೆಯ ಸುಲಭತೆಗಾಗಿ ನಿಮಗೆ ಬೇಕಾಗುವ ಸೇವೆಗಳು ಪ್ರಕಾರ ಹಲವಾರು ನಿಯೋಜನೆ ತಂತ್ರಗಳನ್ನು ಪರಿಗಣಿಸಬಹುದು. ಕೆಳಗೆ ನೀವು ಸ್ಥಳೀಯವಾಗಿ, ಕಂಟೇನರ್‌ಗಳಲ್ಲಿ ಮತ್ತು ಕ್ಲೌಡ್‌ಗೆ MCP ಸರ್ವರ್‌ಗಳನ್ನು ನಿಯೋಜಿಸುವ ಮಾರ್ಗದರ್ಶನವನ್ನು ಕಾಣಬಹುದು.

## ಅವಲೋಕನ

ಈ ಪಾಠವು ನಿಮ್ಮ MCP ಸರ್ವರ್ ಅಪ್ಲಿಕೇಶನ್ನನ್ನು ಹೇಗೂ ನಿಯೋಜಿಸುವುದನ್ನು ಒಳಗೊಂಡಿದೆ.

## ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳು

ಈ ಪಾಠದ ಅಂತ್ಯಕ್ಕೆ, ನೀವು ಈ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಹೊಂದಿರುತ್ತೀರಿ:

- ಬಗೆಯಾದ ನಿಯೋಜನೆ ವಿಧಾನಗಳನ್ನು ಮೌಲ್ಯಮಾಪನ ಮಾಡುವುದು.
- ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ನನ್ನು ನಿಯೋಜಿಸುವುದು.

## ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ಮತ್ತು ನಿಯೋಜನೆ

ನಿಮ್ಮ ಸರ್ವರ್ ಬಳಕೆದಾರರ ಯಂತ್ರದಲ್ಲಿ ನಡೆಯಬೇಕಾದಿದ್ದರೆ, ಕೆಳಗಿನ ಕ್ರಮಗಳನ್ನು ಅನುಸರಿಸಬಹುದು:

1. **ಸರ್ವರ್ ಅನ್ನು ಡೌನ್ಲೋಡ್ ಮಾಡಿಕೊಳ್ಳಿ**. ನೀವು ಸರ್ವರ್ ಅನ್ನು ಬರಹ ಮಾಡದಿದ್ದರೆ, ಮೊದಲು ಅದನ್ನು ನಿಮ್ಮ ಯಂತ್ರಕ್ಕೆ ಡೌನ್ಲೋಡ್ ಮಾಡಿಕೊಳ್ಳಿ.  
1. **ಸರ್ವರ್ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಪ್ರಾರಂಭಿಸಿ**: ನಿಮ್ಮ MCP ಸರ್ವರ್ ಅಪ್ಲಿಕೇಶನ್ನನ್ನು ನಡಿಸಿ

SSE ಗಾಗಿ (stdio ಪ್ರಕಾರದ ಸರ್ವರ್‌ಗಳಿಗೆ ಅಗತ್ಯವಿಲ್ಲ)

1. **ನೆಟ್ವರ್ಕಿಂಗ್ ಅನ್ನು ಸಂರಚಿಸಿ**: ಸರ್ವರ್ ನಿರೀಕ್ಷಿತ ಪೋರ್ಟ್ ನಲ್ಲಿ ಪ್ರವೇಶ ಮಾಡಲು ಲಭ್ಯವಿರಬೇಕೆಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ  
1. **ಗ್ರಾಹಕರನ್ನು ಸಂಪರ್ಕಿಸಿ**: `http://localhost:3000` ಮುಂತಾದ ಸ್ಥಳೀಯ ಸಂಪರ್ಕ URLಗಳನ್ನು ಉಪಯೋಗಿಸಿ

## ಕ್ಲೌಡ್ ನಿಯೋಜನೆ

MCP ಸರ್ವರ್‌ಗಳನ್ನು ಹಲವಾರು ಕ್ಲೌಡ್ ವೇದಿಕೆಗಳಲ್ಲಿ ನಿಯೋಜಿಸಬಹುದು:

- **ಸರ್ವರ್‌ಲೆಸ FUNCTIONS**: ತೂಕ ಹಿಕ್ಕಿದ MCP ಸರ್ವರ್‌ಗಳನ್ನು ಸರ್ವರ್‌ಲೆಸ್ Functions ಆಗಿ ನಿಯೋಜಿಸಿ
- **ಕಂಟೇನರ್ ಸೇವೆಗಳು**: Azure Container Apps, AWS ECS ಅಥವಾ Google Cloud Run ಸರೀಗೆ ಸೇವೆಗಳನ್ನು ಬಳಸಿ
- **ಕುಬೆರ್ನೇಟೀಸ್**: MCP ಸರ್ವರ್‌ಗಳನ್ನು ಹೆಚ್ಚಿನ ಲಭ್ಯತೆಗಾಗಿ ಕುಬೆರ್ನೇಟೀಸ್ ಕ್ಲಸ್ಟರ್ಗಳಲ್ಲಿ ನಿಯೋಜಿಸಿ ಮತ್ತು ನಿರ್ವಹಿಸಿ

### ಉದಾಹರಣೆ: Azure Container Apps

Azure Container Apps MCP ಸರ್ವರ್ ಗಳ ನಿಯೋಜನೆಯನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ. ಇದು ಈಗಾಗಲೇ ಅಭಿವೃದ್ಧಿಯಲ್ಲಿದೆ ಮತ್ತು ಪ್ರಸ್ತುತ SSE ಸರ್ವರ್‌ಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ.

ಇದನ್ನು ಹೇಗೆ ಮಾಡುವುದು ಎಂಬುದಕ್ಕೆ ಉದಾಹರಣೆ:

1. ಒಂದು ರೆಪೊ ಕ್ಲೋನ್ ಮಾಡಿ:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. ಎಣಿಸಿ, ಸ್ಥಳೀಯವಾಗಿ ಪರೀಕ್ಷಿಸಲು:

  ```sh
  uv venv
  uv sync

  # ಲಿನಕ್ಸ್ಗಳು/macOS
  export API_KEYS=<AN_API_KEY>
  # ವಿಂಡೋಸ್
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. ಸ್ಥಳೀಯವಾಗಿ ಪ್ರಯತ್ನಿಸಲು, *.vscode* ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ *mcp.json* ಫೈಲ್ ರಚಿಸಿ ಮತ್ತು ಈ ವಿಷಯವನ್ನು ಸೇರ್ಸಿರಿ:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  SSE ಸರ್ವರ್ ಪ್ರಾರಂಭಿಸಿದ ನಂತರ, JSON ಫೈಲ್‌ನಲ್ಲಿನ ಪ್ಲೇ ಚಿಹ್ನೆಯನ್ನು ಕ್ಲಿಕ್ ಮಾಡಬಹುದು, ಈಗ GitHub Copilot ಮೂಲಕ ಸರ್ವರ್‌ನ ಸಾಧನಗಳನ್ನು ಕಂಡುಹಿಡಿಯಬಹುದು, Tool ಐಕಾನ್ ನೋಡಿ.

1. ನಿಯೋಜಿಸಲು, ಈ ಕೆಳಗಿನ ಆದೇಶವನ್ನು ನಡೆಸಿ:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

ಇದೀಗ, ನಿಮ್ಮ ಸರ್ವರ್ ಅನ್ನು ಸ್ಥಳೀಯವಾಗಿ ಮತ್ತು Azure ಗೆ ಈ ಕ್ರಮಗಳ ಮೂಲಕ ನಿಯೋಜಿಸಿ.
