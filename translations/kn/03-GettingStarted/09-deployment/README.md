# MCP ಸರ್ವರ್‌ಗಳನ್ನು ನಿಯೋಜಿಸುವುದು

ನಿಮ್ಮ MCP ಸರ್ವರ್ ಅನ್ನು ನಿಯೋಜಿಸುವುದರಿಂದ ಇತರರು ನಿಮ್ಮ ಸ್ಥಳೀಯ ಪರಿಸರದ ಹೊರಗೆ ಅದರ ಸಾಧನಗಳು ಮತ್ತು ಸಂಪನ್ಮೂಲಗಳನ್ನು ಪ್ರವೇಶಿಸಬಹುದು. ನಿಮ್ಮ ಅಗತ್ಯಗಳಿಗೆ ಅನುಗುಣವಾಗಿ, ವಿಸ್ತರಣೆ, ನಂಬಿಕೆ ಮತ್ತು ನಿರ್ವಹಣೆಯ ಸುಲಭತೆಯಿಗಾಗಿ ಹಲವು ನಿಯೋಜನೆ ತಂತ್ರಗಳನ್ನು ಪರಿಗಣಿಸಬಹುದು. ಕೆಳಗೆ ನೀವು ಸ್ಥಳೀಯವಾಗಿ, ಕಂಟೈನರ್‌ಗಳಲ್ಲಿ ಮತ್ತು ಕ್ಲೌಡ್‌ಗೆ MCP ಸರ್ವರ್‌ಗಳನ್ನು ನಿಯೋಜಿಸುವ ಮಾರ್ಗದರ್ಶನವನ್ನು ಕಾಣಬಹುದು.

## ಅವಲೋಕನ

ಈ ಪಾಠವು ನಿಮ್ಮ MCP ಸರ್ವರ್ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಹೇಗೆ ನಿಯೋಜಿಸುವುದನ್ನು ಒಳಗೊಂಡಿದೆ.

## ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳು

ಈ ಪಾಠದ ಅಂತ್ಯಕ್ಕೆ, ನೀವು ಈ ಕೆಳಗಿನವುಗಳನ್ನು ಮಾಡಲು ಸಾಧ್ಯವಾಗುತ್ತದೆ:

- ವಿಭಿನ್ನ ನಿಯೋಜನೆ ವಿಧಾನಗಳನ್ನು ಮೌಲ್ಯಮಾಪನ ಮಾಡುವುದು.
- ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ನಿಯೋಜಿಸುವುದು.

## ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ಮತ್ತು ನಿಯೋಜನೆ

ನಿಮ್ಮ ಸರ್ವರ್ ಬಳಕೆದಾರರ ಯಂತ್ರದಲ್ಲಿ ಚಾಲನೆಗೊಳ್ಳಲು ಉದ್ದೇಶಿಸಿದರೆ, ನೀವು ಕೆಳಗಿನ ಹಂತಗಳನ್ನು ಅನುಸರಿಸಬಹುದು:

1. **ಸರ್ವರ್ ಅನ್ನು ಡೌನ್‌ಲೋಡ್ ಮಾಡಿ**. ನೀವು ಸರ್ವರ್ ಅನ್ನು ಬರೆಯದಿದ್ದರೆ, ಮೊದಲು ಅದನ್ನು ನಿಮ್ಮ ಯಂತ್ರಕ್ಕೆ ಡೌನ್‌ಲೋಡ್ ಮಾಡಿ.
1. **ಸರ್ವರ್ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಪ್ರಾರಂಭಿಸಿ**: ನಿಮ್ಮ MCP ಸರ್ವರ್ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಚಾಲನೆ ಮಾಡಿ

SSE ಗಾಗಿ (stdio ಪ್ರಕಾರದ ಸರ್ವರ್‌ಗೆ ಅಗತ್ಯವಿಲ್ಲ)

1. **ನೆಟ್ವರ್ಕಿಂಗ್ ಅನ್ನು ಸಂರಚಿಸಿ**: ಸರ್ವರ್ ನಿರೀಕ್ಷಿತ ಪೋರ್ಟ್‌ನಲ್ಲಿ ಪ್ರವೇಶಿಸಬಹುದಾಗಿರಲಿ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
1. **ಗ್ರಾಹಕರನ್ನು ಸಂಪರ್ಕಿಸಿ**: `http://localhost:3000` ಮುಂತಾದ ಸ್ಥಳೀಯ ಸಂಪರ್ಕ URL ಗಳನ್ನು ಬಳಸಿ

## ಕ್ಲೌಡ್ ನಿಯೋಜನೆ

MCP ಸರ್ವರ್‌ಗಳನ್ನು ವಿವಿಧ ಕ್ಲೌಡ್ ವೇದಿಕೆಗಳಿಗೆ ನಿಯೋಜಿಸಬಹುದು:

- **ಸರ್ವರ್‌ಲೆಸ್ ಫಂಕ್ಷನ್ಸ್**: ಲಘು ತೂಕದ MCP ಸರ್ವರ್‌ಗಳನ್ನು ಸರ್ವರ್‌ಲೆಸ್ ಫಂಕ್ಷನ್ಸ್ ಆಗಿ ನಿಯೋಜಿಸಿ
- **ಕಂಟೈನರ್ ಸೇವೆಗಳು**: Azure Container Apps, AWS ECS, ಅಥವಾ Google Cloud Run ಮುಂತಾದ ಸೇವೆಗಳನ್ನು ಬಳಸಿ
- **ಕುಬೆರ್ನೇಟಿಸ್**: ಹೆಚ್ಚಿನ ಲಭ್ಯತೆಗಾಗಿ ಕುಬೆರ್ನೇಟಿಸ್ ಕ್ಲಸ್ಟರ್‌ಗಳಲ್ಲಿ MCP ಸರ್ವರ್‌ಗಳನ್ನು ನಿಯೋಜಿಸಿ ಮತ್ತು ನಿರ್ವಹಿಸಿ

### ಉದಾಹರಣೆ: Azure Container Apps

Azure Container Apps MCP ಸರ್ವರ್‌ಗಳ ನಿಯೋಜನೆಯನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ. ಇದು ಇನ್ನೂ ಅಭಿವೃದ್ಧಿಯಲ್ಲಿದೆ ಮತ್ತು ಪ್ರಸ್ತುತ SSE ಸರ್ವರ್‌ಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ.

ನೀವು ಇದನ್ನು ಹೇಗೆ ಮಾಡಬಹುದು:

1. ಒಂದು ರೆಪೊ ಕ್ಲೋನ್ ಮಾಡಿ:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. ಸ್ಥಳೀಯವಾಗಿ ಪರೀಕ್ಷಿಸಲು ಅದನ್ನು ಚಾಲನೆ ಮಾಡಿ:

  ```sh
  uv venv
  uv sync

  # ಲಿನಕ್ಸ್ನ/ಮ್ಯಾಕ್‌ಒಎಸ್
  export API_KEYS=<AN_API_KEY>
  # ವಿಂಡೋಸ್
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. ಸ್ಥಳೀಯವಾಗಿ ಪ್ರಯತ್ನಿಸಲು, *.vscode* ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ *mcp.json* ಫೈಲ್ ರಚಿಸಿ ಮತ್ತು ಕೆಳಗಿನ ವಿಷಯವನ್ನು ಸೇರಿಸಿ:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  SSE ಸರ್ವರ್ ಪ್ರಾರಂಭವಾದ ನಂತರ, ನೀವು JSON ಫೈಲ್‌ನಲ್ಲಿ ಪ್ಲೇ ಐಕಾನ್ ಕ್ಲಿಕ್ ಮಾಡಬಹುದು, ಈಗ GitHub Copilot ಮೂಲಕ ಸರ್ವರ್上的 ಸಾಧನಗಳನ್ನು ಕಾಣಬಹುದು, ಸಾಧನ ಐಕಾನ್ ನೋಡಿ.

1. ನಿಯೋಜಿಸಲು, ಕೆಳಗಿನ ಆಜ್ಞೆಯನ್ನು ಚಾಲನೆ ಮಾಡಿ:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

ಇಲ್ಲಿ ನೀವು ಹೊಂದಿದ್ದೀರಿ, ಇದನ್ನು ಸ್ಥಳೀಯವಾಗಿ ನಿಯೋಜಿಸಿ, ಈ ಹಂತಗಳ ಮೂಲಕ Azure ಗೆ ನಿಯೋಜಿಸಿ.
