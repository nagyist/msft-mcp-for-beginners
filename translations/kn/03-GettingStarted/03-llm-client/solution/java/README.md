# ಕ್ಯಾಲ್ಕುಲೇಟರ್ LLM ಕ್ಲೈಂಟ್

GitHub ಮಾದರಿಗಳ ಏಕೀಕರಣದೊಂದಿಗೆ MCP (ಮಾದರಿ ಸಂದರ್ಭ ಪ್ರೋಟೋಕಾಲ್) ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಸೇವೆಗೆ ಸಂಪರ್ಕಿಸಲು LangChain4j ಅನ್ನು ಹೇಗೆ ಬಳಸುವುದು ಎಂಬುದನ್ನು ತೋರಿಸುವ ಜಾವಾ ಅಪ್ಲಿಕೇಶನ್.

## ಪೂರ್ವಾಪೇಕ್ಷೆಗಳು

- ಜಾವಾ 21 ಅಥವಾ ಹೆಚ್ಚಿನ ಆವೃತ್ತಿ
- ಮೇವೆನ್ 3.6+ (ಅಥವಾ ಒಳಗೊಂಡ ಮೇವೆನ್ ರ್ಯಾಪರ್ ಬಳಸಿ)
- GitHub ಮಾದರಿಗಳಿಗೆ ಪ್ರವೇಶ ಹೊಂದಿರುವ GitHub ಖಾತೆ
- `http://localhost:8080` ನಲ್ಲಿ ಚಾಲನೆಯಲ್ಲಿರುವ MCP ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಸೇವೆ

## GitHub ಟೋಕನ್ ಪಡೆಯುವುದು

ಈ ಅಪ್ಲಿಕೇಶನ್ GitHub ಮಾದರಿಗಳನ್ನು ಬಳಸುತ್ತದೆ, ಇದಕ್ಕೆ GitHub ವೈಯಕ್ತಿಕ ಪ್ರವೇಶ ಟೋಕನ್ ಅಗತ್ಯವಿದೆ. ನಿಮ್ಮ ಟೋಕನ್ ಪಡೆಯಲು ಈ ಹಂತಗಳನ್ನು ಅನುಸರಿಸಿ:

### 1. GitHub ಮಾದರಿಗಳಿಗೆ ಪ್ರವೇಶಿಸಿ
1. [GitHub Models](https://github.com/marketplace/models) ಗೆ ಹೋಗಿ
2. ನಿಮ್ಮ GitHub ಖಾತೆಯಿಂದ ಸೈನ್ ಇನ್ ಆಗಿ
3. ನೀವು ಈಗಾಗಲೇ ಮಾಡದಿದ್ದರೆ GitHub ಮಾದರಿಗಳಿಗೆ ಪ್ರವೇಶವನ್ನು ವಿನಂತಿಸಿ

### 2. ವೈಯಕ್ತಿಕ ಪ್ರವೇಶ ಟೋಕನ್ ರಚಿಸಿ
1. [GitHub ಸೆಟ್ಟಿಂಗ್ಸ್ → ಡೆವಲಪರ್ ಸೆಟ್ಟಿಂಗ್ಸ್ → ವೈಯಕ್ತಿಕ ಪ್ರವೇಶ ಟೋಕನ್ಗಳು → ಟೋಕನ್ಗಳು (ಕ್ಲಾಸಿಕ್)](https://github.com/settings/tokens) ಗೆ ಹೋಗಿ
2. "Generate new token" → "Generate new token (classic)" ಕ್ಲಿಕ್ ಮಾಡಿ
3. ನಿಮ್ಮ ಟೋಕನಿಗೆ ವಿವರಣಾತ್ಮಕ ಹೆಸರು ನೀಡಿ (ಉದಾ: "MCP Calculator Client")
4. ಅವಧಿ ಅವಶ್ಯಕತೆ ಪ್ರಕಾರ ಸೆಟ್ ಮಾಡಿ
5. ಕೆಳಗಿನ ವ್ಯಾಪ್ತಿಗಳನ್ನು ಆಯ್ಕೆಮಾಡಿ:
   - `repo` (ಖಾಸಗಿ ರೆಪೊಗಳ ಪ್ರವೇಶಕ್ಕೆ)
   - `user:email`
6. "Generate token" ಕ್ಲಿಕ್ ಮಾಡಿ
7. **ಮುಖ್ಯ**: ಟೋಕನ್ ಅನ್ನು ತಕ್ಷಣ ನಕಲಿಸಿ - ನೀವು ಅದನ್ನು ಮತ್ತೆ ನೋಡಲು ಸಾಧ್ಯವಿಲ್ಲ!

### 3. ಪರಿಸರ ಚರವನ್ನು ಸೆಟ್ ಮಾಡಿ

#### ವಿಂಡೋಸ್ (ಕಮಾಂಡ್ ಪ್ರಾಂಪ್ಟ್) ನಲ್ಲಿ:
```cmd
set GITHUB_TOKEN=your_github_token_here
```

#### ವಿಂಡೋಸ್ (ಪವರ್‌ಶೆಲ್) ನಲ್ಲಿ:
```powershell
$env:GITHUB_TOKEN="your_github_token_here"
```

#### ಮ್ಯಾಕ್‌ಒಎಸ್/ಲಿನಕ್ಸ್ನಲ್ಲಿ:
```bash
export GITHUB_TOKEN=your_github_token_here
```

## ಸೆಟಪ್ ಮತ್ತು ಸ್ಥಾಪನೆ

1. **ಪ್ರಾಜೆಕ್ಟ್ ಡೈರೆಕ್ಟರಿಯನ್ನು ಕ್ಲೋನ್ ಅಥವಾ ನ್ಯಾವಿಗೇಟ್ ಮಾಡಿ**

2. **ಆವಶ್ಯಕತೆಗಳನ್ನು ಸ್ಥಾಪಿಸಿ**:
   ```cmd
   mvnw clean install
   ```
   ಅಥವಾ ನೀವು ಮೇವೆನ್ ಅನ್ನು ಜಾಗತಿಕವಾಗಿ ಸ್ಥಾಪಿಸಿರುವರೆ:
   ```cmd
   mvn clean install
   ```

3. **ಪರಿಸರ ಚರವನ್ನು ಸೆಟ್ ಮಾಡಿ** ("GitHub ಟೋಕನ್ ಪಡೆಯುವುದು" ವಿಭಾಗವನ್ನು ನೋಡಿ)

4. **MCP ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಸೇವೆಯನ್ನು ಪ್ರಾರಂಭಿಸಿ**:
   ಅಧ್ಯಾಯ 1 ರ MCP ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಸೇವೆ `http://localhost:8080/sse` ನಲ್ಲಿ ಚಾಲನೆಯಲ್ಲಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ. ಕ್ಲೈಂಟ್ ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು ಇದು ಚಾಲನೆಯಲ್ಲಿರಬೇಕು.

## ಅಪ್ಲಿಕೇಶನ್ ಚಾಲನೆ

```cmd
mvnw clean package
java -jar target\calculator-llm-client-0.0.1-SNAPSHOT.jar
```

## ಅಪ್ಲಿಕೇಶನ್ ಏನು ಮಾಡುತ್ತದೆ

ಅಪ್ಲಿಕೇಶನ್ ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಸೇವೆಯೊಂದಿಗೆ ಮೂರು ಪ್ರಮುಖ ಸಂವಹನಗಳನ್ನು ತೋರಿಸುತ್ತದೆ:

1. **ಸೇರಿಸುವಿಕೆ**: 24.5 ಮತ್ತು 17.3 ರ ಮೊತ್ತವನ್ನು ಲೆಕ್ಕಹಾಕುತ್ತದೆ
2. **ವರ್ಗಮೂಲ**: 144 ರ ವರ್ಗಮೂಲವನ್ನು ಲೆಕ್ಕಹಾಕುತ್ತದೆ
3. **ಸಹಾಯ**: ಲಭ್ಯವಿರುವ ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಕಾರ್ಯಗಳನ್ನು ತೋರಿಸುತ್ತದೆ

## ನಿರೀಕ್ಷಿತ ಔಟ್‌ಪುಟ್

ಯಶಸ್ವಿಯಾಗಿ ಚಾಲನೆಯಾಗುವಾಗ, ನೀವು ಹೀಗೊಂದು ಔಟ್‌ಪುಟ್ ನೋಡಬಹುದು:

```
The sum of 24.5 and 17.3 is 41.8.
The square root of 144 is 12.
The calculator service provides the following functions: add, subtract, multiply, divide, sqrt, power...
```

## ಸಮಸ್ಯೆ ಪರಿಹಾರ

### ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳು

1. **"GITHUB_TOKEN ಪರಿಸರ ಚರ ಸೆಟ್ ಆಗಿಲ್ಲ"**
   - ನೀವು `GITHUB_TOKEN` ಪರಿಸರ ಚರವನ್ನು ಸೆಟ್ ಮಾಡಿದ್ದೀರಾ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
   - ಚರವನ್ನು ಸೆಟ್ ಮಾಡಿದ ನಂತರ ನಿಮ್ಮ ಟರ್ಮಿನಲ್/ಕಮಾಂಡ್ ಪ್ರಾಂಪ್ಟ್ ಅನ್ನು ಮರುಪ್ರಾರಂಭಿಸಿ

2. **"localhost:8080 ಗೆ ಸಂಪರ್ಕ ನಿರಾಕರಿಸಲಾಗಿದೆ"**
   - MCP ಕ್ಯಾಲ್ಕುಲೇಟರ್ ಸೇವೆ 8080 ಪೋರ್ಟ್‌ನಲ್ಲಿ ಚಾಲನೆಯಲ್ಲಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ
   - ಇನ್ನೊಂದು ಸೇವೆ 8080 ಪೋರ್ಟ್ ಬಳಸುತ್ತಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ

3. **"ಪ್ರಮಾಣೀಕರಣ ವಿಫಲವಾಗಿದೆ"**
   - ನಿಮ್ಮ GitHub ಟೋಕನ್ ಮಾನ್ಯವಾಗಿದ್ದು ಸರಿಯಾದ ಅನುಮತಿಗಳನ್ನು ಹೊಂದಿದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ
   - GitHub ಮಾದರಿಗಳಿಗೆ ನಿಮ್ಮ ಪ್ರವೇಶವನ್ನು ಪರಿಶೀಲಿಸಿ

4. **ಮೇವನ್ ನಿರ್ಮಾಣ ದೋಷಗಳು**
   - ನೀವು ಜಾವಾ 21 ಅಥವಾ ಹೆಚ್ಚಿನ ಆವೃತ್ತಿಯನ್ನು ಬಳಸುತ್ತಿದ್ದೀರಾ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ: `java -version`
   - ನಿರ್ಮಾಣವನ್ನು ಕ್ಲೀನ್ ಮಾಡಲು ಪ್ರಯತ್ನಿಸಿ: `mvnw clean`

### ಡಿಬಗ್

ಡಿಬಗ್ ಲಾಗಿಂಗ್ ಸಕ್ರಿಯಗೊಳಿಸಲು, ಚಾಲನೆಯಾಗುವಾಗ ಕೆಳಗಿನ JVM ಆರ್ಗ್ಯುಮೆಂಟ್ ಅನ್ನು ಸೇರಿಸಿ:
```cmd
java -Dlogging.level.dev.langchain4j=DEBUG -jar target\calculator-llm-client-0.0.1-SNAPSHOT.jar
```

## ಸಂರಚನೆ

ಅಪ್ಲಿಕೇಶನ್ ಈ ಕೆಳಗಿನಂತೆ ಸಂರಚಿಸಲಾಗಿದೆ:
- GitHub ಮಾದರಿಗಳನ್ನು `gpt-4.1-nano` ಮಾದರಿಯೊಂದಿಗೆ ಬಳಸುವುದು
- MCP ಸೇವೆಗೆ `http://localhost:8080/sse` ನಲ್ಲಿ ಸಂಪರ್ಕಿಸುವುದು
- ವಿನಂತಿಗಳಿಗೆ 60 ಸೆಕೆಂಡುಗಳ ಟೈಮೌಟ್ ಬಳಸುವುದು
- ಡಿಬಗ್‌ಗಾಗಿ ವಿನಂತಿ/ಪ್ರತಿಕ್ರಿಯೆ ಲಾಗಿಂಗ್ ಸಕ್ರಿಯಗೊಳಿಸುವುದು

## ಅವಲಂಬನೆಗಳು

ಈ ಪ್ರಾಜೆಕ್ಟ್‌ನಲ್ಲಿ ಬಳಸಲಾಗುವ ಪ್ರಮುಖ ಅವಲಂಬನೆಗಳು:
- **LangChain4j**: AI ಏಕೀಕರಣ ಮತ್ತು ಉಪಕರಣ ನಿರ್ವಹಣೆಗೆ
- **LangChain4j MCP**: ಮಾದರಿ ಸಂದರ್ಭ ಪ್ರೋಟೋಕಾಲ್ ಬೆಂಬಲಕ್ಕೆ
- **LangChain4j GitHub Models**: GitHub ಮಾದರಿಗಳ ಏಕೀಕರಣಕ್ಕೆ
- **Spring Boot**: ಅಪ್ಲಿಕೇಶನ್ ಫ್ರೇಮ್ವರ್ಕ್ ಮತ್ತು ಅವಲಂಬನೆ ಇಂಜೆಕ್ಷನ್‌ಗೆ
