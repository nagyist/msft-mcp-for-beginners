# ಪ್ರಕರಣ ಅಧ್ಯಯನ: MCP ಸರ್ವರ್ ಆಗಿ API ನಿರ್ವಹಣೆಯಲ್ಲಿ REST API ಅನ್ನು ಬಹಿರಂಗಪಡಿಸು

Azure API ನಿರ್ವಹಣೆ, ನಿಮ್ಮ API ಅಂತಿಮ ಬಿಂದುಗಳ ಮೇಲಿನ ಗೇಟ್‌ವೇ ಒದಗಿಸುವ ಸೇವೆಯಾಗಿದೆ. ಇದವು ಹೇಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆಂದರೆ Azure API ನಿರ್ವಹಣೆ ನಿಮ್ಮ API ಗಳು ಮುಂದೆ ಪ್ರಾಕ್ಸಿ ಆಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ ಮತ್ತು ಬರುವಾಗಿರುವ ವಿನಂತಿಗಳೊಂದಿಗೆ ಏನು ಮಾಡಬೇಕು ಎಂದು ತೀರ್ಮಾನಿಸಬಹುದು.

ಅದರ ಬಳಕೆಯಿಂದ, ನೀವು ಹಲವಾರು ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಸೇರಿಸಬಹುದು, ಉದಾಹರಣೆಗೆ:

- **ಸುರಕ್ಷತೆ**, ನೀವು API ಕೀಯಿಂದ JWT ಗಿಂತಲೂ ಮ್ಯಾನೆಜ್ಡ್ ಐಡೆಂಟಿಟಿ ವರೆಗೆ ಎಲ್ಲವನ್ನು ಬಳಸಬಹುದು.
- **ರೇಟು ಮಿತಿಗೊಳಿಸುವಿಕೆ**, ಒಂದು ಉತ್ತಮ ವೈಶಿಷ್ಟ್ಯ ಇದು, ಇದು ನಿಗದಿತ ಕಾಲಘಟ್ಟಕ್ಕೊಂದು ಎಷ್ಟು ಕರೆಗಳನ್ನು ಅನುಮತಿಸಬೇಕೆಂದು ನಿರ್ಧಾರ ಮಾಡುವುದಕ್ಕೆ ಸಹಾಯ ಮಾಡುತ್ತದೆ. ಇದು ಎಲ್ಲಾ ಬಳಕೆದಾರರಿಗೆ ಒಳ್ಳೆಯ ಅನುಭವ ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಲು ಮತ್ತು ನಿಮ್ಮ ಸೇವೆ ವಿನಂತಿಗಳಿಂದ ಭಾರವಾಗದೇ ಇರುವುದು ಸಹಾಯ ಮಾಡುತ್ತದೆ.
- **ಸ್ಕೇಲಿಂಗ್ & ಲೋಡ್ ಬ್ಯಾಲನ್ಸಿಂಗ್**. ನೀವು ಲೋಡ್ ತೂಕವನ್ನು ಸಮತೋಲಗೊಳಿಸಲು ಹಲವಾರು ಅಂತಿಮ ಬಿಂದುಗಳನ್ನು ಹೊಂದಿಸಬಹುದು ಮತ್ತು "ಲೋಡ್ ಬ್ಯಾಲನ್ಸ್" ಹೇಗೆ ಮಾಡಬೇಕೆಂದು ಕೂಡ ನಿರ್ಧರಿಸಬಹುದು.
- **ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆಯ ವೈಶಿಷ್ಟ್ಯಗಳು**ಂತೆ ಸೆಮಾಂಟಿಕ್ ಕ್ಯಾಶಿಂಗ್, ಟೋಕನ್ ಮಿತಿಗಳು ಮತ್ತು ಟೋಕನ್ ಮಾನಿಟರಿಂಗ್ ಹಾಗೂ ಇನ್ನಷ್ಟು. ಇವುಗಳಾಗಿಯೇ ಉತ್ತಮ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ನೀಡುತ್ತವೆ, ಪ್ರತಿಸ್ಪಂದನೆಯನ್ನು ಸುಧಾರಿಸುವಂತೆ ಮತ್ತು ನಿಮ್ಮ ಟೋಕನ್ ಖರ್ಚುಗಳ ಮೇಲೆ ಮೇಲೆ ನಿಗಾ ಇಡುವುದಕ್ಕೆ ಸಹಾಯಕ. [ಇದನ್ನು ಇಲ್ಲಿ ಓದಿ](https://learn.microsoft.com/en-us/azure/api-management/genai-gateway-capabilities).

## MCP + Azure API ನಿರ್ವಹಣೆ ಯಾಕೆ?

Model Context Protocol ವೇಗವಾಗಿ ಏಜೆಂಟಿಕ್ ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಮತ್ತು ಸಾಧನಗಳ ಮಾಹಿತಿ ನಿಯಮಿತವಾಗಿ ಬಹಿರಂಗಪಡಿಸುವ ಉದ್ಧೇಶಕ್ಕೆ ಮಾನ್ಯತೆ ಪಡೆಯುತ್ತಿದೆ. ನೀವು API ಗಳು "ನಿರ್ವಹಿಸಬೇಕಾದ" ಸಂದರ್ಭದಲ್ಲಿ Azure API ನಿರ್ವಹಣೆ ಸ್ವಾಭಾವಿಕ ಆಯ್ಕೆಯಾಗಿದೆ. MCP ಸರ್ವರ್‌ಗಳು ಬಯಲಿನಿಂದ ಇತರ API ಗಳು ಸಹ ಒಟ್ಟಿಗೆ ಕೆಲಸಮಾಡಿ ಒಂದೊಂದು ಸಾಧನಕ್ಕೆ ವಿನಂತಿಗಳನ್ನು ಪರಿಹರಿಸುತ್ತವೆ. ಆದ್ದರಿಂದ Azure API ನಿರ್ವಹಣೆ ಮತ್ತು MCP ಅನ್ನು ಒಟ್ಟಿಗೆ ಬಳಸುವುದು ಸೂಕ್ತವಾಗಿದೆ.

## ಅವಲೋಕನ

ಈ ವಿಶೇಷ ಬಳಕೆದ ಉಪಯೋಗದಲ್ಲಿ ನಾವು API ಅಂತಿಮ ಬಿಂದುಗಳನ್ನು MCP ಸರ್ವರ್ ಆಗಿ ಬಹಿರಂಗಪಡಿಸುವುದನ್ನು ಕಲಿಯುತ್ತೇವೆ. ಇದರಿಂದ, ನಾವು ಈ ಅಂತಿಮ ಬಿಂದುಗಳನ್ನು ಏಜೆಂಟಿಕ್ ಅಪ್ಲಿಕೇಶನ್‌ನ ಭಾಗವಾಗಿ ಸುಲಭವಾಗಿ ಬಳಸಬಹುದು ಮತ್ತು ಜೊತೆಗೆ Azure API ನಿರ್ವಹಣೆಯಿಂದಲೂ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಲಾಭ ಮಾಡಿಕೊಳ್ಳಬಹುದು.

## ಮುಖ್ಯ ವೈಶಿಷ್ಟ್ಯಗಳು

- ನೀವು ಸಾಧನಗಳಾಗಿ ಬಹಿರಂಗಪಡಿಸುವ ಅಂತಿಮ ಬಿಂದು ವಿಧಾನಗಳನ್ನು ಆಯ್ಕೆಮಾಡಬಹುದು.
- ನೀವು ಪಡೆಯುವ ಹೆಚ್ಚುವರಿ ವೈಶಿಷ್ಟ್ಯಗಳು ನಿಮ್ಮ API ಗಾಗಿ ನೀತಿ ವಿಭಾಗದಲ್ಲಿ ಏನು ಸಂರಚಿಸುತ್ತೀರಿ ಎಂಬುದರ ಮೇಲೆ ಅವಲಂಬಿತವಾಗಿದೆ. ಆದರೆ ಇಲ್ಲಿ ನಾವು ನೀವು ರೇಟು ಮಿತಿಗೊಳಿಸುವಿಕೆ ಹೇಗೆ ಸೇರಿಸಬಹುದು ಎಂದು ತೋರಿಸುತ್ತೇವೆ.

## ಪೂರ್ವಚರಿಣಿ: API ಅನ್ನು ಆಮದು ಮಾಡಿ

ನೀವು ಈಗಾಗಲೇ Azure API ನಿರ್ವಹಣೆಯಲ್ಲಿ API ಹೊಂದಿದ್ದರೆ ಉತ್ತಮ, ಈ ಹಂತವನ್ನು ಕಡೆಹಾಕಬಹುದು. ಇಲ್ಲದಿದ್ದರೆ, ಈ ಲಿಂಗನ್ನು ನೋಡಿ, [Azure API ನಿರ್ವಹಣೆಗೆ API ಅನ್ನು ಆಮದು ಮಾಡುವುದು](https://learn.microsoft.com/en-us/azure/api-management/import-and-publish#import-and-publish-a-backend-api).

## MCP ಸರ್ವರ್ ಆಗಿ API ಅನ್ನು ಬಹಿರಂಗಪಡಿಸಿ

API ಅಂತಿಮ ಬಿಂದುಗಳನ್ನು ಬಹಿರಂಗಪಡಿಸಲು, ಈ ಹಂತಗಳನ್ನು ಅನುಸರಿಸಿ:

1. Azure ಪೋರ್ಟಲ್ ಮತ್ತು ಈ ವಿಳಾಸಕ್ಕೆ (<https://portal.azure.com/?Microsoft_Azure_ApiManagement=mcp>) ನ್ಯಾವಿಗೇಟ್ ಮಾಡಿ.
ನಿಮ್ಮ API ನಿರ್ವಹಣಾ ಉದಾಹರಣೆಗೆ ಹೋಗಿ.

1. ಎಡ ಬದಿ ಮೆನುದಲ್ಲಿ, APIs > MCP Servers > + Create new MCP Server ಆಯ್ಕೆಮಾಡಿ.

1. API ವಿಭಾಗದಲ್ಲಿ, MCP ಸರ್ವರ್ ಆಗಿ ಬಹಿರಂಗಪಡಿಸಲು REST API ಆಯ್ಕೆಮಾಡಿ.

1. ಸಾಧನಗಳಾಗಿ ಬಹಿರಂಗಪಡಿಸಲು ಒಂದು ಅಥವಾ ಹೆಚ್ಚಿನ API ಕಾರ್ಯಗಳನ್ನು (Operations) ಆಯ್ಕೆಮಾಡಿ. ನೀವು ಎಲ್ಲಾ ಕಾರ್ಯಗಳನ್ನು ಅಥವಾ ಕೆಲವು ವಿಶೇಷ ಕಾರ್ಯಗಳನ್ನು ಆಯ್ಕೆಮಾಡಬಹುದು.

    ![ಬಹಿರಂಗಪಡಿಸುವ ವಿಧಾನಗಳನ್ನು ಆಯ್ಕೆಮಾಡಿ](https://learn.microsoft.com/en-us/azure/api-management/media/export-rest-mcp-server/create-mcp-server-small.png)


1. **Create** ಆಯ್ಕೆಮಾಡಿ.

1. **APIs** ಮತ್ತು **MCP Servers** ಮೆನು ಆಯ್ಕೆಮಾಡಿ, ಕೆಳಕಂಡಂತೆ ಕಾಣಿಸಿಕೊಳ್ಳಬೇಕು:

    ![ಮುಖ್ಯ ವಿಂಡೋದಲ್ಲಿ MCP ಸರ್ವರ್ ನೋಡಿ](https://learn.microsoft.com/en-us/azure/api-management/media/export-rest-mcp-server/mcp-server-list.png)

    MCP ಸರ್ವರ್ ರಚಿಸಲಾಗಿದೆ ಮತ್ತು API ಕಾರ್ಯಗಳು ಸಾಧನಗಳಾಗಿ ಬಹಿರಂಗಪಡಿಸಲ್ಪಟ್ಟಿವೆ. MCP ಸರ್ವರ್ MCP Servers ಪಟ್ಟಿಯಲ್ಲಿ ಕಾಣಿಸುತ್ತದೆ. URL ಕಾಲಮ್ MCP ಸರ್ವರ್‌ನ ಅಂತಿಮ ಬಿಂದುವನ್ನು ತೋರಿಸುತ್ತದೆ, ಇದನ್ನು ಪರೀಕ್ಷೆಗಾಗಿ ಅಥವಾ ಕ್ಲೈಂಟ್ ಅಪ್ಲಿಕೇಶನ್ ಒಳಗೆ ಕರೆ ಮಾಡಬಹುದು.

## ಐಚ್ಛಿಕ: ನೀತಿಗಳನ್ನು ಸಂರಚನೆ ಮಾಡಿ

Azure API ನಿರ್ವಹಣೆಯಲ್ಲಿ ನೀತಿಗಳು ಎಂಬ ಮುಖ್ಯ概念ವಿದೆ, ಇದು ನೀವು ರೇಟು ಮಿತಿಗೊಳಿಸುವಿಕೆ ಅಥವಾ ಸೆಮಾಂಟಿಕ್ ಕ್ಯಾಶಿಂಗ್ ಮುಂತಾದಂತೆ ನಿಮ್ಮ ಅಂತಿಮ ಬಿಂದುಗಳಿಗೆ ಬೇರ್ಪಡಿಸಬಹುದಾದ ನಿಯಮಗಳನ್ನು ಹೊಂದಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ಈ ನೀತಿಗಳು XML ನಲ್ಲಿ ಬರೆಯಲ್ಪಡುತ್ತವೆ.

ನಿಮ್ಮ MCP ಸರ್ವರ್‌ಗೆ ರೇಟು ಮಿತಿಗೊಳಿಸುವ ನೀತಿಯನ್ನು ಹೇಗೆ ಸೇರಿಸಬಹುದು ಅನ್ನೋದನ್ನು ಇಲ್ಲಿದೆ:

1. ಪೋರ್ಟಲ್ನಲ್ಲಿ, APIs ಅಡಿಯಲ್ಲಿ **MCP Servers** ಆಯ್ಕೆಮಾಡಿ.

1. ನೀವು ರಚಿಸಿದ MCP ಸರ್ವರ್ ಆಯ್ಕೆಮಾಡಿ.

1. ಎಡ ಅರ್ತದಲ್ಲಿ, MCP ಅಡಿಯಲ್ಲಿ **Policies** ಆಯ್ಕೆಮಾಡಿ.

1. ನೀತಿ ಸಂಪಾದಕದಲ್ಲಿ, MCP ಸರ್ವರ್ ಸಾಧನಗಳಿಗೆ ಅನ್ವಯಿಸಬೇಕಾದ ನೀತಿಗಳನ್ನು ಸೇರಿಸಿ ಅಥವಾ ಸಂಪಾದಿಸಿ. ನೀತಿಗಳು XML ಸ್ವರೂಪದಲ್ಲಿ ವ್ಯಾಖ್ಯಾನಿಸಲ್ಪಡುತ್ತವೆ. ಉದಾಹರಣೆಗೆ, MCP ಸರ್ವರ್ ಸಾಧನಗಳಿಗೆ ಕರೆಗಳನ್ನು ಮಿತಿಗೊಳಿಸುವ ನೀತಿ (ಈ ಉದಾಹರಣೆಯಲ್ಲಿ, ಪ್ರತಿ ಕ್ಲೈಂಟ್ IP ವಿಳಾಸಕ್ಕೆ 30 ಸೆಕೆಂಡಿನಲ್ಲಿ 5 ಕರೆಗಳು) ಸೇರಿಸಬಹುದು. ಈ ಕೆಳಗಿನ XML ರೇಟು ಮಿತಿ ಹೊಂದುತ್ತದೆ:

    ```xml
     <rate-limit-by-key calls="5" 
       renewal-period="30" 
       counter-key="@(context.Request.IpAddress)" 
       remaining-calls-variable-name="remainingCallsPerIP" 
    />
    ```

    ನೀತಿ ಸಂಪಾದಕದ ಚಿತ್ರ:

    ![ನೀತಿ ಸಂಪಾದಕ](https://learn.microsoft.com/en-us/azure/api-management/media/export-rest-mcp-server/mcp-server-policies-small.png)
 
## ಪ್ರಯತ್ನಿಸಿ

ನಮ್ಮ MCP ಸರ್ವರ್ ನಿರೀಕ್ಷಿಸಿದಂತೆ ಕೆಲಸ ಮಾಡುತ್ತದೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸೋಣ.

ಇದರಿಗಾಗಿ, ನಾವು Visual Studio Code ಮತ್ತು GitHub ಕೋಪಿಲಟ್ ಹಾಗೂ ಅದರ ಏಜೆಂಟ್ ಮೋಡ್ ಅನ್ನು ಬಳಸುತ್ತೇವೆ. ನಾವು MCP ಸರ್ವರ್ ಅನ್ನು *mcp.json* ನಲ್ಲಿ ಸೇರಿಸುವೆವು. ಇದರಿಂದ Visual Studio Code ಏಜೆಂಟಿಕ್ ಸಾಮರ್ಥ್ಯಗಳೊಂದಿಗೆ ಕ್ಲೈಂಟ್ ಆಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸಿ ಅಂತಿಮ ಬಳಕೆದಾರರು ಪ್ರಾಂಪ್ಟ್ ಟೈಪ್ ಮಾಡಿ ಸರ್ವರ್‌ನೊಂದಿಗೆ ಸಂವಹನ ಮಾಡಬಹುದು.

Visual Studio Code ನಲ್ಲಿ MCP ಸರ್ವರ್ ಸೇರಿಸುವ ವಿಧಾನ:

1. Command Palette ನಿಂದ MCP: **Add Server ಆಜ್ಞೆ** ಬಳಸಿ.

1. ಪ್ರಾಂಪ್ಟ್ ಬಂದಾಗ, ಸರ್ವರ್ ಪ್ರಕಾರ ಆಯ್ಕೆಮಾಡಿ: **HTTP (HTTP ಅಥವಾ Server Sent Events)**.

1. API ನಿರ್ವಹಣೆಯಲ್ಲಿ MCP ಸರ್ವರ್ URL ನಮೂದಿಸಿ. ಉದಾಹರಣೆ: **https://<apim-service-name>.azure-api.net/<api-name>-mcp/sse** (SSE ಬಿಂದುಗಾಗಿ) ಅಥವಾ **https://<apim-service-name>.azure-api.net/<api-name>-mcp/mcp** (MCP ಬಿಂದುಗಾಗಿ), ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ನಡುವಿನ ವ್ಯತ್ಯಾಸ `/sse` ಅಥವಾ `/mcp` ಆಗಿದೆಂದು ಗಮನಿಸಿ.

1. ನಿಮ್ಮ ಇಚ್ಛೆಯ ಸರ್ವರ್ ID ನಮೂದಿಸಿ. ಇದು ಮುಖ್ಯಮೌಲ್ಯವಲ್ಲ, ಆದರೆ ಈ ಸರ್ವರ್ ಉದಾಹರಣೆಯನ್ನು ನೀವು ನೆನಪಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

1. ಸಂರಚನೆಯನ್ನು ನಿಮ್ಮ ವರ್ಕ್‌ಸ್ಪೇಸ್ ಸೆಟ್ಟಿಂಗ್ಸ್ ಅಥವಾ ಯೂಸರ್ ಸೆಟ್ಟಿಂಗ್ಸ್‌ಗೆ ಉಳಿಸುವಿರಾ ಎಂಬುದನ್ನು ಆಯ್ಕೆಮಾಡಿ.

  - **ವರ್ಕ್‌ಸ್ಪೇಸ್ ಸೆಟ್ಟಿಂಗ್ಸ್** - ಸರ್ವರ್ ಸಂರಚನೆ .vscode/mcp.json ಫೈಲ್‌ಗೆ ಉಳಿಯುತ್ತದೆ, ಇದು ಪ್ರಸ್ತುತ ವರ್ಕ್‌ಸ್ಪೇಸ್‌ಗೆ ಮಾತ್ರ ಲಭ್ಯ.

    *mcp.json*

    ```json
    "servers": {
        "APIM petstore" : {
            "type": "sse",
            "url": "url-to-mcp-server/sse"
        }
    }
    ```

    ಅಥವಾ ನೀವು ಸ್ಟ್ರೀಮಿಂಗ್ HTTP ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಆಯ್ಕೆಮಾಡಿದರೆ ಸ್ವಲ್ಪ ವ್ಯತ್ಯಾಸವಿರುತ್ತದೆ:

    ```json
    "servers": {
        "APIM petstore" : {
            "type": "http",
            "url": "url-to-mcp-server/mcp"
        }
    }
    ```

  - **ಯೂಸರ್ ಸೆಟ್ಟಿಂಗ್ಸ್** - ಸರ್ವರ್ ಸಂರಚನೆ ನಿಮ್ಮ ಜಾಗತಿಕ *settings.json* ಫೈಲ್‌ಗೆ ಸೇರ್ಪಡೆಗೊಳ್ಳುತ್ತದೆ ಮತ್ತು ಎಲ್ಲಾ ವರ್ಕ್‌ಸ್ಪೇಸ್‌ಗಳಲ್ಲಿ ಲಭ್ಯವಾಗುತ್ತದೆ. ಸಂರಚನೆ ಹೀಗೆ ಕಾಣುತ್ತದೆ:

    ![ಯೂಸರ್ ಸೆಟ್ಟಿಂಗ್](https://learn.microsoft.com/en-us/azure/api-management/media/export-rest-mcp-server/mcp-servers-visual-studio-code.png)

1. ನೀವು Azure API ನಿರ್ವಹಣೆಯ ಕಡೆಗೆ ಸರಿಯಾಗಿ ದೃಢೀಕರಿಸಲು ಒಂದು ಹೆಡರ್ ಅನ್ನು ಕೂಡ ಸೇರಿಸಬೇಕು. ಇದು **Ocp-Apim-Subscription-Key** ಹೆಡರ್ ಅನ್ನು ಬಳಸುತ್ತದೆ.

    - ಹೇಗೆ ಸೆಟ್ಟಿಂಗ್ಸ್‌ಗೆ ಸೇರಿಸುವುದು:

    ![ಪ್ರಮಾಣೀಕರಣಕ್ಕಾಗಿ ಹೆಡರ್ ಸೇರಿಸುವುದು](https://learn.microsoft.com/en-us/azure/api-management/media/export-rest-mcp-server/mcp-server-with-header-visual-studio-code.png), ಇದರಿಂದ ನೀವು API ಕೀ ಮೌಲ್ಯವನ್ನು ಕೇಳುವ ಪ್ರಾಂಪ್ಟ್ ಕಾಣಿಸುತ್ತದೆ, ಇದು Azure ಪೋರ್ಟಲ್ ನಲ್ಲಿ ನಿಮ್ಮ Azure API ನಿರ್ವಹಣೆ ಉದಾಹರಣೆಯಿಂದ ಪಡೆಯಬಹುದು.

   - ಬದಲಿಗೆ *mcp.json* ಗೆ ಸೇರಿಸಲು ಹೇಗೆ ಮಾಡಬಹುದು:

    ```json
    "inputs": [
      {
        "type": "promptString",
        "id": "apim_key",
        "description": "API Key for Azure API Management",
        "password": true
      }
    ]
    "servers": {
        "APIM petstore" : {
            "type": "http",
            "url": "url-to-mcp-server/mcp",
            "headers": {
                "Ocp-Apim-Subscription-Key": "Bearer ${input:apim_key}"
            }
        }
    }
    ```

### ಏಜೆಂಟ್ ಮೋಡ್ ಬಳಸಿ

ಈಗ ನಾವು ಸೆಟ್ಟಿಂಗ್ಸ್ ಅಥವಾ *.vscode/mcp.json* ನಲ್ಲಿ ಸಿದ್ಧವಾಗಿರುವೆವು. ಪ್ರಯತ್ನಿಸಿ.

ಸರ್ವರ್‌ನಿಂದ ಬಹಿರಂಗಪಡಿಸಲ್ಪಟ್ಟ ಸಾಧನಗಳು ಪಟ್ಟಿಯ ರೂಪದಲ್ಲಿ ಇರುತ್ತದೆ ಎಂದು ಸಾಧನಗಳ ಐಕಾನ್ ಇರಬೇಕು:

![ಸರ್ವರ್‌ನ ಸಾಧನಗಳು](https://learn.microsoft.com/en-us/azure/api-management/media/export-rest-mcp-server/tools-button-visual-studio-code.png)

1. ಸಾಧನಗಳ ಐಕಾನ್ ಮೇಲೆ ಕ್ಲಿಕ್ ಮಾಡಿ, ಸಾಧನಗಳ ಪಟ್ಟಿಯನ್ನು ಕಾಣಿಸಿಕೊಳ್ಳಬೇಕು:

    ![ಸಾಧನಗಳು](https://learn.microsoft.com/en-us/azure/api-management/media/export-rest-mcp-server/select-tools-visual-studio-code.png)

1. ಸಾಧನವನ್ನು ಕರೆ ಮಾಡಲು ಚಾಟ್ ನಲ್ಲಿ ಪ್ರಾಂಪ್ಟ್ ನಮೂದಿಸಿ. ಉದಾಹರಣೆಗೆ, ನೀವು ಆರ್ಡರ್ ಬಗ್ಗೆ ಮಾಹಿತಿ ಪಡೆಯಲಿರುವ ಸಾಧನ ಆಯ್ಕೆಮಾಡಿದ್ದರೆ, ಏಜೆಂಟ್‌ಗೆ ಆರ್ಡರ್ ಬಗ್ಗೆ ಕೇಳಬಹುದು. ಕೆಳಗಿನ ಉದಾಹರಣೆಯ ಪ್ರಾಂಪ್ಟ್:

    ```text
    get information from order 2
    ```

    ಇದೀಗ ನಿಮ್ಮ ಬಳಿ ಸಾಧನವನ್ನು ಕರೆ ಮಾಡಲು ಕೇಳುವ ಸಾಧನಗಳ ಐಕಾನ್ ಪ್ರಸ್ತುತವಾಗುತ್ತದೆ. ಸಾಧನವನ್ನು ಬಳಸಲು ಮುಂದುವರಿಯಿರಿ, ನೀವು ಕೆಳಗಿನ ರೀತಿಯ ಔಟ್‌ಪುಟ್ ನೋಡಬಹುದು:

    ![ಪ್ರಾಂಪ್ಟ್‌ನಿಂದ ಫಲಿತಾಂಶ](https://learn.microsoft.com/en-us/azure/api-management/media/export-rest-mcp-server/chat-results-visual-studio-code.png)

    **ಮೀಡಿಯಲ್ಲಿಯೂ ನೀವು ಹೊಂದಿರುವ ಸಾಧನಗಳ ಆಧಾರದ ಮೇಲೆ ಇದು ಬದಲಾಗಬಹುದು, ಆದರೆ ಉದ್ದೇಶವು ಮೇಲಿನಂತೆಯೇ ಪಠ್ಯಾತ್ಮಕ ಪ್ರತಿಕ್ರಿಯೆಯನ್ನು ಪಡೆಯುವುದು.**
