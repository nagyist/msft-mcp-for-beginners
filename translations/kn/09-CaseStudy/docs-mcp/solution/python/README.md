# ಚೈನ್‌ಲಿಟ್ ಮತ್ತು ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಸ್ MCP ಜೊತೆಗೆ ಅಧ್ಯಯನ ಯೋಜನೆ ಜನರೇಟರ್

## ಪೂರ್ವಾಪೇಕ್ಷಿತಗಳು

- ಪೈಥಾನ್ 3.8 ಅಥವಾ ಹೆಚ್ಚಿನ ಆವೃತ್ತಿ
- ಪಿಪ್ (ಪೈಥಾನ್ ಪ್ಯಾಕೇಜ್ ಮ್ಯಾನೇಜರ್)
- ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಸ್ MCP ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕಿಸಲು ಇಂಟರ್ನೆಟ್ ಪ್ರವೇಶ

## ಸ್ಥಾಪನೆ

1. ಈ ರೆಪೊಸಿಟರಿಯನ್ನು ಕ್ಲೋನ್ ಮಾಡಿ ಅಥವಾ ಪ್ರಾಜೆಕ್ಟ್ ಫೈಲ್‌ಗಳನ್ನು ಡೌನ್‌ಲೋಡ್ ಮಾಡಿ.
2. ಅಗತ್ಯವಿರುವ ಅವಲಂಬನೆಗಳನ್ನು ಸ್ಥಾಪಿಸಿ:

   ```bash
   pip install -r requirements.txt
   ```

## ಬಳಕೆ

### ದೃಶ್ಯ 1: ಡಾಕ್ಸ್ MCP ಗೆ ಸರಳ ಪ್ರಶ್ನೆ
ಡಾಕ್ಸ್ MCP ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕಿಸುವ, ಪ್ರಶ್ನೆಯನ್ನು ಕಳುಹಿಸುವ ಮತ್ತು ಫಲಿತಾಂಶವನ್ನು ಮುದ್ರಿಸುವ ಕಮಾಂಡ್-ಲೈನ್ ಕ್ಲೈಂಟ್.

1. ಸ್ಕ್ರಿಪ್ಟ್ ಅನ್ನು ಚಾಲನೆ ಮಾಡಿ:
   ```bash
   python scenario1.py
   ```
2. ಪ್ರಾಂಪ್ಟ್‌ನಲ್ಲಿ ನಿಮ್ಮ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಪ್ರಶ್ನೆಯನ್ನು ನಮೂದಿಸಿ.

### ದೃಶ್ಯ 2: ಅಧ್ಯಯನ ಯೋಜನೆ ಜನರೇಟರ್ (ಚೈನ್‌ಲಿಟ್ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್)
ಬಳಕೆದಾರರಿಗೆ ಯಾವುದೇ ತಾಂತ್ರಿಕ ವಿಷಯಕ್ಕಾಗಿ ವೈಯಕ್ತಿಕ, ವಾರದಿಂದ ವಾರಕ್ಕೆ ಅಧ್ಯಯನ ಯೋಜನೆಯನ್ನು ರಚಿಸಲು ಅನುಮತಿಸುವ ವೆಬ್ ಆಧಾರಿತ ಇಂಟರ್ಫೇಸ್ (ಚೈನ್‌ಲಿಟ್ ಬಳಸಿ).

1. ಚೈನ್‌ಲಿಟ್ ಅಪ್ಲಿಕೇಶನ್ ಪ್ರಾರಂಭಿಸಿ:
   ```bash
   chainlit run scenario2.py
   ```
2. ನಿಮ್ಮ ಟರ್ಮಿನಲ್‌ನಲ್ಲಿ ನೀಡಲಾದ ಸ್ಥಳೀಯ URL (ಉದಾ: http://localhost:8000) ಅನ್ನು ಬ್ರೌಸರ್‌ನಲ್ಲಿ ತೆರೆಯಿರಿ.
3. ಚಾಟ್ ವಿಂಡೋದಲ್ಲಿ ನಿಮ್ಮ ಅಧ್ಯಯನ ವಿಷಯ ಮತ್ತು ನೀವು ಅಧ್ಯಯನ ಮಾಡಲು ಬಯಸುವ ವಾರಗಳ ಸಂಖ್ಯೆಯನ್ನು ನಮೂದಿಸಿ (ಉದಾ: "AI-900 ಪ್ರಮಾಣಪತ್ರ, 8 ವಾರಗಳು").
4. ಅಪ್ಲಿಕೇಶನ್ ವಾರದಿಂದ ವಾರಕ್ಕೆ ಅಧ್ಯಯನ ಯೋಜನೆಯನ್ನು ಪ್ರತಿಕ್ರಿಯಿಸುತ್ತದೆ, ಸಂಬಂಧಿತ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಲಿಂಕ್‌ಗಳೊಂದಿಗೆ.

**ಅವಶ್ಯಕ ಪರಿಸರ ಚರಗಳು:**

ದೃಶ್ಯ 2 (ಏಜ್ಯೂರ್ ಓಪನ್‌ಎಐ ಜೊತೆಗೆ ಚೈನ್‌ಲಿಟ್ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್) ಬಳಸಲು, ನೀವು `python` ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ `.env` ಫೈಲ್‌ನಲ್ಲಿ ಕೆಳಗಿನ ಪರಿಸರ ಚರಗಳನ್ನು ಹೊಂದಿರಬೇಕು:

```
AZURE_OPENAI_CHAT_DEPLOYMENT_NAME=
AZURE_OPENAI_API_KEY=
AZURE_OPENAI_ENDPOINT=
AZURE_OPENAI_API_VERSION=
```

ಅಪ್ಲಿಕೇಶನ್ ಚಾಲನೆಗೆ ಮುಂಚೆ ನಿಮ್ಮ ಏಜ್ಯೂರ್ ಓಪನ್‌ಎಐ ಸಂಪನ್ಮೂಲ ವಿವರಗಳನ್ನು ಈ ಮೌಲ್ಯಗಳಲ್ಲಿ ತುಂಬಿ.

> [!TIP]
> ನೀವು ಸುಲಭವಾಗಿ ನಿಮ್ಮ ಸ್ವಂತ ಮಾದರಿಗಳನ್ನು [Azure AI Foundry](https://ai.azure.com/) ಬಳಸಿ ನಿಯೋಜಿಸಬಹುದು.

### ದೃಶ್ಯ 3: VS ಕೋಡ್‌ನಲ್ಲಿ MCP ಸರ್ವರ್ ಜೊತೆಗೆ ಇನ್-ಎಡಿಟರ್ ಡಾಕ್ಸ್

ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಹುಡುಕಲು ಬ್ರೌಸರ್ ಟ್ಯಾಬ್‌ಗಳನ್ನು ಬದಲಾಯಿಸುವ ಬದಲು, ನೀವು ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಸ್ ಅನ್ನು ನೇರವಾಗಿ VS ಕೋಡ್‌ನಲ್ಲಿ MCP ಸರ್ವರ್ ಬಳಸಿ ತರಬಹುದು. ಇದರಿಂದ ನೀವು:
- ನಿಮ್ಮ ಕೋಡಿಂಗ್ ಪರಿಸರವನ್ನು ಬಿಟ್ಟುಬಿಡದೆ VS ಕೋಡ್ ಒಳಗೆ ಡಾಕ್ಸ್ ಹುಡುಕಬಹುದು ಮತ್ತು ಓದಬಹುದು.
- ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಉಲ್ಲೇಖಿಸಿ ಮತ್ತು README ಅಥವಾ ಕೋರ್ಸ್ ಫೈಲ್‌ಗಳಲ್ಲಿ ನೇರವಾಗಿ ಲಿಂಕ್‌ಗಳನ್ನು ಸೇರಿಸಬಹುದು.
- GitHub Copilot ಮತ್ತು MCP ಅನ್ನು ಒಟ್ಟಿಗೆ ಬಳಸಿಕೊಂಡು ಸುಗಮ, AI-ಚಾಲಿತ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಕಾರ್ಯಪ್ರವಾಹವನ್ನು ಅನುಭವಿಸಬಹುದು.

**ಉದಾಹರಣಾ ಬಳಕೆ ಪ್ರಕರಣಗಳು:**
- ಕೋರ್ಸ್ ಅಥವಾ ಪ್ರಾಜೆಕ್ಟ್ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಬರೆಯುವಾಗ README ಗೆ ತ್ವರಿತವಾಗಿ ಉಲ್ಲೇಖ ಲಿಂಕ್‌ಗಳನ್ನು ಸೇರಿಸಿ.
- ಕೋಡ್ ರಚಿಸಲು Copilot ಬಳಸಿ ಮತ್ತು ಸಂಬಂಧಿತ ಡಾಕ್ಸ್ ತಕ್ಷಣ ಹುಡುಕಲು MCP ಬಳಸಿ.
- ನಿಮ್ಮ ಸಂಪಾದಕದಲ್ಲಿ ಕೇಂದ್ರೀಕರಿಸಿ ಉತ್ಪಾದಕತೆಯನ್ನು ಹೆಚ್ಚಿಸಿ.

> [!IMPORTANT]
> ನಿಮ್ಮ ವರ್ಕ್‌ಸ್ಪೇಸ್‌ನಲ್ಲಿ ಮಾನ್ಯ [`mcp.json`](../../../../../../09-CaseStudy/docs-mcp/solution/scenario3/mcp.json) ಸಂರಚನೆ (ಸ್ಥಳ `.vscode/mcp.json`) ಹೊಂದಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ.

## ದೃಶ್ಯ 2ಕ್ಕೆ ಚೈನ್‌ಲಿಟ್ ಯಾಕೆ?

ಚೈನ್‌ಲಿಟ್ ಸಂವಾದಾತ್ಮಕ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಲು ಆಧುನಿಕ ಓಪನ್-ಸೋರ್ಸ್ ಫ್ರೇಮ್ವರ್ಕ್. ಇದು ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಸ್ MCP ಸರ್ವರ್ ಮುಂತಾದ ಬ್ಯಾಕೆಂಡ್ ಸೇವೆಗಳಿಗೆ ಸಂಪರ್ಕಿಸುವ ಚಾಟ್ ಆಧಾರಿತ ಬಳಕೆದಾರ ಇಂಟರ್ಫೇಸ್‌ಗಳನ್ನು ಸುಲಭವಾಗಿ ರಚಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ಈ ಪ್ರಾಜೆಕ್ಟ್ ವೈಯಕ್ತಿಕ ಅಧ್ಯಯನ ಯೋಜನೆಗಳನ್ನು ರಿಯಲ್ ಟೈಮ್‌ನಲ್ಲಿ ತಯಾರಿಸಲು ಸರಳ, ಸಂವಾದಾತ್ಮಕ ವಿಧಾನವನ್ನು ಒದಗಿಸಲು ಚೈನ್‌ಲಿಟ್ ಅನ್ನು ಬಳಸುತ್ತದೆ. ಚೈನ್‌ಲಿಟ್ ಬಳಸಿ, ನೀವು ತ್ವರಿತವಾಗಿ ಚಾಟ್ ಆಧಾರಿತ ಉಪಕರಣಗಳನ್ನು ನಿರ್ಮಿಸಿ ನಿಯೋಜಿಸಬಹುದು, ಇದು ಉತ್ಪಾದಕತೆ ಮತ್ತು ಕಲಿಕೆಯನ್ನು ಹೆಚ್ಚಿಸುತ್ತದೆ.

## ಇದು ಏನು ಮಾಡುತ್ತದೆ

ಈ ಅಪ್ಲಿಕೇಶನ್ ಬಳಕೆದಾರರಿಗೆ ವಿಷಯ ಮತ್ತು ಅವಧಿಯನ್ನು ನಮೂದಿಸುವ ಮೂಲಕ ವೈಯಕ್ತಿಕ ಅಧ್ಯಯನ ಯೋಜನೆಯನ್ನು ರಚಿಸಲು ಅನುಮತಿಸುತ್ತದೆ. ಅಪ್ಲಿಕೇಶನ್ ನಿಮ್ಮ ಇನ್‌ಪುಟ್ ಅನ್ನು ವಿಶ್ಲೇಷಿಸಿ, ಸಂಬಂಧಿತ ವಿಷಯಕ್ಕಾಗಿ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಸ್ MCP ಸರ್ವರ್‌ಗೆ ಪ್ರಶ್ನೆ ಕೇಳುತ್ತದೆ ಮತ್ತು ಫಲಿತಾಂಶಗಳನ್ನು ವಾರದಿಂದ ವಾರಕ್ಕೆ ವಿನ್ಯಾಸಗೊಳಿಸಿದ ಯೋಜನೆಯಲ್ಲಿ ಸಂಘಟಿಸುತ್ತದೆ. ಪ್ರತಿ ವಾರದ ಶಿಫಾರಸುಗಳನ್ನು ಚಾಟ್‌ನಲ್ಲಿ ಪ್ರದರ್ಶಿಸಲಾಗುತ್ತದೆ, ಇದರಿಂದ ಅನುಸರಿಸಲು ಮತ್ತು ನಿಮ್ಮ ಪ್ರಗತಿಯನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡಲು ಸುಲಭವಾಗುತ್ತದೆ. ಈ ಸಂಯೋಜನೆ ನಿಮಗೆ ಯಾವಾಗಲೂ ಇತ್ತೀಚಿನ, ಅತ್ಯಂತ ಸಂಬಂಧಿತ ಕಲಿಕಾ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ.

## ಮಾದರಿ ಪ್ರಶ್ನೆಗಳು

ಅಪ್ಲಿಕೇಶನ್ ಹೇಗೆ ಪ್ರತಿಕ್ರಿಯಿಸುತ್ತದೆ ಎಂದು ನೋಡಲು ಈ ಪ್ರಶ್ನೆಗಳನ್ನು ಚಾಟ್ ವಿಂಡೋದಲ್ಲಿ ಪ್ರಯತ್ನಿಸಿ:

- `AI-900 ಪ್ರಮಾಣಪತ್ರ, 8 ವಾರಗಳು`
- `ಏಜ್ಯೂರ್ ಫಂಕ್ಷನ್ಸ್ ಕಲಿಯಿರಿ, 4 ವಾರಗಳು`
- `ಏಜ್ಯೂರ್ ಡೆವ್ಓಪ್ಸ್, 6 ವಾರಗಳು`
- `ಏಜ್ಯೂರ್‌ನಲ್ಲಿ ಡೇಟಾ ಎಂಜಿನಿಯರಿಂಗ್, 10 ವಾರಗಳು`
- `ಮೈಕ್ರೋಸಾಫ್ಟ್ ಭದ್ರತಾ ಮೂಲತತ್ವಗಳು, 5 ವಾರಗಳು`
- `ಪವರ್ ಪ್ಲಾಟ್‌ಫಾರ್ಮ್, 7 ವಾರಗಳು`
- `ಏಜ್ಯೂರ್ AI ಸೇವೆಗಳು, 12 ವಾರಗಳು`
- `ಕ್ಲೌಡ್ ವಾಸ್ತುಶಿಲ್ಪ, 9 ವಾರಗಳು`

ಈ ಉದಾಹರಣೆಗಳು ವಿಭಿನ್ನ ಕಲಿಕಾ ಗುರಿಗಳು ಮತ್ತು ಅವಧಿಗಳಿಗಾಗಿ ಅಪ್ಲಿಕೇಶನ್‌ನ ಲವಚಿಕತೆಯನ್ನು ತೋರಿಸುತ್ತವೆ.
