# ಪ್ರಕರಣ ಅಧ್ಯಯನ: ಕ್ಲೈಂಟ್‌ನಿಂದ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಸ್ MCP ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕಿಸುವುದು

ನೀವು ಎಂದಾದರೂ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಸೈಟ್‌ಗಳು, ಸ್ಟ್ಯಾಕ್ ಓವರ್ಫ್ಲೋ ಮತ್ತು ಅಸಂಖ್ಯಾತ ಸರ್ಚ್ ಎಂಜಿನ್ ಟ್ಯಾಬ್‌ಗಳ ನಡುವೆ ಜಗ್ಗಲಿಂಗ್ ಮಾಡುತ್ತಿರುವಂತೆ ಅನುಭವಿಸಿದ್ದು ಇದೆಯೇ, ನಿಮ್ಮ ಕೋಡ್‌ನಲ್ಲಿ ಸಮಸ್ಯೆಯನ್ನು ಬಗೆಹರಿಸುವ ಪ್ರಯತ್ನ ಮಾಡುವಾಗ? ನೀವು ಡಾಕ್ಸ್‌ಗಳಿಗಾಗಿ ಎರಡನೇ ಮಾನಿಟರ್ ಅನ್ನು ಇಡುತ್ತೀರಾ, ಅಥವಾ ನಿಮ್ಮ ಐಡಿಇ ಮತ್ತು ಬ್ರೌಸರ್ ನಡುವೆ ನಿರಂತರವಾಗಿ ಆಲ್ಟ್-ಟ್ಯಾಬ್ ಮಾಡುತ್ತೀರಾ? ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಅನ್ನು ನೇರವಾಗಿ ನಿಮ್ಮ ವರ್ಕ್‌ಫ್ಲೋಗೆ—ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು, ನಿಮ್ಮ ಐಡಿಇ, ಅಥವಾ ನಿಮ್ಮ ಸ್ವಂತ ಕಸ್ಟಮ್ ಸಾಧನಗಳಲ್ಲಿ ಅಂತರ್ಗತ ಮಾಡಿಕೊಳ್ಳುವುದು ಉತ್ತಮವಾಗುವುದಾಗಿರಲ್ವಾ? ಈ ಪ್ರಕರಣ ಅಧ್ಯಯನದಲ್ಲಿ, ನಾವು ನಿಮ್ಮ ಸ್ವಂತ ಕ್ಲೈಂಟ್ ಅಪ್ಲಿಕೇಶನ್‌ನಿಂದ ನೇರವಾಗಿ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಸ್ MCP ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕಿಸುವ ಮೂಲಕ ಅದನ್ನು ಹೇಗೆ ಮಾಡುವುದು ಎಂದು ಅನ್ವೇಷಿಸುವೆವು.

## ಅವಲೋಕನ

आಧುನಿಕ ಅಭಿವೃದ್ಧಿ ಕೇವಲ ಕೋಡ್ ಬರೆಯುವುದೇ ಅಲ್ಲ—ಸರಿಯಾದ ಸಮಯದಲ್ಲಿ ಸರಿಯಾದ ಮಾಹಿತಿಯನ್ನು ಕಂಡುಹಿಡಿಯುವುದಾಗಿಯೂ ಆಗಿದೆ. ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಎಲ್ಲೆಡೆ ಇದೆ, ಆದರೆ ಅದನ್ನು ನೀವು ಅತ್ಯಂತ ಅಗತ್ಯವಿರುವ ಜಾಗಗಳಲ್ಲಿ ಕಳೆದುಕೊಳ್ಳುವುದು ಅಪರೂಪ—ಅದು ನಿಮ್ಮ ಸಾಧನಗಳು ಮತ್ತು ವರ್ಕ್‌ಫ್ಲೋಗಳಲ್ಲಿ ಇರುವಾಗ. ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಪಡೆಯುವಿಕೆಯನ್ನು ನೇರವಾಗಿ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಲ್ಲಿ ಒಳಗೊಳ್ಳಿಸುವ ಮೂಲಕ, ನೀವು ಸಮಯ ಉಳಿತಾಯ ಮಾಡಬಹುದು, ಸಂಧರ್ಭ ಬದಲಾವಣೆಯನ್ನು ಕಡಿಮೆ ಮಾಡಬಹುದು ಮತ್ತು ಉತ್ಪಾದಕತೆಯನ್ನು ಹೆಚ್ಚಿಸಬಹುದು. ಈ ಭಾಗದಲ್ಲಿ, ನಾವು ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಬಿಟ್ಟು ಹೋಗದೆ ನೇರವಾಗಿ, ಸanduರ್ಭ ಪಟ್ಟಿ ಡಾಕ್ಯುಮೆಂಟೇಶನ್‌ಗೆ ಪ್ರವೇಶ ಪಡೆಯಲು ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಸ್ MCP ಸರ್ವರ್‌ಗೆ ಕ್ಲೈಂಟ್ ಅನ್ನು ಸಂಪರ್ಕಿಸುವುದು ಹೇಗೆ ಎಂದು ತೋರಿಸುವೆವು.

ನಾವು ಸಂಪರ್ಕ ಸ್ಥಾಪಿಸುವ ಪ್ರಕ್ರಿಯೆ, ವಿನಂತಿಯನ್ನು ಕಳುಹಿಸುವುದು ಮತ್ತು ಸ್ಟ್ರೀಮಿಂಗ್ ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ಹ್ಯಾಡಲ್ ಮಾಡುವುದನ್ನು ನಡೆಸಿ ತೋರಿಸುತ್ತೇವೆ. ಈ ವಿಧಾನವು ನಿಮ್ಮ ವರ್ಕ್‌ಫ್ಲೋವನ್ನು ಸರಳಗೊಳಿಸುವುದಲ್ಲದೆ, ಹೆಚ್ಚು ಬುದ್ಧಿವಂತ ಮತ್ತು ಸಹಾಯಕ ಅಭಿವೃದ್ಧಿಗಾರ ಸಾಧನಗಳನ್ನು ನಿರ್ಮಿಸುವ ಎಡಗಿ ತೆರೆಯುತ್ತದೆ.

## ಕಲಿಕಾ ಉದ್ದೇಶಗಳು

ನಾವು ಇದನ್ನು ಏಕೆ ಮಾಡುತಿದ್ದೇವೆ? ಅತ್ಯುತ್ತಮ ಡೆವೆಲಪರ್ ಅನುಭವಗಳು ಅವುಗಳ ನಡುವಿನ ಅಡಚಣೆಯನ್ನು ತೆಗೆದುಹಾಕುತ್ತವೆ. ನಿಮ್ಮ ಕೋಡ್ ಸಂಪಾದಕ, ಚಾಟ್‌ಬಾಟ್ ಅಥವಾ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್‌ನ ನವೀನ ವಿಷಯಗಳನ್ನು ಬಳಸಿಕೊಂಡು ನಿಮ್ಮ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಪ್ರಶ್ನೆಗಳಿಗೆ ಕ್ಷಣಿಕವಾಗಿ ಉತ್ತರಿಸಬಹುದು ಎಂದು ಕಲ್ಪಿಸಿಕೊಳ್ಳಿ. ಈ ಅಧ್ಯಾಯದ ಕೊನೆಯಲ್ಲಿ, ನೀವು ಹೇಗೆ:

- ಡಾಕ್ಯುಮೆಂಟೇಶನ್‌ಗೆ MCP ಸರ್ವರ್-ಕ್ಲೈಂಟ್ ಸಂವಹನದ ಮೂಲಭೂತಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು
- ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಸ್ MCP ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕಿಸಲು ಕಾನ್ಸೋಲ್ ಅಥವಾ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಜಾರಿಗೆ ತರುವುದು
- ನೇರ-ಕಾಲದ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಪಡೆಯಲು ಸ್ಟ್ರೀಮಿಂಗ್ HTTP ಕ್ಲೈಂಟ್‌ಗಳನ್ನು ಬಳಸುವುದು
- ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್‌ನಲ್ಲಿ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಲಾಗ್ ಮತ್ತು ವಿವೇಚಿಸುವುದು

ಇವು ನಿಮ್ಮ ಅಂಗಾಳಿನಲ್ಲಿ ಪ್ರತಿಕ್ರಿಯಾತ್ಮಕವಾಗಿರುವುದಲ್ಲದೆ ನಿಜವಾದ ಸಂಧರ್ಭ ಪರಿಚಯ ಹೊಂದಿರುವ ಮತ್ತು ವ್ಯವಹಾರಾತ್ಮಕ ಸಾಧನಗಳನ್ನು ನಿರ್ಮಿಸುವಲ್ಲಿ ಸಹಾಯ ಮಾಡುತ್ತದೆ.

## ಪರಿಸ್ಥಿತಿ 1 - MCP ಜೊತೆಗೆ ನೇರ-ಕಾಲದ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಪಡೆದುಕೊಳ್ಳುವುದು

ಈ ಸಂದರ್ಭದಲ್ಲ, ನಾವು ನಿಮಗೆ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಸ್ MCP ಸರ್ವರ್‌ಗೆ ಕ್ಲೈಂಟ್ ಸಂಪರ್ಕಿಸುವುದನ್ನು ತೋರಿಸುತ್ತೇವೆ, ಇದರಿಂದ ನೀವು ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಬಿಟ್ಟು ಹೋಗದೆ ನೇರ-ಕಾಲದ, ಸಂಧರ್ಭ ಪಟ್ಟಿ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಪಡೆಯಬಹುದು.

ಇದನ್ನು ಕಾರ್ಯಗತಗೊಳಿಸೋಣ. ನಿಮ್ಮ ಕಾರ್ಯವೆಂದರೆ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಸ್ MCP ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕಿಸುವ, `microsoft_docs_search` ಉಪಕರಣವನ್ನು ಕರೆಯುವ ಮತ್ತು ಸಂಚರಿಸುವ ಪ್ರತಿಕ್ರಿಯೆಯನ್ನು ಕಾನ್ಸೋಲ್‌ಗೆ ಲಾಗ್ ಮಾಡುವ ಅಪ್ಲಿಕೇಶನ್ ಬರೆಯುವುದು.

import asyncio
from mcp.client.streamable_http import streamablehttp_client
from mcp import ClientSession

async def main():
    async with streamablehttp_client("https://learn.microsoft.com/api/mcp") as (read_stream, write_stream, _):
        async with ClientSession(read_stream, write_stream) as session:
            await session.initialize()
            result = await session.call_tool("microsoft_docs_search", {"query": "Azure Functions best practices"})
            print(result.content)

if __name__ == "__main__":
    asyncio.run(main())

## ಪರಿಸ್ಥಿತಿ 2 - MCP ಜೊತೆಗೆ ಸಂವಹನಾತ್ಮಕ ಅಧ್ಯಯನ ಯೋಜನೆ ಜನರೇಟರ್ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್

ಚೈನ್‌ಲಿಟ್ ಸಂಭಾಷಣಾತ್ಮಕ AI ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳ ನಿರ್ಮಾಣಕ್ಕೆ ಫ್ರೇಮ್‌ವರ್ಕ್. ಇದು ಸಂವಹನಾತ್ಮಕ ಚಾಟ್‌ಬಾಟ್‌ಗಳು ಮತ್ತು ಸಹಾಯಕಗಳನ್ನು ಎಂಬುವ ನೀವು ಸರಳವಾಗಿ ನಿರ್ಮಿಸಬಹುದು, ಅವು MCP ಉಪಕರಣಗಳನ್ನು ಕರೆಮಾಡಿ ನೇರ-ಕಾಲದಲ್ಲಿ ಫಲಿತಾಂಶಗಳನ್ನು ಪ್ರದರ್ಶಿಸಬಹುದು. ಇದು ವೆಗವಾಗಿ ಪ್ರೋಟೋಟೈಪಿಂಗ್ ಮತ್ತು ಬಳಕೆದಾರ ಸ್ನೇಹಿ ಇಂಟರ್ಫೇಸ್‌ಗಳಿಗೆ ಸೂಕ್ತ.

```python
import chainlit as cl
import requests

MCP_URL = "https://learn.microsoft.com/api/mcp"

@cl.on_message
def handle_message(message):
    query = {"question": message}
    response = requests.post(MCP_URL, json=query)
    if response.ok:
        result = response.json()
        cl.Message(content=result.get("answer", "No answer found.")).send()
    else:
        cl.Message(content="Error: " + response.text).send()
```

## ಪರಿಸ್ಥಿತಿ 3: VS ಕೋಡ್‌ನಲ್ಲಿನ MCP ಸರ್ವರ್ ಮೂಲಕ ಎಡಿಟರ್ ಒಳಗೆ ಡಾಕ್ಸ್

ನೀವು ಬ್ರೌಸರ್ ಟ್ಯಾಬ್‌ಗಳನ್ನು ಬದಲಾಯಿಸದೆ ನಿಮ್ಮ VS ಕೋಡ್ ಆಧಾರಿತ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಸ್ ಪಡೆಯಲು ಬಯಸಿದರೆ, MCP ಸರ್ವರ್ ಅನ್ನು ನಿಮ್ಮ ಎಡಿಟರ್‌ನಲ್ಲಿ ಬಳಸಬಹುದು. ಇದರಿಂದ ನೀವು:
- VS ಕೋಡ್‌ನಲ್ಲಿ ನೀವು ಕೋಡ್ ಮಾಡುವ ಪರಿಸರ ಬಿಟ್ಟು ಹೋಗದೇ ಡಾಕ್ಸ್ ಹುಡುಕಿ ಓದಲು ಸಾಧ್ಯ.
- ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಉಲ್ಲೇಖಿಸಿ, README ಅಥವಾ ಕೋರ್ಸ್ ಫೈಲ್‌ಗಳಿಗೆ ಲಿಂಕ್ ನೇರವಾಗಿ ಸೇರಿಸಬಹುದು.
- ಗಿಥಬ್ ಕೊಪೈಲಟ್ ಮತ್ತು MCP ಅನ್ನು ಸೇರಿಸಿಕೊಂಡು, ಸಮಗ್ರ AI ಚಾಲಿತ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ವರ್ಕ್‌ಫ್ಲೋ ರಚಿಸಬಹುದು.

**ನೀವು ತಿಳಿದುಕೊಳ್ಳುವ ವಿಷಯಗಳು:**
- ನಿಮ್ಮ ವರ್ಕ್‌ಸ್ಪೇಸ್ ಮೂಲದಲ್ಲಿಗೆ ಮಾನ್ಯವಾದ `.vscode/mcp.json` ಫೈಲ್ ಸೇರಿಸುವುದು (ಕೆಳಗಿನ ಉದಾಹರಣೆ ನೋಡಿ).
- MCP ಪ್ಯಾನೆಲ್ ತೆರೆಯುವುದು ಅಥವಾ VS ಕೋಡ್‌ನ ಕಮಾಂಡ್ ಪ್ಯಾಲೇಟ್ ಬಳಸಿಕೊಂಡು ಡಾಕ್ಸ್ ಹುಡುಕುವುದು ಮತ್ತು ಸೇರಿಸುವುದು.
- ಕೆಲಸ ಮಾಡುವಾಗ ನಿಮ್ಮ ಮಾರ್ಕ್‌ಡೌನ್ ಫೈಲ್‌ಗಳಲ್ಲಿ ನೇರವಾಗಿ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಉಲ್ಲೇಖಿಸುವುದು.
- ಈ ವರ್ಕ್‌ಫ್ಲೋವನ್ನು ಗಿಥಬ್ ಕೊಪೈಲಟ್ ಜೊತೆ ಸಂಯೋಜಿಸಲು ಮತ್ತಷ್ಟು ಉತ್ಪಾದಕತೆಗಾಗಿ.

VS ಕೋಡ್‌ನಲ್ಲಿ MCP ಸರ್ವರ್ ಸೆಟ್‌ಅಪ್ ಮಾಡುವುದರ ಉದಾಹರಣೆ ಇಲ್ಲಿದೆ:

## ಮುಖ್ಯ ಅಂಶಗಳು

ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಅನ್ನು ನೇರವಾಗಿ ನಿಮ್ಮ ಸಾಧನಗಳಲ್ಲಿ ಸಂಯೋಜಿಸುವುದು ಕೇವಲ ಸೌಲಭ್ಯವಲ್ಲ—ಇದು ಉತ್ಪಾದಕತೆಯ ಪರಿಪೂರ್ಣತೆಗಾಗಿ ಕ್ರಾಂತಿಯಾಗಿದೆ. ನಿಮ್ಮ ಕ್ಲೈಂಟ್‌ನಿಂದ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಲರ್ನ್ ಡಾಕ್ಸ್ MCP ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕಿಸುವ ಮೂಲಕ, ನೀವು:

- ನಿಮ್ಮ ಕೋಡ್ ಮತ್ತು ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ನಡುವಿನ ಸಂಧರ್ಭ ಬದಲಾವಣೆಗಳನ್ನು ನಿರ್ಮೂಲಗೊಳಿಸಬಹುದು
- ನೇರ-ಕಾಲದ, ಸಂಧರ್ಭ ಪಟ್ಟಿ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಪಡೆಯಬಹುದು
- ಹೆಚ್ಚು ಬುದ್ಧಿವಂತ, ಸಂವಹನಾತ್ಮಕ ಅಭಿವೃದ್ಧಿಗಾರ ಸಾಧನಗಳನ್ನು ನಿರ್ಮಿಸಬಹುದು

ಈ ಕೌಶಲ್ಯಗಳು ನಿಮಗೆ ಪರಿಣಾಮಕಾರಿ ಮತ್ತು ಬಳಕೆಗೆ ಆನಂದಕಾರಿ ಪರಿಹಾರಗಳನ್ನು ನಿರ್ಮಿಸುವಲ್ಲಿ ಸಹಾಯಮಾಡುತ್ತವೆ.
