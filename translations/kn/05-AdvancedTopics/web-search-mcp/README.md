# ಪಾಠ: ವೆಬ್ ಹುಡುಕಾಟ MCP ಸರ್ವರ್ ನಿರ್ಮಾಣ

ಈ ಅಧ್ಯಾಯವು ಹೊರಗಿನ API ಗಳೊಂದಿಗೆ ಸಂಯೋಜಿಸುವ, ವಿಭಿನ್ನ ಡೇಟಾ ಪ್ರಕಾರಗಳನ್ನು ನಿರ್ವಹಿಸುವ, ದೋಷಗಳನ್ನು ನಿರ್ವಹಿಸುವ ಮತ್ತು ಹಲವಾರು ಸಾಧನಗಳನ್ನು ಒಟ್ಟಿಗೆ ಸಂಚಾಲಿಸುವ ನೈಜ ಜಗತ್ತಿನ AI ಏಜೆಂಟ್ ಅನ್ನು ನಿರ್ಮಿಸುವ ವಿಧಾನವನ್ನು ತೋರಿಸುತ್ತದೆ—ಎಲ್ಲವೂ ಉತ್ಪಾದನೆಗೆ ಸಿದ್ಧವಾದ ಸ್ವರೂಪದಲ್ಲಿ. ನೀವು ನೋಡಲಿರುವುದು:

- **ಪ್ರಮಾಣೀಕರಣ ಅಗತ್ಯವಿರುವ ಹೊರಗಿನ API ಗಳೊಂದಿಗೆ ಸಂಯೋಜನೆ**
- **ಬಹು ಎಂಡ್ಪಾಯಿಂಟ್ ಗಳಿಂದ ವಿಭಿನ್ನ ಡೇಟಾ ಪ್ರಕಾರಗಳನ್ನು ನಿರ್ವಹಣೆ**
- **ದೃಢ ದೋಷ ನಿರ್ವಹಣೆ ಮತ್ತು ಲಾಗಿಂಗ್ ತಂತ್ರಗಳು**
- **ಒಂದು ಸರ್ವರ್‌ನಲ್ಲಿ ಬಹು ಸಾಧನಗಳ ಸಂಚಾಲನೆ**

ಅಂತ್ಯದಲ್ಲಿ, ನೀವು ಉನ್ನತ AI ಮತ್ತು LLM ಚಾಲಿತ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಅಗತ್ಯವಿರುವ ಮಾದರಿಗಳು ಮತ್ತು ಉತ್ತಮ ಅಭ್ಯಾಸಗಳೊಂದಿಗೆ ಪ್ರಾಯೋಗಿಕ ಅನುಭವವನ್ನು ಹೊಂದಿರುತ್ತೀರಿ.

## ಪರಿಚಯ

ಈ ಪಾಠದಲ್ಲಿ, ನೀವು SerpAPI ಬಳಸಿ ನೈಜ ಸಮಯದ ವೆಬ್ ಡೇಟಾ ಮೂಲಕ LLM ಸಾಮರ್ಥ್ಯಗಳನ್ನು ವಿಸ್ತರಿಸುವ ಉನ್ನತ MCP ಸರ್ವರ್ ಮತ್ತು ಕ್ಲೈಂಟ್ ಅನ್ನು ನಿರ್ಮಿಸುವುದನ್ನು ಕಲಿಯುತ್ತೀರಿ. ಇದು ವೆಬ್‌ನಿಂದ ನವೀಕೃತ ಮಾಹಿತಿಯನ್ನು ಪ್ರವೇಶಿಸಬಹುದಾದ ಚಲನೆಯ AI ಏಜೆಂಟ್‌ಗಳನ್ನು ಅಭಿವೃದ್ಧಿಪಡಿಸಲು ಅತ್ಯಂತ ಮುಖ್ಯ ಕೌಶಲ್ಯವಾಗಿದೆ.

## ಕಲಿಕೆಯ ಗುರಿಗಳು

ಈ ಪಾಠದ ಅಂತ್ಯಕ್ಕೆ, ನೀವು ಈ ಕೆಳಗಿನವುಗಳನ್ನು ಮಾಡಲು ಸಾಧ್ಯವಾಗುತ್ತದೆ:

- ಹೊರಗಿನ API ಗಳನ್ನು (SerpAPI ಮುಂತಾದ) ಸುರಕ್ಷಿತವಾಗಿ MCP ಸರ್ವರ್‌ಗೆ ಸಂಯೋಜಿಸುವುದು
- ವೆಬ್, ಸುದ್ದಿ, ಉತ್ಪನ್ನ ಹುಡುಕಾಟ ಮತ್ತು ಪ್ರಶ್ನೋತ್ತರಕ್ಕಾಗಿ ಹಲವಾರು ಸಾಧನಗಳನ್ನು ಜಾರಿಗೆ ತರುವುದು
- LLM ಬಳಕೆಗೆ ರಚನಾತ್ಮಕ ಡೇಟಾವನ್ನು ಪಾರ್ಸ್ ಮತ್ತು ಫಾರ್ಮ್ಯಾಟ್ ಮಾಡುವುದು
- ದೋಷಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು ಮತ್ತು API ದರ ಮಿತಿಗಳನ್ನು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ನಿರ್ವಹಿಸುವುದು
- ಸ್ವಯಂಚಾಲಿತ ಮತ್ತು ಸಂವಹನಾತ್ಮಕ MCP ಕ್ಲೈಂಟ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಿ ಪರೀಕ್ಷಿಸುವುದು

## ವೆಬ್ ಹುಡುಕಾಟ MCP ಸರ್ವರ್

ಈ ವಿಭಾಗವು ವೆಬ್ ಹುಡುಕಾಟ MCP ಸರ್ವರ್‌ನ ವಾಸ್ತುಶಿಲ್ಪ ಮತ್ತು ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಪರಿಚಯಿಸುತ್ತದೆ. ನೀವು FastMCP ಮತ್ತು SerpAPI ಅನ್ನು ಒಟ್ಟಿಗೆ ಬಳಸಿಕೊಂಡು LLM ಸಾಮರ್ಥ್ಯಗಳನ್ನು ನೈಜ ಸಮಯದ ವೆಬ್ ಡೇಟಾ ಮೂಲಕ ವಿಸ್ತರಿಸುವುದನ್ನು ನೋಡುತ್ತೀರಿ.

### ಅವಲೋಕನ

ಈ ಜಾರಿಗೆ ನಾಲ್ಕು ಸಾಧನಗಳಿವೆ, ಅವು MCP ಯು ವಿಭಿನ್ನ, ಹೊರಗಿನ API ಚಾಲಿತ ಕಾರ್ಯಗಳನ್ನು ಸುರಕ್ಷಿತವಾಗಿ ಮತ್ತು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ನಿರ್ವಹಿಸುವ ಸಾಮರ್ಥ್ಯವನ್ನು ಪ್ರದರ್ಶಿಸುತ್ತವೆ:

- **general_search**: ವ್ಯಾಪಕ ವೆಬ್ ಫಲಿತಾಂಶಗಳಿಗಾಗಿ
- **news_search**: ಇತ್ತೀಚಿನ ಶೀರ್ಷಿಕೆಗಳಿಗಾಗಿ
- **product_search**: ಇ-ಕಾಮರ್ಸ್ ಡೇಟಾಗಾಗಿ
- **qna**: ಪ್ರಶ್ನೆ-ಉತ್ತರ ತುಣುಕುಗಳಿಗಾಗಿ

### ವೈಶಿಷ್ಟ್ಯಗಳು
- **ಕೋಡ್ ಉದಾಹರಣೆಗಳು**: ಪೈಥಾನ್ (ಮತ್ತು ಇತರ ಭಾಷೆಗಳಿಗೆ ಸುಲಭವಾಗಿ ವಿಸ್ತರಿಸಬಹುದಾದ) ಭಾಷಾ-ನಿರ್ದಿಷ್ಟ ಕೋಡ್ ಬ್ಲಾಕ್‌ಗಳನ್ನು ಒಳಗೊಂಡಿದೆ, ಸ್ಪಷ್ಟತೆಯಿಗಾಗಿ ಕೋಡ್ ಪಿವಟ್ಗಳನ್ನು ಬಳಸಿಕೊಂಡು

### ಪೈಥಾನ್

```python
# general_search ಉಪಕರಣದ ಉದಾಹರಣೆಯ ಬಳಕೆ
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "open source LLMs"})
            print(result)
```

---

ಕ್ಲೈಂಟ್ ಅನ್ನು ಚಾಲನೆ ಮಾಡುವ ಮೊದಲು, ಸರ್ವರ್ ಏನು ಮಾಡುತ್ತದೆ ಎಂಬುದನ್ನು ತಿಳಿದುಕೊಳ್ಳುವುದು ಸಹಾಯಕ. [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ಫೈಲ್ MCP ಸರ್ವರ್ ಅನ್ನು ಜಾರಿಗೆ ತರುತ್ತದೆ, SerpAPI ಜೊತೆಗೆ ಸಂಯೋಜಿಸಿ ವೆಬ್, ಸುದ್ದಿ, ಉತ್ಪನ್ನ ಹುಡುಕಾಟ ಮತ್ತು ಪ್ರಶ್ನೋತ್ತರ ಸಾಧನಗಳನ್ನು ಬಹಿರಂಗಪಡಿಸುತ್ತದೆ. ಇದು ಬರುವ ವಿನಂತಿಗಳನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ, API ಕರೆಗಳನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ, ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಪಾರ್ಸ್ ಮಾಡುತ್ತದೆ ಮತ್ತು ರಚನಾತ್ಮಕ ಫಲಿತಾಂಶಗಳನ್ನು ಕ್ಲೈಂಟ್‌ಗೆ ಹಿಂತಿರುಗಿಸುತ್ತದೆ.

ನೀವು ಸಂಪೂರ್ಣ ಜಾರಿಗೆ [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ನಲ್ಲಿ ಪರಿಶೀಲಿಸಬಹುದು.

ಇಲ್ಲಿ ಸರ್ವರ್ ಸಾಧನವನ್ನು ಹೇಗೆ ವ್ಯಾಖ್ಯಾನಿಸಿ ನೋಂದಾಯಿಸುತ್ತದೆ ಎಂಬ ಸಂಕ್ಷಿಪ್ತ ಉದಾಹರಣೆ ಇದೆ:

### ಪೈಥಾನ್ ಸರ್ವರ್

```python
# server.py (ಉದ್ಘಾಟನೆ)
from mcp.server import MCPServer, Tool

async def general_search(query: str):
    # ...ಕಾರ್ಯಾನ್ವಯಿಕೆ...

server = MCPServer()
server.add_tool(Tool("general_search", general_search))

if __name__ == "__main__":
    server.run()
```

---

- **ಹೊರಗಿನ API ಸಂಯೋಜನೆ**: API ಕೀಗಳು ಮತ್ತು ಹೊರಗಿನ ವಿನಂತಿಗಳನ್ನು ಸುರಕ್ಷಿತವಾಗಿ ನಿರ್ವಹಿಸುವುದನ್ನು ತೋರಿಸುತ್ತದೆ
- **ರಚನಾತ್ಮಕ ಡೇಟಾ ಪಾರ್ಸಿಂಗ್**: API ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು LLM ಗೆ ಅನುಕೂಲಕರ ಸ್ವರೂಪಗಳಿಗೆ ಪರಿವರ್ತಿಸುವುದನ್ನು ತೋರಿಸುತ್ತದೆ
- **ದೋಷ ನಿರ್ವಹಣೆ**: ಸೂಕ್ತ ಲಾಗಿಂಗ್ ಜೊತೆಗೆ ದೃಢ ದೋಷ ನಿರ್ವಹಣೆ
- **ಸಂವಹನಾತ್ಮಕ ಕ್ಲೈಂಟ್**: ಸ್ವಯಂಚಾಲಿತ ಪರೀಕ್ಷೆಗಳು ಮತ್ತು ಪರೀಕ್ಷೆಗಾಗಿ ಸಂವಹನಾತ್ಮಕ ಮೋಡ್ ಎರಡನ್ನೂ ಒಳಗೊಂಡಿದೆ
- **ಸಂದರ್ಭ ನಿರ್ವಹಣೆ**: MCP Context ಅನ್ನು ಲಾಗಿಂಗ್ ಮತ್ತು ವಿನಂತಿಗಳನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡಲು ಬಳಸುತ್ತದೆ

## ಪೂರ್ವಾಪೇಕ್ಷಿತಗಳು

ನೀವು ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು, ನಿಮ್ಮ ಪರಿಸರವನ್ನು ಸರಿಯಾಗಿ ಹೊಂದಿಸಲು ಈ ಹಂತಗಳನ್ನು ಅನುಸರಿಸಿ. ಇದರಿಂದ ಎಲ್ಲಾ ಅವಲಂಬನೆಗಳು ಸ್ಥಾಪಿತವಾಗುತ್ತವೆ ಮತ್ತು ನಿಮ್ಮ API ಕೀಗಳು ಸರಿಯಾಗಿ ಸಂರಚಿತವಾಗುತ್ತವೆ, ಸುಗಮ ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಪರೀಕ್ಷೆಗೆ.

- Python 3.8 ಅಥವಾ ಹೆಚ್ಚಿನ ಆವೃತ್ತಿ
- SerpAPI API ಕೀ (ಸೈನ್ ಅಪ್ ಮಾಡಲು [SerpAPI](https://serpapi.com/) - ಉಚಿತ ಮಟ್ಟ ಲಭ್ಯವಿದೆ)

## ಸ್ಥಾಪನೆ

ಪ್ರಾರಂಭಿಸಲು, ನಿಮ್ಮ ಪರಿಸರವನ್ನು ಹೊಂದಿಸಲು ಈ ಹಂತಗಳನ್ನು ಅನುಸರಿಸಿ:

1. uv (ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ) ಅಥವಾ pip ಬಳಸಿ ಅವಲಂಬನೆಗಳನ್ನು ಸ್ಥಾಪಿಸಿ:

```bash
# uv ಬಳಕೆ (ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ)
uv pip install -r requirements.txt

# ಪಿಪ್ ಬಳಕೆ
pip install -r requirements.txt
```

2. ಪ್ರಾಜೆಕ್ಟ್ ರೂಟ್‌ನಲ್ಲಿ `.env` ಫೈಲ್ ರಚಿಸಿ ಮತ್ತು ನಿಮ್ಮ SerpAPI ಕೀ ಸೇರಿಸಿ:

```
SERPAPI_KEY=your_serpapi_key_here
```

## ಬಳಕೆ

ವೆಬ್ ಹುಡುಕಾಟ MCP ಸರ್ವರ್ ಮುಖ್ಯ ಘಟಕವಾಗಿದೆ, ಇದು SerpAPI ಜೊತೆಗೆ ಸಂಯೋಜಿಸಿ ವೆಬ್, ಸುದ್ದಿ, ಉತ್ಪನ್ನ ಹುಡುಕಾಟ ಮತ್ತು ಪ್ರಶ್ನೋತ್ತರ ಸಾಧನಗಳನ್ನು ಬಹಿರಂಗಪಡಿಸುತ್ತದೆ. ಇದು ಬರುವ ವಿನಂತಿಗಳನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ, API ಕರೆಗಳನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ, ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಪಾರ್ಸ್ ಮಾಡುತ್ತದೆ ಮತ್ತು ರಚನಾತ್ಮಕ ಫಲಿತಾಂಶಗಳನ್ನು ಕ್ಲೈಂಟ್‌ಗೆ ಹಿಂತಿರುಗಿಸುತ್ತದೆ.

ನೀವು ಸಂಪೂರ್ಣ ಜಾರಿಗೆ [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) ನಲ್ಲಿ ಪರಿಶೀಲಿಸಬಹುದು.

### ಸರ್ವರ್ ಚಾಲನೆ

MCP ಸರ್ವರ್ ಅನ್ನು ಪ್ರಾರಂಭಿಸಲು, ಕೆಳಗಿನ ಆಜ್ಞೆಯನ್ನು ಬಳಸಿ:

```bash
python server.py
```

ಸರ್ವರ್ stdio ಆಧಾರಿತ MCP ಸರ್ವರ್ ಆಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ, ಇದಕ್ಕೆ ಕ್ಲೈಂಟ್ ನೇರವಾಗಿ ಸಂಪರ್ಕಿಸಬಹುದು.

### ಕ್ಲೈಂಟ್ ಮೋಡ್‌ಗಳು

ಕ್ಲೈಂಟ್ (`client.py`) MCP ಸರ್ವರ್ ಜೊತೆಗೆ ಸಂವಹನ ಮಾಡಲು ಎರಡು ಮೋಡ್‌ಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ:

- **ಸಾಮಾನ್ಯ ಮೋಡ್**: ಎಲ್ಲಾ ಸಾಧನಗಳನ್ನು ಪರೀಕ್ಷಿಸುವ ಸ್ವಯಂಚಾಲಿತ ಪರೀಕ್ಷೆಗಳನ್ನು ನಡೆಸುತ್ತದೆ ಮತ್ತು ಅವುಗಳ ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಪರಿಶೀಲಿಸುತ್ತದೆ. ಇದು ಸರ್ವರ್ ಮತ್ತು ಸಾಧನಗಳು ನಿರೀಕ್ಷೆಯಂತೆ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತಿರುವುದನ್ನು ತ್ವರಿತವಾಗಿ ಪರಿಶೀಲಿಸಲು ಉಪಯುಕ್ತ.
- **ಸಂವಹನಾತ್ಮಕ ಮೋಡ್**: ಮೆನು ಚಾಲಿತ ಇಂಟರ್ಫೇಸ್ ಪ್ರಾರಂಭಿಸುತ್ತದೆ, ಇಲ್ಲಿ ನೀವು ಕೈಯಿಂದ ಸಾಧನಗಳನ್ನು ಆಯ್ಕೆಮಾಡಿ ಕರೆ ಮಾಡಬಹುದು, ಕಸ್ಟಮ್ ಪ್ರಶ್ನೆಗಳನ್ನು ನಮೂದಿಸಬಹುದು ಮತ್ತು ಫಲಿತಾಂಶಗಳನ್ನು ನೈಜ ಸಮಯದಲ್ಲಿ ನೋಡಬಹುದು. ಇದು ಸರ್ವರ್ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಅನ್ವೇಷಿಸಲು ಮತ್ತು ವಿಭಿನ್ನ ಇನ್ಪುಟ್‌ಗಳೊಂದಿಗೆ ಪ್ರಯೋಗ ಮಾಡಲು ಸೂಕ್ತ.

ನೀವು ಸಂಪೂರ್ಣ ಜಾರಿಗೆ [`client.py`](../../../../05-AdvancedTopics/web-search-mcp/client.py) ನಲ್ಲಿ ಪರಿಶೀಲಿಸಬಹುದು.

### ವಿಭಿನ್ನ ವಿಧಾನಗಳೊಂದಿಗೆ ಪರೀಕ್ಷೆ

ನೀವು ಸರ್ವರ್ ಒದಗಿಸುವ ಸಾಧನಗಳನ್ನು ಪರೀಕ್ಷಿಸಲು ಮತ್ತು ಸಂವಹನ ಮಾಡಲು ಹಲವಾರು ವಿಧಾನಗಳಿವೆ, ನಿಮ್ಮ ಅಗತ್ಯಗಳು ಮತ್ತು ಕಾರ್ಯಪ್ರವಾಹದ ಮೇಲೆ ಅವಲಂಬಿತವಾಗಿದೆ.

#### MCP ಪೈಥಾನ್ SDK ಬಳಸಿ ಕಸ್ಟಮ್ ಪರೀಕ್ಷಾ ಸ್ಕ್ರಿಪ್ಟ್ ಬರೆಯುವುದು
ನೀವು MCP ಪೈಥಾನ್ SDK ಬಳಸಿ ನಿಮ್ಮದೇ ಪರೀಕ್ಷಾ ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಬಹುದು:

# [Python](../../../../05-AdvancedTopics/web-search-mcp)

```python
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def test_custom_query():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            # ನಿಮ್ಮ ಕಸ್ಟಮ್ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳೊಂದಿಗೆ ಟೂಲ್ಗಳನ್ನು ಕರೆಮಾಡಿ
            result = await session.call_tool("general_search", 
                                           arguments={"query": "your custom query"})
            # ಫಲಿತಾಂಶವನ್ನು ಪ್ರಕ್ರಿಯೆಗೊಳಿಸಿ
```

---

ಈ ಸನ್ನಿವೇಶದಲ್ಲಿ, "ಪರೀಕ್ಷಾ ಸ್ಕ್ರಿಪ್ಟ್" ಎಂದರೆ ನೀವು MCP ಸರ್ವರ್‌ಗೆ ಕ್ಲೈಂಟ್ ಆಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸಲು ಬರೆಯುವ ಕಸ್ಟಮ್ ಪೈಥಾನ್ ಪ್ರೋಗ್ರಾಂ. ಇದು ಅಧಿಕೃತ ಯೂನಿಟ್ ಟೆಸ್ಟ್ ಅಲ್ಲ, ಬದಲಿಗೆ ಈ ಸ್ಕ್ರಿಪ್ಟ್ ಮೂಲಕ ನೀವು ಪ್ರೋಗ್ರಾಮ್ ಮೂಲಕ ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕಿಸಿ, ಅದರ ಯಾವುದೇ ಸಾಧನವನ್ನು ನಿಮ್ಮ ಆಯ್ಕೆಮಾಡಿದ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳೊಂದಿಗೆ ಕರೆ ಮಾಡಿ, ಫಲಿತಾಂಶಗಳನ್ನು ಪರಿಶೀಲಿಸಬಹುದು. ಈ ವಿಧಾನವು ಉಪಯುಕ್ತವಾಗುತ್ತದೆ:
- ಸಾಧನ ಕರೆಗಳನ್ನು ಪ್ರೋಟೋಟೈಪ್ ಮಾಡುವುದು ಮತ್ತು ಪ್ರಯೋಗಿಸುವುದು
- ಸರ್ವರ್ ವಿಭಿನ್ನ ಇನ್ಪುಟ್‌ಗಳಿಗೆ ಹೇಗೆ ಪ್ರತಿಕ್ರಿಯಿಸುತ್ತದೆ ಎಂದು ಮಾನ್ಯತೆ ನೀಡುವುದು
- ಪುನರಾವರ್ತಿತ ಸಾಧನ ಕರೆಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತಗೊಳಿಸುವುದು
- MCP ಸರ್ವರ್ ಮೇಲೆ ನಿಮ್ಮದೇ ಕಾರ್ಯಪ್ರವಾಹಗಳು ಅಥವಾ ಸಂಯೋಜನೆಗಳನ್ನು ನಿರ್ಮಿಸುವುದು

ನೀವು ಪರೀಕ್ಷಾ ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳನ್ನು ಬಳಸಿ ಹೊಸ ಪ್ರಶ್ನೆಗಳನ್ನು ತ್ವರಿತವಾಗಿ ಪ್ರಯತ್ನಿಸಬಹುದು, ಸಾಧನ ವರ್ತನೆಯನ್ನು ಡಿಬಗ್ ಮಾಡಬಹುದು, ಅಥವಾ ಹೆಚ್ಚು ಉನ್ನತ ಸ್ವಯಂಚಾಲಿತಗೊಳಿಸುವಿಕೆಯ ಪ್ರಾರಂಭ ಬಿಂದುವಾಗಿ ಬಳಸಬಹುದು. ಕೆಳಗಿನ ಉದಾಹರಣೆ MCP ಪೈಥಾನ್ SDK ಬಳಸಿ ಇಂತಹ ಸ್ಕ್ರಿಪ್ಟ್ ರಚಿಸುವ ವಿಧಾನವನ್ನು ತೋರಿಸುತ್ತದೆ:

## ಸಾಧನ ವಿವರಣೆಗಳು

ನೀವು ಸರ್ವರ್ ಒದಗಿಸುವ ಕೆಳಗಿನ ಸಾಧನಗಳನ್ನು ವಿಭಿನ್ನ ರೀತಿಯ ಹುಡುಕಾಟ ಮತ್ತು ಪ್ರಶ್ನೆಗಳಿಗೆ ಬಳಸಬಹುದು. ಪ್ರತಿ ಸಾಧನವು ಅದರ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳು ಮತ್ತು ಉದಾಹರಣೆಯ ಬಳಕೆಯೊಂದಿಗೆ ಕೆಳಗಿನಂತೆ ವಿವರಿಸಲಾಗಿದೆ.

ಈ ವಿಭಾಗವು ಲಭ್ಯವಿರುವ ಪ್ರತಿಯೊಂದು ಸಾಧನ ಮತ್ತು ಅವುಗಳ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳ ವಿವರಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ.

## ಈ ಪಾಠದ ಉನ್ನತ ಸಂಪ್ರದಾಯಗಳು

ನೀವು ನಿರ್ಮಾಣ ಪ್ರಾರಂಭಿಸುವ ಮೊದಲು, ಈ ಅಧ್ಯಾಯದಲ್ಲಿ ಕಾಣಿಸಿಕೊಳ್ಳುವ ಕೆಲವು ಪ್ರಮುಖ ಉನ್ನತ ಸಂಪ್ರದಾಯಗಳನ್ನು ತಿಳಿದುಕೊಳ್ಳಿ. ಇವುಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದರಿಂದ ನೀವು ಸುಲಭವಾಗಿ ಅನುಸರಿಸಬಹುದು, ನೀವು ಹೊಸವರಾಗಿದ್ದರೂ ಸಹ:

- **ಬಹು ಸಾಧನ ಸಂಚಾಲನೆ**: ಇದು ಒಂದೇ MCP ಸರ್ವರ್‌ನಲ್ಲಿ ಹಲವಾರು ವಿಭಿನ್ನ ಸಾಧನಗಳನ್ನು (ವೆಬ್ ಹುಡುಕಾಟ, ಸುದ್ದಿ ಹುಡುಕಾಟ, ಉತ್ಪನ್ನ ಹುಡುಕಾಟ ಮತ್ತು ಪ್ರಶ್ನೋತ್ತರ) ನಡೆಸುವುದನ್ನು ಅರ್ಥಮಾಡುತ್ತದೆ. ಇದು ನಿಮ್ಮ ಸರ್ವರ್‌ಗೆ ಒಂದೇ ಕಾರ್ಯವಲ್ಲದೆ ವಿವಿಧ ಕಾರ್ಯಗಳನ್ನು ನಿರ್ವಹಿಸಲು ಅವಕಾಶ ನೀಡುತ್ತದೆ.
- **API ದರ ಮಿತಿ ನಿರ್ವಹಣೆ**: ಬಹು API ಗಳು (SerpAPI ಮುಂತಾದ) ನೀವು ನಿರ್ದಿಷ್ಟ ಸಮಯದಲ್ಲಿ ಎಷ್ಟು ವಿನಂತಿಗಳನ್ನು ಮಾಡಬಹುದು ಎಂಬ ಮಿತಿಯನ್ನು ಹೊಂದಿವೆ. ಉತ್ತಮ ಕೋಡ್ ಈ ಮಿತಿಗಳನ್ನು ಪರಿಶೀಲಿಸಿ, ಅವುಗಳನ್ನು ಸೌಮ್ಯವಾಗಿ ನಿರ್ವಹಿಸುತ್ತದೆ, ಹೀಗಾಗಿ ನೀವು ಮಿತಿಯನ್ನು ತಲುಪಿದಾಗ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಮುರಿಯುವುದಿಲ್ಲ.
- **ರಚನಾತ್ಮಕ ಡೇಟಾ ಪಾರ್ಸಿಂಗ್**: API ಪ್ರತಿಕ್ರಿಯೆಗಳು ಬಹುಶಃ ಸಂಕೀರ್ಣ ಮತ್ತು ನೆಸ್ಟೆಡ್ ಆಗಿರುತ್ತವೆ. ಈ ಸಂಪ್ರದಾಯವು ಆ ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಸ್ವಚ್ಛ, ಸುಲಭವಾಗಿ ಬಳಸಬಹುದಾದ ಸ್ವರೂಪಗಳಿಗೆ ಪರಿವರ್ತಿಸುವುದರ ಬಗ್ಗೆ.
- **ದೋಷ ಪುನಃಪಡೆಯುವಿಕೆ**: ಕೆಲವೊಮ್ಮೆ ಸಮಸ್ಯೆಗಳು ಸಂಭವಿಸಬಹುದು—ಶೃಂಖಲೆ ವಿಫಲವಾಗಬಹುದು, ಅಥವಾ API ನಿರೀಕ್ಷಿತ ಪ್ರತಿಕ್ರಿಯೆ ನೀಡದಿರಬಹುದು. ದೋಷ ಪುನಃಪಡೆಯುವಿಕೆ ಎಂದರೆ ನಿಮ್ಮ ಕೋಡ್ ಈ ಸಮಸ್ಯೆಗಳನ್ನು ನಿರ್ವಹಿಸಿ, ಅಪ್ಲಿಕೇಶನ್ ಕುಸಿಯದೆ ಉಪಯುಕ್ತ ಪ್ರತಿಕ್ರಿಯೆ ನೀಡುತ್ತದೆ.
- **ಪ್ಯಾರಾಮೀಟರ್ ಮಾನ್ಯತೆ**: ಇದು ನಿಮ್ಮ ಸಾಧನಗಳಿಗೆ ಎಲ್ಲಾ ಇನ್ಪುಟ್‌ಗಳು ಸರಿಯಾಗಿವೆ ಮತ್ತು ಸುರಕ್ಷಿತವಾಗಿವೆ ಎಂದು ಪರಿಶೀಲಿಸುವುದರ ಬಗ್ಗೆ. ಇದರಲ್ಲಿ ಡೀಫಾಲ್ಟ್ ಮೌಲ್ಯಗಳನ್ನು ನಿಗದಿಪಡಿಸುವುದು ಮತ್ತು ಪ್ರಕಾರಗಳನ್ನು ಖಚಿತಪಡಿಸುವುದು ಸೇರಿದೆ, ಇದು ದೋಷಗಳು ಮತ್ತು ಗೊಂದಲಗಳನ್ನು ತಡೆಯಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

## ಸಮಸ್ಯೆ ಪರಿಹಾರ

ವೆಬ್ ಹುಡುಕಾಟ MCP ಸರ್ವರ್ ಜೊತೆಗೆ ಕೆಲಸ ಮಾಡುವಾಗ ನೀವು ಕೆಲವೊಮ್ಮೆ ಸಮಸ್ಯೆಗಳನ್ನು ಎದುರಿಸಬಹುದು—ಹೊರಗಿನ API ಗಳು ಮತ್ತು ಹೊಸ ಸಾಧನಗಳೊಂದಿಗೆ ಅಭಿವೃದ್ಧಿಪಡಿಸುವಾಗ ಇದು ಸಾಮಾನ್ಯ. ಈ ವಿಭಾಗವು ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳಿಗೆ ಪ್ರಾಯೋಗಿಕ ಪರಿಹಾರಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ, ಹೀಗಾಗಿ ನೀವು ತ್ವರಿತವಾಗಿ ಹಾದಿ ತಲುಪಬಹುದು. ನೀವು ದೋಷವನ್ನು ಎದುರಿಸಿದರೆ, ಇಲ್ಲಿ ಪ್ರಾರಂಭಿಸಿ: ಕೆಳಗಿನ ಸಲಹೆಗಳು ಬಹುಮಾನವಾಗಿ ಬಳಕೆದಾರರು ಎದುರಿಸುವ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸುತ್ತವೆ ಮತ್ತು ಹೆಚ್ಚುವರಿ ಸಹಾಯವಿಲ್ಲದೆ ನಿಮ್ಮ ಸಮಸ್ಯೆಯನ್ನು ಪರಿಹರಿಸಬಹುದು.

### ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳು

ಕೆಳಗಿನವು ಬಳಕೆದಾರರು ಸಾಮಾನ್ಯವಾಗಿ ಎದುರಿಸುವ ಕೆಲವು ಸಮಸ್ಯೆಗಳು, ಸ್ಪಷ್ಟ ವಿವರಣೆಗಳು ಮತ್ತು ಅವುಗಳನ್ನು ಪರಿಹರಿಸುವ ಹಂತಗಳೊಂದಿಗೆ:

1. **.env ಫೈಲ್‌ನಲ್ಲಿ SERPAPI_KEY ಇಲ್ಲದಿರುವುದು**
   - ನೀವು `SERPAPI_KEY environment variable not found` ಎಂಬ ದೋಷವನ್ನು ನೋಡಿದರೆ, ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ SerpAPI ಪ್ರವೇಶಿಸಲು ಅಗತ್ಯವಿರುವ API ಕೀ ಅನ್ನು ಕಂಡುಹಿಡಿಯಲಾಗುತ್ತಿಲ್ಲ. ಇದನ್ನು ಸರಿಪಡಿಸಲು, ನಿಮ್ಮ ಪ್ರಾಜೆಕ್ಟ್ ರೂಟ್‌ನಲ್ಲಿ `.
DEBUG ಮೋಡ್ ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಲು, ನಿಮ್ಮ `client.py` ಅಥವಾ `server.py` ಫೈಲ್‌ನ ಮೇಲ್ಭಾಗದಲ್ಲಿ ಲಾಗಿಂಗ್ ಮಟ್ಟವನ್ನು DEBUG ಗೆ ಸೆಟ್ ಮಾಡಿ:
