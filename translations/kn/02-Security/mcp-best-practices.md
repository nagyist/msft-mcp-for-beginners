# MCP ಭದ್ರತಾ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು 2025

ಈ ಸಮಗ್ರ ಮಾರ್ಗದರ್ಶಿ ಇತ್ತೀಚಿನ **MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-06-18** ಮತ್ತು ಪ್ರಸ್ತುತ ಕೈಗಾರಿಕಾ ಮಾನದಂಡಗಳ ಆಧಾರದ ಮೇಲೆ ಮಾದರಿ ಸಾಂದರ್ಭಿಕ ಪ್ರೋಟೋಕಾಲ್ (MCP) ವ್ಯವಸ್ಥೆಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುವುದಕ್ಕಾಗಿ ಅಗತ್ಯವಾದ ಭದ್ರತಾ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳನ್ನು ವಿವರಿಸುತ್ತದೆ. ಈ ಅಭ್ಯಾಸಗಳು ಪರಂಪರাগত ಭದ್ರತಾ ಚಿಂತೆಗಳು ಮತ್ತು MCP ನಿಯೋಜನೆಗಳಿಗೆ ವಿಶೇಷವಾದ AI-ಸಂಬಂಧಿತ ಬೆದರಿಕೆಗಳನ್ನು ಎದುರಿಸುತ್ತವೆ.

## ಪ್ರಮುಖ ಭದ್ರತಾ ಅಗತ್ಯತೆಗಳು

### ಕಡ್ಡಾಯ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳು (MUST ಅಗತ್ಯತೆಗಳು)

1. **ಟೋಕನ್ ಮಾನ್ಯತೆ**: MCP ಸರ್ವರ್‌ಗಳು ಸ್ಪಷ್ಟವಾಗಿ MCP ಸರ್ವರ್‌ಗೆ ಮಾತ್ರ ನೀಡಲಾದ ಯಾವುದೇ ಟೋಕನ್‌ಗಳನ್ನು **ಸ್ವೀಕರಿಸಬಾರದು**
2. **ಅಧಿಕಾರ ಪರಿಶೀಲನೆ**: ಪ್ರಾಧಿಕಾರವನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುವ MCP ಸರ್ವರ್‌ಗಳು ಎಲ್ಲಾ ಒಳಬರುವ ವಿನಂತಿಗಳನ್ನು ಪರಿಶೀಲಿಸಬೇಕು ಮತ್ತು **ಪ್ರಮಾಣೀಕರಣಕ್ಕಾಗಿ ಸೆಷನ್‌ಗಳನ್ನು ಬಳಸಬಾರದು**  
3. **ಬಳಕೆದಾರ ಅನುಮತಿ**: ಸ್ಥಿರ ಕ್ಲೈಂಟ್ ಐಡಿಗಳನ್ನು ಬಳಸುವ MCP ಪ್ರಾಕ್ಸಿ ಸರ್ವರ್‌ಗಳು ಪ್ರತಿ ಡೈನಾಮಿಕ್ ನೋಂದಾಯಿತ ಕ್ಲೈಂಟ್‌ಗೆ ಸ್ಪಷ್ಟ ಬಳಕೆದಾರ ಅನುಮತಿಯನ್ನು ಪಡೆಯಬೇಕು
4. **ಭದ್ರ ಸೆಷನ್ ಐಡಿಗಳು**: MCP ಸರ್ವರ್‌ಗಳು ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್‌ವಾಗಿ ಭದ್ರ, ನಿರ್ಧಾರಾತ್ಮಕವಲ್ಲದ ಸೆಷನ್ ಐಡಿಗಳನ್ನು ಭದ್ರ ರ್ಯಾಂಡಮ್ ಸಂಖ್ಯೆ ಜನರೇಟರ್‌ಗಳೊಂದಿಗೆ ಬಳಸಬೇಕು

## ಮೂಲ ಭದ್ರತಾ ಅಭ್ಯಾಸಗಳು

### 1. ಇನ್‌ಪುಟ್ ಮಾನ್ಯತೆ ಮತ್ತು ಶುದ್ಧೀಕರಣ  
- **ಸಮಗ್ರ ಇನ್‌ಪುಟ್ ಮಾನ್ಯತೆ**: ಇಂಜೆಕ್ಷನ್ ದಾಳಿಗಳು, ಗೊಂದಲದ ಡೆಪ್ಯೂ ಸಮಸ್ಯೆಗಳು ಮತ್ತು ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್ ದುರ್ಬಲತೆಗಳನ್ನು ತಡೆಯಲು ಎಲ್ಲಾ ಇನ್‌ಪುಟ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಿ ಮತ್ತು ಶುದ್ಧೀಕರಿಸಿ  
- **ಪ್ಯಾರಾಮೀಟರ್ ಸ್ಕೀಮಾ ಅನುಷ್ಠಾನ**: ಎಲ್ಲಾ ಟೂಲ್ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳು ಮತ್ತು API ಇನ್‌ಪುಟ್‌ಗಳಿಗೆ ಕಠಿಣ JSON ಸ್ಕೀಮಾ ಮಾನ್ಯತೆ ಅನುಷ್ಠಾನಗೊಳಿಸಿ  
- **ವಿಷಯ ಫಿಲ್ಟರಿಂಗ್**: ಪ್ರಾಂಪ್ಟ್‌ಗಳು ಮತ್ತು ಪ್ರತಿಕ್ರಿಯೆಗಳಲ್ಲಿ ದುಷ್ಟ ವಿಷಯವನ್ನು ಫಿಲ್ಟರ್ ಮಾಡಲು Microsoft Prompt Shields ಮತ್ತು Azure Content Safety ಬಳಸಿ  
- **ಔಟ್‌ಪುಟ್ ಶುದ್ಧೀಕರಣ**: ಬಳಕೆದಾರರಿಗೆ ಅಥವಾ ಡೌನ್‌ಸ್ಟ್ರೀಮ್ ವ್ಯವಸ್ಥೆಗಳಿಗೆ ಪ್ರದರ್ಶಿಸುವ ಮೊದಲು ಎಲ್ಲಾ ಮಾದರಿ ಔಟ್‌ಪುಟ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಿ ಮತ್ತು ಶುದ್ಧೀಕರಿಸಿ  

### 2. ಪ್ರಮಾಣೀಕರಣ ಮತ್ತು ಪ್ರಾಧಿಕಾರ ಶ್ರೇಷ್ಠತೆ  
- **ಬಾಹ್ಯ ಗುರುತಿನ ಒದಗಿಸುವವರು**: ಕಸ್ಟಮ್ ಪ್ರಮಾಣೀಕರಣವನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುವ ಬದಲು ಸ್ಥಾಪಿತ ಗುರುತಿನ ಒದಗಿಸುವವರ (Microsoft Entra ID, OAuth 2.1 ಒದಗಿಸುವವರು) ಬಳಿ ಪ್ರಮಾಣೀಕರಣವನ್ನು ನಿಯೋಜಿಸಿ  
- **ಸೂಕ್ಷ್ಮ ಅನುಮತಿಗಳು**: ಕನಿಷ್ಠ привಿಲೇಜ್ ತತ್ವವನ್ನು ಅನುಸರಿಸಿ ಸೂಕ್ಷ್ಮ, ಟೂಲ್-ನಿರ್ದಿಷ್ಟ ಅನುಮತಿಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ  
- **ಟೋಕನ್ ಜೀವನಚಕ್ರ ನಿರ್ವಹಣೆ**: ಭದ್ರ ತಿರುಗಾಟ ಮತ್ತು ಸರಿಯಾದ ಪ್ರೇಕ್ಷಕ ಮಾನ್ಯತೆಯೊಂದಿಗೆ ಕಡಿಮೆ ಅವಧಿಯ ಪ್ರವೇಶ ಟೋಕನ್‌ಗಳನ್ನು ಬಳಸಿ  
- **ಬಹು-ಘಟಕ ಪ್ರಮಾಣೀಕರಣ**: ಎಲ್ಲಾ ಆಡಳಿತ ಪ್ರವೇಶ ಮತ್ತು ಸಂವೇದನಶೀಲ ಕಾರ್ಯಾಚರಣೆಗಳಿಗೆ MFA ಅಗತ್ಯವಿದೆ  

### 3. ಭದ್ರ ಸಂವಹನ ಪ್ರೋಟೋಕಾಲ್‌ಗಳು  
- **ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಲೇಯರ್ ಭದ್ರತೆ**: ಸರಿಯಾದ ಪ್ರಮಾಣಪತ್ರ ಮಾನ್ಯತೆಯೊಂದಿಗೆ ಎಲ್ಲಾ MCP ಸಂವಹನಗಳಿಗೆ HTTPS/TLS 1.3 ಬಳಸಿ  
- **ಎಂಡ್-ಟು-ಎಂಡ್ ಎನ್‌ಕ್ರಿಪ್ಷನ್**: ಸಂವೇದನಶೀಲ ಡೇಟಾಗಾಗಿ ಸಂಚಾರ ಮತ್ತು ವಿಶ್ರಾಂತಿಯಲ್ಲಿ ಹೆಚ್ಚುವರಿ ಎನ್‌ಕ್ರಿಪ್ಷನ್ ಪದರಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ  
- **ಪ್ರಮಾಣಪತ್ರ ನಿರ್ವಹಣೆ**: ಸ್ವಯಂಚಾಲಿತ ನವೀಕರಣ ಪ್ರಕ್ರಿಯೆಗಳೊಂದಿಗೆ ಸರಿಯಾದ ಪ್ರಮಾಣಪತ್ರ ಜೀವನಚಕ್ರ ನಿರ್ವಹಣೆಯನ್ನು ಕಾಪಾಡಿ  
- **ಪ್ರೋಟೋಕಾಲ್ ಆವೃತ್ತಿ ಅನುಷ್ಠಾನ**: ಪ್ರಸ್ತುತ MCP ಪ್ರೋಟೋಕಾಲ್ ಆವೃತ್ತಿ (2025-06-18) ಅನ್ನು ಸರಿಯಾದ ಆವೃತ್ತಿ ಮಾತುಕತೆ ಜೊತೆಗೆ ಬಳಸಿ  

### 4. ಸುಧಾರಿತ ದರ ನಿಯಂತ್ರಣ ಮತ್ತು ಸಂಪನ್ಮೂಲ ರಕ್ಷಣೆ  
- **ಬಹು-ಪದರ ದರ ನಿಯಂತ್ರಣ**: ದುರುಪಯೋಗವನ್ನು ತಡೆಯಲು ಬಳಕೆದಾರ, ಸೆಷನ್, ಟೂಲ್ ಮತ್ತು ಸಂಪನ್ಮೂಲ ಮಟ್ಟದಲ್ಲಿ ದರ ನಿಯಂತ್ರಣವನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ  
- **ಅನುಕೂಲ ದರ ನಿಯಂತ್ರಣ**: ಬಳಕೆ ಮಾದರಿಗಳು ಮತ್ತು ಬೆದರಿಕೆ ಸೂಚಕಗಳಿಗೆ ಹೊಂದಿಕೊಳ್ಳುವ ಯಂತ್ರ ಅಧ್ಯಯನ ಆಧಾರಿತ ದರ ನಿಯಂತ್ರಣವನ್ನು ಬಳಸಿ  
- **ಸಂಪನ್ಮೂಲ ಕೊಟಾ ನಿರ್ವಹಣೆ**: ಗಣನೀಯ ಸಂಪನ್ಮೂಲಗಳು, ಮೆಮೊರಿ ಬಳಕೆ ಮತ್ತು ಕಾರ್ಯಾಚರಣೆ ಸಮಯಕ್ಕೆ ಸೂಕ್ತ ಮಿತಿಗಳನ್ನು ನಿಗದಿಪಡಿಸಿ  
- **DDoS ರಕ್ಷಣೆ**: ಸಮಗ್ರ DDoS ರಕ್ಷಣೆ ಮತ್ತು ಟ್ರಾಫಿಕ್ ವಿಶ್ಲೇಷಣಾ ವ್ಯವಸ್ಥೆಗಳನ್ನು ನಿಯೋಜಿಸಿ  

### 5. ಸಮಗ್ರ ಲಾಗಿಂಗ್ ಮತ್ತು ಮೇಲ್ವಿಚಾರಣೆ  
- **ಸಂರಚಿತ ಆಡಿಟ್ ಲಾಗಿಂಗ್**: ಎಲ್ಲಾ MCP ಕಾರ್ಯಾಚರಣೆಗಳು, ಟೂಲ್ ಕಾರ್ಯಾಚರಣೆಗಳು ಮತ್ತು ಭದ್ರತಾ ಘಟನೆಗಳಿಗಾಗಿ ವಿವರವಾದ, ಹುಡುಕಬಹುದಾದ ಲಾಗ್‌ಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ  
- **ರಿಯಲ್-ಟೈಮ್ ಭದ್ರತಾ ಮೇಲ್ವಿಚಾರಣೆ**: MCP ಕಾರ್ಯಭಾರಗಳಿಗೆ AI-ಚಾಲಿತ ಅನೋಮಲಿ ಪತ್ತೆಗಾಗಿ SIEM ವ್ಯವಸ್ಥೆಗಳನ್ನು ನಿಯೋಜಿಸಿ  
- **ಗೌಪ್ಯತೆ-ಅನುಕೂಲ ಲಾಗಿಂಗ್**: ಡೇಟಾ ಗೌಪ್ಯತೆ ಅಗತ್ಯತೆಗಳು ಮತ್ತು ನಿಯಮಾವಳಿಗಳನ್ನು ಗೌರವಿಸುವಂತೆ ಭದ್ರತಾ ಘಟನೆಗಳನ್ನು ಲಾಗ್ ಮಾಡಿ  
- **ಘಟನೆ ಪ್ರತಿಕ್ರಿಯೆ ಏಕೀಕರಣ**: ಲಾಗಿಂಗ್ ವ್ಯವಸ್ಥೆಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತ ಘಟನೆ ಪ್ರತಿಕ್ರಿಯೆ ಕಾರ್ಯಪ್ರವಾಹಗಳಿಗೆ ಸಂಪರ್ಕಿಸಿ  

### 6. ಸುಧಾರಿತ ಭದ್ರ ಸಂಗ್ರಹ ಅಭ್ಯಾಸಗಳು  
- **ಹಾರ್ಡ್‌ವೇರ್ ಭದ್ರತಾ ಘಟಕಗಳು**: ಪ್ರಮುಖ ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್ ಕಾರ್ಯಾಚರಣೆಗಳಿಗೆ HSM-ಬೆಂಬಲಿತ ಕೀ ಸಂಗ್ರಹಣೆ (Azure Key Vault, AWS CloudHSM) ಬಳಸಿ  
- **ಎನ್‌ಕ್ರಿಪ್ಷನ್ ಕೀ ನಿರ್ವಹಣೆ**: ಎನ್‌ಕ್ರಿಪ್ಷನ್ ಕೀಗಳಿಗೆ ಸರಿಯಾದ ಕೀ ತಿರುಗಾಟ, ವಿಭಜನೆ ಮತ್ತು ಪ್ರವೇಶ ನಿಯಂತ್ರಣಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ  
- **ರಹಸ್ಯ ನಿರ್ವಹಣೆ**: ಎಲ್ಲಾ API ಕೀಗಳು, ಟೋಕನ್‌ಗಳು ಮತ್ತು ಪ್ರಮಾಣಪತ್ರಗಳನ್ನು ಸಮರ್ಪಿತ ರಹಸ್ಯ ನಿರ್ವಹಣಾ ವ್ಯವಸ್ಥೆಗಳಲ್ಲಿ ಸಂಗ್ರಹಿಸಿ  
- **ಡೇಟಾ ವರ್ಗೀಕರಣ**: ಸಂವೇದನಶೀಲತೆ ಮಟ್ಟಗಳ ಆಧಾರದ ಮೇಲೆ ಡೇಟಾವನ್ನು ವರ್ಗೀಕರಿಸಿ ಮತ್ತು ಸೂಕ್ತ ರಕ್ಷಣೆ ಕ್ರಮಗಳನ್ನು ಅನ್ವಯಿಸಿ  

### 7. ಸುಧಾರಿತ ಟೋಕನ್ ನಿರ್ವಹಣೆ  
- **ಟೋಕನ್ ಪಾಸ್ತ್ರೂ ನಿರೋಧ**: ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳನ್ನು ಬಿಟ್ಟುಹೋಗುವ ಟೋಕನ್ ಪಾಸ್ತ್ರೂ ಮಾದರಿಗಳನ್ನು ಸ್ಪಷ್ಟವಾಗಿ ನಿಷೇಧಿಸಿ  
- **ಪ್ರೇಕ್ಷಕ ಮಾನ್ಯತೆ**: ಟೋಕನ್ ಪ್ರೇಕ್ಷಕ ಹಕ್ಕುಗಳು ಉದ್ದೇಶಿತ MCP ಸರ್ವರ್ ಗುರುತಿಗೆ ಹೊಂದಿಕೆಯಾಗಿರುವುದನ್ನು ಯಾವಾಗಲೂ ಪರಿಶೀಲಿಸಿ  
- **ಹಕ್ಕು ಆಧಾರಿತ ಪ್ರಾಧಿಕಾರ**: ಟೋಕನ್ ಹಕ್ಕುಗಳು ಮತ್ತು ಬಳಕೆದಾರ ಗುಣಲಕ್ಷಣಗಳ ಆಧಾರದ ಮೇಲೆ ಸೂಕ್ಷ್ಮ ಪ್ರಾಧಿಕಾರವನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ  
- **ಟೋಕನ್ ಬಾಂಧನ**: ಸೂಕ್ತವಾದಲ್ಲಿ ಟೋಕನ್‌ಗಳನ್ನು ನಿರ್ದಿಷ್ಟ ಸೆಷನ್‌ಗಳು, ಬಳಕೆದಾರರು ಅಥವಾ ಸಾಧನಗಳಿಗೆ ಬಾಂಧಿಸಿ  

### 8. ಭದ್ರ ಸೆಷನ್ ನಿರ್ವಹಣೆ  
- **ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್ ಸೆಷನ್ ಐಡಿಗಳು**: ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್‌ವಾಗಿ ಭದ್ರ ರ್ಯಾಂಡಮ್ ಸಂಖ್ಯೆ ಜನರೇಟರ್‌ಗಳನ್ನು ಬಳಸಿ ಸೆಷನ್ ಐಡಿಗಳನ್ನು ರಚಿಸಿ (ಅಂದಾಜು ಮಾಡಬಹುದಾದ ಕ್ರಮವಲ್ಲ)  
- **ಬಳಕೆದಾರ-ನಿರ್ದಿಷ್ಟ ಬಾಂಧನ**: `<user_id>:<session_id>` ಮುಂತಾದ ಭದ್ರ ಸ್ವರೂಪಗಳನ್ನು ಬಳಸಿ ಸೆಷನ್ ಐಡಿಗಳನ್ನು ಬಳಕೆದಾರ-ನಿರ್ದಿಷ್ಟ ಮಾಹಿತಿಗೆ ಬಾಂಧಿಸಿ  
- **ಸೆಷನ್ ಜೀವನಚಕ್ರ ನಿಯಂತ್ರಣಗಳು**: ಸರಿಯಾದ ಸೆಷನ್ ಅವಧಿ, ತಿರುಗಾಟ ಮತ್ತು ಅಮಾನ್ಯಗೊಳಿಸುವ ಯಂತ್ರಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ  
- **ಸೆಷನ್ ಭದ್ರತಾ ಹೆಡರ್‌ಗಳು**: ಸೆಷನ್ ರಕ್ಷಣೆಗೆ ಸೂಕ್ತ HTTP ಭದ್ರತಾ ಹೆಡರ್‌ಗಳನ್ನು ಬಳಸಿ  

### 9. AI-ಸಂಬಂಧಿತ ಭದ್ರತಾ ನಿಯಂತ್ರಣಗಳು  
- **ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್ ರಕ್ಷಣಾ**: Microsoft Prompt Shields ಅನ್ನು ಸ್ಪಾಟ್‌ಲೈಟಿಂಗ್, ಡೆಲಿಮಿಟರ್‌ಗಳು ಮತ್ತು ಡೇಟಾಮಾರ್ಕಿಂಗ್ ತಂತ್ರಜ್ಞಾನಗಳೊಂದಿಗೆ ನಿಯೋಜಿಸಿ  
- **ಟೂಲ್ ವಿಷಕಾರಕ ತಡೆ**: ಟೂಲ್ ಮೆಟಾಡೇಟಾವನ್ನು ಪರಿಶೀಲಿಸಿ, ಡೈನಾಮಿಕ್ ಬದಲಾವಣೆಗಳಿಗೆ ಮೇಲ್ವಿಚಾರಣೆ ಮಾಡಿ ಮತ್ತು ಟೂಲ್ ಅಖಂಡತೆಯನ್ನು ಪರಿಶೀಲಿಸಿ  
- **ಮಾದರಿ ಔಟ್‌ಪುಟ್ ಮಾನ್ಯತೆ**: ಸಾಧ್ಯವಿರುವ ಡೇಟಾ ಲೀಕೇಜ್, ಹಾನಿಕಾರಕ ವಿಷಯ ಅಥವಾ ಭದ್ರತಾ ನೀತಿ ಉಲ್ಲಂಘನೆಗಳಿಗಾಗಿ ಮಾದರಿ ಔಟ್‌ಪುಟ್‌ಗಳನ್ನು ಸ್ಕ್ಯಾನ್ ಮಾಡಿ  
- **ಸಂದರ್ಭ ವಿಂಡೋ ರಕ್ಷಣಾ**: ಸಂದರ್ಭ ವಿಂಡೋ ವಿಷಕಾರಕತೆ ಮತ್ತು манಿಪ್ಯುಲೇಶನ್ ದಾಳಿಗಳನ್ನು ತಡೆಯಲು ನಿಯಂತ್ರಣಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ  

### 10. ಟೂಲ್ ಕಾರ್ಯಾಚರಣೆ ಭದ್ರತೆ  
- **ಕಾರ್ಯಾಚರಣೆ ಸ್ಯಾಂಡ್‌ಬಾಕ್ಸಿಂಗ್**: ಸಂಪನ್ಮೂಲ ಮಿತಿಗಳೊಂದಿಗೆ ಕಂಟೈನರ್‌ಗೊಳಿಸಿದ, ವಿಭಜಿತ ಪರಿಸರಗಳಲ್ಲಿ ಟೂಲ್ ಕಾರ್ಯಾಚರಣೆಗಳನ್ನು ನಡೆಸಿ  
- **ಪ್ರಿವಿಲೇಜ್ ವಿಭಜನೆ**: ಕನಿಷ್ಠ ಅಗತ್ಯ ಪ್ರಿವಿಲೇಜ್‌ಗಳೊಂದಿಗೆ ಮತ್ತು ವಿಭಜಿತ ಸೇವಾ ಖಾತೆಗಳೊಂದಿಗೆ ಟೂಲ್‌ಗಳನ್ನು ಕಾರ್ಯಗೊಳಿಸಿ  
- **ನೆಟ್‌ವರ್ಕ್ ವಿಭಜನೆ**: ಟೂಲ್ ಕಾರ್ಯಾಚರಣೆ ಪರಿಸರಗಳಿಗೆ ನೆಟ್‌ವರ್ಕ್ ವಿಭಾಗವನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ  
- **ಕಾರ್ಯಾಚರಣೆ ಮೇಲ್ವಿಚಾರಣೆ**: ಅನೋಮಲಿ ವರ್ತನೆ, ಸಂಪನ್ಮೂಲ ಬಳಕೆ ಮತ್ತು ಭದ್ರತಾ ಉಲ್ಲಂಘನೆಗಳಿಗಾಗಿ ಟೂಲ್ ಕಾರ್ಯಾಚರಣೆಯನ್ನು ಮೇಲ್ವಿಚಾರಣೆ ಮಾಡಿ  

### 11. ನಿರಂತರ ಭದ್ರತಾ ಮಾನ್ಯತೆ  
- **ಸ್ವಯಂಚಾಲಿತ ಭದ್ರತಾ ಪರೀಕ್ಷೆ**: GitHub Advanced Security ಮುಂತಾದ ಟೂಲ್‌ಗಳೊಂದಿಗೆ CI/CD ಪೈಪ್ಲೈನ್‌ಗಳಲ್ಲಿ ಭದ್ರತಾ ಪರೀಕ್ಷೆಯನ್ನು ಏಕೀಕರಿಸಿ  
- **ದುರ್ಬಲತೆ ನಿರ್ವಹಣೆ**: AI ಮಾದರಿಗಳು ಮತ್ತು ಬಾಹ್ಯ ಸೇವೆಗಳನ್ನು ಒಳಗೊಂಡಂತೆ ಎಲ್ಲಾ ಅವಲಂಬನೆಗಳನ್ನು ನಿಯಮಿತವಾಗಿ ಸ್ಕ್ಯಾನ್ ಮಾಡಿ  
- **ಪ್ರವೇಶ ಪರೀಕ್ಷೆ**: ವಿಶೇಷವಾಗಿ MCP ಅನುಷ್ಠಾನಗಳನ್ನು ಗುರಿಯಾಗಿಸಿಕೊಂಡು ನಿಯಮಿತ ಭದ್ರತಾ ಮೌಲ್ಯಮಾಪನಗಳನ್ನು ನಡೆಸಿ  
- **ಭದ್ರತಾ ಕೋಡ್ ವಿಮರ್ಶೆಗಳು**: ಎಲ್ಲಾ MCP-ಸಂಬಂಧಿತ ಕೋಡ್ ಬದಲಾವಣೆಗಳಿಗೆ ಕಡ್ಡಾಯ ಭದ್ರತಾ ವಿಮರ್ಶೆಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ  

### 12. AIಗಾಗಿ ಸರಬರಾಜು ಸರಣಿ ಭದ್ರತೆ  
- **ಘಟಕ ಪರಿಶೀಲನೆ**: ಎಲ್ಲಾ AI ಘಟಕಗಳ (ಮಾದರಿಗಳು, ಎಂಬೆಡ್ಡಿಂಗ್‌ಗಳು, APIಗಳು) ಮೂಲ, ಅಖಂಡತೆ ಮತ್ತು ಭದ್ರತೆಯನ್ನು ಪರಿಶೀಲಿಸಿ  
- **ಅವಲಂಬನೆ ನಿರ್ವಹಣೆ**: ದುರ್ಬಲತೆ ಟ್ರ್ಯಾಕಿಂಗ್ ಜೊತೆಗೆ ಎಲ್ಲಾ ಸಾಫ್ಟ್‌ವೇರ್ ಮತ್ತು AI ಅವಲಂಬನೆಗಳ ಪ್ರಸ್ತುತ ಇನ್ವೆಂಟರಿಗಳನ್ನು ಕಾಪಾಡಿ  
- **ನಂಬಿಗಸ್ತ ಸಂಗ್ರಹಾಲಯಗಳು**: ಎಲ್ಲಾ AI ಮಾದರಿಗಳು, ಗ್ರಂಥಾಲಯಗಳು ಮತ್ತು ಟೂಲ್‌ಗಳಿಗೆ ಪರಿಶೀಲಿತ, ನಂಬಿಗಸ್ತ ಮೂಲಗಳನ್ನು ಬಳಸಿ  
- **ಸರಬರಾಜು ಸರಣಿ ಮೇಲ್ವಿಚಾರಣೆ**: AI ಸೇವಾ ಒದಗಿಸುವವರು ಮತ್ತು ಮಾದರಿ ಸಂಗ್ರಹಾಲಯಗಳಲ್ಲಿ ಉಲ್ಲಂಘನೆಗಳಿಗಾಗಿ ನಿರಂತರವಾಗಿ ಮೇಲ್ವಿಚಾರಣೆ ಮಾಡಿ  

## ಸುಧಾರಿತ ಭದ್ರತಾ ಮಾದರಿಗಳು

### MCPಗಾಗಿ ಶೂನ್ಯ ನಂಬಿಕೆ ವಾಸ್ತುಶಿಲ್ಪ  
- **ಯಾವಾಗಲೂ ನಂಬಬೇಡಿ, ಯಾವಾಗಲೂ ಪರಿಶೀಲಿಸಿ**: ಎಲ್ಲಾ MCP ಭಾಗವಹಿಸುವವರಿಗಾಗಿ ನಿರಂತರ ಪರಿಶೀಲನೆಯನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ  
- **ಮೈಕ್ರೋ-ವಿಭಾಗೀಕರಣ**: ಸೂಕ್ಷ್ಮ ನೆಟ್‌ವರ್ಕ್ ಮತ್ತು ಗುರುತಿನ ನಿಯಂತ್ರಣಗಳೊಂದಿಗೆ MCP ಘಟಕಗಳನ್ನು ವಿಭಜಿಸಿ  
- **ಶರತಾಧಾರಿತ ಪ್ರವೇಶ**: ಸಂದರ್ಭ ಮತ್ತು ವರ್ತನೆಗೆ ಹೊಂದಿಕೊಳ್ಳುವ ಅಪಾಯ ಆಧಾರಿತ ಪ್ರವೇಶ ನಿಯಂತ್ರಣಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ  
- **ನಿರಂತರ ಅಪಾಯ ಮೌಲ್ಯಮಾಪನ**: ಪ್ರಸ್ತುತ ಬೆದರಿಕೆ ಸೂಚಕಗಳ ಆಧಾರದ ಮೇಲೆ ಭದ್ರತಾ ಸ್ಥಿತಿಯನ್ನು ಗತಿಶೀಲವಾಗಿ ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ  

### ಗೌಪ್ಯತೆ-ರಕ್ಷಿಸುವ AI ಅನುಷ್ಠಾನ  
- **ಡೇಟಾ ಕನಿಷ್ಠೀಕರಣ**: ಪ್ರತಿ MCP ಕಾರ್ಯಾಚರಣೆಗೆ ಅಗತ್ಯವಿರುವ ಕನಿಷ್ಠ ಡೇಟಾವನ್ನು ಮಾತ್ರ ಬಹಿರಂಗಪಡಿಸಿ  
- **ವಿಭಿನ್ನ ಗೌಪ್ಯತೆ**: ಸಂವೇದನಶೀಲ ಡೇಟಾ ಪ್ರಕ್ರಿಯೆಗೆ ಗೌಪ್ಯತೆ-ರಕ್ಷಿಸುವ ತಂತ್ರಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ  
- **ಹೋಮೋಮಾರ್ಫಿಕ್ ಎನ್‌ಕ್ರಿಪ್ಷನ್**: ಎನ್‌ಕ್ರಿಪ್ಟ್ ಮಾಡಲಾದ ಡೇಟಾದ ಮೇಲೆ ಭದ್ರ ಗಣನೆಗಾಗಿ ಸುಧಾರಿತ ಎನ್‌ಕ್ರಿಪ್ಷನ್ ತಂತ್ರಗಳನ್ನು ಬಳಸಿ  
- **ಫೆಡರೇಟೆಡ್ ಲರ್ನಿಂಗ್**: ಡೇಟಾ ಸ್ಥಳೀಯತೆ ಮತ್ತು ಗೌಪ್ಯತೆಯನ್ನು ಕಾಪಾಡುವ ವಿತರಿತ ಅಧ್ಯಯನ ವಿಧಾನಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ  

### AI ವ್ಯವಸ್ಥೆಗಳಿಗೆ ಘಟನೆ ಪ್ರತಿಕ್ರಿಯೆ  
- **AI-ಸಂಬಂಧಿತ ಘಟನೆ ಪ್ರಕ್ರಿಯೆಗಳು**: AI ಮತ್ತು MCP-ಸಂಬಂಧಿತ ಬೆದರಿಕೆಗಳಿಗೆ ಹೊಂದಿಕೊಂಡ ಘಟನೆ ಪ್ರತಿಕ್ರಿಯೆ ಪ್ರಕ್ರಿಯೆಗಳನ್ನು ಅಭಿವೃದ್ಧಿಪಡಿಸಿ  
- **ಸ್ವಯಂಚಾಲಿತ ಪ್ರತಿಕ್ರಿಯೆ**: ಸಾಮಾನ್ಯ AI ಭದ್ರತಾ ಘಟನೆಗಳಿಗೆ ಸ್ವಯಂಚಾಲಿತ ನಿಯಂತ್ರಣ ಮತ್ತು ಪರಿಹಾರವನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸಿ  
- **ಫೋರೆನ್ಸಿಕ್ ಸಾಮರ್ಥ್ಯಗಳು**: AI ವ್ಯವಸ್ಥೆ ಉಲ್ಲಂಘನೆಗಳು ಮತ್ತು ಡೇಟಾ ಉಲ್ಲಂಘನೆಗಳಿಗೆ ಫೋರೆನ್ಸಿಕ್ ಸಿದ್ಧತೆಯನ್ನು ಕಾಪಾಡಿ  
- **ಪುನರುದ್ದಾರ ಪ್ರಕ್ರಿಯೆಗಳು**: AI ಮಾದರಿ ವಿಷಕಾರಕತೆ, ಪ್ರಾಂಪ್ಟ್ ಇಂಜೆಕ್ಷನ್ ದಾಳಿಗಳು ಮತ್ತು ಸೇವಾ ಉಲ್ಲಂಘನೆಗಳಿಂದ ಪುನರುದ್ದಾರಕ್ಕೆ ಪ್ರಕ್ರಿಯೆಗಳನ್ನು ಸ್ಥಾಪಿಸಿ
