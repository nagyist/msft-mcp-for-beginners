# AGENTS.md

## ಪ್ರಾಜೆಕ್ಟ್ ಅವಲೋಕನ

**MCP for Beginners** ಎಂಬುದು Model Context Protocol (MCP) ಅನ್ನು ಕಲಿಯಲು ತೆರೆಯಲ್ಪಟ್ಟ ಮೂಲಸೂತ್ರ ಶಿಕ್ಷಣ ಪಠ್ಯಕ್ರಮವಾಗಿದೆ - ಇದು AI ಮಾದರಿಗಳು ಮತ್ತು ಕ್ಲೈಂಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳ ನಡುವಿನ ಸಂವಹನಗಳಿಗೆ ಮಾನಕೀಕೃತ ಚಟುವಟಿಕೆ ರೂಪರೇಖೆ. ಈ ರೆಪೊಸಿಟರಿ ವಿವಿಧ ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳಲ್ಲಿ ಕೈಯಿಂದ ಮಾಡಬಹುದಾದ ಕೋಡ್ ಉದಾಹರಣೆಗಳೊಂದಿಗೆ ಸಮಗ್ರ ಅಧ್ಯಯನ ಸಾಮಗ್ರಿಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ.

### ಪ್ರಮುಖ ತಂತ್ರಜ್ಞಾನಗಳು

- **ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳು**: C#, Java, JavaScript, TypeScript, Python, Rust
- **ಫ್ರೇಮ್ವರ್ಕ್‌ಗಳು ಮತ್ತು SDKಗಳು**: 
  - MCP SDK (`@modelcontextprotocol/sdk`)
  - Spring Boot (Java)
  - FastMCP (Python)
  - LangChain4j (Java)
- **ಡೇಟಾಬೇಸ್‌ಗಳು**: PostgreSQL ಜೊತೆಗೆ pgvector ವಿಸ್ತರಣೆ
- **ಕ್ಲೌಡ್ ಪ್ಲಾಟ್‌ಫಾರ್ಮ್‌ಗಳು**: Azure (Container Apps, OpenAI, Content Safety, Application Insights)
- **ಬಿಲ್ಡ್ ಟೂಲ್‌ಗಳು**: npm, Maven, pip, Cargo
- **ದಾಖಲೆಗಳು**: Markdown ಸ್ವಯಂಚಾಲಿತ ಬಹುಭಾಷಾ ಅನುವಾದದೊಂದಿಗೆ (48+ ಭಾಷೆಗಳು)

### ವಾಸ್ತುಶಿಲ್ಪ

- **11 ಕೋರ್ ಮೋಡ್ಯೂಲ್‌ಗಳು (00-11)**: ಮೂಲಭೂತಗಳಿಂದ ಪ್ರಗತಿಶೀಲ ವಿಷಯಗಳವರೆಗೆ ಕ್ರಮಬದ್ಧ ಅಧ್ಯಯನ ಮಾರ್ಗ
- **ಹ್ಯಾಂಡ್ಸ್-ಆನ್ ಲ್ಯಾಬ್‌ಗಳು**: ಬಹುಭಾಷಾ ಸಂಪೂರ್ಣ ಪರಿಹಾರ ಕೋಡ್‌ನೊಂದಿಗೆ ಪ್ರಾಯೋಗಿಕ ವ್ಯಾಯಾಮಗಳು
- **ನಮೂನಾ ಪ್ರಾಜೆಕ್ಟ್‌ಗಳು**: ಕಾರ್ಯನಿರ್ವಹಿಸುವ MCP ಸರ್ವರ್ ಮತ್ತು ಕ್ಲೈಂಟ್ ಅನುಷ್ಠಾನಗಳು
- **ಅನುವಾದ ವ್ಯವಸ್ಥೆ**: ಬಹುಭಾಷಾ ಬೆಂಬಲಕ್ಕಾಗಿ ಸ್ವಯಂಚಾಲಿತ GitHub Actions ವರ್ಕ್‌ಫ್ಲೋ
- **ಚಿತ್ರ ಸಂಪತ್ತುಗಳು**: ಅನುವಾದಿತ ಆವೃತ್ತಿಗಳೊಂದಿಗೆ ಕೇಂದ್ರಿತ ಚಿತ್ರ ಡೈರೆಕ್ಟರಿ

## ಸೆಟಪ್ ಕಮಾಂಡ್‌ಗಳು

ಇದು ಡಾಕ್ಯುಮೆಂಟೇಶನ್-ಕೇಂದ್ರೀಕೃತ ರೆಪೊಸಿಟರಿ. ಬಹುತೇಕ ಸೆಟಪ್ ವೈಯಕ್ತಿಕ ನಮೂನಾ ಪ್ರಾಜೆಕ್ಟ್‌ಗಳು ಮತ್ತು ಲ್ಯಾಬ್‌ಗಳಲ್ಲಿ ನಡೆಯುತ್ತದೆ.

### ರೆಪೊಸಿಟರಿ ಸೆಟಪ್

```bash
# ರೆಪೊಸಿಟರಿಯನ್ನು ಕ್ಲೋನ್ ಮಾಡಿ
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### ನಮೂನಾ ಪ್ರಾಜೆಕ್ಟ್‌ಗಳೊಂದಿಗೆ ಕೆಲಸ

ನಮೂನಾ ಪ್ರಾಜೆಕ್ಟ್‌ಗಳು ಈ ಸ್ಥಳಗಳಲ್ಲಿ ಇವೆ:
- `03-GettingStarted/samples/` - ಭಾಷಾ-ನಿರ್ದಿಷ್ಟ ಉದಾಹರಣೆಗಳು
- `03-GettingStarted/01-first-server/solution/` - ಮೊದಲ ಸರ್ವರ್ ಅನುಷ್ಠಾನಗಳು
- `03-GettingStarted/02-client/solution/` - ಕ್ಲೈಂಟ್ ಅನುಷ್ಠಾನಗಳು
- `11-MCPServerHandsOnLabs/` - ಸಮಗ್ರ ಡೇಟಾಬೇಸ್ ಇಂಟಿಗ್ರೇಶನ್ ಲ್ಯಾಬ್‌ಗಳು

ಪ್ರತಿ ನಮೂನಾ ಪ್ರಾಜೆಕ್ಟ್ ತನ್ನದೇ ಆದ ಸೆಟಪ್ ಸೂಚನೆಗಳನ್ನು ಹೊಂದಿದೆ:

#### TypeScript/JavaScript ಪ್ರಾಜೆಕ್ಟ್‌ಗಳು
```bash
cd <project-directory>
npm install
npm start
```

#### Python ಪ್ರಾಜೆಕ್ಟ್‌ಗಳು
```bash
cd <project-directory>
pip install -r requirements.txt
# ಅಥವಾ
pip install -e .
python main.py
```

#### Java ಪ್ರಾಜೆಕ್ಟ್‌ಗಳು
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

## ಅಭಿವೃದ್ಧಿ ವರ್ಕ್‌ಫ್ಲೋ

### ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ರಚನೆ

- **ಮೋಡ್ಯೂಲ್‌ಗಳು 00-11**: ಕ್ರಮಬದ್ಧವಾಗಿ ಕೋರ್ ಪಠ್ಯಕ್ರಮ ವಿಷಯ
- **translations/**: ಭಾಷಾ-ನಿರ್ದಿಷ್ಟ ಆವೃತ್ತಿಗಳು (ಸ್ವಯಂಚಾಲಿತ, ನೇರವಾಗಿ ಸಂಪಾದಿಸಬೇಡಿ)
- **translated_images/**: ಸ್ಥಳೀಯೃತ ಚಿತ್ರ ಆವೃತ್ತಿಗಳು (ಸ್ವಯಂಚಾಲಿತ)
- **images/**: ಮೂಲ ಚಿತ್ರಗಳು ಮತ್ತು ಚಿತ್ರಣಗಳು

### ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಬದಲಾವಣೆಗಳನ್ನು ಮಾಡುವುದು

1. ಮೂಲ ಮೋಡ್ಯೂಲ್ ಡೈರೆಕ್ಟರಿಗಳಲ್ಲಿ (00-11) ಇಂಗ್ಲಿಷ್ ಮಾರ್ಕ್‌ಡೌನ್ ಫೈಲ್‌ಗಳನ್ನು ಮಾತ್ರ ಸಂಪಾದಿಸಿ
2. ಅಗತ್ಯವಿದ್ದರೆ `images/` ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ಚಿತ್ರಗಳನ್ನು ನವೀಕರಿಸಿ
3. ಸಹಕಾರ-ಅನುವಾದಕ GitHub Action ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಅನುವಾದಗಳನ್ನು ರಚಿಸುತ್ತದೆ
4. ಅನುವಾದಗಳು ಮುಖ್ಯ ಶಾಖೆಗೆ ಪುಷ್ ಮಾಡಿದಾಗ ಪುನಃ ರಚಿಸಲಾಗುತ್ತವೆ

### ಅನುವಾದಗಳೊಂದಿಗೆ ಕೆಲಸ

- **ಸ್ವಯಂಚಾಲಿತ ಅನುವಾದ**: GitHub Actions ವರ್ಕ್‌ಫ್ಲೋ ಎಲ್ಲಾ ಅನುವಾದಗಳನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ
- `translations/` ಡೈರೆಕ್ಟರಿಯಲ್ಲಿನ ಫೈಲ್‌ಗಳನ್ನು ಕೈಯಿಂದ ಸಂಪಾದಿಸಬೇಡಿ
- ಅನುವಾದ ಮೆಟಾಡೇಟಾ ಪ್ರತಿ ಅನುವಾದಿತ ಫೈಲ್‌ನಲ್ಲಿ ಸೇರಿಸಲಾಗಿದೆ
- ಬೆಂಬಲಿತ ಭಾಷೆಗಳು: 48+ ಭಾಷೆಗಳು, ಅರೆಬಿಕ್, ಚೈನೀಸ್, ಫ್ರೆಂಚ್, ಜರ್ಮನ್, ಹಿಂದಿ, ಜಪಾನೀಸ್, ಕೊರಿಯನ್, ಪೋರ್ಚುಗೀಸ್, ರಷ್ಯನ್, ಸ್ಪ್ಯಾನಿಷ್ ಮತ್ತು ಇನ್ನಷ್ಟು

## ಪರೀಕ್ಷಾ ಸೂಚನೆಗಳು

### ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಮಾನ್ಯತೆ

ಇದು ಮುಖ್ಯವಾಗಿ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ರೆಪೊಸಿಟರಿ ಆದ್ದರಿಂದ ಪರೀಕ್ಷೆ ಕೆಳಗಿನ ಮೇಲೆ ಕೇಂದ್ರೀಕರಿಸುತ್ತದೆ:

1. **ಲಿಂಕ್ ಮಾನ್ಯತೆ**: ಎಲ್ಲಾ ಆಂತರಿಕ ಲಿಂಕ್‌ಗಳು ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
```bash
# ಮುರಿದ ಮಾರ್ಕ್‌ಡೌನ್ ಲಿಂಕ್‌ಗಳನ್ನು ಪರಿಶೀಲಿಸಿ
find . -name "*.md" -type f | xargs grep -n "\[.*\](../../.*)"
```

2. **ಕೋಡ್ ಉದಾಹರಣೆ ಮಾನ್ಯತೆ**: ಕೋಡ್ ಉದಾಹರಣೆಗಳು ಸಂಯೋಜನೆ/ನಡೆಸಲು ಸಾಧ್ಯವಿರುವುದನ್ನು ಪರೀಕ್ಷಿಸಿ
```bash
# ನಿರ್ದಿಷ್ಟ ಮಾದರಿಯನ್ನು ನಾವಿಗೇಟ್ ಮಾಡಿ ಮತ್ತು ಅದರ ಪರೀಕ್ಷೆಗಳನ್ನು ನಡೆಸಿ
cd 03-GettingStarted/samples/typescript
npm install && npm test
```

3. **ಮಾರ್ಕ್‌ಡೌನ್ ಲಿಂಟಿಂಗ್**: ಸ್ವರೂಪದ ಸुसಂಗತತೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
```bash
# ಅಗತ್ಯವಿದ್ದರೆ markdownlint ಅನ್ನು ಬಳಸಿ
npx markdownlint-cli2 "**/*.md" "#node_modules"
```

## ನಿರ್ಮಾಣ ಮತ್ತು ನಿಯೋಜನೆ

### ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ನಿಯೋಜನೆ

ರೆಪೊಸಿಟರಿ GitHub Pages ಅಥವಾ ಸಮಾನ ಸೇವೆಯನ್ನು ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಹೋಸ್ಟಿಂಗ್‌ಗೆ ಬಳಸುತ್ತದೆ (ಅನ್ವಯಿಸಿದರೆ). ಮುಖ್ಯ ಶಾಖೆಯಲ್ಲಿ ಬದಲಾವಣೆಗಳು ಈ ಕೆಳಗಿನವುಗಳನ್ನು ಪ್ರೇರೇಪಿಸುತ್ತವೆ:

1. ಅನುವಾದ ವರ್ಕ್‌ಫ್ಲೋ (`.github/workflows/co-op-translator.yml`)
2. ಎಲ್ಲಾ ಇಂಗ್ಲಿಷ್ ಮಾರ್ಕ್‌ಡೌನ್ ಫೈಲ್‌ಗಳ ಸ್ವಯಂಚಾಲಿತ ಅನುವಾದ
3. ಅಗತ್ಯವಿದ್ದರೆ ಚಿತ್ರ ಸ್ಥಳೀಯೀಕರಣ

### ಯಾವುದೇ ನಿರ್ಮಾಣ ಪ್ರಕ್ರಿಯೆ ಅಗತ್ಯವಿಲ್ಲ

ಈ ರೆಪೊಸಿಟರಿ ಮುಖ್ಯವಾಗಿ ಮಾರ್ಕ್‌ಡೌನ್ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಹೊಂದಿದೆ. ಕೋರ್ ಪಠ್ಯಕ್ರಮ ವಿಷಯಕ್ಕೆ ಸಂಯೋಜನೆ ಅಥವಾ ನಿರ್ಮಾಣ ಹಂತ ಅಗತ್ಯವಿಲ್ಲ.
