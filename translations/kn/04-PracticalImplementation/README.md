# ಪ್ರಾಯೋಗಿಕ ಅನುಷ್ಠಾನ

[![ವಾಸ್ತವಿಕ ಉಪಕರಣಗಳು ಮತ್ತು ವರ್ಕ್‌ಫ್ಲೋಗಳೊಂದಿಗೆ MCP ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಹೇಗೆ ನಿರ್ಮಿಸಬೇಕು, ಪರೀಕ್ಷಿಸಬೇಕು ಮತ್ತು ನಿಯೋಜಿಸಬೇಕು](../../../translated_images/kn/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(ಈ ಪಾಠದ ವಿಡಿಯೋವನ್ನು ನೋಡುವುದಕ್ಕೆ ಮೇಲಿನ ಚಿತ್ರವನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ)_

ಪ್ರಾಯೋಗಿಕ ಅನುಷ್ಠಾನವು Model Context Protocol (MCP) ಶಕ್ತಿಯನ್ನು ಸ್ಪಷ್ಟಗೊಳಿಸುತ್ತದೆ. MCP ನ ಹಿಂದೆ ಇದ್ದ ಸಿದ್ಧಾಂತ ಮತ್ತು স্থಾಪत्यವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಮುಖ್ಯವಾಗಿದ್ದರೆ, ನಿಜವಾದ ಮೌಲ್ಯವು ಈ අදರ್ಶಗಳನ್ನು ಅನ್ವಯಿಸಿ ನೈಜ ಜಾಗತಿಕ ಸಮಸ್ಯೆಗಳಿಗೆ ಪರಿಹಾರಗಳನ್ನು ನಿರ್ಮಿಸು, ಪರೀಕ್ಷಿಸು ಮತ್ತು ನಿಯೋಜಿಸುವಾಗ ಕಾಣುತ್ತದೆ. ಈ ಅಧ್ಯಾಯವು ಕಲ್ಪನಾತ್ಮಕ ಜ್ಞಾನ ಮತ್ತು ಕೈಯಿಂದ ಅಭಿವೃದ್ಧಿಯ ನಡುವೆ ಸೇತುವೆ ಸಾಗಿಸುತ್ತದೆ, ನಿನಗೆ MCP ಆಧಾರಿತ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಜೀವಂತ ಮಾಡಲು ಮಾರ್ಗದರ್ಶನ ಮಾಡುತ್ತದೆ.

ನೀವು ಬುದ್ಧಿವಂತ ಸಹಾಯಕರನ್ನು ಅಭಿವೃದ್ಧಿಪಡಿಸುತ್ತಿದ್ದೀರಾ, ವ್ಯವಹಾರ ವಾಕ್ಫ್ಲೋಗಳಿಗೆ AI ಅನ್ನು ಸಂಯೋಜಿಸುತ್ತಿದ್ದೀರಾ ಅಥವಾ ಡೇಟಾ ಪ್ರಕ್ರಿಯೆಗೆ ಕಸ್ಟಮ್ ಉಪಕರಣಗಳನ್ನು ನಿರ್ಮಿಸುತ್ತಿದ್ದೀರಾ, MCP ಒಂದೊಂದು ಅನುವುತ್ತದ ಅನುಸ್ಥಾಪನೆ ನೀಡುತ್ತದೆ. ಇದರ ಭಾಷಾ-ಸ್ವತಂತ್ರ ವಿನ್ಯಾಸ ಮತ್ತು ಜನಪ್ರಿಯ ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳಿಗಾಗಿ ಅಧಿಕೃತ SDKಗಳು ವಿಶಾಲ ಡೆವಲಪರ್ ಪರಿಧಿಗೆ ಪ್ರಾಪ್ಯವಾಗುವಂತೆ ಮಾಡುತ್ತವೆ. ಈ SDKಗಳನ್ನು ಬಳಸಿಕೊಂಡು, ನೀವು ಬೇಗPrototype ಮಾರ್ಗವಾಗಿ ನಿಮ್ಮ ಪರಿಹಾರಗಳನ್ನು ವಿಭಿನ್ನ ವೇದಿಕೆಗಳಿಗೆ ಮತ್ತು ಪರಿಸರಗಳಿಗೆ ವರ್ಗಾವಣೆ ಮಾಡಬಹುದು.

ಮುಂದಿನ ವಿಭಾಗಗಳಲ್ಲಿ, ನೀವು MCP ನ C#, Java with Spring, TypeScript, JavaScript, ಮತ್ತು Python ನಲ್ಲಿ ಹೇಗೆ ಅನುಷ್ಠಾನಗೊಳಿಸಬಹುದು ಎಂಬುದನ್ನು ತೋರಿಸುವ ಪ್ರಾಯೋಗಿಕ ಉದಾಹರಣೆಗಳು, ಮಾದರಿ ಕೋಡ್ ಮತ್ತು ನಿಯೋಜನೆ ತಂತ್ರಗಳನ್ನು ಕಾಣಬಹುದು. ನೀವು MCP ಸರ್ವರ್ ಗಳನ್ನು ಡೀಬಗ್ ಮಾಡುವುದು ಮತ್ತು ಪರೀಕ್ಷಿಸುವುದು, APIಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು ಮತ್ತು Azure ಬಳಸಿಕೊಂಡು ಕ್ಲೌಡ್‌ಗೆ ಪರಿಹಾರಗಳನ್ನು ನಿಯೋಜಿಸುವ ವಿಧಾನವನ್ನು ತಿಳಿಯುತ್ತೀರಿ. ಈ ಕೈ-ಮೇಲೆ ಸಂಪನ್ಮೂಲಗಳು ನಿಮಗೆ ಕಲಿಕೆ ದ್ರುತಗೊಳ್ಳಲು ಮತ್ತು ನಂಬಿಕೆಯಿಂದ ಅಪಾರ್ಡಿದ, ಉತ್ಪಾದನಾ ಸಿದ್ಧ MCP ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತವೆ.

## ಅವಲೋಕನ

ಈ ಪಾಠವು MCP ಅನುಷ್ಠಾನದ ಪ್ರಾಯೋಗಿಕ ತತ್ವಗಳನ್ನು ಹಲವಾರು ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳಾದಿವೆ. ನಾವು C#, Java with Spring, TypeScript, JavaScript ಮತ್ತು Python ನಲ್ಲಿ MCP SDKಗಳನ್ನು ಬಳಸುವ ವಿಧಾನವನ್ನು ಅನ್ವೇಷಿಸುವೆವು, ಹೀಗಾಗಿ ವಿಶ್ವಾಸಾರ್ಹ ಅಪ್ಲಿಕೇಶನ್ ಗಳನ್ನು ನಿರ್ಮಿಸುವುದು, MCP ಸರ್ವರ್ ಗಳನ್ನು ಡೀಬಗ್ ಮತ್ತು ಪರೀಕ್ಷಿಸುವುದು ಮತ್ತು ಮರುಬಳಕೆ ಯೋಗ್ಯ ಸಂಪನ್ಮೂಲಗಳು, ಪ್ರಾಂಪ್ಟ್ ಗಳು ಮತ್ತು ಉಪಕರಣಗಳನ್ನು ರಚಿಸುವುದನ್ನು ಕಾಣಬಹುದು.

## ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳು

ಈ ಪಾಠದ ಕೊನೆಯಲ್ಲಿ, ನೀವು ಈ ವಿಷಯಗಳನ್ನು ಮುಂದುವರಿಸಬಲ್ಲಿರಿ:

- ವಿವಿಧ ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಭಾಷೆಗಳ ಅಧಿಕೃತ SDKಗಳನ್ನು ಬಳಸಿಕೊಂಡು MCP ಪರಿಹಾರಗಳನ್ನು ಅನುಷ್ಠಾನ ಮಾಡುವುದು
- ದೃಢತೆಯಿಂದ MCP ಸರ್ವರ್ ಗಳನ್ನು ಡೀಬಗ್ ಮತ್ತು ಪರೀಕ್ಷಿಸುವುದು
- ಸರ್ವರ್ ವೈಶಿಷ್ಟ್ಯಗಳು (ಸಂಪನ್ಮೂಲಗಳು, ಪ್ರಾಂಪ್ಟ್ ಗಳು ಮತ್ತು ಉಪಕರಣಗಳು) ರಚಿಸುವುದು ಮತ್ತು ಬಳಸುವುದು
- ಸಂಕೀರ್ಣ ಕಾರ್ಯಗಳಿಗಾಗಿ ಪರಿಣಾಮಕಾರಿ MCP ವರ್ಕ್‌ಫ್ಲೋಗಳನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸುವುದು
- ಕಾರ್ಯಕ್ಷಮತೆ ಮತ್ತು ವಿಶ್ವಾಸಾರ್ಹತೆಯಿಗಾಗಿ MCP ಅನುಷ್ಠಾನಗಳನ್ನು ಹೆಚ್ಚಿಸುವುದು

- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk)

## ಮೂಲ ಸರ್ವರ್ ವೈಶಿಷ್ಟ್ಯಗಳು

MCP ಸರ್ವರ್ ಗಳು ಈ ವೈಶಿಷ್ಟ್ಯಗಳ ಯಾವ ಸಂಯೋಜನೆಯನ್ನು ಬೇಕಾದರೂ ಅನುಷ್ಠಾನ ಮಾಡಬಹುದು:

### ಸಂಪನ್ಮೂಲಗಳು

ಸಂಪನ್ಮೂಲಗಳು ಬಳಕೆದಾರಿಗೆ ಅಥವಾ AI ಮಾದರಿಗೆ ಉಪಯೋಗಿಸುವ ಸಂದರ್‌ಭ ಮತ್ತು ಡೇಟಾದನ್ನು ಒದಗಿಸುತ್ತವೆ:

- ಲಿಖಿತ ದಾಖಲೆ ಸಂಗ್ರಹಾಲಯಗಳು
- ಜ್ಞಾನ ಭಂಡಾರಗಳು
- ರಚಿಸಲಾಗಿದೆ ಡೇಟಾ ಮೂಲಗಳು
- ಫೈಲ್ ವ್ಯವಸ್ಥೆಗಳು

### ಪ್ರಾಂಪ್ಟ್ ಗಳು

ಪ್ರಾಂಪ್ಟ್ ಗಳು ಬಳಕೆದಾರರಿಗೆ ರೂಪುಗೊಂಡ ಸಂದೇಶಗಳು ಮತ್ತು ವರ್ಕ್‌ಫ್ಲೋಗಳಾಗಿವೆ:

- ಪೂರ್ವನಿರ್ಧಾರಿತ ಸಂಭಾಷಣೆ ರೂಪಗಳು
- ಮಾರ್ಗದರ್ಶನಾಧಾರಿತ ಸಂವಾದ ಮಾದರಿಗಳು
- ವಿಶೇಷ ಸಂವಾದ ರಚನೆಗಳು

### ಉಪಕರಣಗಳು

ಉಪಕರಣಗಳು AI ಮಾದರಿ ಅನುಷ್ಠಾನ ಮಾಡುವ ಕಾರ್ಯಚಟುವಟಿಕೆಗಳು:

- ಡೇಟಾ ಪ್ರಕ್ರಿಯೆ ಉಪಕರಣಗಳು
- ಬಾಹ್ಯ API ಸಂಯೋಜನೆಗಳು
- ಗಣನಾಶಕ್ತಿಗಳು
- ಹುಡುಕಾಟ ಕಾರ್ಯಾಚರಣೆಗಳು

## ಮಾದರಿ ಅನುಷ್ಠಾನಗಳು: C# ಅನುಷ್ಠಾನ

ಅಧಿಕೃತ C# SDK ರೆಪೊಸಿಟರಿಯಲ್ಲಿ MCP ನ ವಿಭಿನ್ನ ಅಂಶಗಳನ್ನು ತೋರಿಸುವ ಕೆಲವು ಮಾದರಿ ಅನುಷ್ಠಾನಗಳಿವೆ:

- **ಮೂಲ MCP ಕ್ಲೈಂಟ್**: MCP ಕ್ಲೈಂಟ್ ನಿರ್ಮಿಸಿ ಉಪಕರಣಗಳನ್ನು ಕರೆ ಮಾಡುವ ಸರಳ ಉದಾಹರಣೆ
- **ಮೂಲ MCP ಸರ್ವರ್**: ಸರಳ ಉಪಕರಣ ನೋಂದಣಿಯೊಂದಿಗೆ ಕನಿಷ್ಟ ಸರ್ವರ್ ಅನುಷ್ಠಾನ
- **ಅ್ಯಡ್ವಾನ್ಸ್ಡ್ MCP ಸರ್ವರ್**: ಉಪಕರಣ ನೋಂದಣಿ, ದೃಢೀಕರಣ ಮತ್ತು ದೋಷ ನಿರ್ವಹಣೆ ಯಿಂದ ಪೂರ್ಣ ಸರ್ವರ್
- **ASP.NET ಸಂಯೋಜನೆ**: ASP.NET ಕೋರ್ ಅನ್ನು ಹೊಂದಿಸುವ ಉದಾಹರಣೆಗಳು
- **ಉಪಕರಣ ಅನುಷ್ಠಾನ ಮಾದರಿಗಳು**: ವಿಭಿನ್ನ ಸಂಕೀರ್ಣತೆ ಮಟ್ಟಗಳ ಉಪಕರಣಗಳನ್ನು ಅನುಷ್ಠಾನ ಮಾಡುವ ವಿವಿಧ ಮಾದರಿಗಳು

MCP C# SDK ಪೂರ್ವವೀಕ್ಷಣೆಯಲ್ಲಿದೆ ಮತ್ತು APIಗಳು ಬದಲಾಯಿಸಬಹುದು. SDK ಅಭಿವೃದ್ಧಿಯೊಂದಿಗೆ ನಾವು ಈ ಬ್ಲಾಗ್ ಅನ್ನು ನಿರಂತರವಾಗಿ ನವೀಕರಿಸುವೆವು.

## ರಿಮೋಟ್ MCP ಸರ್ವರ್ ಅನ್ನು Azure ಗೆ ನಿಯೋಜಿಸುವುದು

ಮುಂದೆ ನೀಡಿದ ಮಾದರಿಯನ್ನ ನಾವು ನಿಯೋಜಿಸಬಹುದೇ ನೋಡೋಣ:

1. ರೆಪೊ ಅನ್ನು ಕ್ಲೋನ್ ಮಾಡಿರಿ

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

1. `Microsoft.App` ಸಂಪನ್ಮೂಲ ಪ್ರೊವೈಡರ್ ಅನ್ನು ನೋಂದಣಿ ಮಾಡಿರಿ.

   - ನೀವು Azure CLI ಬಳಸುತ್ತಿದ್ದರೆ, `az provider register --namespace Microsoft.App --wait` ಅನ್ನು ಚಲಾಯಿಸಿ.
   - ನೀವು Azure PowerShell ಬಳಸುತ್ತಿದ್ದರೆ, `Register-AzResourceProvider -ProviderNamespace Microsoft.App` ಅನ್ನು ಚಲಾಯಿಸಿ. ನಂತರ ಕೆಲವು ಸಮಯದ ನಂತರ `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` ಅನ್ನು ಪರೀಕ್ಷಿಸಿ ನೋಂದಣಿಯ ಸ್ಥಿತಿಯನ್ನು ಪಡೆದುಕೊಳ್ಳಿ.

1. ಈ [azd](https://aka.ms/azd) ಕಮಾಂಡ್ ಅನ್ನು ಚಲಾಯಿಸಿ API ನಿರ್ವಹಣಾ ಸೇವೆ, ಕಾರ್ಯ ಅಪ್ಲಿಕೇಶನ್ (ಕೋಡಿನೊಂದಿಗೆ) ಮತ್ತು ಬೇರೆ ಎಲ್ಲಾ ಅಗತ್ಯ Azure ಸಂಪನ್ಮೂಲಗಳನ್ನು ಸಿದ್ಧಪಡಿಸಲು

    ```shell
    azd up
    ```

    ಈ ಕಮಾಂಡ್‌ಗಳು ಎಲ್ಲಾ ಕ್ಲೌಡ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು Azure ನಲ್ಲಿ ನಿಯೋಜಿಸಬೇಕು

### MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಬಳಸಿ ನಿಮ್ಮ ಸರ್ವರ್ ಅನ್ನು ಪರೀಕ್ಷೆ ಮಾಡುವುದು

1. **ಹೊಸ ಟರ್ಮಿನಲ್ ವಿಂಡೋದಲ್ಲಿ**, MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಅನ್ನು ಸ್ಥಾಪಿಸಿ ಮತ್ತು ಚಲಾಯಿಸಿ

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    ನೀವು ಈ ರೀತಿ ಇಂಟರ್ಫೇಸ್ ಕಾಣಬೇಕು:

    ![Connect to Node inspector](../../../translated_images/kn/connect.141db0b2bd05f096.webp)

1. MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು URL ಮೂಲಕ ಲೋಡ್ ಮಾಡಲು CTRL ಕ್ಲಿಕ್ ಮಾಡಲು (ಉದಾ: [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources))
1. ಸಾರಿದ ಸಂಪರ್ಕ ವಿಧವನ್ನು `SSE` ಗೆ ಸೆಟ್ ಮಾಡಿ
1. ನಿಮ್ಮ ಚಾಲನೆಯಲ್ಲಿರುವ API ನಿರ್ವಹಣೆ SSE ಅಂತ್ಯದ URL ಅನ್ನು ಸೆಟ್ ಮಾಡಿ ನಂತರ **Connect**:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

1. **ಉಪಕರಣಗಳನ್ನು ಪಟ್ಟಿ ಮಾಡಿ**. ಒಂದು ಉಪಕರಣ ಕ್ಲಿಕ್ ಮಾಡಿ ಮತ್ತು **Run Tool** ಮಾಡಿ.  

ಎಲ್ಲಾ ಹಂತಗಳು ಯಶಸ್ವಿಯಾಗಿ ಇದ್ದರೆ, ನೀವು ಈಗ MCP ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕ ಹೊಂದಿದ್ದು, ಉಪಕರಣವನ್ನು ಕರೆಯಲು ಸಾದ್ಯವಾಗಿದೆ.

## Azure ಗಾಗಿ MCP ಸರ್ವರ್ ಗಳು

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): ಈ ರೆಪೊಗಳನ್ನು ಬಳಸಿಕೊಂಡು Python, C# .NET ಅಥವಾ Node/TypeScript ಬಳಸಿಕೊಂಡು ಕಸ್ಟಮ್ ರಿಮೋಟ್ MCP ಸರ್ವರ್‌ಗಳನ್ನು Azure Functions ಮೂಲಕ ನಿರ್ಮಿಸಲು ಮತ್ತು ನಿಯೋಜಿಸಲು ತ್ವರಿತ ಆರಂಭಿಕ ಟೆಂಪ್ಲೇಟುಗಳಿವೆ.

ಸ್ಯಾಂಪಲ್ಸ್ ಡೆವಲಪರ್‌ಗಳಿಗೆ ಪೂರ್ಣ ಪರಿಹಾರ ಒದಗಿಸುತ್ತದೆ, ಇದು:

- ಸ್ಥಳೀಯವಾಗಿ ನಿರ್ಮಿಸಿ ಮತ್ತು ಚಲಾಯಿಸಿ: ಸ್ಥಳೀಯ ಯಂತ್ರದಲ್ಲಿ MCP ಸರ್ವರ್ ಅನ್ನು ಅಭಿವೃದ್ಧಿಪಡಿಸಿ ಮತ್ತು ಡೀಬಗ್ ಮಾಡಿ
- Azure ಗೆ ನಿಯೋಜಿಸಿ: ಸರಳ azd up ಕಮಾಂಡ್ ಮೂಲಕ ಕ್ಲೌಡ್ ಗೆ ಸುಲಭ ನಿಯೋಜನೆ
- ಗ್ರಾಹಕರಿಂದ ಸಂಪರ್ಕ ಸಾಧಿಸಿ: VS Code ನ Copilot ಏಜೆಂಟ್ ಮೋಡ್ ಮತ್ತು MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಉಪಕರಣ ಸೇರಿದಂತೆ ವಿವಿಧ ಗ್ರಾಹಕರಿಂದ MCP ಸರ್ವರ್‌ಗೆ ಸಂಪರ್ಕ

### ಪ್ರಮುಖ ವೈಶಿಷ್ಟ್ಯಗಳು

- ವಿನ್ಯಾಸದಿಂದ ಭದ್ರತೆ: MCP ಸರ್ವರ್ ಕೀ ಮತ್ತು HTTPS ಬಳಸಿ ಸುರಕ್ಷಿತವಾಗಿದೆ
- ಪ್ರಮಾಣೀಕರಣ ಆಯ್ಕೆಗಳು: ಕೆಲವು ಒಳಗೆ ಬರಿದ ಆಯ್ಕೆಗಳ ಮೂಲಕ OAuth ನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ ಮತ್ತು/ಅಥವಾ API ನಿರ್ವಹಣೆ
- ನೆಟ್‌ವರ್ಕ್ ವಿಭಜನೆ: Azure ವೀರ್ಚುವಲ್ ನೆಟ್‌ವರ್ಕ್ (VNET) ಬಳಸಿ ನೆಟ್‌ವರ್ಕ್ ವಿಭಜನೆ ನೀಡುತ್ತದೆ
- ಸರ್ವರ್‌ಲೆಸ್ ಸ್ಥಾಪত্য: Azure Functions ಬಳಸಿ ಸ್ಕೇಲಬಲ್, ಈವೆಂಟ್ ಚಾಲಿತ ಕಾರ್ಯಗತಗೊಳಿಸುವಿಕೆ
- ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ: ಸ umfassಿತ ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಡೀಬಗ್ ಬೆಂಬಲ

4. ಮಾದರಿಗೆ ನಿಮ್ಮ ಉಪಕರಣ ಸದುಪಯೋಗವಾಗಲು ಸಹಾಯ ಮಾಡುವ ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟನ್ನು ರಚಿಸಿ
