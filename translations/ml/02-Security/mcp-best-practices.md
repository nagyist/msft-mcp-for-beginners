# MCP സുരക്ഷാ മികച്ച പ്രാക്ടീസുകൾ 2025

ഈ സമഗ്ര ഗൈഡ് ഏറ്റവും പുതിയ **MCP സ്പെസിഫിക്കേഷൻ 2025-06-18** ആധാരമാക്കി, നിലവിലെ വ്യവസായ മാനദണ്ഡങ്ങൾ അനുസരിച്ച് മോഡൽ കോൺടെക്സ്റ്റ് പ്രോട്ടോക്കോൾ (MCP) സിസ്റ്റങ്ങൾ നടപ്പിലാക്കുന്നതിനുള്ള അനിവാര്യമായ സുരക്ഷാ മികച്ച പ്രാക്ടീസുകൾ വിശദീകരിക്കുന്നു. ഈ പ്രാക്ടീസുകൾ പരമ്പരാഗത സുരക്ഷാ പ്രശ്നങ്ങളും MCP വിന്യാസങ്ങൾക്ക് പ്രത്യേകമായ AI-സംബന്ധമായ ഭീഷണികളും പരിഹരിക്കുന്നു.

## നിർണായക സുരക്ഷാ ആവശ്യകതകൾ

### നിർബന്ധമായ സുരക്ഷാ നിയന്ത്രണങ്ങൾ (MUST ആവശ്യകതകൾ)

1. **ടോക്കൺ പരിശോധന**: MCP സെർവറുകൾക്ക് വ്യക്തമായി MCP സെർവറിനായി മാത്രമേ പുറപ്പെടുവിച്ച ടോക്കണുകൾ സ്വീകരിക്കാവൂ; മറ്റ് ടോക്കണുകൾ **MUST NOT** സ്വീകരിക്കരുത്  
2. **അധികാര പരിശോധന**: അധികാരം നടപ്പിലാക്കുന്ന MCP സെർവറുകൾ എല്ലാ ഇൻബൗണ്ട് അഭ്യർത്ഥനകളും പരിശോധിക്കണം, സെഷനുകൾ പ്രാമാണീകരണത്തിന് ഉപയോഗിക്കരുത്  
3. **ഉപയോക്തൃ സമ്മതി**: സ്റ്റാറ്റിക് ക്ലയന്റ് ഐഡികൾ ഉപയോഗിക്കുന്ന MCP പ്രോക്സി സെർവറുകൾ ഓരോ ഡൈനാമിക് രജിസ്റ്റർ ചെയ്ത ക്ലയന്റിനും വ്യക്തമായ ഉപയോക്തൃ സമ്മതി നേടണം  
4. **സുരക്ഷിത സെഷൻ ഐഡികൾ**: MCP സെർവറുകൾ ക്രിപ്‌റ്റോഗ്രാഫിക് സുരക്ഷിതവും നിർണായകമല്ലാത്ത സെഷൻ ഐഡികൾ സുരക്ഷിത റാൻഡം നമ്പർ ജനറേറ്ററുകൾ ഉപയോഗിച്ച് സൃഷ്ടിക്കണം  

## പ്രധാന സുരക്ഷാ പ്രാക്ടീസുകൾ

### 1. ഇൻപുട്ട് പരിശോധനയും ശുദ്ധീകരണവും  
- **സമഗ്ര ഇൻപുട്ട് പരിശോധന**: ഇൻജക്ഷൻ ആക്രമണങ്ങൾ, കൺഫ്യൂസ്ഡ് ഡെപ്യൂട്ടി പ്രശ്നങ്ങൾ, പ്രോംപ്റ്റ് ഇൻജക്ഷൻ ദുർബലതകൾ തടയാൻ എല്ലാ ഇൻപുട്ടുകളും പരിശോധിച്ച് ശുദ്ധീകരിക്കുക  
- **പാരാമീറ്റർ സ്കീമ നിർബന്ധനം**: എല്ലാ ടൂൾ പാരാമീറ്ററുകൾക്കും API ഇൻപുട്ടുകൾക്കും കർശനമായ JSON സ്കീമ പരിശോധന നടപ്പിലാക്കുക  
- **കണ്ടന്റ് ഫിൽറ്ററിംഗ്**: പ്രോംപ്റ്റുകളിലും പ്രതികരണങ്ങളിലും ദുഷ്പ്രവർത്തന ഉള്ളടക്കം Microsoft Prompt Shields, Azure Content Safety എന്നിവ ഉപയോഗിച്ച് ഫിൽട്ടർ ചെയ്യുക  
- **ഔട്ട്പുട്ട് ശുദ്ധീകരണം**: ഉപയോക്താക്കൾക്കും ഡൗൺസ്ട്രീം സിസ്റ്റങ്ങൾക്കും മുമ്പ് എല്ലാ മോഡൽ ഔട്ട്പുട്ടുകളും പരിശോധിച്ച് ശുദ്ധീകരിക്കുക  

### 2. പ്രാമാണീകരണവും അധികാരവും  
- **ബാഹ്യ ഐഡന്റിറ്റി പ്രൊവൈഡർമാർ**: കസ്റ്റം പ്രാമാണീകരണം നടപ്പിലാക്കുന്നതിന് പകരം സ്ഥാപിതമായ ഐഡന്റിറ്റി പ്രൊവൈഡർമാർ (Microsoft Entra ID, OAuth 2.1 പ്രൊവൈഡർമാർ) പ്രാമാണീകരണം കൈമാറുക  
- **സൂക്ഷ്മ അനുമതികൾ**: കുറഞ്ഞ അധികാര സിദ്ധാന്തം പാലിച്ച് ടൂൾ-നിർദിഷ്ട അനുമതികൾ നടപ്പിലാക്കുക  
- **ടോക്കൺ ലൈഫ്‌സൈക്കിൾ മാനേജ്മെന്റ്**: സുരക്ഷിത റോട്ടേഷൻ, ശരിയായ പ്രേക്ഷക പരിശോധന എന്നിവയോടെ ചെറുനാൾ കാലാവധി ഉള്ള ആക്സസ് ടോക്കണുകൾ ഉപയോഗിക്കുക  
- **മൾട്ടി-ഫാക്ടർ പ്രാമാണീകരണം**: എല്ലാ അഡ്മിനിസ്ട്രേറ്റീവ് ആക്സസിനും സങ്കീർണ്ണ പ്രവർത്തനങ്ങൾക്കും MFA ആവശ്യമാണ്  

### 3. സുരക്ഷിത കമ്മ്യൂണിക്കേഷൻ പ്രോട്ടോക്കോളുകൾ  
- **ട്രാൻസ്പോർട്ട് ലെയർ സുരക്ഷ**: എല്ലാ MCP കമ്മ്യൂണിക്കേഷനുകൾക്കും HTTPS/TLS 1.3 ഉപയോഗിച്ച് ശരിയായ സർട്ടിഫിക്കറ്റ് പരിശോധന നടത്തുക  
- **എൻഡ്-ടു-എൻഡ് എൻക്രിപ്ഷൻ**: യാത്രയിലും സ്റ്റോറേജിലും അത്യന്തം സങ്കീർണ്ണമായ ഡാറ്റയ്ക്ക് അധിക എൻക്രിപ്ഷൻ ലെയറുകൾ നടപ്പിലാക്കുക  
- **സർട്ടിഫിക്കറ്റ് മാനേജ്മെന്റ്**: ഓട്ടോമേറ്റഡ് പുതുക്കൽ പ്രക്രിയകളോടെ ശരിയായ സർട്ടിഫിക്കറ്റ് ലൈഫ്‌സൈക്കിൾ മാനേജ്മെന്റ് നടത്തുക  
- **പ്രോട്ടോക്കോൾ പതിപ്പ് നിർബന്ധനം**: നിലവിലെ MCP പ്രോട്ടോക്കോൾ പതിപ്പ് (2025-06-18) ശരിയായ പതിപ്പ് ചർച്ചയോടെ ഉപയോഗിക്കുക  

### 4. പുരോഗമന റേറ്റ് ലിമിറ്റിംഗ് & റിസോഴ്‌സ് സംരക്ഷണം  
- **മൾട്ടി-ലെയർ റേറ്റ് ലിമിറ്റിംഗ്**: ഉപയോക്താവ്, സെഷൻ, ടൂൾ, റിസോഴ്‌സ് തലങ്ങളിൽ റേറ്റ് ലിമിറ്റിംഗ് നടപ്പിലാക്കുക  
- **അഡാപ്റ്റീവ് റേറ്റ് ലിമിറ്റിംഗ്**: ഉപയോഗ പാറ്റേണുകൾക്കും ഭീഷണി സൂചനകൾക്കും അനുസരിച്ച് മെഷീൻ ലേണിംഗ് അടിസ്ഥാനമാക്കിയ റേറ്റ് ലിമിറ്റിംഗ് ഉപയോഗിക്കുക  
- **റിസോഴ്‌സ് ക്വോട്ട മാനേജ്മെന്റ്**: കംപ്യൂട്ടേഷൻ, മെമ്മറി ഉപയോഗം, എക്സിക്യൂഷൻ സമയം എന്നിവയ്ക്ക് അനുയോജ്യമായ പരിധികൾ നിശ്ചയിക്കുക  
- **DDoS സംരക്ഷണം**: സമഗ്രമായ DDoS സംരക്ഷണവും ട്രാഫിക് വിശകലന സംവിധാനങ്ങളും വിന്യസിക്കുക  

### 5. സമഗ്രമായ ലോഗിംഗ് & മോണിറ്ററിംഗ്  
- **സംഘടിത ഓഡിറ്റ് ലോഗിംഗ്**: എല്ലാ MCP പ്രവർത്തനങ്ങൾക്കും, ടൂൾ എക്സിക്യൂഷനുകൾക്കും, സുരക്ഷാ സംഭവങ്ങൾക്കും വിശദമായ, തിരയാവുന്ന ലോഗുകൾ നടപ്പിലാക്കുക  
- **റിയൽ-ടൈം സുരക്ഷാ മോണിറ്ററിംഗ്**: MCP വർക്ക്‌ലോഡുകൾക്കായി AI-ശക്തിയുള്ള അനോമലി കണ്ടെത്തൽ SIEM സിസ്റ്റങ്ങൾ വിന്യസിക്കുക  
- **സ്വകാര്യത പാലിക്കുന്ന ലോഗിംഗ്**: ഡാറ്റാ സ്വകാര്യതാ ആവശ്യകതകളും നിയമങ്ങളും മാനിച്ച് സുരക്ഷാ സംഭവങ്ങൾ ലോഗ് ചെയ്യുക  
- **ഇൻസിഡന്റ് പ്രതികരണ സംയോജനം**: ഓട്ടോമേറ്റഡ് ഇൻസിഡന്റ് പ്രതികരണ പ്രവാഹങ്ങളുമായി ലോഗിംഗ് സിസ്റ്റങ്ങൾ ബന്ധിപ്പിക്കുക  

### 6. മെച്ചപ്പെട്ട സുരക്ഷിത സംഭരണ പ്രാക്ടീസുകൾ  
- **ഹാർഡ്‌വെയർ സുരക്ഷാ മോഡ്യൂളുകൾ**: Azure Key Vault, AWS CloudHSM പോലുള്ള HSM പിന്തുണയുള്ള കീ സംഭരണം പ്രധാന ക്രിപ്‌റ്റോഗ്രാഫിക് പ്രവർത്തനങ്ങൾക്ക് ഉപയോഗിക്കുക  
- **എൻക്രിപ്ഷൻ കീ മാനേജ്മെന്റ്**: കീ റോട്ടേഷൻ, വേർതിരിക്കൽ, ആക്സസ് നിയന്ത്രണങ്ങൾ ശരിയായി നടപ്പിലാക്കുക  
- **സീക്രട്ട് മാനേജ്മെന്റ്**: എല്ലാ API കീകൾ, ടോക്കണുകൾ, ക്രെഡൻഷ്യലുകൾ സമർപ്പിത സീക്രട്ട് മാനേജ്മെന്റ് സിസ്റ്റങ്ങളിൽ സൂക്ഷിക്കുക  
- **ഡാറ്റാ വർഗ്ഗീകരണം**: ഡാറ്റയുടെ സങ്കീർണ്ണത അടിസ്ഥാനമാക്കി വർഗ്ഗീകരിച്ച് അനുയോജ്യമായ സംരക്ഷണ നടപടികൾ സ്വീകരിക്കുക  

### 7. പുരോഗമന ടോക്കൺ മാനേജ്മെന്റ്  
- **ടോക്കൺ പാസ്ത്രൂ പ്രിവെൻഷൻ**: സുരക്ഷാ നിയന്ത്രണങ്ങൾ മറികടക്കുന്ന ടോക്കൺ പാസ്ത്രൂ മാതൃകകൾ വ്യക്തമായി നിരോധിക്കുക  
- **പ്രേക്ഷക പരിശോധന**: ടോക്കൺ പ്രേക്ഷക അവകാശങ്ങൾ MCP സെർവർ ഐഡന്റിറ്റിയുമായി പൊരുത്തപ്പെടുന്നുണ്ടെന്ന് എല്ലായ്പ്പോഴും പരിശോധിക്കുക  
- **ക്ലെയിംസ് അടിസ്ഥാനത്തിലുള്ള അധികാരം**: ടോക്കൺ ക്ലെയിംസും ഉപയോക്തൃ ഗുണങ്ങളുമെ അടിസ്ഥാനമാക്കി സൂക്ഷ്മ അധികാരം നടപ്പിലാക്കുക  
- **ടോക്കൺ ബൈൻഡിംഗ്**: ടോക്കണുകൾ പ്രത്യേക സെഷനുകൾ, ഉപയോക്താക്കൾ, ഉപകരണങ്ങൾ എന്നിവയുമായി ബന്ധിപ്പിക്കുക  

### 8. സുരക്ഷിത സെഷൻ മാനേജ്മെന്റ്  
- **ക്രിപ്‌റ്റോഗ്രാഫിക് സെഷൻ ഐഡികൾ**: പ്രവചിക്കാനാകാത്ത ക്രിപ്‌റ്റോഗ്രാഫിക് സുരക്ഷിത റാൻഡം നമ്പർ ജനറേറ്ററുകൾ ഉപയോഗിച്ച് സെഷൻ ഐഡികൾ സൃഷ്ടിക്കുക  
- **ഉപയോക്തൃ-നിർദിഷ്ട ബൈൻഡിംഗ്**: `<user_id>:<session_id>` പോലുള്ള സുരക്ഷിത ഫോർമാറ്റുകൾ ഉപയോഗിച്ച് സെഷൻ ഐഡികൾ ഉപയോക്തൃ-നിർദിഷ്ട വിവരങ്ങളുമായി ബന്ധിപ്പിക്കുക  
- **സെഷൻ ലൈഫ്‌സൈക്കിൾ നിയന്ത്രണങ്ങൾ**: ശരിയായ സെഷൻ കാലാവധി, റോട്ടേഷൻ, അസാധുവാക്കൽ സംവിധാനങ്ങൾ നടപ്പിലാക്കുക  
- **സെഷൻ സുരക്ഷാ ഹെഡറുകൾ**: സെഷൻ സംരക്ഷണത്തിന് അനുയോജ്യമായ HTTP സുരക്ഷാ ഹെഡറുകൾ ഉപയോഗിക്കുക  

### 9. AI-സംബന്ധമായ സുരക്ഷാ നിയന്ത്രണങ്ങൾ  
- **പ്രോംപ്റ്റ് ഇൻജക്ഷൻ പ്രതിരോധം**: Microsoft Prompt Shields, spotlighting, delimiters, datamarking സാങ്കേതികവിദ്യകൾ വിന്യസിക്കുക  
- **ടൂൾ വിഷവൽക്കരണം തടയൽ**: ടൂൾ മെറ്റാഡേറ്റ പരിശോധിക്കുക, ഡൈനാമിക് മാറ്റങ്ങൾ നിരീക്ഷിക്കുക, ടൂൾ അഖണ്ഡത ഉറപ്പാക്കുക  
- **മോഡൽ ഔട്ട്പുട്ട് പരിശോധന**: ഡാറ്റാ ലീക്കേജ്, ഹാനികരമായ ഉള്ളടക്കം, സുരക്ഷാ നയം ലംഘനങ്ങൾ കണ്ടെത്താൻ മോഡൽ ഔട്ട്പുട്ടുകൾ സ്കാൻ ചെയ്യുക  
- **കോൺടെക്സ്റ്റ് വിൻഡോ സംരക്ഷണം**: കോൺടെക്സ്റ്റ് വിൻഡോ വിഷവൽക്കരണം, മാനിപ്പുലേഷൻ ആക്രമണങ്ങൾ തടയാൻ നിയന്ത്രണങ്ങൾ നടപ്പിലാക്കുക  

### 10. ടൂൾ എക്സിക്യൂഷൻ സുരക്ഷ  
- **എക്സിക്യൂഷൻ സാൻഡ്‌ബോക്സിംഗ്**: ടൂൾ എക്സിക്യൂഷനുകൾ കണ്ടെയ്‌നറൈസ്ഡ്, വേർതിരിച്ച പരിസരങ്ങളിൽ റിസോഴ്‌സ് പരിധികളോടെ നടത്തുക  
- **പ്രിവിലേജ് വേർതിരിക്കൽ**: കുറഞ്ഞ ആവശ്യമായ പ്രിവിലേജുകളോടെ ടൂളുകൾ പ്രവർത്തിപ്പിക്കുക, സേവന അക്കൗണ്ടുകൾ വേർതിരിക്കുക  
- **നെറ്റ്‌വർക്ക് വേർതിരിക്കൽ**: ടൂൾ എക്സിക്യൂഷൻ പരിസരങ്ങൾക്ക് നെറ്റ്‌വർക്ക് സെഗ്മെന്റേഷൻ നടപ്പിലാക്കുക  
- **എക്സിക്യൂഷൻ മോണിറ്ററിംഗ്**: അനോമലസ് പെരുമാറ്റം, റിസോഴ്‌സ് ഉപയോഗം, സുരക്ഷാ ലംഘനങ്ങൾ നിരീക്ഷിക്കുക  

### 11. തുടർച്ചയായ സുരക്ഷാ പരിശോധന  
- **ഓട്ടോമേറ്റഡ് സുരക്ഷാ പരിശോധന**: GitHub Advanced Security പോലുള്ള ടൂളുകൾ ഉപയോഗിച്ച് CI/CD പൈപ്പ്ലൈനുകളിൽ സുരക്ഷാ പരിശോധന സംയോജിപ്പിക്കുക  
- **ദുർബലത മാനേജ്മെന്റ്**: AI മോഡലുകളും ബാഹ്യ സേവനങ്ങളും ഉൾപ്പെടെ എല്ലാ ആശ്രിതത്വങ്ങളും নিয়മിതമായി സ്കാൻ ചെയ്യുക  
- **പെനെട്രേഷൻ ടെസ്റ്റിംഗ്**: MCP നടപ്പാക്കലുകൾ ലക്ഷ്യമാക്കി സ്ഥിരമായി സുരക്ഷാ വിലയിരുത്തലുകൾ നടത്തുക  
- **സുരക്ഷാ കോഡ് റിവ്യൂസ്**: MCP-സംബന്ധമായ എല്ലാ കോഡ് മാറ്റങ്ങൾക്കും നിർബന്ധമായ സുരക്ഷാ റിവ്യൂസ് നടപ്പിലാക്കുക  

### 12. AI-ക്കുള്ള സപ്ലൈ ചെയിൻ സുരക്ഷ  
- **ഘടക പരിശോധന**: എല്ലാ AI ഘടകങ്ങളുടെ (മോഡലുകൾ, എംബെഡിംഗുകൾ, APIകൾ) ഉറവിടം, അഖണ്ഡത, സുരക്ഷ പരിശോധിക്കുക  
- **ആശ്രിതത്വ മാനേജ്മെന്റ്**: എല്ലാ സോഫ്റ്റ്‌വെയർ, AI ആശ്രിതത്വങ്ങളുടെ നിലവിലെ ഇൻവെന്ററികൾ സൂക്ഷിച്ച് ദുർബലതകൾ ട്രാക്ക് ചെയ്യുക  
- **വിശ്വസനീയ റിപോസിറ്ററികൾ**: എല്ലാ AI മോഡലുകൾക്കും ലൈബ്രറികൾക്കും ടൂളുകൾക്കും സ്ഥിരീകരിച്ച, വിശ്വസനീയമായ ഉറവിടങ്ങൾ ഉപയോഗിക്കുക  
- **സപ്ലൈ ചെയിൻ മോണിറ്ററിംഗ്**: AI സേവന ദാതാക്കളിലും മോഡൽ റിപോസിറ്ററികളിലും ഉണ്ടാകാവുന്ന തട്ടിപ്പുകൾ നിരന്തരം നിരീക്ഷിക്കുക  

## പുരോഗമന സുരക്ഷാ മാതൃകകൾ

### MCP-ക്കുള്ള സീറോ ട്രസ്റ്റ് ആർക്കിടെക്ചർ  
- **ഒരിക്കലും വിശ്വസിക്കരുത്, എല്ലായ്പ്പോഴും പരിശോധിക്കുക**: എല്ലാ MCP പങ്കാളികളുടെയും തുടർച്ചയായ പരിശോധന നടപ്പിലാക്കുക  
- **മൈക്രോ-സെഗ്മെന്റേഷൻ**: MCP ഘടകങ്ങളെ സൂക്ഷ്മമായ നെറ്റ്‌വർക്ക്, ഐഡന്റിറ്റി നിയന്ത്രണങ്ങളോടെ വേർതിരിക്കുക  
- **ശരതപ്പെട്ട ആക്സസ്**: സാഹചര്യവും പെരുമാറ്റവും അനുസരിച്ച് റിസ്ക് അടിസ്ഥാനമാക്കിയ ആക്സസ് നിയന്ത്രണങ്ങൾ നടപ്പിലാക്കുക  
- **തുടർച്ചയായ റിസ്ക് വിലയിരുത്തൽ**: നിലവിലെ ഭീഷണി സൂചനകളുടെ അടിസ്ഥാനത്തിൽ സുരക്ഷാ നില വിലയിരുത്തുക  

### സ്വകാര്യത സംരക്ഷിക്കുന്ന AI നടപ്പാക്കൽ  
- **ഡാറ്റാ കുറവ്**: ഓരോ MCP പ്രവർത്തനത്തിനും ആവശ്യമായ ഏറ്റവും കുറഞ്ഞ ഡാറ്റ മാത്രം പുറത്തുവിടുക  
- **ഡിഫറൻഷ്യൽ പ്രൈവസി**: സങ്കീർണ്ണ ഡാറ്റാ പ്രോസസ്സിംഗിനായി സ്വകാര്യത സംരക്ഷിക്കുന്ന സാങ്കേതികവിദ്യകൾ നടപ്പിലാക്കുക  
- **ഹോമോമോർഫിക് എൻക്രിപ്ഷൻ**: എൻക്രിപ്റ്റ് ചെയ്ത ഡാറ്റയിൽ സുരക്ഷിത കംപ്യൂട്ടേഷൻക്കായി പുരോഗമന എൻക്രിപ്ഷൻ സാങ്കേതികവിദ്യകൾ ഉപയോഗിക്കുക  
- **ഫെഡറേറ്റഡ് ലേണിംഗ്**: ഡാറ്റാ ലൊക്കാലിറ്റി, സ്വകാര്യത സംരക്ഷിക്കുന്ന വിതരണ പഠന സമീപനങ്ങൾ നടപ്പിലാക്കുക  

### AI സിസ്റ്റങ്ങൾക്കുള്ള ഇൻസിഡന്റ് പ്രതികരണം  
- **AI-സംബന്ധമായ ഇൻസിഡന്റ് നടപടിക്രമങ്ങൾ**: AI, MCP-സംബന്ധമായ ഭീഷണികൾക്കായി പ്രത്യേകിച്ചുള്ള ഇൻസിഡന്റ് പ്രതികരണ നടപടിക്രമങ്ങൾ വികസിപ്പിക്കുക  
- **ഓട്ടോമേറ്റഡ് പ്രതികരണം**: സാധാരണ AI സുരക്ഷാ സംഭവങ്ങൾക്ക് ഓട്ടോമേറ്റഡ് തടയൽ, പരിഹാര നടപടികൾ നടപ്പിലാക്കുക  
- **ഫോറൻസിക് ശേഷികൾ**: AI സിസ്റ്റം തട്ടിപ്പുകൾക്കും ഡാറ്റാ ലംഘനങ്ങൾക്കും ഫോറൻസിക് തയ്യാറെടുപ്പ് നിലനിർത്തുക  
- **പുനരുദ്ധാരണ നടപടിക്രമങ്ങൾ**: AI മോഡൽ വിഷവൽക്കരണം, പ്രോംപ്റ്റ് ഇൻജക്ഷൻ ആക്രമണങ്ങൾ, സേവന തട്ടിപ്പുകൾ എന്നിവയിൽ നിന്നുള്ള പുനരുദ്ധാരണ നടപടികൾ സ്ഥാപിക്കുക
