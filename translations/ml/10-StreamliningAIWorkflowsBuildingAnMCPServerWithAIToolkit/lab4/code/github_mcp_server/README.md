# Weather MCP സെർവർ

ഇത് പൊത്തുള്ള പ്രതികരണങ്ങളോടെ കാലാവസ്ഥാ ഉപകരണങ്ങൾ നടപ്പിലാക്കുന്ന പൈത്തൺ സാംപിൾ MCP സെർവർ ആണ്. നിങ്ങളുടെ സ്വന്തം MCP സെർവറിനു തീർച്ച ചെയ്യാനുള്ള സ്‌കാഫോൾഡ് ആയി ഇത് ഉപയോഗിക്കാനാകും. ഇതിൽ താഴെപ്പറയുന്ന സവിശേഷതകൾ ഉൾപ്പെടുന്നു:

- **കാലാവസ്ഥാ ഉപകരണം**: നൽകിയ സ്ഥലം അടിസ്ഥാനമാക്കി പൊത്തുള്ള കാലാവസ്ഥാ വിവരങ്ങൾ നൽകുന്ന ഉപകരണം.
- **Git ക്ലോൺ ഉപകരണം**: git റീപോസിറ്ററി ഒരു നിർദ്ദിഷ്ട ഫോൾഡറിലേക്ക് ക്ലോൺ ചെയ്യുന്നു.
- **VS കോഡ് ഓപ്പൺ ഉപകരണം**: ഒരു ഫോൾഡർ VS കോഡ് അല്ലെങ്കിൽ VS കോഡ് ഇൻസൈറ്റേഴ്സ്-ൽ തുറക്കുന്നു.
- **എജന്റ് ബിൽഡറിലേക്ക് കണക്ട് ചെയ്യുക**: ടെസ്റ്റിംഗിനും ഡീബഗിംഗിനും MCP സെർവർ എജന്റ് ബിൽഡറുമായി ബന്ധിപ്പിക്കാൻ സാധിക്കും.
- **[MCP ഇൻസ്പക്ടർ](https://github.com/modelcontextprotocol/inspector) ൽ ഡീബഗ് ചെയ്യുക**: MCP ഇൻസ്പക്ടർ ഉപയോഗിച്ച് MCP സെർവർ ഡീബഗ് ചെയ്യാനുള്ള സവിശേഷത.

## Weather MCP സെർവർ ടെംപ്ലേറ്റുമായി തുടക്കം എടുക്കുക

> **ആവശ്യങ്ങൾ**
>
> നിങ്ങളുടെ ലൊക്കൽ ഡെവ് машина-യിൽ MCP സെർവർ പ്രവർത്തിപ്പിക്കാൻ ഇതുകൾ ആവശ്യമാണ്:
>
> - [Python](https://www.python.org/)
> - [Git](https://git-scm.com/) (git_clone_repo ടൂൾക്കായി ആവശ്യമാണ്)
> - [VS Code](https://code.visualstudio.com/) അല്ലെങ്കിൽ [VS Code Insiders](https://code.visualstudio.com/insiders/) (open_in_vscode ടൂൾക്കായി ആവശ്യമാണ്)
> - (*ഐച്ഛികം - നിങ്ങൾക്ക് uv മുൻഗണനയെങ്കിൽ*) [uv](https://github.com/astral-sh/uv)
> - [Python Debugger Extension](https://marketplace.visualstudio.com/items?itemName=ms-python.debugpy)

## പരിസ്ഥിതി ഒരുക്കുക

ഈ പ്രോജക്ടിനായി പരിസ്ഥിതിയെ ക്രമീകരിക്കാൻ രണ്ട് മാർഗ്ഗങ്ങളുണ്ട്. നിങ്ങളുടെ ഇഷ്ടാനുസരണം ഒരു മാർഗം തിരഞ്ഞെടുക്കാം.

> കുറിപ്പ്: വർച്വൽ പരിസ്ഥിതി സൃഷ്ടിച്ച ശേഷം വിൻഡ്‌സ് കോഡ് അല്ലെങ്കിൽ ടെർമിനൽ റീലോഡ് ചെയ്യുക, വിർച്ച്വൽ പരിസ്ഥിതിയിലെ പൈത്തൺ ഉപയോഗിക്കുന്നതിന്.

| മാർഗ്ഗം | പടികൾ |
| -------- | ----- |
| `uv` ഉപയോഗിച്ച് | 1. വിർച്ച്വൽ പരിസ്ഥിതി സൃഷ്ടിക്കുക: `uv venv` <br>2. VSCode കമാൻഡ് "***Python: Select Interpreter***" പ്രവർത്തിപ്പിച്ച് സൃഷ്ടിച്ച വിർച്ച്വൽ പരിസ്ഥിതിയിലെ പൈത്തൺ തിരഞ്ഞെടുക്കുക <br>3. ആവശ്യമായ ഡിപ്പൻഡൻസികൾ ഇൻസ്റ്റാൾ ചെയ്യുക (ഡെവ് ഡിപ്പൻഡൻസികൾ ഉൾപ്പെടെ): `uv pip install -r pyproject.toml --extra dev` |
| `pip` ഉപയോഗിച്ച് | 1. വിർച്ച്വൽ പരിസ്ഥിതി സൃഷ്ടിക്കുക: `python -m venv .venv` <br>2. VSCode കമാൻഡ് "***Python: Select Interpreter***" പ്രവർത്തിപ്പിച്ച് സൃഷ്ടിച്ച വിർച്ച്വൽ പരിസ്ഥിതിയിലെ പൈത്തൺ തിരഞ്ഞെടുക്കുക <br>3. ആവശ്യമായ ഡിപ്പൻഡൻസികൾ ഇൻസ്റ്റാൾ ചെയ്യുക (ഡെവ് ഡിപ്പൻഡൻസികൾ ഉൾപ്പെടെ): `pip install -e .[dev]` |

പരിസ്ഥിതി ക്രമീകരിച്ച് കഴിഞ്ഞാൽ, MCP ക്ലയന്റ് ആയി എജന്റ് ബിൽഡർ വഴി നിങ്ങളുടെ ലൊക്കൽ ഡെവ് машина-യിൽ സെർവർ ഓടിക്കാം:

1. VS കോഡ് ഡീബഗ് പാനൽ തുറക്കുക. `Debug in Agent Builder` തിരഞ്ഞെടുക്കുക അല്ലെങ്കിൽ MCP സെർവർ ഡീബഗ് ആരംഭിക്കാൻ `F5` അമർത്തുക.
2. AI Toolkit എജന്റ് ബിൽഡർ ഉപയോഗിച്ച് [ഈ പ്രോമ്പ്റ്റ്](../../../../../../../../../../../open_prompt_builder) ഉപയോഗിച്ച് സെർവർ പരീക്ഷിക്കുക. സെർവർ സ്വയം എജന്റ് ബിൽഡറുമായി കണക്ട് ചെയ്യും.
3. പ്രോമ്പ്റ്റ് ഉപയോഗിച്ച് സെർവർ ടെസ്റ്റ് ചെയ്യാൻ `Run` ക്ലിക്ക് ചെയ്യുക.

**അഭിനന്ദനങ്ങൾ**! എജന്റ് ബിൽഡർ വഴി MCP ക്ലയന്റ് ആയി നിങ്ങളുടെ ലൊക്കൽ ഡെവ് машина-യിൽ Weather MCP സെർവർ വിജയകരമായി ഓടിച്ചു.

## Weather MCP സെർവർ ഡീബഗ് എങ്ങനെ ചെയ്യാം

## ലഭ്യമായ ടൂളുകൾ

### കാലാവസ്ഥാ ടൂൾ
`get_weather` ടൂൾ ഒരു പ്രവചിക്കപ്പെട്ട സ്ഥലത്തിനുള്ള പൊത്തുള്ള കാലാവസ്ഥാ വിവരങ്ങൾ നൽകുന്നു.

| പാരാമീറ്റർ | തരം | വിവരണം |
| --------- | ---- | ----------- |
| `location` | string | കാലാവസ്ഥ അറിയേണ്ട സ്ഥലം (Ex: നഗരം, സംസ്ഥാനം, കോർഡിനേറ്റ്സ്) |

### Git ക്ലോൺ ടൂൾ
`git_clone_repo` ടൂൾ git റീപോസിറ്ററി നിർദ്ദിഷ്ട ഫോൾഡറിലേക്ക് ക്ലോൺ ചെയ്യുന്നു.

| പാരാമീറ്റർ | തരം | വിവരണം |
| --------- | ---- | ----------- |
| `repo_url` | string | ക്ലോൺ ചെയ്യാനുള്ള git റീപോസിറ്ററിയുടെ URL |
| `target_folder` | string | റീപോസിറ്ററി എവിടേക്ക് ക്ലോൺ ചെയ്യണം എന്ന് ഫോൾഡറിന്റെ പാത |

ഉപകരണം ഒരു JSON ഒബ്ജക്റ്റ് തിരിച്ചുകൊടുക്കുന്നു:
- `success`: പ്രവർത്തി വിജയിച്ചോ എന്ന ബൂളിയൻ
- `target_folder` അല്ലെങ്കിൽ `error`: ക്ലോൺ ചെയ്ത ഫോൾഡറിന്റെ പാത്തോ അല്ലെങ്കിൽ ഒരു തകരാർ സന്ദേശം

### VS കോഡ് ഓപ്പ് ടൂൾ
`open_in_vscode` ടൂൾ ഒരു ഫോൾഡർ VS കോഡ് അല്ലെങ്കിൽ VS കോഡ് ഇൻസൈറ്റേഴ്സ് ആപ്പ്ലിക്കേഷനിൽ തുറക്കുന്നു.

| പാരാമീറ്റർ | തരം | വിവരണം |
| --------- | ---- | ----------- |
| `folder_path` | string | തുറക്കേണ്ട ഫോൾഡറിന്റെ പാത |
| `use_insiders` | boolean (ഐച്ഛികം) | സാധാരണ VS കോഡിന് പകരം VS കോഡ് ഇൻസൈറ്റേഴ്സ് ഉപയോഗിക്കണോ എന്ന് |

ടൂൾ ഒരു JSON ഒബ്ജക്റ്റ് തിരിച്ചുന്നു:
- `success`: പ്രവർത്തി വിജയിച്ചോ എന്ന ബൂളിയൻ
- `message` അല്ലെങ്കിൽ `error`: സ്ഥിരീകരണ സന്ദേശം അല്ലെങ്കിൽ പിശക് സന്ദേശം

| ഡീബഗ് മോഡ് | വിവരണം | ഡീബഗ് ചെയ്യാനുള്ള പടികൾ |
| ---------- | ----------- | --------------- |
| എജന്റ് ബിൽഡർ | AI Toolkit വഴി MCP സെർവർ എജന്റ് ബിൽഡറിൽ ഡീബഗ് ചെയ്യുന്നവ. | 1. VS കോഡ് ഡീബഗ് പാനൽ തുറക്കുക. `Debug in Agent Builder` തിരഞ്ഞെടുക്കുകയും MCP സെർവർ ഡീബഗ് തുടങ്ങാൻ `F5` അമർത്തുകയും ചെയ്യുക.<br>2. AI Toolkit എജന്റ് ബിൽഡർ ഉപയോഗിച്ച് [ഈ പ്രോമ്പ്റ്റ്](../../../../../../../../../../../open_prompt_builder) ഉപയോഗിച്ച് സെർവർ പരീക്ഷിക്കുക. സെർവർ സ്വയം എജന്റ് ബിൽഡറുമായി കണക്ട് ചെയ്യും.<br>3. പ്രോമ്പ്റ്റ് ഉപയോഗിച്ച് സെർവർ പരിശോധിക്കാൻ `Run` ക്ലിക്ക് ചെയ്യുക. |
| MCP ഇൻസ്പക്ടർ | MCP ഇൻസ്പക്ടർ ഉപയോഗിച്ച് MCP സെർവർ ഡീബഗ് ചെയ്യുക. | 1. [Node.js](https://nodejs.org/) ഇൻസ്റ്റാൾ ചെയ്യുക<br>2. ഇൻസ്പക്ടർ ക്രമീകരിക്കുക: `cd inspector` && `npm install` <br>3. VS കോഡ് ഡീബഗ് പാനൽ തുറക്കുക. `Debug SSE in Inspector (Edge)` അല്ലെങ്കിൽ `Debug SSE in Inspector (Chrome)` തിരഞ്ഞെടുക്കുക, F5 അമർത്തി ഡീബഗ് തുടങ്ങുക.<br>4. MCP ഇൻസ്പക്ടർ ബ്രൗസറിൽ തുടങ്ങിയപ്പോൾ `Connect` ബട്ടൺ ക്ലിക്ക് ചെയ്തു ഈ MCP സെർവറുമായി കണക്റ്റ് ചെയ്യുക.<br>5. പിന്നെ `List Tools` തിരഞ്ഞെടുക്കുക, ടൂൾ തിരഞ്ഞെടുക്കുക, പാരാമീറ്ററുകൾ നൽകുക, `Run Tool` ചെയ്ത് നിങ്ങളുടെ സെർവർ കോഡ് ഡീബഗ് ചെയ്യാം.<br> |
