# കാൽക്കുലേറ്റർ HTTP സ്ട്രീമിംഗ് ഡെമോ

ഈ പ്രോജക്ട് Spring Boot WebFlux ഉപയോഗിച്ച് Server-Sent Events (SSE) ഉപയോഗിച്ച് HTTP സ്ട്രീമിംഗ് പ്രദർശിപ്പിക്കുന്നു. ഇത് രണ്ട് ആപ്ലിക്കേഷനുകളായി വിഭജിച്ചിരിക്കുന്നു:

- **കാൽക്കുലേറ്റർ സർവർ**: കാൽക്കുലേഷനുകൾ നടത്തുകയും SSE ഉപയോഗിച്ച് ഫലങ്ങൾ സ്ട്രീം ചെയ്യുകയും ചെയ്യുന്ന റിയാക്ടീവ് വെബ് സർവീസ്
- **കാൽക്കുലേറ്റർ ക്ലയന്റ്**: സ്ട്രീമിംഗ് എൻഡ്‌പോയിന്റ് ഉപയോഗിക്കുന്ന ക്ലയന്റ് ആപ്ലിക്കേഷൻ

## മുൻ‌അവശ്യങ്ങൾ

- ജാവ 17 അല്ലെങ്കിൽ അതിനുമുകളിൽ
- മേവൻ 3.6 അല്ലെങ്കിൽ അതിനുമുകളിൽ

## ഇത് എങ്ങനെ പ്രവർത്തിക്കുന്നു

1. **കാൽക്കുലേറ്റർ സർവർ** ഒരു `/calculate` എൻഡ്‌പോയിന്റ് എക്സ്പോസ് ചെയ്യുന്നു, അത്:
   - ക്വറി പാരാമീറ്ററുകൾ സ്വീകരിക്കുന്നു: `a` (സംഖ്യ), `b` (സംഖ്യ), `op` (ഓപ്പറേഷൻ)
   - പിന്തുണയുള്ള ഓപ്പറേഷനുകൾ: `add`, `sub`, `mul`, `div`
   - കാൽക്കുലേഷൻ പുരോഗതിയും ഫലവും Server-Sent Events ആയി റിട്ടേൺ ചെയ്യുന്നു

2. **കാൽക്കുലേറ്റർ ക്ലയന്റ്** സർവറുമായി കണക്ട് ചെയ്യുന്നു, അത്:
   - `7 * 5` കാൽക്കുലേറ്റ് ചെയ്യാൻ അഭ്യർത്ഥന അയയ്ക്കുന്നു
   - സ്ട്രീമിംഗ് പ്രതികരണം ഉപയോഗിക്കുന്നു
   - ഓരോ ഇവന്റും കൺസോളിൽ പ്രിന്റ് ചെയ്യുന്നു

## ആപ്ലിക്കേഷനുകൾ പ്രവർത്തിപ്പിക്കൽ

### ഓപ്ഷൻ 1: മേവൻ ഉപയോഗിച്ച് (ശുപാർശ ചെയ്യുന്നു)

#### 1. കാൽക്കുലേറ്റർ സർവർ ആരംഭിക്കുക

ടെർമിനൽ തുറന്ന് സർവർ ഡയറക്ടറിയിലേക്ക് പോകുക:

```bash
cd calculator-server
mvn clean package

```

നിങ്ങൾക്ക് ഇങ്ങനെ ഔട്ട്പുട്ട് കാണാം:
```

```

#### 2. കാൽക്കുലേറ്റർ ക്ലയന്റ് പ്രവർത്തിപ്പിക്കുക

പുതിയ **ടെർമിനൽ** തുറന്ന് ക്ലയന്റ് ഡയറക്ടറിയിലേക്ക് പോകുക:

```bash
cd calculator-client
mvn clean package
mvn spring-boot:run
```

ക്ലയന്റ് സർവറുമായി കണക്ട് ചെയ്ത് കാൽക്കുലേഷൻ നടത്തുകയും സ്ട്രീമിംഗ് ഫലങ്ങൾ പ്രദർശിപ്പിക്കുകയും ചെയ്യും.

## സർവർ മാനുവലായി ടെസ്റ്റ് ചെയ്യൽ

വെബ് ബ്രൗസർ അല്ലെങ്കിൽ കർൾ ഉപയോഗിച്ച് സർവർ ടെസ്റ്റ് ചെയ്യാം:

### വെബ് ബ്രൗസർ ഉപയോഗിച്ച്:
വിസിറ്റ് ചെയ്യുക: `http://localhost:8080/calculate?a=10&b=5&op=add`

### കർൾ ഉപയോഗിച്ച്:
```bash
curl "http://localhost:8080/calculate?a=10&b=5&op=add" -H "Accept: text/event-stream"
```

## പ്രതീക്ഷിക്കുന്ന ഔട്ട്പുട്ട്

ക്ലയന്റ് പ്രവർത്തിപ്പിക്കുമ്പോൾ, നിങ്ങൾക്ക് ഇങ്ങനെ സ്ട്രീമിംഗ് ഔട്ട്പുട്ട് കാണാം:

```
event:info
data:Calculating: 7.0 mul 5.0

event:result
data:35.0
```

## പിന്തുണയുള്ള ഓപ്പറേഷനുകൾ

- `add` - കൂട്ടിച്ചേർക്കൽ (a + b)
- `sub` - കുറയ്ക്കൽ (a - b)
- `mul` - ഗുണനം (a * b)
- `div` - വിഭജനം (a / b, b = 0 ആണെങ്കിൽ NaN റിട്ടേൺ ചെയ്യും)

## API റഫറൻസ്

### GET /calculate

**പാരാമീറ്ററുകൾ:**
- `a` (ആവശ്യമാണ്): ആദ്യ സംഖ്യ (ഡബിൾ)
- `b` (ആവശ്യമാണ്): രണ്ടാം സംഖ്യ (ഡബിൾ)
- `op` (ആവശ്യമാണ്): ഓപ്പറേഷൻ (`add`, `sub`, `mul`, `div`)

**പ്രതികരണം:**
- Content-Type: `text/event-stream`
- കാൽക്കുലേഷൻ പുരോഗതിയും ഫലവും Server-Sent Events ആയി റിട്ടേൺ ചെയ്യുന്നു

**ഉദാഹരണ അഭ്യർത്ഥന:**
```
GET /calculate?a=7&b=5&op=mul HTTP/1.1
Host: localhost:8080
Accept: text/event-stream
```

**ഉദാഹരണ പ്രതികരണം:**
```
event: info
data: Calculating: 7.0 mul 5.0

event: result
data: 35.0
```

## പ്രശ്നപരിഹാരം

### സാധാരണ പ്രശ്നങ്ങൾ

1. **പോർട്ട് 8080 ഇതിനകം ഉപയോഗത്തിലാണ്**
   - 8080 പോർട്ട് ഉപയോഗിക്കുന്ന മറ്റ് ആപ്ലിക്കേഷനുകൾ നിർത്തുക
   - അല്ലെങ്കിൽ `calculator-server/src/main/resources/application.yml` ൽ സർവർ പോർട്ട് മാറ്റുക

2. **കണക്ഷൻ നിരസിച്ചു**
   - ക്ലയന്റ് ആരംഭിക്കുന്നതിന് മുമ്പ് സർവർ പ്രവർത്തിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക
   - സർവർ 8080 പോർട്ടിൽ വിജയകരമായി ആരംഭിച്ചതായി പരിശോധിക്കുക

3. **പാരാമീറ്റർ നാമ പ്രശ്നങ്ങൾ**
   - ഈ പ്രോജക്ടിൽ മേവൻ കോമ്പൈലർ കോൺഫിഗറേഷൻ `-parameters` ഫ്ലാഗ് ഉൾപ്പെടുത്തിയിട്ടുണ്ട്
   - പാരാമീറ്റർ ബൈൻഡിംഗ് പ്രശ്നങ്ങൾ ഉണ്ടെങ്കിൽ, ഈ കോൺഫിഗറേഷൻ ഉപയോഗിച്ച് പ്രോജക്ട് ബിൽഡ് ചെയ്തിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക

### ആപ്ലിക്കേഷനുകൾ നിർത്തൽ

- ഓരോ ആപ്ലിക്കേഷനും പ്രവർത്തിക്കുന്ന ടെർമിനലിൽ `Ctrl+C` അമർത്തുക
- അല്ലെങ്കിൽ ബാക്ക്ഗ്രൗണ്ട് പ്രോസസ്സ് ആയി പ്രവർത്തിക്കുന്നെങ്കിൽ `mvn spring-boot:stop` ഉപയോഗിക്കുക
