# ഈ സാമ്പിൾ പ്രവർത്തിപ്പിക്കൽ

## -1- ആശ്രിതങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്യുക

```bash
dotnet restore
```

## -3- സാമ്പിൾ പ്രവർത്തിപ്പിക്കുക


```bash
dotnet run
```

## -4- സാമ്പിൾ പരിശോധന

ഒരു ടെർമിനലിൽ സെർവർ പ്രവർത്തിക്കുന്നതിനിടെ, മറ്റൊരു ടെർമിനൽ തുറന്ന് താഴെ കാണുന്ന കമാൻഡ് പ്രവർത്തിപ്പിക്കുക:

```bash
npx @modelcontextprotocol/inspector dotnet run
```

ഇത് ഒരു വെബ് സെർവർ ആരംഭിക്കും, ഒരു ദൃശ്യ ഇന്റർഫേസ് നൽകുന്നു, ഇത് ഉപയോഗിച്ച് നിങ്ങൾക്ക് സാമ്പിൾ പരിശോധിക്കാം.

സെർവർ കണക്റ്റ് ആയ ശേഷം:

- ടൂളുകൾ ലിസ്റ്റ് ചെയ്യാൻ ശ്രമിക്കുക, `add` പ്രവർത്തിപ്പിക്കുക, 2, 4 എന്ന ആർഗ്യുമെന്റുകൾ നൽകി, ഫലത്തിൽ 6 കാണണം.
- resources-ലും resource template-ലും "greeting" വിളിക്കുക, ഒരു പേര് ടൈപ്പ് ചെയ്യുക, നിങ്ങൾ നൽകിയ പേരോടെ ഒരു അഭിവാദ്യം കാണണം.

### CLI മോഡിൽ പരിശോധന

നിങ്ങൾക്ക് നേരിട്ട് CLI മോഡിൽ ഇത് ആരംഭിക്കാം താഴെ കാണുന്ന കമാൻഡ് പ്രവർത്തിപ്പിച്ച്:

```bash
npx @modelcontextprotocol/inspector --cli dotnet run --method tools/list
```

ഇത് സെർവറിൽ ലഭ്യമായ എല്ലാ ടൂളുകളും ലിസ്റ്റ് ചെയ്യും. നിങ്ങൾക്ക് താഴെ കാണുന്ന ഔട്ട്പുട്ട് കാണാം:

```text
{
  "tools": [
    {
      "name": "Add",
      "description": "Adds two numbers",
      "inputSchema": {
        "type": "object",
        "properties": {
          "a": {
            "type": "integer"
          },
          "b": {
            "type": "integer"
          }
        },
        "title": "Add",
        "description": "Adds two numbers",
        "required": [
          "a",
          "b"
        ]
      }
    }
  ]
}
```

ഒരു ടൂൾ വിളിക്കാൻ ടൈപ്പ് ചെയ്യുക:

```bash
npx @modelcontextprotocol/inspector --cli dotnet run --method tools/call --tool-name Add --tool-arg a=1 --tool-arg b=2
```

നിങ്ങൾക്ക് താഴെ കാണുന്ന ഔട്ട്പുട്ട് കാണാം:

```text
{
  "content": [
    {
      "type": "text",
      "text": "Sum 3"
    }
  ],
  "isError": false
}
```

> [!TIP]
> ബ്രൗസറിൽ നിന്ന് CLI മോഡിൽ ഇൻസ്പെക്ടർ പ്രവർത്തിപ്പിക്കുന്നത് സാധാരണയായി വളരെ വേഗമാണ്.
> ഇൻസ്പെക്ടറിനെക്കുറിച്ച് കൂടുതൽ വായിക്കാൻ [ഇവിടെ](https://github.com/modelcontextprotocol/inspector) ക്ലിക്ക് ചെയ്യുക.
