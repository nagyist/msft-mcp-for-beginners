# ഈ സാമ്പിൾ പ്രവർത്തിപ്പിക്കൽ

നിങ്ങൾക്ക് `uv` ഇൻസ്റ്റാൾ ചെയ്യാൻ ശുപാർശ ചെയ്യുന്നു, പക്ഷേ അത് നിർബന്ധമല്ല, [നിർദ്ദേശങ്ങൾ](https://docs.astral.sh/uv/#highlights) കാണുക

## -1- ആശ്രിതങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്യുക

```bash
npm install
```

## -3- സാമ്പിൾ പ്രവർത്തിപ്പിക്കുക


```bash
npm run build
```

## -4- സാമ്പിൾ പരിശോധന

ഒരു ടെർമിനലിൽ സർവർ പ്രവർത്തിക്കുന്നതിനിടെ, മറ്റൊരു ടെർമിനൽ തുറന്ന് താഴെ കാണുന്ന കമാൻഡ് പ്രവർത്തിപ്പിക്കുക:

```bash
npm run inspector
```

ഇത് ഒരു ദൃശ്യ ഇന്റർഫേസ് ഉള്ള വെബ് സർവർ ആരംഭിക്കും, ഇത് ഉപയോഗിച്ച് നിങ്ങൾക്ക് സാമ്പിൾ പരിശോധിക്കാം.

സർവർ കണക്ട് ചെയ്തശേഷം:

- ടൂളുകൾ ലിസ്റ്റ് ചെയ്യാൻ ശ്രമിക്കുക, പിന്നെ `add` ഓപ്പറേഷൻ 2, 4 എന്ന ആർഗ്യുമെന്റുകളോടെ പ്രവർത്തിപ്പിക്കുക, ഫലത്തിൽ 6 കാണണം.
- resources-ലും resource template-ലും "greeting" വിളിക്കുക, ഒരു പേര് ടൈപ്പ് ചെയ്യുക, നിങ്ങൾ നൽകിയ പേരോടെ ഒരു അഭിവാദ്യം കാണണം.

### CLI മോഡിൽ പരിശോധന

നിങ്ങൾ പ്രവർത്തിപ്പിച്ച ഇൻസ്പെക്ടർ യഥാർത്ഥത്തിൽ ഒരു Node.js ആപ്പ് ആണ്, `mcp dev` അതിന്റെ ഒരു റാപ്പർ ആണ്.

താഴെ കാണുന്ന കമാൻഡ് പ്രവർത്തിപ്പിച്ച് CLI മോഡിൽ നേരിട്ട് ഇത് ആരംഭിക്കാം:

```bash
npx @modelcontextprotocol/inspector --cli node ./build/index.js --method tools/list
```

ഇത് സർവറിൽ ലഭ്യമായ എല്ലാ ടൂളുകളും ലിസ്റ്റ് ചെയ്യും. നിങ്ങൾക്ക് താഴെ കാണുന്ന ഔട്ട്പുട്ട് കാണണം:

```text
{
  "tools": [
    {
      "name": "add",
      "description": "Add two numbers",
      "inputSchema": {
        "type": "object",
        "properties": {
          "a": {
            "title": "A",
            "type": "integer"
          },
          "b": {
            "title": "B",
            "type": "integer"
          }
        },
        "required": [
          "a",
          "b"
        ],
        "title": "addArguments"
      }
    }
  ]
}
```

ഒരു ടൂൾ വിളിക്കാൻ ടൈപ്പ് ചെയ്യുക:

```bash
nnpx @modelcontextprotocol/inspector --cli node ./build/index.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

നിങ്ങൾക്ക് താഴെ കാണുന്ന ഔട്ട്പുട്ട് കാണണം:

```text
{
  "content": [
    {
      "type": "text",
      "text": "3"
    }
  ],
  "isError": false
}
```

> [!TIP]
> ഇൻസ്പെക്ടർ ബ്രൗസറിൽ പ്രവർത്തിപ്പിക്കുന്നതിനേക്കാൾ CLI മോഡിൽ പ്രവർത്തിപ്പിക്കുന്നത് സാധാരണയായി വളരെ വേഗമാണ്.
> ഇൻസ്പെക്ടറിനെക്കുറിച്ച് കൂടുതൽ വായിക്കാൻ [ഇവിടെ](https://github.com/modelcontextprotocol/inspector) കാണുക.
