# കാൽക്കുലേറ്റർ LLM ക്ലയന്റ്

GitHub മോഡലുകൾ ഇന്റഗ്രേഷനോടുകൂടിയ MCP (Model Context Protocol) കാൽക്കുലേറ്റർ സർവീസുമായി ബന്ധിപ്പിക്കാൻ LangChain4j ഉപയോഗിക്കുന്ന വിധം കാണിക്കുന്ന ഒരു ജാവ ആപ്ലിക്കേഷൻ.

## മുൻകൂട്ടി ആവശ്യങ്ങൾ

- ജാവ 21 അല്ലെങ്കിൽ അതിനുമുകളിൽ
- മേവൻ 3.6+ (അല്ലെങ്കിൽ ഉൾപ്പെടുത്തിയ മേവൻ റാപ്പർ ഉപയോഗിക്കുക)
- GitHub മോഡലുകളിലേക്ക് ആക്‌സസ് ഉള്ള GitHub അക്കൗണ്ട്
- `http://localhost:8080`-ൽ പ്രവർത്തിക്കുന്ന MCP കാൽക്കുലേറ്റർ സർവീസ്

## GitHub ടോക്കൺ എടുക്കൽ

ഈ ആപ്ലിക്കേഷൻ GitHub മോഡലുകൾ ഉപയോഗിക്കുന്നു, അതിനായി GitHub വ്യക്തിഗത ആക്‌സസ് ടോക്കൺ ആവശ്യമാണ്. നിങ്ങളുടെ ടോക്കൺ എടുക്കാൻ താഴെ പറയുന്ന ഘട്ടങ്ങൾ പിന്തുടരുക:

### 1. GitHub മോഡലുകൾ ആക്‌സസ് ചെയ്യുക
1. [GitHub Models](https://github.com/marketplace/models) സന്ദർശിക്കുക
2. നിങ്ങളുടെ GitHub അക്കൗണ്ടിൽ സൈൻ ഇൻ ചെയ്യുക
3. GitHub മോഡലുകൾക്ക് ആക്‌സസ് അഭ്യർത്ഥിക്കുക, ഇതുവരെ ചെയ്തിട്ടില്ലെങ്കിൽ

### 2. വ്യക്തിഗത ആക്‌സസ് ടോക്കൺ സൃഷ്ടിക്കുക
1. [GitHub Settings → Developer settings → Personal access tokens → Tokens (classic)](https://github.com/settings/tokens) സന്ദർശിക്കുക
2. "Generate new token" → "Generate new token (classic)" ക്ലിക്ക് ചെയ്യുക
3. നിങ്ങളുടെ ടോക്കണിന് വിവരണാത്മകമായ പേര് നൽകുക (ഉദാ: "MCP Calculator Client")
4. ആവശ്യമുള്ളതുപോലെ കാലഹരണി സജ്ജമാക്കുക
5. താഴെ പറയുന്ന സ്കോപ്പുകൾ തിരഞ്ഞെടുക്കുക:
   - `repo` (പ്രൈവറ്റ് റിപോസിറ്ററികൾ ആക്‌സസ് ചെയ്യുമ്പോൾ)
   - `user:email`
6. "Generate token" ക്ലിക്ക് ചെയ്യുക
7. **പ്രധാനമാണ്**: ടോക്കൺ ഉടൻ കോപ്പി ചെയ്യുക - പിന്നീട് കാണാൻ സാധിക്കില്ല!

### 3. പരിസ്ഥിതി വ്യത്യാസം സജ്ജമാക്കുക

#### Windows (Command Prompt)ൽ:
```cmd
set GITHUB_TOKEN=your_github_token_here
```

#### Windows (PowerShell)ൽ:
```powershell
$env:GITHUB_TOKEN="your_github_token_here"
```

#### macOS/Linuxൽ:
```bash
export GITHUB_TOKEN=your_github_token_here
```

## സജ്ജീകരണവും ഇൻസ്റ്റാളേഷനും

1. **പ്രോജക്ട് ഡയറക്ടറിയിലേക്ക് ക്ലോൺ ചെയ്യുക അല്ലെങ്കിൽ നാവിഗേറ്റ് ചെയ്യുക**

2. **ഡിപ്പൻഡൻസികൾ ഇൻസ്റ്റാൾ ചെയ്യുക**:
   ```cmd
   mvnw clean install
   ```
   അല്ലെങ്കിൽ മേവൻ ഗ്ലോബലായി ഇൻസ്റ്റാൾ ചെയ്തിട്ടുണ്ടെങ്കിൽ:
   ```cmd
   mvn clean install
   ```

3. **പരിസ്ഥിതി വ്യത്യാസം സജ്ജമാക്കുക** ("Getting the GitHub Token" വിഭാഗം കാണുക)

4. **MCP കാൽക്കുലേറ്റർ സർവീസ് ആരംഭിക്കുക**:
   `http://localhost:8080/sse`-ൽ ചാപ്റ്റർ 1-ലെ MCP കാൽക്കുലേറ്റർ സർവീസ് പ്രവർത്തിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക. ക്ലയന്റ് ആരംഭിക്കുന്നതിന് മുമ്പ് ഇത് പ്രവർത്തിക്കണം.

## ആപ്ലിക്കേഷൻ പ്രവർത്തിപ്പിക്കൽ

```cmd
mvnw clean package
java -jar target\calculator-llm-client-0.0.1-SNAPSHOT.jar
```

## ആപ്ലിക്കേഷൻ ചെയ്യുന്നത്

ആപ്ലിക്കേഷൻ കാൽക്കുലേറ്റർ സർവീസുമായി മൂന്ന് പ്രധാന ഇടപെടലുകൾ കാണിക്കുന്നു:

1. **കൂട്ടിച്ചേർക്കൽ**: 24.5നും 17.3നും കൂട്ടിച്ചേർക്കൽ കണക്കാക്കുന്നു
2. **സ്ക്വയർ റൂട്ട്**: 144-ന്റെ സ്ക്വയർ റൂട്ട് കണക്കാക്കുന്നു
3. **സഹായം**: ലഭ്യമായ കാൽക്കുലേറ്റർ ഫംഗ്ഷനുകൾ കാണിക്കുന്നു

## പ്രതീക്ഷിക്കുന്ന ഔട്ട്പുട്ട്

സഫലമായി പ്രവർത്തിക്കുമ്പോൾ, താഴെപോലുള്ള ഔട്ട്പുട്ട് കാണാം:

```
The sum of 24.5 and 17.3 is 41.8.
The square root of 144 is 12.
The calculator service provides the following functions: add, subtract, multiply, divide, sqrt, power...
```

## പ്രശ്നപരിഹാരം

### സാധാരണ പ്രശ്നങ്ങൾ

1. **"GITHUB_TOKEN environment variable not set"**
   - `GITHUB_TOKEN` പരിസ്ഥിതി വ്യത്യാസം സജ്ജമാക്കിയിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക
   - വ്യത്യാസം സജ്ജമാക്കിയ ശേഷം ടെർമിനൽ/കമാൻഡ് പ്രോംപ്റ്റ് റീസ്റ്റാർട്ട് ചെയ്യുക

2. **"Connection refused to localhost:8080"**
   - MCP കാൽക്കുലേറ്റർ സർവീസ് പോർട്ട് 8080-ൽ പ്രവർത്തിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക
   - മറ്റൊരു സർവീസ് പോർട്ട് 8080 ഉപയോഗിക്കുന്നുണ്ടോ എന്ന് പരിശോധിക്കുക

3. **"Authentication failed"**
   - GitHub ടോക്കൺ സാധുവാണെന്നും ശരിയായ അനുമതികൾ ഉള്ളതാണെന്നും പരിശോധിക്കുക
   - GitHub മോഡലുകളിലേക്ക് ആക്‌സസ് ഉള്ളതാണോ എന്ന് പരിശോധിക്കുക

4. **മേവൻ ബിൽഡ് പിശകുകൾ**
   - ജാവ 21 അല്ലെങ്കിൽ അതിനുമുകളിൽ ഉപയോഗിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക: `java -version`
   - ബിൽഡ് ക്ലീൻ ചെയ്യാൻ ശ്രമിക്കുക: `mvnw clean`

### ഡീബഗ്ഗിംഗ്

ഡീബഗ് ലോഗിംഗ് സജ്ജമാക്കാൻ, പ്രവർത്തിപ്പിക്കുമ്പോൾ താഴെ പറയുന്ന JVM ആർഗുമെന്റ് ചേർക്കുക:
```cmd
java -Dlogging.level.dev.langchain4j=DEBUG -jar target\calculator-llm-client-0.0.1-SNAPSHOT.jar
```

## കോൺഫിഗറേഷൻ

ആപ്ലിക്കേഷൻ താഴെപറയുന്നവയ്ക്ക് സജ്ജീകരിച്ചിരിക്കുന്നു:
- GitHub മോഡലുകൾ `gpt-4.1-nano` മോഡലുമായി ഉപയോഗിക്കുക
- MCP സർവീസുമായി `http://localhost:8080/sse`-ൽ ബന്ധിപ്പിക്കുക
- അഭ്യർത്ഥനകൾക്ക് 60 സെക്കൻഡ് ടൈംഔട്ട് ഉപയോഗിക്കുക
- ഡീബഗ്ഗിംഗിനായി അഭ്യർത്ഥന/പ്രതികരണം ലോഗിംഗ് സജ്ജമാക്കുക

## ഡിപ്പൻഡൻസികൾ

ഈ പ്രോജക്ടിൽ ഉപയോഗിക്കുന്ന പ്രധാന ഡിപ്പൻഡൻസികൾ:
- **LangChain4j**: AI ഇന്റഗ്രേഷൻക്കും ടൂൾ മാനേജ്മെന്റിനും
- **LangChain4j MCP**: Model Context Protocol പിന്തുണയ്ക്കായി
- **LangChain4j GitHub Models**: GitHub മോഡലുകൾ ഇന്റഗ്രേഷനായി
- **Spring Boot**: ആപ്ലിക്കേഷൻ ഫ്രെയിംവർക്ക്, ഡിപ്പൻഡൻസി ഇൻജക്ഷനായി
