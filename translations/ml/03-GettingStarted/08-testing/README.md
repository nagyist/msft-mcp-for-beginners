## ടെസ്റ്റിംഗ് ആൻഡ് ഡീബഗ്ഗിംഗ്

നിങ്ങളുടെ MCP സെർവർ ടെസ്റ്റ് ചെയ്യാൻ തുടങ്ങുന്നതിന് മുമ്പ്, ലഭ്യമായ ഉപകരണങ്ങളും ഡീബഗ്ഗിംഗിനുള്ള മികച്ച പ്രാക്ടീസുകളും മനസ്സിലാക്കുന്നത് പ്രധാനമാണ്. ഫലപ്രദമായ ടെസ്റ്റിംഗ് നിങ്ങളുടെ സെർവർ പ്രതീക്ഷിച്ചതുപോലെ പ്രവർത്തിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുകയും പ്രശ്നങ്ങൾ വേഗത്തിൽ കണ്ടെത്തി പരിഹരിക്കാൻ സഹായിക്കുകയും ചെയ്യുന്നു. താഴെക്കാണുന്ന വിഭാഗം നിങ്ങളുടെ MCP നടപ്പാക്കലിന്റെ സാധുത പരിശോധിക്കുന്നതിനുള്ള ശുപാർശ ചെയ്ത സമീപനങ്ങൾ വിശദീകരിക്കുന്നു.

## അവലോകനം

ഈ പാഠം ശരിയായ ടെസ്റ്റിംഗ് സമീപനം തിരഞ്ഞെടുക്കുന്നതും ഏറ്റവും ഫലപ്രദമായ ടെസ്റ്റിംഗ് ഉപകരണം എന്താണെന്നും ഉൾക്കൊള്ളുന്നു.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ പാഠം അവസാനിക്കുമ്പോൾ, നിങ്ങൾക്ക് കഴിയും:

- ടെസ്റ്റിംഗിനുള്ള വിവിധ സമീപനങ്ങൾ വിവരണം ചെയ്യുക.
- നിങ്ങളുടെ കോഡ് ഫലപ്രദമായി ടെസ്റ്റ് ചെയ്യാൻ വ്യത്യസ്ത ഉപകരണങ്ങൾ ഉപയോഗിക്കുക.

## MCP സെർവറുകൾ ടെസ്റ്റ് ചെയ്യൽ

MCP നിങ്ങളുടെ സെർവറുകൾ ടെസ്റ്റ് ചെയ്യാനും ഡീബഗ് ചെയ്യാനും സഹായിക്കുന്ന ഉപകരണങ്ങൾ നൽകുന്നു:

- **MCP ഇൻസ്പെക്ടർ**: CLI ഉപകരണമായും ദൃശ്യ ഉപകരണമായും പ്രവർത്തിപ്പിക്കാവുന്ന കമാൻഡ് ലൈൻ ടൂൾ.
- **മാനുവൽ ടെസ്റ്റിംഗ്**: curl പോലുള്ള ഒരു ഉപകരണം ഉപയോഗിച്ച് വെബ് അഭ്യർത്ഥനകൾ നടത്താം, എന്നാൽ HTTP പ്രവർത്തിപ്പിക്കാൻ കഴിയുന്ന ഏതെങ്കിലും ഉപകരണം ഉപയോഗിക്കാം.
- **യൂണിറ്റ് ടെസ്റ്റിംഗ്**: സെർവർ, ക്ലയന്റ് എന്നിവയുടെ ഫീച്ചറുകൾ ടെസ്റ്റ് ചെയ്യാൻ നിങ്ങൾ ഇഷ്ടപ്പെടുന്ന ടെസ്റ്റിംഗ് ഫ്രെയിംവർക്ക് ഉപയോഗിക്കാം.

### MCP ഇൻസ്പെക്ടർ ഉപയോഗിക്കൽ

ഈ ഉപകരണത്തിന്റെ ഉപയോഗം മുമ്പത്തെ പാഠങ്ങളിൽ വിശദീകരിച്ചിട്ടുണ്ട്, എന്നാൽ ഇതിനെ കുറിച്ച് ഉയർന്ന തലത്തിൽ കുറച്ച് സംസാരിക്കാം. ഇത് Node.js-ൽ നിർമ്മിച്ച ഒരു ഉപകരണമാണ്, നിങ്ങൾ `npx` എക്സിക്യൂട്ടബിൾ വിളിച്ച് ഇത് ഉപയോഗിക്കാം, ഇത് താൽക്കാലികമായി ടൂൾ ഡൗൺലോഡ് ചെയ്ത് ഇൻസ്റ്റാൾ ചെയ്ത് നിങ്ങളുടെ അഭ്യർത്ഥന പൂർത്തിയാക്കിയ ശേഷം സ്വയം ക്ലീൻ അപ്പ് ചെയ്യും.

[MCP ഇൻസ്പെക്ടർ](https://github.com/modelcontextprotocol/inspector) നിങ്ങളെ സഹായിക്കുന്നു:

- **സെർവർ ശേഷികൾ കണ്ടെത്തുക**: ലഭ്യമായ റിസോഴ്‌സുകൾ, ഉപകരണങ്ങൾ, പ്രോംപ്റ്റുകൾ സ്വയം കണ്ടെത്തുക
- **ടൂൾ എക്സിക്യൂഷൻ ടെസ്റ്റ് ചെയ്യുക**: വ്യത്യസ്ത പാരാമീറ്ററുകൾ പരീക്ഷിച്ച് റിയൽ-ടൈം പ്രതികരണങ്ങൾ കാണുക
- **സെർവർ മെറ്റാഡേറ്റ കാണുക**: സെർവർ വിവരങ്ങൾ, സ്കീമകൾ, കോൺഫിഗറേഷനുകൾ പരിശോധിക്കുക

ടൂളിന്റെ സാധാരണ പ്രവർത്തനം ഇങ്ങനെ കാണാം:

```bash
npx @modelcontextprotocol/inspector node build/index.js
```

മുകളിൽ കാണിച്ച കമാൻഡ് ഒരു MCP ആരംഭിക്കുകയും അതിന്റെ ദൃശ്യ ഇന്റർഫേസ് ആരംഭിക്കുകയും നിങ്ങളുടെ ബ്രൗസറിൽ ഒരു ലോക്കൽ വെബ് ഇന്റർഫേസ് തുറക്കുകയും ചെയ്യുന്നു. രജിസ്റ്റർ ചെയ്ത MCP സെർവറുകൾ, അവയുടെ ലഭ്യമായ ഉപകരണങ്ങൾ, റിസോഴ്‌സുകൾ, പ്രോംപ്റ്റുകൾ ഡാഷ്ബോർഡിൽ കാണാൻ കഴിയും. ഇന്റർഫേസ് ഉപകരണങ്ങളുടെ പ്രവർത്തനം ഇന്ററാക്ടീവ് ആയി ടെസ്റ്റ് ചെയ്യാനും, സെർവർ മെറ്റാഡേറ്റ പരിശോധിക്കാനും, റിയൽ-ടൈം പ്രതികരണങ്ങൾ കാണാനും സഹായിക്കുന്നു, ഇത് MCP സെർവർ നടപ്പാക്കലുകൾ സാധുത പരിശോധിക്കാനും ഡീബഗ് ചെയ്യാനും എളുപ്പമാക്കുന്നു.

ഇത് ഇങ്ങനെ കാണാം: ![Inspector](../../../../translated_images/ml/connect.141db0b2bd05f096.webp)

നിങ്ങൾക്ക് ഈ ടൂൾ CLI മോഡിൽ പ്രവർത്തിപ്പിക്കാനും കഴിയും, അപ്പോൾ `--cli` ആട്രിബ്യൂട്ട് ചേർക്കണം. സെർവറിലെ എല്ലാ ടൂളുകളും പട്ടികപ്പെടുത്തുന്ന "CLI" മോഡിൽ ടൂൾ പ്രവർത്തിപ്പിക്കുന്ന ഉദാഹരണം ഇതാ:

```sh
npx @modelcontextprotocol/inspector --cli node build/index.js --method tools/list
```

### മാനുവൽ ടെസ്റ്റിംഗ്

സെർവർ ശേഷികൾ പരിശോധിക്കാൻ ഇൻസ്പെക്ടർ ടൂൾ പ്രവർത്തിപ്പിക്കുന്നതിനു പുറമേ, HTTP ഉപയോഗിക്കാൻ കഴിയുന്ന ക്ലയന്റ് പ്രവർത്തിപ്പിക്കുന്ന മറ്റൊരു സമാന സമീപനമാണ് curl പോലുള്ള ഉപകരണം ഉപയോഗിക്കുക.

curl ഉപയോഗിച്ച് MCP സെർവറുകൾ നേരിട്ട് HTTP അഭ്യർത്ഥനകൾ വഴി ടെസ്റ്റ് ചെയ്യാം:

```bash
# ഉദാഹരണം: ടെസ്റ്റ് സെർവർ മെറ്റാഡേറ്റ
curl http://localhost:3000/v1/metadata

# ഉദാഹരണം: ഒരു ടൂൾ പ്രവർത്തിപ്പിക്കുക
curl -X POST http://localhost:3000/v1/tools/execute \
  -H "Content-Type: application/json" \
  -d '{"name": "calculator", "parameters": {"expression": "2+2"}}'
```

മുകളിൽ കാണുന്ന curl ഉപയോഗത്തിൽ നിന്ന് നിങ്ങൾക്ക് കാണാം, ടൂൾയുടെ പേര്, പാരാമീറ്ററുകൾ അടങ്ങിയ പേലോഡ് ഉപയോഗിച്ച് POST അഭ്യർത്ഥനയിലൂടെ ടൂൾ പ്രവർത്തിപ്പിക്കുന്നു. നിങ്ങൾക്ക് ഏറ്റവും അനുയോജ്യമായ സമീപനം തിരഞ്ഞെടുക്കുക. CLI ടൂളുകൾ സാധാരണയായി വേഗത്തിൽ ഉപയോഗിക്കാനാകും, സ്ക്രിപ്റ്റ് ചെയ്യാനും എളുപ്പമാണ്, ഇത് CI/CD പരിസ്ഥിതിയിൽ ഉപകാരപ്രദമാണ്.

### യൂണിറ്റ് ടെസ്റ്റിംഗ്

നിങ്ങളുടെ ടൂളുകൾക്കും റിസോഴ്‌സുകൾക്കും യൂണിറ്റ് ടെസ്റ്റുകൾ സൃഷ്ടിച്ച് അവ പ്രതീക്ഷിച്ചതുപോലെ പ്രവർത്തിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക. ചില ഉദാഹരണ ടെസ്റ്റിംഗ് കോഡ് ഇതാ.

```python
import pytest

from mcp.server.fastmcp import FastMCP
from mcp.shared.memory import (
    create_connected_server_and_client_session as create_session,
)

# അസിങ്ക്രൺ ടെസ്റ്റുകൾക്കായി മുഴുവൻ മോഡ്യൂൾ അടയാളപ്പെടുത്തുക
pytestmark = pytest.mark.anyio


async def test_list_tools_cursor_parameter():
    """Test that the cursor parameter is accepted for list_tools.

    Note: FastMCP doesn't currently implement pagination, so this test
    only verifies that the cursor parameter is accepted by the client.
    """

 server = FastMCP("test")

    # രണ്ട് ടെസ്റ്റ് ഉപകരണങ്ങൾ സൃഷ്ടിക്കുക
    @server.tool(name="test_tool_1")
    async def test_tool_1() -> str:
        """First test tool"""
        return "Result 1"

    @server.tool(name="test_tool_2")
    async def test_tool_2() -> str:
        """Second test tool"""
        return "Result 2"

    async with create_session(server._mcp_server) as client_session:
        # കേഴ്സർ പാരാമീറ്റർ ഇല്ലാതെ ടെസ്റ്റ് ചെയ്യുക (വിട്ടുവെച്ചിരിക്കുന്നു)
        result1 = await client_session.list_tools()
        assert len(result1.tools) == 2

        # കേഴ്സർ=None ആയി ടെസ്റ്റ് ചെയ്യുക
        result2 = await client_session.list_tools(cursor=None)
        assert len(result2.tools) == 2

        # കേഴ്സർ സ്ട്രിംഗ് ആയി ടെസ്റ്റ് ചെയ്യുക
        result3 = await client_session.list_tools(cursor="some_cursor_value")
        assert len(result3.tools) == 2

        # ശൂന്യമായ സ്ട്രിംഗ് കേഴ്സർ ഉപയോഗിച്ച് ടെസ്റ്റ് ചെയ്യുക
        result4 = await client_session.list_tools(cursor="")
        assert len(result4.tools) == 2
    
```

മുകളിൽ കാണുന്ന കോഡ് ചെയ്യുന്നത്:

- pytest ഫ്രെയിംവർക്ക് ഉപയോഗിക്കുന്നു, ഇത് ഫംഗ്ഷനുകളായി ടെസ്റ്റുകൾ സൃഷ്ടിക്കാനും assert സ്റ്റേറ്റ്മെന്റുകൾ ഉപയോഗിക്കാനും അനുവദിക്കുന്നു.
- രണ്ട് വ്യത്യസ്ത ടൂളുകളുള്ള MCP സെർവർ സൃഷ്ടിക്കുന്നു.
- ചില നിബന്ധനകൾ പാലിക്കപ്പെടുന്നുണ്ടെന്ന് പരിശോധിക്കാൻ `assert` സ്റ്റേറ്റ്മെന്റ് ഉപയോഗിക്കുന്നു.

[പൂർണ്ണ ഫയൽ ഇവിടെ കാണുക](https://github.com/modelcontextprotocol/python-sdk/blob/main/tests/client/test_list_methods_cursor.py)

മുകളിൽ കാണിച്ച ഫയൽ ഉപയോഗിച്ച്, നിങ്ങളുടെ സ്വന്തം സെർവർ ടെസ്റ്റ് ചെയ്ത് ശേഷികൾ ശരിയായി സൃഷ്ടിക്കപ്പെട്ടിട്ടുണ്ടെന്ന് ഉറപ്പാക്കാം.

എല്ലാ പ്രധാന SDK-കൾക്കും സമാനമായ ടെസ്റ്റിംഗ് വിഭാഗങ്ങൾ ഉണ്ട്, അതിനാൽ നിങ്ങൾ തിരഞ്ഞെടുക്കുന്ന റൺടൈമിനനുസരിച്ച് ക്രമീകരിക്കാം.
