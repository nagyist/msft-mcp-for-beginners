# MCP stdio Server - Python പരിഹാരം

> **⚠️ പ്രധാനപ്പെട്ടത്**: MCP Specification 2025-06-18 പ്രകാരം ശുപാർശ ചെയ്ത **stdio transport** ഉപയോഗിക്കാൻ ഈ പരിഹാരം പുതുക്കിയിട്ടുണ്ട്. മുൾ SSE transport ഇനി ഉപയോഗിക്കരുത്.

## അവലോകനം

ഈ Python പരിഹാരം നിലവിലുള്ള stdio transport ഉപയോഗിച്ച് MCP സെർവർ എങ്ങനെ നിർമ്മിക്കാമെന്ന് കാണിക്കുന്നു. stdio transport ലളിതവും, കൂടുതൽ സുരക്ഷിതവുമാണ്, പഴയ SSE രീതിയേക്കാൾ മികച്ച പ്രകടനം നൽകുന്നു.

## മുൻകൂട്ടി ആവശ്യങ്ങൾ

- Python 3.8 അല്ലെങ്കിൽ അതിനുശേഷം
- പാക്കേജ് മാനേജ്മെന്റിനായി `uv` ഇൻസ്റ്റാൾ ചെയ്യാൻ ശുപാർശ ചെയ്യുന്നു, [നിർദ്ദേശങ്ങൾ](https://docs.astral.sh/uv/#highlights) കാണുക

## സജ്ജീകരണ നിർദ്ദേശങ്ങൾ

### ഘട്ടം 1: ഒരു virtual environment സൃഷ്ടിക്കുക

```bash
python -m venv venv
```

### ഘട്ടം 2: virtual environment സജീവമാക്കുക

**Windows:**
```bash
venv\Scripts\activate
```

**macOS/Linux:**
```bash
source venv/bin/activate
```

### ഘട്ടം 3: ആശ്രിതങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്യുക

```bash
pip install mcp
```

## സെർവർ പ്രവർത്തിപ്പിക്കൽ

stdio സെർവർ പഴയ SSE സെർവറിനേക്കാൾ വ്യത്യസ്തമായി പ്രവർത്തിക്കുന്നു. ഒരു വെബ് സെർവർ ആരംഭിക്കുന്നതിന് പകരം, stdin/stdout വഴി ആശയവിനിമയം നടത്തുന്നു:

```bash
python server.py
```

**പ്രധാനപ്പെട്ടത്**: സെർവർ ഹാംഗായിരിക്കുന്നതുപോലെയാണ് തോന്നുക - ഇത് സാധാരണമാണ്! stdin-ൽ നിന്ന് JSON-RPC സന്ദേശങ്ങൾക്കായി കാത്തിരിക്കുന്നു.

## സെർവർ പരിശോധന

### രീതി 1: MCP ഇൻസ്പെക്ടർ ഉപയോഗിച്ച് (ശുപാർശ ചെയ്യുന്നു)

```bash
npx @modelcontextprotocol/inspector python server.py
```

ഇത് ചെയ്യും:
1. നിങ്ങളുടെ സെർവർ subprocess ആയി ആരംഭിക്കും
2. പരിശോധനയ്ക്കായി ഒരു വെബ് ഇന്റർഫേസ് തുറക്കും
3. എല്ലാ സെർവർ ടൂളുകളും ഇന്ററാക്ടീവ് ആയി പരീക്ഷിക്കാൻ അനുവദിക്കും

### രീതി 2: നേരിട്ട് JSON-RPC പരിശോധന

നേരിട്ട് JSON-RPC സന്ദേശങ്ങൾ അയച്ച് നിങ്ങൾക്ക് പരീക്ഷിക്കാം:

1. സെർവർ ആരംഭിക്കുക: `python server.py`
2. JSON-RPC സന്ദേശം അയയ്ക്കുക (ഉദാഹരണം):

```json
{"jsonrpc": "2.0", "id": 1, "method": "tools/list"}
```

3. സെർവർ ലഭ്യമായ ടൂളുകൾക്ക് പ്രതികരിക്കും

### ലഭ്യമായ ടൂളുകൾ

സെർവർ ഈ ടൂളുകൾ നൽകുന്നു:

- **add(a, b)**: രണ്ട് സംഖ്യകൾ കൂട്ടുക
- **multiply(a, b)**: രണ്ട് സംഖ്യകൾ ഗുണിക്കുക  
- **get_greeting(name)**: വ്യക്തിഗതമായ ഒരു അഭിവാദ്യം സൃഷ്ടിക്കുക
- **get_server_info()**: സെർവറിന്റെ വിവരങ്ങൾ നേടുക

### Claude Desktop ഉപയോഗിച്ച് പരിശോധന

ഈ സെർവർ Claude Desktop-നൊപ്പം ഉപയോഗിക്കാൻ, നിങ്ങളുടെ `claude_desktop_config.json`-ൽ ഈ കോൺഫിഗറേഷൻ ചേർക്കുക:

```json
{
  "mcpServers": {
    "example-stdio-server": {
      "command": "python",
      "args": ["path/to/server.py"]
    }
  }
}
```

## SSE-യുമായി പ്രധാന വ്യത്യാസങ്ങൾ

**stdio transport (നിലവിലുള്ളത്):**
- ✅ ലളിതമായ സജ്ജീകരണം - വെബ് സെർവർ ആവശ്യമില്ല
- ✅ മെച്ചപ്പെട്ട സുരക്ഷ - HTTP എൻഡ്‌പോയിന്റുകൾ ഇല്ല
- ✅ subprocess അടിസ്ഥാനത്തിലുള്ള ആശയവിനിമയം
- ✅ stdin/stdout വഴി JSON-RPC
- ✅ മെച്ചപ്പെട്ട പ്രകടനം

**SSE transport (പഴയത്):**
- ❌ HTTP സെർവർ സജ്ജീകരണം ആവശ്യമായിരുന്നു
- ❌ വെബ് ഫ്രെയിംവർക്ക് (Starlette/FastAPI) ആവശ്യമായിരുന്നു
- ❌ കൂടുതൽ സങ്കീർണ്ണമായ റൂട്ടിംഗ്, സെഷൻ മാനേജ്മെന്റ്
- ❌ അധിക സുരക്ഷാ പരിഗണനകൾ
- ❌ MCP 2025-06-18-ൽ ഇനി ഉപയോഗിക്കരുതെന്ന് പ്രഖ്യാപിച്ചു

## ഡീബഗ്ഗിംഗ് ടിപ്പുകൾ

- ലോഗിംഗിന് `stderr` ഉപയോഗിക്കുക (`stdout` അല്ല)
- ദൃശ്യ ഡീബഗ്ഗിംഗിന് ഇൻസ്പെക്ടർ ഉപയോഗിച്ച് പരീക്ഷിക്കുക
- എല്ലാ JSON സന്ദേശങ്ങളും newline-ൽ വേർതിരിച്ചിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക
- സെർവർ പിശകുകൾ കൂടാതെ ആരംഭിക്കുന്നുവെന്ന് പരിശോധിക്കുക

ഈ പരിഹാരം നിലവിലുള്ള MCP സ്പെസിഫിക്കേഷൻ അനുസരിച്ച് stdio transport നടപ്പിലാക്കലിനുള്ള മികച്ച രീതികൾ കാണിക്കുന്നു.
