# MCP stdio Server - .NET പരിഹാരം

> **⚠️ പ്രധാനപ്പെട്ടത്**: MCP സ്പെസിഫിക്കേഷൻ 2025-06-18 പ്രകാരം ശുപാർശ ചെയ്ത **stdio ട്രാൻസ്പോർട്ട്** ഉപയോഗിക്കാൻ ഈ പരിഹാരം അപ്ഡേറ്റ് ചെയ്തിരിക്കുന്നു. മുൾ SSE ട്രാൻസ്പോർട്ട് ഇനി ഉപയോഗിക്കരുത്.

## അവലോകനം

ഈ .NET പരിഹാരം നിലവിലുള്ള stdio ട്രാൻസ്പോർട്ട് ഉപയോഗിച്ച് MCP സെർവർ എങ്ങനെ നിർമ്മിക്കാമെന്ന് കാണിക്കുന്നു. stdio ട്രാൻസ്പോർട്ട് ലളിതവും, കൂടുതൽ സുരക്ഷിതവുമാണ്, പഴയ SSE രീതിയേക്കാൾ മികച്ച പ്രകടനം നൽകുന്നു.

## മുൻകൂട്ടി ആവശ്യങ്ങൾ

- .NET 9.0 SDK അല്ലെങ്കിൽ അതിനുശേഷം
- .NET ഡിപ്പൻഡൻസി ഇൻജക്ഷൻ അടിസ്ഥാന അറിവ്

## സജ്ജീകരണ നിർദ്ദേശങ്ങൾ

### ഘട്ടം 1: ഡിപ്പൻഡൻസികൾ പുനഃസ്ഥാപിക്കുക

```bash
dotnet restore
```

### ഘട്ടം 2: പ്രോജക്ട് നിർമ്മിക്കുക

```bash
dotnet build
```

## സെർവർ പ്രവർത്തിപ്പിക്കൽ

stdio സെർവർ പഴയ HTTP അടിസ്ഥാനത്തിലുള്ള സെർവറിനേക്കാൾ വ്യത്യസ്തമായി പ്രവർത്തിക്കുന്നു. വെബ് സെർവർ ആരംഭിക്കുന്നതിന് പകരം, ഇത് stdin/stdout വഴി ആശയവിനിമയം നടത്തുന്നു:

```bash
dotnet run
```

**പ്രധാനപ്പെട്ടത്**: സെർവർ ഹാംഗായിരിക്കുന്നതുപോലെയാണ് കാണപ്പെടുക - ഇത് സാധാരണമാണ്! ഇത് stdin-ൽ നിന്നുള്ള JSON-RPC സന്ദേശങ്ങൾക്കായി കാത്തിരിക്കുന്നു.

## സെർവർ പരിശോധന

### രീതി 1: MCP ഇൻസ്പെക്ടർ ഉപയോഗിച്ച് (ശുപാർശ ചെയ്യുന്നു)

```bash
npx @modelcontextprotocol/inspector dotnet run
```

ഇത് ചെയ്യും:
1. നിങ്ങളുടെ സെർവർ subprocess ആയി ആരംഭിക്കും
2. പരിശോധനയ്ക്കായി വെബ് ഇന്റർഫേസ് തുറക്കും
3. എല്ലാ സെർവർ ടൂളുകളും ഇന്ററാക്ടീവ് ആയി പരീക്ഷിക്കാൻ അനുവദിക്കും

### രീതി 2: നേരിട്ട് കമാൻഡ് ലൈൻ പരിശോധന

ഇൻസ്പെക്ടർ നേരിട്ട് ആരംഭിച്ച് നിങ്ങൾക്ക് പരീക്ഷിക്കാം:

```bash
npx @modelcontextprotocol/inspector dotnet run --project .
```

### ലഭ്യമായ ടൂളുകൾ

സെർവർ ഈ ടൂളുകൾ നൽകുന്നു:

- **AddNumbers(a, b)**: രണ്ട് സംഖ്യകൾ കൂട്ടുക
- **MultiplyNumbers(a, b)**: രണ്ട് സംഖ്യകൾ ഗുണിക്കുക  
- **GetGreeting(name)**: വ്യക്തിഗതമായ ഒരു അഭിവാദ്യം സൃഷ്ടിക്കുക
- **GetServerInfo()**: സെർവറിന്റെ വിവരങ്ങൾ നേടുക

### Claude Desktop ഉപയോഗിച്ച് പരിശോധന

ഈ സെർവർ Claude Desktop-നൊപ്പം ഉപയോഗിക്കാൻ, നിങ്ങളുടെ `claude_desktop_config.json`-ൽ ഈ കോൺഫിഗറേഷൻ ചേർക്കുക:

```json
{
  "mcpServers": {
    "example-stdio-server": {
      "command": "dotnet",
      "args": ["run", "--project", "path/to/server.csproj"]
    }
  }
}
```

## HTTP/SSE-യുമായി പ്രധാന വ്യത്യാസങ്ങൾ

**stdio ട്രാൻസ്പോർട്ട് (നിലവിലുള്ളത്):**
- ✅ ലളിതമായ സജ്ജീകരണം - വെബ് സെർവർ ആവശ്യമില്ല
- ✅ മെച്ചപ്പെട്ട സുരക്ഷ - HTTP എൻഡ്‌പോയിന്റുകൾ ഇല്ല
- ✅ `WebApplication.CreateBuilder()`-ന്റെ പകരം `Host.CreateApplicationBuilder()` ഉപയോഗിക്കുന്നു
- ✅ `WithHttpTransport()`-ന്റെ പകരം `WithStdioTransport()` ഉപയോഗിക്കുന്നു
- ✅ വെബ് അപ്ലിക്കേഷനിന് പകരം കൺസോൾ അപ്ലിക്കേഷൻ
- ✅ മെച്ചപ്പെട്ട പ്രകടനം

**HTTP/SSE ട്രാൻസ്പോർട്ട് (പഴയതും ഉപേക്ഷിച്ചതും):**
- ❌ ASP.NET കോർ വെബ് സെർവർ ആവശ്യമായിരുന്നു
- ❌ `app.MapMcp()` റൂട്ടിംഗ് സജ്ജീകരണം ആവശ്യമായിരുന്നു
- ❌ കൂടുതൽ സങ്കീർണ്ണമായ കോൺഫിഗറേഷൻ, ഡിപ്പൻഡൻസികൾ
- ❌ അധിക സുരക്ഷാ പരിഗണനകൾ
- ❌ MCP 2025-06-18-ൽ ഇത് ഉപേക്ഷിക്കപ്പെട്ടു

## വികസന സവിശേഷതകൾ

- **ഡിപ്പൻഡൻസി ഇൻജക്ഷൻ**: സർവീസുകളും ലോഗിംഗും പൂർണ്ണ DI പിന്തുണ
- **സ്ട്രക്ചർഡ് ലോഗിംഗ്**: `ILogger<T>` ഉപയോഗിച്ച് stderr-ലേക്ക് ശരിയായ ലോഗിംഗ്
- **ടൂൾ ആട്രിബ്യൂട്ടുകൾ**: `[McpServerTool]` ആട്രിബ്യൂട്ടുകൾ ഉപയോഗിച്ച് ക്ലീൻ ടൂൾ നിർവചനം
- **അസിങ്ക് പിന്തുണ**: എല്ലാ ടൂളുകളും അസിങ്ക് പ്രവർത്തനങ്ങൾ പിന്തുണയ്ക്കുന്നു
- **പിശക് കൈകാര്യം**: സുന്ദരമായ പിശക് കൈകാര്യം ചെയ്യലും ലോഗിംഗും

## വികസന ടിപ്പുകൾ

- ലോഗിംഗിന് `ILogger` ഉപയോഗിക്കുക (stdout-ലേക്ക് നേരിട്ട് എഴുതരുത്)
- പരീക്ഷിക്കുന്നതിന് മുമ്പ് `dotnet build` ഉപയോഗിച്ച് നിർമ്മിക്കുക
- ദൃശ്യ ഡീബഗിംഗിനായി ഇൻസ്പെക്ടർ ഉപയോഗിച്ച് പരീക്ഷിക്കുക
- എല്ലാ ലോഗിംഗും സ്വയം stderr-ലേക്ക് പോകും
- സെർവർ സുന്ദരമായ ഷട്ട്ഡൗൺ സിഗ്നലുകൾ കൈകാര്യം ചെയ്യുന്നു

ഈ പരിഹാരം നിലവിലുള്ള MCP സ്പെസിഫിക്കേഷൻ അനുസരിച്ച് stdio ട്രാൻസ്പോർട്ട് നടപ്പിലാക്കലിനുള്ള മികച്ച രീതികൾ കാണിക്കുന്നു, .NET ഉപയോഗിച്ച്.
