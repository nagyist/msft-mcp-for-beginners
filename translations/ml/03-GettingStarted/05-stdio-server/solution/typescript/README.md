# MCP stdio Server - TypeScript പരിഹാരം

> **⚠️ പ്രധാനപ്പെട്ടത്**: MCP സ്പെസിഫിക്കേഷൻ 2025-06-18 പ്രകാരം ശുപാർശ ചെയ്ത **stdio ട്രാൻസ്പോർട്ട്** ഉപയോഗിക്കാൻ ഈ പരിഹാരം അപ്ഡേറ്റ് ചെയ്തിരിക്കുന്നു. മുൽ SSE ട്രാൻസ്പോർട്ട് ഡിപ്രിക്കേറ്റഡ് ആയി.

## അവലോകനം

ഈ TypeScript പരിഹാരം നിലവിലുള്ള stdio ട്രാൻസ്പോർട്ട് ഉപയോഗിച്ച് MCP സെർവർ എങ്ങനെ നിർമ്മിക്കാമെന്ന് കാണിക്കുന്നു. stdio ട്രാൻസ്പോർട്ട് ലളിതവും, കൂടുതൽ സുരക്ഷിതവുമാണ്, ഡിപ്രിക്കേറ്റഡ് SSE സമീപനത്തേക്കാൾ മികച്ച പ്രകടനം നൽകുന്നു.

## മുൻകൂട്ടി ആവശ്യങ്ങൾ

- Node.js 18+ അല്ലെങ്കിൽ അതിനുശേഷം
- npm അല്ലെങ്കിൽ yarn പാക്കേജ് മാനേജർ

## സെറ്റപ്പ് നിർദ്ദേശങ്ങൾ

### ഘട്ടം 1: ആശ്രിതങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്യുക

```bash
npm install
```

### ഘട്ടം 2: പ്രോജക്ട് നിർമ്മിക്കുക

```bash
npm run build
```

## സെർവർ പ്രവർത്തിപ്പിക്കൽ

stdio സെർവർ പഴയ SSE സെർവറിനേക്കാൾ വ്യത്യസ്തമായി പ്രവർത്തിക്കുന്നു. വെബ് സെർവർ ആരംഭിക്കുന്നതിന് പകരം, ഇത് stdin/stdout വഴി ആശയവിനിമയം നടത്തുന്നു:

```bash
npm start
```

**പ്രധാനപ്പെട്ടത്**: സെർവർ ഹാംഗ് ആകുന്ന പോലെ തോന്നും - ഇത് സാധാരണമാണ്! ഇത് stdin-ൽ നിന്നുള്ള JSON-RPC സന്ദേശങ്ങൾക്കായി കാത്തിരിക്കുന്നു.

## സെർവർ ടെസ്റ്റിംഗ്

### രീതി 1: MCP ഇൻസ്പെക്ടർ ഉപയോഗിച്ച് (ശുപാർശ ചെയ്യുന്നു)

```bash
npm run inspector
```

ഇത് ചെയ്യും:
1. നിങ്ങളുടെ സെർവർ subprocess ആയി ആരംഭിക്കും
2. ടെസ്റ്റിംഗിനായി വെബ് ഇന്റർഫേസ് തുറക്കും
3. എല്ലാ സെർവർ ടൂളുകളും ഇന്ററാക്ടീവ് ആയി ടെസ്റ്റ് ചെയ്യാൻ അനുവദിക്കും

### രീതി 2: നേരിട്ട് കമാൻഡ് ലൈൻ ടെസ്റ്റിംഗ്

ഇൻസ്പെക്ടർ നേരിട്ട് ആരംഭിച്ച് നിങ്ങൾക്ക് ടെസ്റ്റ് ചെയ്യാം:

```bash
npx @modelcontextprotocol/inspector node build/index.js
```

### ലഭ്യമായ ടൂളുകൾ

സെർവർ ഈ ടൂളുകൾ നൽകുന്നു:

- **add(a, b)**: രണ്ട് സംഖ്യകൾ ചേർക്കുക
- **multiply(a, b)**: രണ്ട് സംഖ്യകൾ ഗുണിക്കുക  
- **get_greeting(name)**: വ്യക്തിഗതമായ ഒരു അഭിവാദ്യം സൃഷ്ടിക്കുക
- **get_server_info()**: സെർവറിന്റെ വിവരങ്ങൾ നേടുക

### Claude Desktop ഉപയോഗിച്ച് ടെസ്റ്റിംഗ്

ഈ സെർവർ Claude Desktop-നൊപ്പം ഉപയോഗിക്കാൻ, നിങ്ങളുടെ `claude_desktop_config.json`-ൽ ഈ കോൺഫിഗറേഷൻ ചേർക്കുക:

```json
{
  "mcpServers": {
    "example-stdio-server": {
      "command": "node",
      "args": ["path/to/build/index.js"]
    }
  }
}
```

## SSE-യുമായി പ്രധാന വ്യത്യാസങ്ങൾ

**stdio ട്രാൻസ്പോർട്ട് (നിലവിലുള്ളത്):**
- ✅ ലളിതമായ സെറ്റപ്പ് - HTTP സെർവർ ആവശ്യമില്ല
- ✅ മെച്ചപ്പെട്ട സുരക്ഷ - HTTP എൻഡ്‌പോയിന്റുകൾ ഇല്ല
- ✅ subprocess അടിസ്ഥാനത്തിലുള്ള ആശയവിനിമയം
- ✅ stdin/stdout വഴി JSON-RPC
- ✅ മെച്ചപ്പെട്ട പ്രകടനം

**SSE ട്രാൻസ്പോർട്ട് (ഡിപ്രിക്കേറ്റഡ്):**
- ❌ Express സെർവർ സെറ്റപ്പ് ആവശ്യമായിരുന്നു
- ❌ സങ്കീർണ്ണമായ റൂട്ടിംഗ്, സെഷൻ മാനേജ്മെന്റ് ആവശ്യമായിരുന്നു
- ❌ കൂടുതൽ ആശ്രിതങ്ങൾ (Express, HTTP ഹാൻഡ്ലിംഗ്)
- ❌ അധിക സുരക്ഷാ പരിഗണനകൾ
- ❌ MCP 2025-06-18-ൽ ഇപ്പോൾ ഡിപ്രിക്കേറ്റഡ്

## ഡെവലപ്പ്മെന്റ് ടിപ്പുകൾ

- ലോഗിംഗിന് `console.error()` ഉപയോഗിക്കുക (`console.log()` ഉപയോഗിക്കരുത്, അത് stdout-ലേക്ക് എഴുതുന്നു)
- ടെസ്റ്റിംഗിന് മുമ്പ് `npm run build` ഉപയോഗിച്ച് നിർമ്മിക്കുക
- ദൃശ്യ ഡീബഗിംഗിന് ഇൻസ്പെക്ടർ ഉപയോഗിച്ച് ടെസ്റ്റ് ചെയ്യുക
- എല്ലാ JSON സന്ദേശങ്ങളും ശരിയായി ഫോർമാറ്റ് ചെയ്തിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക
- സെർവർ സ്വയം SIGINT/SIGTERM-ൽ ഗ്രേസ്‌ഫുൾ ഷട്ട്ഡൗൺ കൈകാര്യം ചെയ്യുന്നു

ഈ പരിഹാരം നിലവിലുള്ള MCP സ്പെസിഫിക്കേഷൻ പാലിക്കുകയും TypeScript ഉപയോഗിച്ച് stdio ട്രാൻസ്പോർട്ട് നടപ്പിലാക്കുന്നതിൽ മികച്ച പ്രാക്ടീസുകൾ കാണിക്കുകയും ചെയ്യുന്നു.
