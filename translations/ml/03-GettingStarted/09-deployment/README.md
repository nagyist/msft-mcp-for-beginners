# MCP സേർവർസ് വിന്യസിക്കൽ

നിങ്ങളുടെ MCP സേർവർ വിന്യസിക്കുന്നത് അതിലുളള ഉപകരണങ്ങളും വിഭവങ്ങളും നിങ്ങളുടെ പ്രാദേശിക പരിസരത്തിനപ്പുറം മറ്റ് പേർക്ക് ആക്സസ് ചെയ്യാനാകുന്നതായി ചെയ്യും. വർദ്ധിച്ച തോതിലുള്ള കേന്ദ്രീകൃതത, വിശ്വാസ്യത, മാനേജുമെന്റ് എളുപ്പം തുടങ്ങിയ ആവശ്യകതകളുടെ അടിസ്ഥാനത്തിൽ നിരവധി വിന്യസിക്കൽ തന്ത്രങ്ങൾ പരിഗണിക്കാം. താഴെ MCP സേർവർസ് പ്രാദേശികമായി, കണ്ടെയ്‌നറുകളിലായി, ക്ലൗഡിലായി വിന്യസിക്കുന്നതിനുള്ള മാർഗ നിർദ്ദേശങ്ങൾ കാണാം.

## അവലോകനം

ഈ പാഠം നിങ്ങളുടെ MCP Server ആപ്പ് എങ്ങനെ വിന്യസിക്കാമെന്നത് ഉൾകൊള്ളുന്നു.

## പഠനലക്ഷ്യങ്ങൾ

ഈ പാഠത്തിന്റെ അവസാനം, നിങ്ങൾക്ക് കഴിയുന്നതു:

- വ്യത്യസ്ത വിന്യസിക്കൽ സമീപനങ്ങൾ വിലയിരുത്തുക.
- നിങ്ങളുടെ ആപ്പ് വിന്യസിക്കുക.

## പ്രാദേശിക വികസനം മയും വിന്യസിക്കൽ

നിങ്ങളുടെ സേർവർ ഉപഭോക്താക്കളുടെ യന്ത്രത്തിൽ പ്രവർത്തിക്കുന്നതിനായി ഉള്ളതെങ്കിൽ, താഴെ പറയുന്ന ഘട്ടങ്ങൾ പിന്തുടരാം:

1. **സേർവർ ഡൗൺലോകഡ് ചെയ്യുക**. നിങ്ങൾ സേർവർ എഴുതാത്തെങ്കിൽ, ആദ്യം അത് നിങ്ങളുടെ യന്ത്രത്തിൽ ഡൗൺലോഡ് ചെയ്യുക.  
1. **സേർവർ പ്രോസസ്സ് ആരംഭിക്കുക**: നിങ്ങളുടെ MCP സേർവർ അപ്ലിക്കേഷൻ പ്രവർത്തിപ്പിക്കുക 

SSE നായി (stdio തരത്തിലുള്ള സേർവർക്ക് വേണ്ടതല്ല)

1. **നെറ്റ്വർക്കിംഗ് ക്രമീകരിക്കുക**: സേർവർ പ്രതീക്ഷിച്ച പോർട്ടിൽ ആക്സസ് ചെയ്യാവുന്ന രീതിയിലേക്ക് ഉറപ്പാക്കുക  
1. **ക്ലയന്റുകൾ കണക്റ്റ് ചെയ്യുക**: `http://localhost:3000` പോലുള്ള പ്രാദേശിക കണക്ഷൻ URLs ഉപയോഗിക്കുക

## ക്ലൗഡ് വിന്യസിക്കൽ

MCP സേർവർസ് വിവിധ ക്ലൗഡ് പ്ലാറ്റ്ഫോമുകളിൽ വിന്യസിക്കാം:

- **സർവർലെസ്സ് ഫങ്ക്ഷനുകൾ**: MCP സേർവർസ് ലഘുവായവർഗ്ഗത്തിൽ സർവർലെസ്സ് ഫങ്ക്ഷനുകളായി വിന്യസിക്കുക
- **കണ്ടെയ്‌നർ സേവനങ്ങൾ**: Azure Container Apps, AWS ECS, Google Cloud Run പോലുള്ള സേവനങ്ങൾ ഉപയോഗിക്കുക
- **കുബേർണറ്റീസ്**: MCP സേർവർസ് കുബേർണറ്റീസ് ക്ലസ്റ്ററുകളിൽ ഉയർന്ന ലഭ്യതക്ക് വിന്യസിക്കുകയും മാനേജ് ചെയ്യുകയും ചെയ്യുക

### ഉദാഹരണം: Azure Container Apps

Azure Container Apps MCP സേർവർസ് വിന്യസിക്കൽ പിന്തുണയ്ക്കുന്നു. ഇത് ഇപ്പോഴും പുരോഗമനത്തിലുള്ള ഒരു പ്രവർത്തനമാണ്, ഇപ്പോൾ SSE സേർവർസ് മാത്രമേ ഇത് പിന്തുണയ്ക്കൂ.

ഇവിടെ നിങ്ങളുടെ പ്രവർത്തന മാർഗ്ഗങ്ങൾ:

1. ഒരു റിപ്പൊ ക്ലോൺ ചെയ്യുക:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. പ്രവർത്തികൾ പ്രാദേശികമായി പരിശോധിക്കാൻ ഇത് ഓടിക്കുക:

  ```sh
  uv venv
  uv sync

  # ലിനക്സ്സ്/മാക്‌ഒഎസ്
  export API_KEYS=<AN_API_KEY>
  # വിൻഡോസ്
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. പ്രാദേശികമായി പരീക്ഷിക്കാൻ, *.vscode* ഡയറക്ടറിയിൽ *mcp.json* ഫയൽ ഉണ്ടാക്കി താഴെ പറയുന്ന ഉള്ളടക്കം ചേർക്കുക:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  SSE സേർവർ ആരംഭിക്കുന്നതോടെ, JSON ഫയലിൽ പ്ലേ ഐകണിൽ ക്ലിക്ക് ചെയ്താൽ ഗിറ്റ്‌ഹബ് കോപൈലറ്റ് വഴി സേർവറിൽ ഉള്ള ഉപകരണങ്ങളുടെ ഐകൺ കാണാൻ കഴിയും.

1. വിന്യസിക്കാൻ, താഴെ പറയുന്ന കമാൻഡ് നടത്തുക:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

ഇതാണ്, പ്രാദേശികമായി വിന്യസിക്കുക, Azure ൽ ഈ ഘട്ടങ്ങളിലൂടെ വിന്യസിക്കുക.
