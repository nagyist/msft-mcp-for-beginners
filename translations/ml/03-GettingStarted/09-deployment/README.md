# MCP സെർവറുകൾ വിന്യസിക്കൽ

നിങ്ങളുടെ MCP സെർവർ വിന്യസിക്കുന്നത്, അതിന്റെ ഉപകരണങ്ങളും വിഭവങ്ങളും നിങ്ങളുടെ പ്രാദേശിക പരിസരത്തിന് പുറത്തും മറ്റുള്ളവർക്ക് ആക്‌സസ് ചെയ്യാൻ അനുവദിക്കുന്നു. സ്കെയിലബിലിറ്റി, വിശ്വാസ്യത, മാനേജ്മെന്റ് എളുപ്പം എന്നിവയുടെ ആവശ്യകതകൾ അനുസരിച്ച് പരിഗണിക്കേണ്ട വിവിധ വിന്യസന തന്ത്രങ്ങൾ ഉണ്ട്. താഴെ നിങ്ങൾക്ക് MCP സെർവറുകൾ പ്രാദേശികമായി, കണ്ടെയ്‌നറുകളിൽ, ക്ലൗഡിലേക്ക് വിന്യസിക്കുന്നതിനുള്ള മാർഗ്ഗനിർദ്ദേശങ്ങൾ ലഭിക്കും.

## അവലോകനം

ഈ പാഠം നിങ്ങളുടെ MCP സെർവർ ആപ്പ് എങ്ങനെ വിന്യസിക്കാമെന്ന് ഉൾക്കൊള്ളുന്നു.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ പാഠം അവസാനിക്കുമ്പോൾ, നിങ്ങൾക്ക് സാധിക്കും:

- വ്യത്യസ്ത വിന്യസന സമീപനങ്ങൾ വിലയിരുത്തുക.
- നിങ്ങളുടെ ആപ്പ് വിന്യസിക്കുക.

## പ്രാദേശിക വികസനവും വിന്യസനവും

നിങ്ങളുടെ സെർവർ ഉപയോക്താക്കളുടെ യന്ത്രത്തിൽ പ്രവർത്തിക്കുന്നതിനായി ഉപയോഗിക്കപ്പെടുകയാണെങ്കിൽ, താഴെ പറയുന്ന ഘട്ടങ്ങൾ പിന്തുടരാം:

1. **സെർവർ ഡൗൺലോഡ് ചെയ്യുക**. നിങ്ങൾ സെർവർ എഴുതിയിട്ടില്ലെങ്കിൽ, ആദ്യം അത് നിങ്ങളുടെ യന്ത്രത്തിലേക്ക് ഡൗൺലോഡ് ചെയ്യുക.
1. **സെർവർ പ്രക്രിയ ആരംഭിക്കുക**: നിങ്ങളുടെ MCP സെർവർ ആപ്പ് പ്രവർത്തിപ്പിക്കുക

SSE (stdio തരം സെർവറിന് ആവശ്യമില്ല)

1. **നെറ്റ്വർക്കിംഗ് ക്രമീകരിക്കുക**: സെർവർ പ്രതീക്ഷിക്കുന്ന പോർട്ടിൽ ആക്‌സസിബിൾ ആണെന്ന് ഉറപ്പാക്കുക
1. **ക്ലയന്റുകൾ കണക്ട് ചെയ്യുക**: `http://localhost:3000` പോലുള്ള പ്രാദേശിക കണക്ഷൻ URLs ഉപയോഗിക്കുക

## ക്ലൗഡ് വിന്യസനം

MCP സെർവറുകൾ വിവിധ ക്ലൗഡ് പ്ലാറ്റ്ഫോമുകളിൽ വിന്യസിക്കാം:

- **സെർവർലെസ് ഫംഗ്ഷനുകൾ**: ലഘുവായ MCP സെർവറുകൾ സെർവർലെസ് ഫംഗ്ഷനുകളായി വിന്യസിക്കുക
- **കണ്ടെയ്‌നർ സർവീസുകൾ**: Azure Container Apps, AWS ECS, Google Cloud Run പോലുള്ള സർവീസുകൾ ഉപയോഗിക്കുക
- **കുബർനെറ്റീസ്**: ഉയർന്ന ലഭ്യതയ്ക്കായി MCP സെർവറുകൾ കുബർനെറ്റീസ് ക്ലസ്റ്ററുകളിൽ വിന്യസിക്കുകയും മാനേജുചെയ്യുകയും ചെയ്യുക

### ഉദാഹരണം: Azure Container Apps

Azure Container Apps MCP സെർവറുകളുടെ വിന്യസനത്തെ പിന്തുണയ്ക്കുന്നു. ഇത് ഇപ്പോഴും പുരോഗമനത്തിലാണ്, ഇപ്പോൾ SSE സെർവറുകൾ മാത്രമേ പിന്തുണയ്ക്കൂ.

ഇങ്ങനെ ചെയ്യാം:

1. ഒരു റിപോ ക്ലോൺ ചെയ്യുക:

  ```sh
  git clone https://github.com/anthonychu/azure-container-apps-mcp-sample.git
  ```

1. പ്രാദേശികമായി പരീക്ഷിക്കാൻ ഇത് പ്രവർത്തിപ്പിക്കുക:

  ```sh
  uv venv
  uv sync

  # ലിനക്സ്/മാക്‌ഓഎസ്
  export API_KEYS=<AN_API_KEY>
  # വിൻഡോസ്
  set API_KEYS=<AN_API_KEY>

  uv run fastapi dev main.py
  ```

1. പ്രാദേശികമായി പരീക്ഷിക്കാൻ, *.vscode* ഡയറക്ടറിയിൽ *mcp.json* ഫയൽ സൃഷ്ടിച്ച് താഴെ പറയുന്ന ഉള്ളടക്കം ചേർക്കുക:

  ```json
  {
      "inputs": [
          {
              "type": "promptString",
              "id": "weather-api-key",
              "description": "Weather API Key",
              "password": true
          }
      ],
      "servers": {
          "weather-sse": {
              "type": "sse",
              "url": "http://localhost:8000/sse",
              "headers": {
                  "x-api-key": "${input:weather-api-key}"
              }
          }
      }
  }
  ```

  SSE സെർവർ ആരംഭിച്ചതിന് ശേഷം, JSON ഫയലിൽ പ്ലേ ഐക്കൺ ക്ലിക്ക് ചെയ്യാം, ഇപ്പോൾ GitHub Copilot സെർവറിൽ ഉപകരണങ്ങൾ കണ്ടെത്തുന്നതായി കാണാം, Tool ഐക്കൺ കാണുക.

1. വിന്യസിക്കാൻ, താഴെ പറയുന്ന കമാൻഡ് പ്രവർത്തിപ്പിക്കുക:

  ```sh
  az containerapp up -g <RESOURCE_GROUP_NAME> -n weather-mcp --environment mcp -l westus --env-vars API_KEYS=<AN_API_KEY> --source .
  ```

ഇവിടെ നിങ്ങൾക്ക് പ്രാദേശികമായി വിന്യസിക്കാം, ഈ ഘട്ടങ്ങളിലൂടെ Azure-ലേക്ക് വിന്യസിക്കാം.
