## Getting Started  

[![Build Your First MCP Server](../../../translated_images/04.0ea920069efd979a0b2dad51e72c1df7ead9c57b3305796068a6cee1f0dd6674.ml.png)](https://youtu.be/sNDZO9N4m9Y)

_(ഈ പാഠത്തിന്റെ വീഡിയോ കാണാൻ മുകളിൽ ചിത്രത്തിൽ ക്ലിക്ക് ചെയ്യുക)_

ഈ വിഭാഗം പല പാഠങ്ങളടങ്ങിയതാണ്:

- **1 നിങ്ങളുടെ ആദ്യ സെർവർ**, ഈ ആദ്യ പാഠത്തിൽ, നിങ്ങളുടെ ആദ്യ സെർവർ എങ്ങനെ സൃഷ്ടിക്കാമെന്ന്, ഇൻസ്പെക്ടർ ടൂൾ ഉപയോഗിച്ച് അത് പരിശോധിക്കാനും പഠിക്കും, ഇത് നിങ്ങളുടെ സെർവർ ടെസ്റ്റ് ചെയ്യാനും ഡീബഗ് ചെയ്യാനും ഒരു വിലപ്പെട്ട മാർഗമാണ്, [പാഠത്തിലേക്ക്](01-first-server/README.md)

- **2 ക്ലയന്റ്**, ഈ പാഠത്തിൽ, നിങ്ങളുടെ സെർവറുമായി കണക്റ്റ് ചെയ്യാൻ കഴിയുന്ന ഒരു ക്ലയന്റ് എങ്ങനെ എഴുതാമെന്ന് പഠിക്കും, [പാഠത്തിലേക്ക്](02-client/README.md)

- **3 LLM ഉള്ള ക്ലയന്റ്**, ക്ലയന്റ് എഴുതാനുള്ള ഒരു മെച്ചപ്പെട്ട മാർഗം LLM ചേർക്കുന്നതാണ്, അതിലൂടെ അത് നിങ്ങളുടെ സെർവറുമായി "നിഗോഷ്യേറ്റ്" ചെയ്ത് എന്ത് ചെയ്യണമെന്ന് തീരുമാനിക്കാം, [പാഠത്തിലേക്ക്](03-llm-client/README.md)

- **4 Visual Studio Code-ൽ GitHub Copilot ഏജന്റ് മോഡ് ഉപയോഗിച്ച് സെർവർ ഉപയോഗിക്കൽ**. ഇവിടെ, Visual Studio Code-ൽ നിന്ന് നമ്മുടെ MCP സെർവർ പ്രവർത്തിപ്പിക്കുന്നതിനെക്കുറിച്ച് നോക്കുന്നു, [പാഠത്തിലേക്ക്](04-vscode/README.md)

- **5 stdio ട്രാൻസ്പോർട്ട് സെർവർ** stdio ട്രാൻസ്പോർട്ട് MCP സെർവർ-ക്ലയന്റ് ആശയവിനിമയത്തിന് നിലവിലെ സ്പെസിഫിക്കേഷനിൽ ശുപാർശ ചെയ്ത സ്റ്റാൻഡേർഡ് ആണ്, സുരക്ഷിതമായ subprocess-അടിസ്ഥാനത്തിലുള്ള ആശയവിനിമയം നൽകുന്നു [പാഠത്തിലേക്ക്](05-stdio-server/README.md)

- **6 MCP ഉപയോഗിച്ച് HTTP സ്ട്രീമിംഗ് (Streamable HTTP)**. ആധുനിക HTTP സ്ട്രീമിംഗ്, പ്രോഗ്രസ് നോട്ടിഫിക്കേഷനുകൾ, Streamable HTTP ഉപയോഗിച്ച് സ്കെയിലബിൾ, റിയൽ-ടൈം MCP സെർവർകളും ക്ലയന്റുകളും എങ്ങനെ നടപ്പിലാക്കാമെന്ന് പഠിക്കുക. [പാഠത്തിലേക്ക്](06-http-streaming/README.md)

- **7 VSCode-ക്കായി AI ടൂൾകിറ്റ് ഉപയോഗിക്കൽ** നിങ്ങളുടെ MCP ക്ലയന്റുകളും സെർവറുകളും ഉപയോഗിച്ച് പരീക്ഷിക്കാനും ടെസ്റ്റ് ചെയ്യാനും [പാഠത്തിലേക്ക്](07-aitk/README.md)

- **8 ടെസ്റ്റിംഗ്**. ഇവിടെ പ്രത്യേകിച്ച് നമ്മുടെ സെർവർ, ക്ലയന്റ് വ്യത്യസ്ത മാർഗങ്ങളിൽ എങ്ങനെ ടെസ്റ്റ് ചെയ്യാമെന്ന് ശ്രദ്ധിക്കും, [പാഠത്തിലേക്ക്](08-testing/README.md)

- **9 ഡിപ്ലോയ്മെന്റ്**. ഈ അധ്യായം നിങ്ങളുടെ MCP പരിഹാരങ്ങൾ ഡിപ്ലോയ് ചെയ്യാനുള്ള വ്യത്യസ്ത മാർഗങ്ങൾ പരിശോധിക്കും, [പാഠത്തിലേക്ക്](09-deployment/README.md)

- **10 അഡ്വാൻസ്ഡ് സെർവർ ഉപയോഗം**. ഈ അധ്യായം അഡ്വാൻസ്ഡ് സെർവർ ഉപയോഗം ഉൾക്കൊള്ളുന്നു, [പാഠത്തിലേക്ക്](./10-advanced/README.md)

- **11 ഓത്ത്**. ഈ അധ്യായം ലളിതമായ ഓത്ത് ചേർക്കുന്നതിനെക്കുറിച്ചാണ്, ബേസിക് ഓത്ത് മുതൽ JWT, RBAC വരെ. നിങ്ങൾക്ക് ഇവിടെ തുടങ്ങാൻ പ്രോത്സാഹനം നൽകുന്നു, പിന്നീട് അധ്യായം 5-ലെ അഡ്വാൻസ്ഡ് വിഷയങ്ങൾ കാണുകയും അധ്യായം 2-ലെ ശുപാർശകൾ വഴി അധിക സുരക്ഷാ ശക്തീകരണം നടത്തുകയും ചെയ്യുക, [പാഠത്തിലേക്ക്](./11-simple-auth/README.md)

Model Context Protocol (MCP) ഒരു തുറന്ന പ്രോട്ടോക്കോൾ ആണ്, ആപ്ലിക്കേഷനുകൾ LLM-കൾക്ക് കോൺടെക്സ്റ്റ് നൽകുന്നത് എങ്ങനെ സ്റ്റാൻഡേർഡൈസ് ചെയ്യാമെന്ന് നിർവചിക്കുന്നു. MCP-യെ AI ആപ്ലിക്കേഷനുകൾക്കുള്ള USB-C പോർട്ട് പോലെ കരുതുക - ഇത് AI മോഡലുകൾ വിവിധ ഡാറ്റാ സ്രോതസ്സുകളിലേക്കും ടൂളുകളിലേക്കും കണക്റ്റ് ചെയ്യാനുള്ള സ്റ്റാൻഡേർഡ് മാർഗം നൽകുന്നു.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ പാഠം അവസാനിക്കുമ്പോൾ, നിങ്ങൾക്ക് കഴിയും:

- C#, Java, Python, TypeScript, JavaScript എന്നിവയിൽ MCP ഡെവലപ്പ്മെന്റ് പരിസ്ഥിതികൾ സജ്ജമാക്കുക
- കസ്റ്റം ഫീച്ചറുകളുള്ള (റിസോഴ്‌സുകൾ, പ്രോംപ്റ്റുകൾ, ടൂളുകൾ) അടിസ്ഥാന MCP സെർവർ നിർമ്മിച്ച് ഡിപ്ലോയ് ചെയ്യുക
- MCP സെർവറുകളുമായി കണക്റ്റ് ചെയ്യുന്ന ഹോസ്റ്റ് ആപ്ലിക്കേഷനുകൾ സൃഷ്ടിക്കുക
- MCP ഇംപ്ലിമെന്റേഷനുകൾ ടെസ്റ്റ് ചെയ്ത് ഡീബഗ് ചെയ്യുക
- സാധാരണ സജ്ജീകരണ പ്രശ്നങ്ങളും അവയുടെ പരിഹാരങ്ങളും മനസിലാക്കുക
- നിങ്ങളുടെ MCP ഇംപ്ലിമെന്റേഷനുകൾ ജനപ്രിയ LLM സേവനങ്ങളുമായി കണക്റ്റ് ചെയ്യുക

## നിങ്ങളുടെ MCP പരിസ്ഥിതി സജ്ജമാക്കൽ

MCP-യുമായി പ്രവർത്തനം ആരംഭിക്കുന്നതിന് മുമ്പ്, നിങ്ങളുടെ ഡെവലപ്പ്മെന്റ് പരിസ്ഥിതിയെ തയ്യാറാക്കുകയും അടിസ്ഥാന പ്രവൃത്തി പ്രവാഹം മനസിലാക്കുകയും ചെയ്യുന്നത് പ്രധാനമാണ്. MCP-യുമായി സുഖപ്രദമായ തുടക്കം ഉറപ്പാക്കാൻ ഈ വിഭാഗം പ്രാഥമിക സജ്ജീകരണ ഘട്ടങ്ങൾ വഴി നിങ്ങളെ നയിക്കും.

### മുൻകൂട്ടി ആവശ്യങ്ങൾ

MCP ഡെവലപ്പ്മെന്റിൽ പ്രവേശിക്കുന്നതിന് മുമ്പ്, നിങ്ങൾക്കുണ്ടാകേണ്ടത്:

- **ഡെവലപ്പ്മെന്റ് പരിസ്ഥിതി**: നിങ്ങളുടെ തിരഞ്ഞെടുക്കുന്ന ഭാഷയ്ക്ക് (C#, Java, Python, TypeScript, അല്ലെങ്കിൽ JavaScript)
- **IDE/എഡിറ്റർ**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm, അല്ലെങ്കിൽ ഏതെങ്കിലും ആധുനിക കോഡ് എഡിറ്റർ
- **പാക്കേജ് മാനേജർമാർ**: NuGet, Maven/Gradle, pip, അല്ലെങ്കിൽ npm/yarn
- **API കീകൾ**: നിങ്ങളുടെ ഹോസ്റ്റ് ആപ്ലിക്കേഷനുകളിൽ ഉപയോഗിക്കാൻ ഉദ്ദേശിക്കുന്ന ഏതെങ്കിലും AI സേവനങ്ങൾക്ക്

### ഔദ്യോഗിക SDKകൾ

അടുത്ത അധ്യായങ്ങളിൽ Python, TypeScript, Java, .NET ഉപയോഗിച്ച് നിർമ്മിച്ച പരിഹാരങ്ങൾ കാണാം. ഇവിടെ എല്ലാ ഔദ്യോഗികമായി പിന്തുണയ്ക്കുന്ന SDKകളും ഉണ്ട്.

MCP പല ഭാഷകൾക്കും ഔദ്യോഗിക SDKകൾ നൽകുന്നു:
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - Microsoft-നൊപ്പം സഹകരിച്ച് പരിപാലിക്കുന്നു
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI-യുമായി സഹകരിച്ച് പരിപാലിക്കുന്നു
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - ഔദ്യോഗിക TypeScript ഇംപ്ലിമെന്റേഷൻ
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - ഔദ്യോഗിക Python ഇംപ്ലിമെന്റേഷൻ
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - ഔദ്യോഗിക Kotlin ഇംപ്ലിമെന്റേഷൻ
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI-യുമായി സഹകരിച്ച് പരിപാലിക്കുന്നു
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - ഔദ്യോഗിക Rust ഇംപ്ലിമെന്റേഷൻ

## പ്രധാനപ്പെട്ട കാര്യങ്ങൾ

- MCP ഡെവലപ്പ്മെന്റ് പരിസ്ഥിതി സജ്ജമാക്കൽ ഭാഷാ-നിർദ്ദിഷ്ട SDKകളോടെ എളുപ്പമാണ്
- MCP സെർവർ നിർമ്മാണം ടൂളുകൾ സൃഷ്ടിച്ച് രജിസ്റ്റർ ചെയ്യുന്നതിൽ ഉൾപ്പെടുന്നു, വ്യക്തമായ സ്കീമകളോടെ
- MCP ക്ലയന്റുകൾ സെർവറുകളുമായി മോഡലുകളുമായി കണക്റ്റ് ചെയ്ത് വിപുലമായ കഴിവുകൾ പ്രയോജനപ്പെടുത്തുന്നു
- ടെസ്റ്റിംഗ്, ഡീബഗ്ഗിംഗ് MCP ഇംപ്ലിമെന്റേഷനുകൾക്ക് അനിവാര്യമാണ്
- ഡിപ്ലോയ്മെന്റ് ഓപ്ഷനുകൾ ലോക്കൽ ഡെവലപ്പ്മെന്റിൽ നിന്ന് ക്ലൗഡ് അടിസ്ഥാനപരമായ പരിഹാരങ്ങളിലേക്കാണ്
