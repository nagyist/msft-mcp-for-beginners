## ആരംഭിക്കുന്നത്  

[![നിങ്ങളുടെ ആദ്യ MCP സെർവർ നിർമ്മിക്കുക](../../../translated_images/ml/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(ഈ പാഠത്തിന്റെ വീഡിയോ കാണാൻ മുകളിൽ ചിത്രത്തിൽ ക്ലിക്കുചെയ്യുക)_

ഈ വിഭാഗം പല പാഠങ്ങളടങ്ങിയതാണ്:

- **1 നിങ്ങളുടെ ആദ്യ സെർവർ**, ഈ ആദ്യ പാഠത്തിൽ, നിങ്ങളുടെ ആദ്യ സെർവർ എങ്ങനെ സൃഷ്ടിക്കാമെന്ന് പഠിക്കുകയും ഇൻസ്പക്ടർ ടൂളിനെ ഉപയോഗിച്ച് അത് സംശോധന ചെയ്യുകയും ചെയ്യും, സെർവർ ടെസ്റ്റ് ചെയ്യാനും ഡീബഗ് ചെയ്യാനും വിലപ്പെട്ട മാർഗം, [പാഠത്തിൽ](01-first-server/README.md)

- **2 ക്ലയന്റ്**, ഈ പാഠത്തിൽ, നിങ്ങളുടെ സെർവറിൽ കണക്ട് ചെയ്യാൻ കഴിയുന്ന ഒരു ക്ലയന്റ് എങ്ങനെ എഴുതാമെന്ന് പഠിക്കും, [പാഠത്തിൽ](02-client/README.md)

- **3 LLM ഉള്ള ക്ലയന്റ്**, ക്ലയന്റ് എഴുതുന്നതിനുള്ള ഏറ്റവും മികച്ച മാർഗം LLM ചേർത്തുകൊണ്ട് സെർവറുമായി "ഏതുചെയ്യേണ്ടതാണെന്ന്" ചർച്ച ചെയ്യാനാകും, [പാഠത്തിൽ](03-llm-client/README.md)

- **4 Visual Studio Code-ൽ GitHub Copilot ഏജന്റ് മോഡിലൂടെ ഒരു സെർവർ ഉപഭോക്താക്കൽ**. ഇവിടെ, Visual Studio Code-ൽ നിന്ന് ഞങ്ങളുടെ MCP സെർവർ ഓടിക്കുന്നതു നോക്കുന്നു, [പാഠത്തിൽ](04-vscode/README.md)

- **5 stdio ട്രാൻспортർ സെർവർ** stdio ട്രാൻസ്പോർട്ട് ലോക്കൽ MCP സെർവർ-ക്ലയന്റ് കമ്മ്യൂണിക്കേഷനിൽ ശുപാർശ ചെയ്യപ്പെടുന്ന സ്റ്റാൻഡേർഡ് ആണ്, സുരക്ഷിതമായ subprocess-അധിഷ്ഠിത കമ്മ്യൂണിക്കേഷൻ പ്രോസസ് ഐസൊലേഷൻ ഉൾക്കൊള്ളുന്നു [പാഠത്തിൽ](05-stdio-server/README.md)

- **6 MCP-യോടുകൂടിയ HTTP സ്റ്റ്രീമിംഗ് (Streamable HTTP)**. ആധുനിക HTTP സ്റ്റ്രീമിംഗ് ട്രാൻസ്പോർട്ടിനെക്കുറിച്ച് പഠിക്കൂ ([MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http) അനുസരിച്ച് വിദൂര MCP സെർവർമാർഗ്ഗത്തിന് ശുപാർശ), പുരോഗതി നോട്ടിഫിക്കേഷനുകൾ, Streamable HTTP ഉപയോഗിച്ച് സ്കേലബിള്‍ റിയൽ-ടെം MCP സെർവറുകളും ക്ലയന്റുകളും എങ്ങനെ നടപ്പാക്കാമെന്നും പഠിക്കൂ. [പാഠത്തിൽ](06-http-streaming/README.md)

- **7 VSCode-ക്ക് AI ടൂൾകിറ്റ് ഉപയോഗിച്ച് MCP ക്ലയന്റുകളെയും സർവറുകളെയും ഉപഭോക്താക്കാനും സോഷ്യാലവ് ചെയ്യാനും** [പാഠത്തിൽ](07-aitk/README.md)

- **8 ടെസ്റ്റിംഗ്**. സെർവർയും ക്ലയന്റും വിവിധ രീതികളിൽ എങ്ങനെ ടെസ്റ്റ് ചെയ്യാമെന്ന് ഇവിടെ ശ്രദ്ധാപൂർവ്വം നോക്കുന്നു, [പാഠത്തിൽ](08-testing/README.md)

- **9 ഡിപ്ലോയ്മെന്റ്**. നിങ്ങളുടെ MCP പരിഹാരങ്ങൾ ഡിപ്ലോയുചെയ്യാനുള്ള വിവിധ മാർഗങ്ങൾ ഇവിടെ പരിശോധിക്കും, [പാഠത്തിൽ](09-deployment/README.md)

- **10 ആഡ്‌വാൻസ് സെർവർ ഉപയോഗം**. ആഡ്‌വാൻസ് സെർവർ ഉപയോഗത്തെ കുറിച്ചുള്ള അധ്യായം, [പാഠത്തിൽ](./10-advanced/README.md)

- **11 ഓത്ത്**. ഇതിൽ ലളിതമായ ഓത്ത് ചേർക്കുന്നത്, ബേസിക് ഓത്ത് മുതൽ JWT, RBAC വരെ ഉൾക്കൊള്ളുന്നു. ആദ്യമായി ഇവിടെ തുടങ്ങാൻ ആഹ്വാനം ചെയ്യുന്നു, തുടർന്ന് അധ്യായം 5-ലെ ആഡ്‌വാൻസ് ടോപിക്സും, അധ്യായം 2യിലെ സുരക്ഷാ മെച്ചപ്പെടുത്തലും പരിശോധിക്കാം, [പാഠത്തിൽ](./11-simple-auth/README.md)

- **12 MCP ഹോസ്റ്റുകൾ**. ക്ലോഡ് ഡെസ്ക്ടോപ്പ്, കേഴ്സർ, ക്ലൈൻ, വിംഡ്സർഫ് പോലുള്ള പ്രശസ്ത MCP ഹോസ്റ്റ് ക്ലയന്റുകളെ കൺഫിഗർ ചെയ്യാനും ഉപയോഗിക്കാനും പഠിക്കുക. ട്രാൻസ്പോർട്ട് തരംകളും പ്രശ്ന പരിഹാരവും, [പാഠത്തിൽ](./12-mcp-hosts/README.md)

- **13 MCP ഇൻസ്പെക്ടർ**. MCP ഇൻസ്പെക്ടർ ടൂൾ ഉപയോഗിച്ച് നിങ്ങളുടെ MCP സെർവറുകൾ ഇന്ററാക്റ്റീവായി ഡീബഗ് ചെയ്യാനും ടെസ്റ്റ് ചെയ്യാനും പഠിക്കുക. ടൂൾസ്, റിസോഴ്‌സസ്ടയും പ്രോട്ടോകോൾ സന്ദേശങ്ങളും പ്രശ്ന പരിഹാരവും മനസിലാക്കുക, [പാഠത്തിൽ](./13-mcp-inspector/README.md)

Model Context Protocol (MCP) എന്നത് ഒരു തുറന്ന പ്രോട്ടോക്കോളാണ്, ആപ്ലിക്കേഷനുകൾ LLM-കളെ 어떻게 കോൺടെക്സ്റ്റ് നൽകണമെന്ന് സ്റ്റാൻഡർഡൈസ് ചെയ്യുന്നു. MCP-യെ USB-C പോർട്ടായി AI ആപ്ലിക്കേഷനുകളിൽ ചിന്തിക്കാം - ഇത് വിവിധ ഡാറ്റ അഭ്യർത്ഥനകൾക്കും ഉപകരണങ്ങൾക്കും AI മോഡലുകൾ കണക്റ്റ് ചെയ്യാനുള്ള സ്റ്റാൻഡേർഡൈസ് മാർഗം നൽകുന്നു.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ പാഠം കഴിഞ്ഞാൽ, നിങ്ങൾ ചെയ്യാൻ കഴിയും:

- C#, ജാവ, പൈറ്റൺ, ടൈപ്പ്‌സ്‌ക്രിപ്റ്റ്, ജാവാസ്ക്രിപ്റ്റ് എന്നിവയിൽ MCP വികസന പരിസ്ഥിതി സജ്ജമാക്കുക
- അടിസ്ഥാന MCP സെർവർകൾ കസ്റ്റം ഫീച്ചറുകളോടെ (റിസോഴ്‌സുകൾ, പ്രോംപ്റ്റുകൾ, ടൂളുകൾ) നിർമ്മിച്ച് ഡിപ്ലോയ് ചെയ്യുക
- MCP സെർവറുകളുമായി കണക്ട് ചെയ്യുന്ന ഹോസ്റ്റ് അപ്ലിക്കേഷനുകൾ സൃഷ്ടിക്കുക
- MCP ഇംപ്ലിമെന്റേഷനുകൾ ടെസ്റ്റ് ചെയ്യുകയും ഡീബഗ് ചെയ്യുകയും ചെയ്യുക
- സാധാരണ സജ്ജീകരണ പ്രശ്നങ്ങളും അവയുടെ പരിഹാരങ്ങളും മനസിലാക്കുക
- നിങ്ങളുടെ MCP ഇംപ്ലിമെന്റേഷനുകൾ പ്രശസ്ത LLM സേവനങ്ങളുമായി കണക്റ്റ് ചെയ്യുക

## നിങ്ങളുടെ MCP പരിസ്ഥിതി സജ്ജീകരിക്കൽ

MCP ഉപയോഗിച്ചുചെയ്യാൻ തുടങ്ങുന്നതിന് മുമ്പ്, നിങ്ങളുടെ വികസന പരിസ്ഥിതി തയ്യാറാക്കുകയും അടിസ്ഥാന പ്രവർ‌ത്തന പ,nextുകൾ മനസിലാക്കുകയും ചെയ്യുന്നത് പ്രധാനമാണ്. ഈ വിഭാഗം MCP ഉപയോഗം സൌകര്യമാക്കാൻ തുടക്ക സജ്ജീകരണ നടപടികൾ വഴി നിങ്ങളെ ഗൈഡ് ചെയ്യും.

### മുൻ‌തയ്യാറുകൾ

MCP വികസനത്തിൽ മുക്കാന്തിയിരിക്കാനുള്ള മുൻ‌ദൃഷ്ടകൾ:

- **വികസന പരിസ്ഥിതി**: നിങ്ങളുടെ തെരഞ്ഞെടുത്ത ഭാഷയിൽ (C#, ജാവ, പൈറ്റൺ, ടൈപ്പ്‌സ്‌ക്രിപ്റ്റ്, ജാവാസ്ക്രിപ്റ്റ്)
- **ഐ.ഡി.ഇ/എഡിറ്റർ**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm, അല്ലെങ്കിൽ ഏതെങ്കിലും ആധുനിക കോഡ് എഡിറ്റർ
- **പാക്കേജ് മാനേജർമാർ**: NuGet, Maven/Gradle, pip, അല്ലെങ്കിൽ npm/yarn
- **API കീകൾ**: നിങ്ങൾ ഹോസ്റ്റ് അപ്ലിക്കേഷനുകളിൽ ഉപയോഗിക്കാൻ ഉദ്ദേശിക്കുന്ന ഏതു AI സേവനത്തിനും

### ഔദ്യോഗിക SDKകൾ

നടുത്ത അധ്യായങ്ങളിൽ Python, TypeScript, Java, .NET ഉപയോഗിച്ച് നിർമ്മിച്ച പരിഹാരങ്ങൾ കാണും. ഇവയാണ് ഔദ്യോഗികമായി പിന്തുണയ്ക്കുന്ന SDKകൾ.

MCP വിവിധ ഭാഷകളിൽ ഔദ്യോഗിക SDKകൾ നൽകുന്നു ([MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/) അനുസരിച്ച്):
- [C# SDK](https://github.com/modelcontextprotocol/csharp-sdk) - Microsoft-ന്റെ സഹകരണത്തോടെ പരിപാലിക്കുന്നു
- [Java SDK](https://github.com/modelcontextprotocol/java-sdk) - Spring AI-ന് സഹകരണം
- [TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) - ഔദ്യോഗിക TypeScript ഇംപ്ലിമെന്റേഷൻ
- [Python SDK](https://github.com/modelcontextprotocol/python-sdk) - ഔദ്യോഗിക Python ഇംപ്ലിമെന്റേഷൻ (FastMCP)
- [Kotlin SDK](https://github.com/modelcontextprotocol/kotlin-sdk) - ഔദ്യോഗിക Kotlin ഇംപ്ലിമെന്റേഷൻ
- [Swift SDK](https://github.com/modelcontextprotocol/swift-sdk) - Loopwork AI-യുടെ സഹകരണത്തോടെ പരിപാലിക്കുന്നു
- [Rust SDK](https://github.com/modelcontextprotocol/rust-sdk) - ഔദ്യോഗിക Rust ഇംപ്ലിമെന്റേഷൻ
- [Go SDK](https://github.com/modelcontextprotocol/go-sdk) - ഔദ്യോഗिक Go ഇംപ്ലിമെന്റേഷൻ

## പ്രധാനപ്പെട്ട ശേഖരങ്ങൾ

- MCP വികസന പരിസ്ഥിതി സജ്ജീകരണം ഭാഷാനുസൃത SDKകൾ ഉപയോഗിച്ച് ലളിതമാണ്
- MCP സെർവറുകൾ ടൂൾസ് സൃഷ്ടിക്കുകയും രജിസ്റ്റർ ചെയ്യുകയും വ്യക്തമായ സ്കീമകൾ ഒപ്പം നിർമിച്ചെടുക്കുന്നതാണ് പ്രധാന ഘടകം
- MCP ക്ലയന്റുകൾ സെർവർമാരും മോഡലുകളും കണക്റ്റ് ചെയ്തു വിപുലമായ ശേഷികൾ പ്രയോജനപ്പെടുത്തുന്നു
- വിശ്വസനീയമായ MCP ഇംപ്ലിമെന്റേഷനുകൾക്ക് ടെസ്റ്റിംഗും ഡീബഗിംഗും അനിവാര്യമാണ്
- ഡിപ്ലോയ്മെന്റ് ഓപ്ഷനുകൾ ലൊക്കൽ ഡെവലപ്പ്മെന്റും ക്ലൗഡ് അടിസ്ഥാന സെർവീസുകളും ഉൾക്കൊള്ളുന്നു
