# പ്രായോഗിക നടപ്പാക്കല്‍

[![How to Build, Test, and Deploy MCP Apps with Real Tools and Workflows](../../../translated_images/05.64bea204e25ca891e3dd8b8f960d2170b9a000d8364305f57db3ec4a2c049a9a.ml.png)](https://youtu.be/vCN9-mKBDfQ)

_(ഈ പാഠത്തിന്റെ വീഡിയോ കാണാന്‍ മുകളില്‍ ചിത്രത്തില്‍ ക്ലിക്ക് ചെയ്യുക)_

പ്രായോഗിക നടപ്പാക്കലാണ് മോഡല്‍ കോണ്‍ടെക്‌സ്‌റ്റ് പ്രോട്ടോക്കോള്‍ (MCP)യുടെ ശക്തി സ്പര്‍ശനീയമാകുന്നത്. MCPയുടെ സിദ്ധാന്തവും ആർക്കിടെക്ചറും മനസ്സിലാക്കുന്നത് പ്രധാനമാണെങ്കിലും, യഥാര്‍ത്ഥ മൂല്യം ഈ ആശയങ്ങള്‍ ഉപയോഗിച്ച് യാഥാര്‍ത്ഥ്യ പ്രശ്നങ്ങള്‍ പരിഹരിക്കുന്ന പരിഹാരങ്ങള്‍ നിര്‍മ്മിച്ച്, പരീക്ഷിച്ച്, വിന്യസിക്കുമ്പോഴാണ് പ്രത്യക്ഷപ്പെടുന്നത്. ഈ അധ്യായം ആശയപരമായ അറിവും പ്രായോഗിക വികസനവും തമ്മിലുള്ള ഇടവേള പൂരിപ്പിച്ച് MCP അടിസ്ഥാനമാക്കിയുള്ള ആപ്ലിക്കേഷനുകള്‍ സജീവമാക്കാനുള്ള പ്രക്രിയയിലൂടെ നിങ്ങളെ നയിക്കുന്നു.

നിങ്ങള്‍ ബുദ്ധിമുട്ടുള്ള അസിസ്റ്റന്റുകളെ വികസിപ്പിക്കുകയാണോ, ബിസിനസ് വര്‍ക്ക്ഫ്ലോകളില്‍ AI സംയോജിപ്പിക്കുകയാണോ, അല്ലെങ്കില്‍ ഡാറ്റ പ്രോസസ്സിംഗിനായി കസ്റ്റം ടൂളുകള്‍ നിര്‍മ്മിക്കുകയാണോ, MCP ഒരു ലളിതമായ അടിസ്ഥാനം നല്‍കുന്നു. അതിന്റെ ഭാഷ-സ്വതന്ത്ര രൂപകല്‍പ്പനയും ജനപ്രിയ പ്രോഗ്രാമിംഗ് ഭാഷകളിലേക്കുള്ള ഔദ്യോഗിക SDKകളും വ്യാപകമായ ഡെവലപ്പര്‍മാര്‍ക്ക് ആക്‌സസ് നല്‍കുന്നു. ഈ SDKകള്‍ ഉപയോഗിച്ച്, നിങ്ങള്‍ വേഗത്തില്‍ പ്രോട്ടോടൈപ്പ് നിര്‍മ്മിച്ച്, പുനരാവര്‍ത്തിച്ച്, വിവിധ പ്ലാറ്റ്‌ഫോമുകളിലും പരിസരങ്ങളിലും നിങ്ങളുടെ പരിഹാരങ്ങള്‍ വ്യാപിപ്പിക്കാം.

താഴെ വരുന്ന വിഭാഗങ്ങളില്‍, MCP C#, Java with Spring, TypeScript, JavaScript, Python എന്നിവയില്‍ എങ്ങനെ നടപ്പിലാക്കാമെന്ന് പ്രായോഗിക ഉദാഹരണങ്ങള്‍, സാമ്പിള്‍ കോഡ്, വിന്യസന തന്ത്രങ്ങള്‍ കാണാം. MCP സര്‍വറുകള്‍ ഡീബഗ് ചെയ്യാനും പരീക്ഷിക്കാനും, APIകള്‍ മാനേജ് ചെയ്യാനും, Azure ഉപയോഗിച്ച് ക്ലൗഡിലേക്ക് പരിഹാരങ്ങള്‍ വിന്യസിക്കാനും നിങ്ങള്‍ പഠിക്കും. ഈ പ്രായോഗിക വിഭവങ്ങള്‍ നിങ്ങളുടെ പഠനത്തെ വേഗത്തിലാക്കുകയും ഉറച്ച, പ്രൊഡക്ഷന്‍-സജ്ജമായ MCP ആപ്ലിക്കേഷനുകള്‍ ആത്മവിശ്വാസത്തോടെ നിര്‍മ്മിക്കാന്‍ സഹായിക്കുകയും ചെയ്യും.

## അവലോകനം

ഈ പാഠം MCP നടപ്പാക്കലിന്റെ പ്രായോഗിക വശങ്ങള്‍ വിവിധ പ്രോഗ്രാമിംഗ് ഭാഷകളില്‍ കേന്ദ്രീകരിക്കുന്നു. C#, Java with Spring, TypeScript, JavaScript, Python എന്നിവയില്‍ MCP SDKകള്‍ ഉപയോഗിച്ച് ഉറച്ച ആപ്ലിക്കേഷനുകള്‍ നിര്‍മ്മിക്കാനും MCP സര്‍വറുകള്‍ ഡീബഗ് ചെയ്യാനും പരീക്ഷിക്കാനും, പുനരുപയോഗയോഗ്യമായ റിസോഴ്‌സുകള്‍, പ്രോംപ്റ്റുകള്‍, ടൂളുകള്‍ സൃഷ്ടിക്കാനും പഠിക്കും.

## പഠന ലക്ഷ്യങ്ങള്‍

ഈ പാഠം പൂര്‍ത്തിയാക്കിയാല്‍, നിങ്ങള്‍ക്ക് കഴിയും:

- വിവിധ പ്രോഗ്രാമിംഗ് ഭാഷകളില്‍ ഔദ്യോഗിക SDKകള്‍ ഉപയോഗിച്ച് MCP പരിഹാരങ്ങള്‍ നടപ്പിലാക്കുക
- MCP സര്‍വറുകള്‍ ക്രമബദ്ധമായി ഡീബഗ് ചെയ്യുകയും പരീക്ഷിക്കുകയും ചെയ്യുക
- സര്‍വര്‍ ഫീച്ചറുകള്‍ (റിസോഴ്‌സുകള്‍, പ്രോംപ്റ്റുകള്‍, ടൂളുകള്‍) സൃഷ്ടിക്കുകയും ഉപയോഗിക്കുകയും ചെയ്യുക
- സങ്കീര്‍ണ്ണമായ ജോലികള്‍ക്കായി ഫലപ്രദമായ MCP വര്‍ക്ക്ഫ്ലോകള്‍ രൂപകല്‍പ്പന ചെയ്യുക
- പ്രകടനവും വിശ്വാസ്യതയും മെച്ചപ്പെടുത്താന്‍ MCP നടപ്പാക്കലുകള്‍ ഒപ്റ്റിമൈസ് ചെയ്യുക

## കോര്‍ സര്‍വര്‍ ഫീച്ചറുകള്‍

MCP സര്‍വറുകള്‍ താഴെ പറയുന്ന ഫീച്ചറുകളുടെ ഏതെങ്കിലും സംയോജനം നടപ്പിലാക്കാം:

### റിസോഴ്‌സുകള്‍

ഉപയോക്താവിനോ AI മോഡലിനോ ഉപയോഗിക്കാനുള്ള കോണ്‍ടെക്‌സ്‌റ്റും ഡാറ്റയും നല്‍കുന്നു:

- ഡോക്യുമെന്റ് റിപോസിറ്ററികള്‍
- നോളജ് ബേസുകള്‍
- ഘടനയുള്ള ഡാറ്റ സ്രോതസ്സുകള്‍
- ഫയല്‍ സിസ്റ്റങ്ങള്‍

### പ്രോംപ്റ്റുകള്‍

ഉപയോക്താക്കള്‍ക്കുള്ള ടെംപ്ലേറ്റുചെയ്ത സന്ദേശങ്ങളും വര്‍ക്ക്ഫ്ലോകളും:

- മുന്‍ നിര്‍ദ്ദേശിച്ച സംഭാഷണ ടെംപ്ലേറ്റുകള്‍
- മാര്‍ഗനിര്‍ദേശമുള്ള ഇടപെടല്‍ മാതൃകകള്‍
- പ്രത്യേക ഡയലോഗ് ഘടനകള്‍

### ടൂളുകള്‍

AI മോഡല്‍ നിര്‍വഹിക്കാനുള്ള ഫംഗ്ഷനുകള്‍:

- ഡാറ്റ പ്രോസസ്സിംഗ് ഉപകരണങ്ങള്‍
- ബാഹ്യ API സംയോജനം
- കണക്കുകൂട്ടല്‍ ശേഷികള്‍
- തിരയല്‍ ഫംഗ്ഷനാലിറ്റി

## സാമ്പിള്‍ നടപ്പാക്കലുകള്‍: C# Implementation

ഔദ്യോഗിക C# SDK റിപോസിറ്ററിയില്‍ MCPയുടെ വിവിധ വശങ്ങള്‍ കാണിക്കുന്ന നിരവധി സാമ്പിള്‍ നടപ്പാക്കലുകള്‍ ഉണ്ട്:

- **ബേസിക് MCP ക്ലയന്റ്**: MCP ക്ലയന്റ് സൃഷ്ടിച്ച് ടൂളുകള്‍ വിളിക്കുന്ന ലളിതമായ ഉദാഹരണം
- **ബേസിക് MCP സര്‍വര്‍**: അടിസ്ഥാന ടൂൾ രജിസ്ട്രേഷനുള്ള ലഘു സര്‍വര്‍ നടപ്പാക്കല്‍
- **അഡ്വാന്‍സ്ഡ് MCP സര്‍വര്‍**: ടൂൾ രജിസ്ട്രേഷന്‍, ഓതന്റിക്കേഷന്‍, പിശക് കൈകാര്യം എന്നിവയുള്ള പൂര്‍ണ്ണസവിശേഷതകള്‍ ഉള്ള സര്‍വര്‍
- **ASP.NET ഇന്റഗ്രേഷന്‍**: ASP.NET കോറുമായി സംയോജിപ്പിക്കുന്ന ഉദാഹരണങ്ങള്‍
- **ടൂൾ നടപ്പാക്കല്‍ മാതൃകകള്‍**: വ്യത്യസ്ത സങ്കീര്‍ണ്ണതകളുള്ള ടൂളുകള്‍ നടപ്പിലാക്കാനുള്ള വിവിധ മാതൃകകള്‍

MCP C# SDK പ്രിവ്യൂ ഘട്ടത്തിലാണ്, APIകള്‍ മാറാം. SDK വികസനവുമായി ഈ ബ്ലോഗ് തുടർച്ചയായി അപ്ഡേറ്റ് ചെയ്യും.

### പ്രധാന ഫീച്ചറുകള്‍

- [C# MCP Nuget ModelContextProtocol](https://www.nuget.org/packages/ModelContextProtocol)
- നിങ്ങളുടെ [ആദ്യ MCP സര്‍വര്‍](https://devblogs.microsoft.com/dotnet/build-a-model-context-protocol-mcp-server-in-csharp/) നിര്‍മ്മിക്കല്‍

പൂര്‍ണ്ണമായ C# നടപ്പാക്കല്‍ സാമ്പിളുകള്‍ക്കായി, [ഔദ്യോഗിക C# SDK സാമ്പിള്‍ റിപോസിറ്ററി](https://github.com/modelcontextprotocol/csharp-sdk) സന്ദര്‍ശിക്കുക

## സാമ്പിള്‍ നടപ്പാക്കല്‍: Java with Spring Implementation

Java with Spring SDK MCP നടപ്പാക്കലിന് എന്റര്‍പ്രൈസ്-ഗ്രേഡ് സവിശേഷതകളോടെ ശക്തമായ ഓപ്ഷനുകള്‍ നല്‍കുന്നു.

### പ്രധാന ഫീച്ചറുകള്‍

- Spring Framework ഇന്റഗ്രേഷന്‍
- ശക്തമായ ടൈപ്പ് സുരക്ഷ

## API മാനേജ്‌മെന്റ്

Azure API മാനേജ്‌മെന്റ് MCP സര്‍വറുകള്‍ എങ്ങനെ സുരക്ഷിതമാക്കാമെന്നതിന് മികച്ച പരിഹാരമാണ്. നിങ്ങളുടെ MCP സര്‍വറിന് മുന്നില്‍ Azure API മാനേജ്‌മെന്റ് ഇന്‍സ്റ്റന്‍സ് സ്ഥാപിച്ച്, താഴെ പറയുന്ന സവിശേഷതകള്‍ കൈകാര്യം ചെയ്യാന്‍ അനുവദിക്കുന്നു:

- നിരക്ക് നിയന്ത്രണം
- ടോക്കണ്‍ മാനേജ്‌മെന്റ്
- നിരീക്ഷണം
- ലോഡ് ബാലന്‍സിംഗ്
- സുരക്ഷ

## റിമോട്ട് MCP സര്‍വര്‍ Azure-ലേക്ക് വിന്യസിക്കല്‍

മുമ്പ് പറഞ്ഞ സാമ്പിള്‍ വിന്യസിക്കാമോ നോക്കാം:

1. റിപോ ക്ലോണ്‍ ചെയ്യുക

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

1. `Microsoft.App` റിസോഴ്‌സ് പ്രൊവൈഡര്‍ രജിസ്റ്റര്‍ ചെയ്യുക.

   - Azure CLI ഉപയോഗിക്കുന്നുവെങ്കില്‍, `az provider register --namespace Microsoft.App --wait` ഓടിക്കുക.
   - Azure PowerShell ഉപയോഗിക്കുന്നുവെങ്കില്‍, `Register-AzResourceProvider -ProviderNamespace Microsoft.App` ഓടിക്കുക. കുറച്ച് സമയം കഴിഞ്ഞ് `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` ഓടിച്ച് രജിസ്ട്രേഷന്‍ പൂര്‍ത്തിയാണോ എന്ന് പരിശോധിക്കുക.

1. API മാനേജ്‌മെന്റ് സര്‍വീസ്, ഫംഗ്ഷന്‍ ആപ്പ് (കോഡോടുകൂടി) എന്നിവയും മറ്റ് ആവശ്യമായ Azure റിസോഴ്‌സുകളും പ്രൊവിഷന്‍ ചെയ്യാന്‍ ഈ [azd](https://aka.ms/azd) കമാന്‍ഡ് ഓടിക്കുക

    ```shell
    azd up
    ```

    ഈ കമാന്‍ഡ് Azure-ല്‍ എല്ലാ ക്ലൗഡ് റിസോഴ്‌സുകളും വിന്യസിക്കും

### MCP ഇന്‍സ്‌പെക്ടറുമായി നിങ്ങളുടെ സര്‍വര്‍ പരീക്ഷിക്കല്‍

1. **പുതിയ ടെര്‍മിനല്‍ വിന്‍ഡോയില്‍**, MCP ഇന്‍സ്‌പെക്ടര്‍ ഇന്‍സ്റ്റാള്‍ ചെയ്ത് ഓടിക്കുക

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    ഇങ്ങനെ ഒരു ഇന്റര്‍ഫേസ് കാണാം:

    ![Connect to Node inspector](../../../translated_images/connect.141db0b2bd05f096fb1dd91273771fd8b2469d6507656c3b0c9df4b3c5473929.ml.png)

1. MCP ഇന്‍സ്‌പെക്ടര്‍ വെബ് ആപ്പ് URL (ഉദാ: [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources)) CTRL ക്ലിക്ക് ചെയ്ത് ലോഡ് ചെയ്യുക
1. ട്രാന്‍സ്‌പോര്‍ട്ട് തരം `SSE` ആയി സെറ്റ് ചെയ്യുക
1. `azd up` കഴിഞ്ഞ് കാണിച്ചിരിക്കുന്ന API മാനേജ്‌മെന്റ് SSE എന്റ്പോയിന്റ് URL സെറ്റ് ചെയ്ത് **Connect** ചെയ്യുക:

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

1. **List Tools**. ഒരു ടൂളില്‍ ക്ലിക്ക് ചെയ്ത് **Run Tool** ചെയ്യുക.

എല്ലാ ഘട്ടങ്ങളും ശരിയായി നടന്നാല്‍, MCP സര്‍വറുമായി ബന്ധിപ്പിക്കുകയും ടൂൾ വിളിക്കാനും കഴിഞ്ഞിരിക്കും.

## Azure-ക്കുള്ള MCP സര്‍വറുകള്‍

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): Python, C# .NET, Node/TypeScript ഉപയോഗിച്ച് Azure Functions ഉപയോഗിച്ച് കസ്റ്റം റിമോട്ട് MCP (Model Context Protocol) സര്‍വറുകള്‍ നിര്‍മ്മിച്ച് വിന്യസിക്കാനുള്ള ക്വിക്‌സ്റ്റാര്‍ട്ട് ടെംപ്ലേറ്റുകള്‍.

സാമ്പിളുകള്‍ ഡെവലപ്പര്‍മാര്‍ക്ക് പൂര്‍ണ്ണ പരിഹാരം നല്‍കുന്നു:

- ലോക്കലായി നിര്‍മ്മിച്ച് ഓടിക്കുക: MCP സര്‍വര്‍ ലോക്കല്‍ മെഷീനില്‍ വികസിപ്പിച്ച് ഡീബഗ് ചെയ്യുക
- Azure-ലേക്ക് വിന്യസിക്കുക: ലളിതമായ azd up കമാന്‍ഡ് ഉപയോഗിച്ച് ക്ലൗഡിലേക്ക് എളുപ്പത്തില്‍ വിന്യസിക്കുക
- ക്ലയന്റുകളില്‍ നിന്ന് കണക്ട് ചെയ്യുക: VS Code-ന്റെ Copilot ഏജന്റ് മോഡ്, MCP ഇന്‍സ്‌പെക്ടര്‍ ടൂൾ തുടങ്ങിയവ ഉള്‍പ്പെടെയുള്ള വിവിധ ക്ലയന്റുകളില്‍ നിന്ന് MCP സര്‍വറുമായി ബന്ധിപ്പിക്കുക

### പ്രധാന ഫീച്ചറുകള്‍

- ഡിസൈനില്‍ സുരക്ഷ: MCP സര്‍വര്‍ കീകളും HTTPS ഉപയോഗിച്ച് സുരക്ഷിതമാക്കുന്നു
- ഓതന്റിക്കേഷന്‍ ഓപ്ഷനുകള്‍: ബില്‍ട്ട്-ഇന്‍ ഓത്ത്, API മാനേജ്‌മെന്റ് എന്നിവ പിന്തുണയ്ക്കുന്നു
- നെറ്റ്‌വര്‍ക്ക് ഐസൊലേഷന്‍: Azure Virtual Networks (VNET) ഉപയോഗിച്ച് നെറ്റ്‌വര്‍ക്ക് ഐസൊലേഷന്‍ അനുവദിക്കുന്നു
- സര്‍വര്‍ലെസ് ആർക്കിടെക്ചര്‍: സ്കെയിലബിള്‍, ഇവന്റ്-ഡ്രിവന്‍ എക്സിക്യൂഷന്‍ വേണ്ടി Azure Functions ഉപയോഗിക്കുന്നു
- ലോക്കല്‍ വികസനം: സമഗ്രമായ ലോക്കല്‍ വികസനവും ഡീബഗിംഗും
- ലളിതമായ വിന്യസനം: Azure-ലേക്ക് streamlined വിന്യസന പ്രക്രിയ
