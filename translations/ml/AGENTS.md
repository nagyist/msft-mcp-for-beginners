# AGENTS.md

## പ്രോജക്ട് അവലോകനം

**MCP for Beginners** മോഡൽ കോൺടെക്സ്റ്റ് പ്രോട്ടോക്കോൾ (MCP) പഠിക്കുന്നതിനുള്ള ഒരു ഓപ്പൺ-സോഴ്‌സ് വിദ്യാഭ്യാസ പാഠ്യപദ്ധതിയാണ് - AI മോഡലുകളും ക്ലയന്റ് ആപ്ലിക്കേഷനുകളും തമ്മിലുള്ള ഇടപെടലുകൾക്കുള്ള ഒരു സ്റ്റാൻഡേർഡൈസ്ഡ് ഫ്രെയിംവർക്ക്. ഈ റിപോസിറ്ററി വിവിധ പ്രോഗ്രാമിംഗ് ഭാഷകളിൽ ഹാൻഡ്‌സ്-ഓൺ കോഡ് ഉദാഹരണങ്ങളോടുകൂടിയ സമഗ്രമായ പഠന സാമഗ്രികൾ നൽകുന്നു.

### പ്രധാന സാങ്കേതികവിദ്യകൾ

- **പ്രോഗ്രാമിംഗ് ഭാഷകൾ**: C#, ജാവ, ജാവാസ്ക്രിപ്റ്റ്, ടൈപ്പ്സ്ക്രിപ്റ്റ്, പൈത്തൺ, റസ്റ്റ്  
- **ഫ്രെയിംവർക്ക് & SDKകൾ**:  
  - MCP SDK (`@modelcontextprotocol/sdk`)  
  - സ്പ്രിംഗ് ബൂട്ട് (ജാവ)  
  - ഫാസ്റ്റ് MCP (പൈത്തൺ)  
  - ലാംഗ്‌ചെയിൻ4ജെ (ജാവ)  
- **ഡാറ്റാബേസുകൾ**: pgvector എക്സ്റ്റൻഷനോടുകൂടിയ പോസ്റ്റ്‌ഗ്രെഎസ്‌ക്യു‌എൽ  
- **ക്ലൗഡ് പ്ലാറ്റ്ഫോമുകൾ**: അസ്യൂർ (കണ്ടെയ്‌നർ ആപ്സ്, ഓപ്പൺഎഐ, കോൺടന്റ് സേഫ്റ്റി, ആപ്ലിക്കേഷൻ ഇൻസൈറ്റ്സ്)  
- **ബിൽഡ് ടൂളുകൾ**: npm, മേവൻ, പിപ്പ്, കാർഗോ  
- **ഡോക്യുമെന്റേഷൻ**: ഓട്ടോമേറ്റഡ് മൾട്ടി-ലാംഗ്വേജ് ട്രാൻസ്ലേഷൻ (48+ ഭാഷകൾ) ഉപയോഗിച്ചുള്ള മാർക്ക്ഡൗൺ  

### ആർക്കിടെക്ചർ

- **11 കോർ മോഡ്യൂളുകൾ (00-11)**: അടിസ്ഥാനങ്ങളിൽ നിന്ന് ഉയർന്ന തല വിഷയങ്ങളിലേക്കുള്ള ക്രമാനുസൃത പഠന പാത  
- **ഹാൻഡ്‌സ്-ഓൺ ലാബുകൾ**: വിവിധ ഭാഷകളിൽ സമ്പൂർണ്ണ പരിഹാര കോഡോടുകൂടിയ പ്രായോഗിക അഭ്യാസങ്ങൾ  
- **സാമ്പിൾ പ്രോജക്ടുകൾ**: പ്രവർത്തനക്ഷമമായ MCP സെർവർ, ക്ലയന്റ് ഇംപ്ലിമെന്റേഷനുകൾ  
- **ട്രാൻസ്ലേഷൻ സിസ്റ്റം**: മൾട്ടി-ലാംഗ്വേജ് പിന്തുണയ്ക്കുള്ള ഓട്ടോമേറ്റഡ് GitHub ആക്ഷൻസ് വർക്ക്‌ഫ്ലോ  
- **ഇമേജ് ആസറ്റുകൾ**: വിവർത്തന പതിപ്പുകളോടുകൂടിയ കേന്ദ്രികൃത ഇമേജ് ഡയറക്ടറി  

## സെറ്റപ്പ് കമാൻഡുകൾ

ഇത് ഡോക്യുമെന്റേഷൻ-കേന്ദ്രിതമായ ഒരു റിപോസിറ്ററിയാണ്. മിക്ക സെറ്റപ്പ് വ്യക്തിഗത സാമ്പിൾ പ്രോജക്ടുകളിലും ലാബുകളിലും നടക്കുന്നു.

### റിപോസിറ്ററി സെറ്റപ്പ്

```bash
# റിപ്പോസിറ്ററി ക്ലോൺ ചെയ്യുക
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### സാമ്പിൾ പ്രോജക്ടുകളുമായി പ്രവർത്തിക്കൽ

സാമ്പിൾ പ്രോജക്ടുകൾ താഴെ കാണുന്ന സ്ഥലങ്ങളിൽ സ്ഥിതിചെയ്യുന്നു:  
- `03-GettingStarted/samples/` - ഭാഷാനുസൃത ഉദാഹരണങ്ങൾ  
- `03-GettingStarted/01-first-server/solution/` - ആദ്യ സെർവർ ഇംപ്ലിമെന്റേഷനുകൾ  
- `03-GettingStarted/02-client/solution/` - ക്ലയന്റ് ഇംപ്ലിമെന്റേഷനുകൾ  
- `11-MCPServerHandsOnLabs/` - സമഗ്രമായ ഡാറ്റാബേസ് ഇന്റഗ്രേഷൻ ലാബുകൾ  

ഓരോ സാമ്പിൾ പ്രോജക്ടിനും സ്വന്തം സെറ്റപ്പ് നിർദ്ദേശങ്ങൾ ഉണ്ട്:

#### ടൈപ്പ്സ്ക്രിപ്റ്റ്/ജാവാസ്ക്രിപ്റ്റ് പ്രോജക്ടുകൾ  
```bash
cd <project-directory>
npm install
npm start
```
  
#### പൈത്തൺ പ്രോജക്ടുകൾ  
```bash
cd <project-directory>
pip install -r requirements.txt
# അല്ലെങ്കിൽ
pip install -e .
python main.py
```
  
#### ജാവ പ്രോജക്ടുകൾ  
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```
  
## ഡെവലപ്പ്മെന്റ് വർക്ക്‌ഫ്ലോ

### ഡോക്യുമെന്റേഷൻ ഘടന

- **മോഡ്യൂളുകൾ 00-11**: ക്രമാനുസൃതമായ കോർ പാഠ്യപദ്ധതി ഉള്ളടക്കം  
- **translations/**: ഭാഷാനുസൃത പതിപ്പുകൾ (ഓട്ടോ-ജനറേറ്റഡ്, നേരിട്ട് തിരുത്തരുത്)  
- **translated_images/**: പ്രാദേശികമാക്കിയ ഇമേജ് പതിപ്പുകൾ (ഓട്ടോ-ജനറേറ്റഡ്)  
- **images/**: ഉറവിട ഇമേജുകളും ഡയഗ്രാമുകളും  

### ഡോക്യുമെന്റേഷൻ മാറ്റങ്ങൾ ചെയ്യുന്നത്

1. റൂട്ട് മോഡ്യൂൾ ഡയറക്ടറികളിലുള്ള ഇംഗ്ലീഷ് മാർക്ക്ഡൗൺ ഫയലുകൾ മാത്രം തിരുത്തുക (00-11)  
2. ആവശ്യമായെങ്കിൽ `images/` ഡയറക്ടറിയിലെ ഇമേജുകൾ അപ്ഡേറ്റ് ചെയ്യുക  
3. കോ-ഓപ്പ്-ട്രാൻസ്ലേറ്റർ GitHub ആക്ഷൻ സ്വയം വിവർത്തനങ്ങൾ സൃഷ്ടിക്കും  
4. മെയിൻ ബ്രാഞ്ചിലേക്ക് പുഷ് ചെയ്തപ്പോൾ വിവർത്തനങ്ങൾ പുനഃസൃഷ്ടിക്കപ്പെടും  

### വിവർത്തനങ്ങളുമായി പ്രവർത്തിക്കൽ

- **ഓട്ടോമേറ്റഡ് ട്രാൻസ്ലേഷൻ**: GitHub ആക്ഷൻസ് വർക്ക്‌ഫ്ലോ എല്ലാ വിവർത്തനങ്ങളും കൈകാര്യം ചെയ്യുന്നു  
- `translations/` ഡയറക്ടറിയിലെ ഫയലുകൾ കൈമാറി തിരുത്തരുത്  
- ഓരോ വിവർത്തന ഫയലിലും വിവർത്തന മെറ്റാഡേറ്റയും ഉൾക്കൊള്ളുന്നു  
- പിന്തുണയുള്ള ഭാഷകൾ: അറബി, ചൈനീസ്, ഫ്രഞ്ച്, ജർമ്മൻ, ഹിന്ദി, ജാപ്പനീസ്, കൊറിയൻ, പോർച്ചുഗീസ്, റഷ്യൻ, സ്പാനിഷ് തുടങ്ങിയ 48+ ഭാഷകൾ  

## ടെസ്റ്റിംഗ് നിർദ്ദേശങ്ങൾ

### ഡോക്യുമെന്റേഷൻ സാധുത പരിശോധന

ഇത് പ്രധാനമായും ഡോക്യുമെന്റേഷൻ റിപോസിറ്ററിയായതിനാൽ, ടെസ്റ്റിംഗ് ശ്രദ്ധ കേന്ദ്രീകരിക്കുന്നത്:

1. **ലിങ്ക് സാധുത**: എല്ലാ ആന്തരിക ലിങ്കുകളും പ്രവർത്തിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക  
```bash
# തകരാറിലായ മാർക്ക്ഡൗൺ ലിങ്കുകൾ പരിശോധിക്കുക
find . -name "*.md" -type f | xargs grep -n "\[.*\](../../.*)"
```
  
2. **കോഡ് സാമ്പിൾ സാധുത**: കോഡ് ഉദാഹരണങ്ങൾ കംപൈൽ/റൺ ചെയ്യുന്നതായി പരിശോധിക്കുക  
```bash
# പ്രത്യേക സാമ്പിളിലേക്ക് നാവിഗേറ്റ് ചെയ്ത് അതിന്റെ ടെസ്റ്റുകൾ നടത്തുക
cd 03-GettingStarted/samples/typescript
npm install && npm test
```
  
3. **മാർക്ക്ഡൗൺ ലിന്റിംഗ്**: ഫോർമാറ്റിംഗ് ഏകോപന പരിശോധന  
```bash
# ആവശ്യമെങ്കിൽ markdownlint ഉപയോഗിക്കുക
npx markdownlint-cli2 "**/*.md" "#node_modules"
```
  
### സാമ്പിൾ പ്രോജക്ട് ടെസ്റ്റിംഗ്

ഓരോ ഭാഷാനുസൃത സാമ്പിൾക്കും സ്വന്തം ടെസ്റ്റിംഗ് സമീപനം ഉണ്ട്:

#### ടൈപ്പ്സ്ക്രിപ്റ്റ്/ജാവാസ്ക്രിപ്റ്റ്  
```bash
npm test
npm run build
```
  
#### പൈത്തൺ  
```bash
pytest
python -m pytest tests/
```
  
#### ജാവ  
```bash
mvn test
mvn verify
```

## ബിൽഡ് & ഡിപ്ലോയ്മെന്റ്

### ഡോക്യുമെന്റേഷൻ ഡിപ്ലോയ്മെന്റ്

റിപോസിറ്ററി ഡോക്യുമെന്റേഷൻ ഹോസ്റ്റിംഗിനായി GitHub പേജസ് അല്ലെങ്കിൽ സമാന സംവിധാനം ഉപയോഗിക്കുന്നു (പ്രയോഗയോഗ്യമെങ്കിൽ). മെയിൻ ബ്രാഞ്ചിൽ മാറ്റം വന്നാൽ:

1. ട്രാൻസ്ലേഷൻ വർക്ക്‌ഫ്ലോ (`.github/workflows/co-op-translator.yml`) പ്രവർത്തിക്കും  
2. എല്ലാ ഇംഗ്ലീഷ് മാർക്ക്ഡൗൺ ഫയലുകളുടെ ഓട്ടോമേറ്റഡ് വിവർത്തനം നടക്കും  
3. ആവശ്യമായപ്പോൾ ഇമേജ് പ്രാദേശികമാക്കൽ നടക്കും  

### ബിൽഡ് പ്രോസസ് ആവശ്യമില്ല

ഈ റിപോസിറ്ററിയിൽ പ്രധാനമായും മാർക്ക്ഡൗൺ ഡോക്യുമെന്റേഷൻ മാത്രമാണ്. കോർ പാഠ്യപദ്ധതി ഉള്ളടക്കത്തിന് കംപൈലേഷൻ അല്ലെങ്കിൽ ബിൽഡ് ഘട്ടം ആവശ്യമില്ല.

### സാമ്പിൾ പ്രോജക്ട് ഡിപ്ലോയ്മെന്റ്

വ്യക്തിഗത സാമ്പിൾ പ്രോജക്ടുകൾക്ക് ഡിപ്ലോയ്മെന്റ് നിർദ്ദേശങ്ങൾ ഉണ്ടാകാം:  
- MCP സെർവർ ഡിപ്ലോയ്മെന്റിനായി `03-GettingStarted/09-deployment/` കാണുക  
- അസ്യൂർ കണ്ടെയ്‌നർ ആപ്സ് ഡിപ്ലോയ്മെന്റ് ഉദാഹരണങ്ങൾ `11-MCPServerHandsOnLabs/`-ൽ  

## സംഭാവന മാർഗ്ഗനിർദ്ദേശങ്ങൾ

### പുൾ റിക്വസ്റ്റ് പ്രക്രിയ

1. **ഫോർക്ക് & ക്ലോൺ**: റിപോസിറ്ററി ഫോർക്ക് ചെയ്ത് നിങ്ങളുടെ ഫോർക്ക് ലോക്കലിൽ ക്ലോൺ ചെയ്യുക  
2. **ബ്രാഞ്ച് സൃഷ്ടിക്കുക**: വിവരണാത്മകമായ ബ്രാഞ്ച് പേരുകൾ ഉപയോഗിക്കുക (ഉദാ: `fix/typo-module-3`, `add/python-example`)  
3. **മാറ്റങ്ങൾ ചെയ്യുക**: ഇംഗ്ലീഷ് മാർക്ക്ഡൗൺ ഫയലുകൾ മാത്രം തിരുത്തുക (വിവർത്തനങ്ങൾ അല്ല)  
4. **ലോക്കലിൽ ടെസ്റ്റ് ചെയ്യുക**: മാർക്ക്ഡൗൺ ശരിയായി റെൻഡർ ചെയ്യുന്നതായി ഉറപ്പാക്കുക  
5. **പുൾ റിക്വസ്റ്റ് സമർപ്പിക്കുക**: വ്യക്തമായ PR തലക്കെട്ടുകളും വിവരണങ്ങളും ഉപയോഗിക്കുക  
6. **CLA**: ആവശ്യപ്പെട്ടാൽ Microsoft Contributor License Agreement ഒപ്പിടുക

- നിലവിലുള്ള ഉള്ളടക്കത്തിന്റെ വ്യക്തതയും മെച്ചപ്പെടുത്തലുകളും  
- പുതിയ കേസ് സ്റ്റഡികളും പ്രായോഗിക ഉദാഹരണങ്ങളും  
- അസ്പഷ്ടതയോ തെറ്റായ ഉള്ളടക്കമോ സംബന്ധിച്ച ഇഷ്യൂ റിപ്പോർട്ടുകൾ

- `translations/` ഡയറക്ടറിയിലെ ഫയലുകൾ നേരിട്ട് തിരുത്തരുത്

- ട്രാൻസ്ലേഷൻ വർക്ക്‌ഫ്ലോ ഫയലുകൾ അനുമതിയില്ലാതെ മാറ്റരുത്

### സാധാരണ പ്രശ്നപരിഹാരങ്ങൾ

**Q: എന്റെ PR വിവർത്തന പരിശോധനയിൽ പരാജയപ്പെടുന്നു**  
A: റൂട്ട് മോഡ്യൂൾ ഡയറക്ടറികളിലുള്ള ഇംഗ്ലീഷ് മാർക്ക്ഡൗൺ ഫയലുകൾ മാത്രമേ തിരുത്തിയിട്ടുള്ളൂ എന്ന് ഉറപ്പാക്കുക, വിവർത്തന പതിപ്പുകൾ അല്ല.

**Q: പുതിയ ഭാഷ എങ്ങനെ ചേർക്കാം?**  
A: ഭാഷ പിന്തുണ കോ-ഓപ്പ്-ട്രാൻസ്ലേറ്റർ വർക്ക്‌ഫ്ലോ വഴി നിയന്ത്രിക്കുന്നു. പുതിയ ഭാഷകൾ ചേർക്കുന്നതിന് ഒരു ഇഷ്യൂ തുറന്ന് ചർച്ച ചെയ്യുക.

**Q: കോഡ് സാമ്പിൾസ് പ്രവർത്തിക്കുന്നില്ല**  
A: പ്രത്യേക സാമ്പിളിന്റെ README-യിൽ നൽകിയ സെറ്റപ്പ് നിർദ്ദേശങ്ങൾ പാലിച്ചിട്ടുണ്ടോ എന്ന് പരിശോധിക്കുക. ആവശ്യമായ ഡിപ്പൻഡൻസികളുടെ ശരിയായ പതിപ്പുകൾ ഇൻസ്റ്റാൾ ചെയ്തിട്ടുണ്ടോ എന്ന് ഉറപ്പാക്കുക.

**Q: ചിത്രങ്ങൾ കാണിക്കുന്നില്ല**  
A: ഇമേജ് പാതകൾ സാപേക്ഷമാണെന്ന് ഉറപ്പാക്കുക, ഫോർവേഡ് സ്ലാഷുകൾ ഉപയോഗിക്കുക. ചിത്രങ്ങൾ `images/` ഡയറക്ടറിയിലോ `translated_images/` പ്രാദേശിക പതിപ്പുകളിലോ ഉണ്ടായിരിക്കണം.
