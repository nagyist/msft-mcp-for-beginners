# മാറ്റങ്ങൾ: MCP for Beginners പാഠ്യപദ്ധതി

ഈ രേഖ Model Context Protocol (MCP) for Beginners പാഠ്യപദ്ധതിയിൽ നടത്തിയ എല്ലാ പ്രധാന മാറ്റങ്ങളും രേഖപ്പെടുത്തുന്നു. മാറ്റങ്ങൾ ക്രമവിരുദ്ധമായ ക്രമത്തിൽ രേഖപ്പെടുത്തിയിരിക്കുന്നു (പുതിയ മാറ്റങ്ങൾ ആദ്യം).

## ഒക്ടോബർ 6, 2025

### Getting Started വിഭാഗം വിപുലീകരണം – അഡ്വാൻസ്ഡ് സെർവർ ഉപയോഗവും ലളിതമായ ഓതന്റിക്കേഷൻ

#### അഡ്വാൻസ്ഡ് സെർവർ ഉപയോഗം (03-GettingStarted/10-advanced)
- **പുതിയ അധ്യായം ചേർത്തു**: അഡ്വാൻസ്ഡ് MCP സെർവർ ഉപയോഗത്തിന് സമഗ്രമായ ഗൈഡ് പരിചയപ്പെടുത്തി, സാധാരണയും ലോ-ലെവൽ സെർവർ ആർക്കിടെക്ചറുകളും ഉൾപ്പെടെ.
  - **സാധാരണ vs. ലോ-ലെവൽ സെർവർ**: രണ്ട് സമീപനങ്ങൾക്കും പൈത്തൺ, ടൈപ്പ്സ്ക്രിപ്റ്റ് കോഡ് ഉദാഹരണങ്ങളോടെ വിശദമായ താരതമ്യം.
  - **ഹാൻഡ്ലർ-അധിഷ്ഠിത ഡിസൈൻ**: സ്കേലബിൾ, ഫ്ലെക്സിബിൾ സെർവർ നടപ്പാക്കലിനായി ഹാൻഡ്ലർ-അധിഷ്ഠിത ടൂൾ/റിസോഴ്‌സ്/പ്രോംപ്റ്റ് മാനേജ്മെന്റ് വിശദീകരണം.
  - **പ്രായോഗിക പാറ്റേണുകൾ**: അഡ്വാൻസ്ഡ് ഫീച്ചറുകൾക്കും ആർക്കിടെക്ചറിനും ലോ-ലെവൽ സെർവർ പാറ്റേണുകൾ എങ്ങനെ പ്രയോജനപ്പെടുന്നു എന്നുള്ള യഥാർത്ഥ ലോക സാഹചര്യങ്ങൾ.

#### ലളിതമായ ഓതന്റിക്കേഷൻ (03-GettingStarted/11-simple-auth)
- **പുതിയ അധ്യായം ചേർത്തു**: MCP സെർവറുകളിൽ ലളിതമായ ഓതന്റിക്കേഷൻ നടപ്പാക്കാനുള്ള ഘട്ടം ഘട്ടമായുള്ള ഗൈഡ്.
  - **ഓത് ആശയങ്ങൾ**: ഓതന്റിക്കേഷൻ vs. ഓതറൈസേഷൻ, ക്രെഡൻഷ്യൽ കൈകാര്യം എന്നിവയുടെ വ്യക്തമായ വിശദീകരണം.
  - **ബേസിക് ഓത് നടപ്പാക്കൽ**: പൈത്തൺ (Starlette)യും ടൈപ്പ്സ്ക്രിപ്റ്റ് (Express)ഉം ഉപയോഗിച്ച് മിഡിൽവെയർ-അധിഷ്ഠിത ഓതന്റിക്കേഷൻ പാറ്റേണുകൾ, കോഡ് സാമ്പിളുകളോടെ.
  - **അഡ്വാൻസ്ഡ് സെക്യൂരിറ്റിയിലേക്ക് പുരോഗതി**: ലളിതമായ ഓത് ആരംഭിച്ച് OAuth 2.1, RBAC എന്നിവയിലേക്ക് മുന്നോട്ട് പോകാനുള്ള മാർഗ്ഗനിർദ്ദേശം, അഡ്വാൻസ്ഡ് സെക്യൂരിറ്റി മോഡ്യൂളുകളിലേക്കുള്ള റഫറൻസുകളോടെ.

ഈ ചേർക്കലുകൾ കൂടുതൽ ശക്തമായ, സുരക്ഷിതമായ, ഫ്ലെക്സിബിൾ MCP സെർവർ നടപ്പാക്കലുകൾ നിർമ്മിക്കാൻ പ്രായോഗിക, കൈകൊണ്ടുള്ള മാർഗ്ഗനിർദ്ദേശം നൽകുന്നു, അടിസ്ഥാന ആശയങ്ങളെ അഡ്വാൻസ്ഡ് പ്രൊഡക്ഷൻ പാറ്റേണുകളുമായി ബന്ധിപ്പിക്കുന്നു.

#### കോർ കോൺസെപ്റ്റ്സ് (01-CoreConcepts/) - പൂർണ്ണമായ ആധുനികീകരണം

- **ആർക്കിടെക്ചർ റിഫൈന്മെന്റ്**: ഹോസ്റ്റുകൾ, ക്ലയന്റുകൾ, സർവറുകൾ എന്നിവയുടെ വിവരണങ്ങൾ മെച്ചപ്പെടുത്തി MCP ആർക്കിടെക്ചർ പാറ്റേണുകൾ പ്രതിഫലിപ്പിക്കാൻ
  - ഹോസ്റ്റുകൾ ഇപ്പോൾ MCP ക്ലയന്റ് കണക്ഷനുകൾ ഏകോപിപ്പിക്കുന്ന AI ആപ്ലിക്കേഷനുകളായി വ്യക്തമായി നിർവചിച്ചു
  - ക്ലയന്റുകൾ പ്രോട്ടോക്കോൾ കണക്ടറുകളായി, ഒറ്റ-ഓരോ സർവറുമായി ബന്ധം നിലനിർത്തുന്നു
  - സർവറുകൾ ലൊക്കൽ vs. റിമോട്ട് ഡിപ്ലോയ്മെന്റ് സീനാരിയോകളോടെ മെച്ചപ്പെടുത്തി
- **പ്രിമിറ്റീവ് റീസ്ട്രക്ചറിംഗ്**: സർവർ, ക്ലയന്റ് പ്രിമിറ്റീവുകളുടെ പൂർണ്ണ പുനർനിർമ്മാണം
  - സർവർ പ്രിമിറ്റീവുകൾ: റിസോഴ്‌സുകൾ (ഡാറ്റാ സോഴ്‌സുകൾ), പ്രോംപ്റ്റുകൾ (ടെംപ്ലേറ്റുകൾ), ടൂളുകൾ (എക്സിക്യൂട്ടബിൾ ഫംഗ്ഷനുകൾ) വിശദീകരണങ്ങളോടും ഉദാഹരണങ്ങളോടും
  - ക്ലയന്റ് പ്രിമിറ്റീവുകൾ: സാമ്പ്ലിംഗ് (LLM പൂർത്തീകരണങ്ങൾ), എലിസിറ്റേഷൻ (ഉപയോക്തൃ ഇൻപുട്ട്), ലോഗിംഗ് (ഡീബഗ്ഗിംഗ്/നിരീക്ഷണം)
  - നിലവിലെ ഡിസ്കവറി (`*/list`), റിട്രീവൽ (`*/get`), എക്സിക്യൂഷൻ (`*/call`) മെത്തഡ് പാറ്റേണുകൾ ഉപയോഗിച്ച് അപ്ഡേറ്റ് ചെയ്തു
- **പ്രോട്ടോക്കോൾ ആർക്കിടെക്ചർ**: രണ്ട് ലെയർ ആർക്കിടെക്ചർ മോഡൽ അവതരിപ്പിച്ചു
  - ഡാറ്റാ ലെയർ: JSON-RPC 2.0 അടിസ്ഥാനവും ലൈഫ്‌സൈക്കിൾ മാനേജ്മെന്റും പ്രിമിറ്റീവുകളും
  - ട്രാൻസ്പോർട്ട് ലെയർ: STDIO (ലൊക്കൽ) & Streamable HTTP SSE (റിമോട്ട്) ട്രാൻസ്പോർട്ട് മെക്കാനിസങ്ങൾ
- **സെക്യൂരിറ്റി ഫ്രെയിംവർക്ക്**: വ്യക്തമായ ഉപയോക്തൃ സമ്മതം, ഡാറ്റാ പ്രൈവസി സംരക്ഷണം, ടൂൾ എക്സിക്യൂഷൻ സുരക്ഷ, ട്രാൻസ്പോർട്ട് ലെയർ സെക്യൂരിറ്റി ഉൾപ്പെടെയുള്ള സമഗ്ര സെക്യൂരിറ്റി സിദ്ധാന്തങ്ങൾ
- **കമ്മ്യൂണിക്കേഷൻ പാറ്റേണുകൾ**: പ്രോട്ടോക്കോൾ സന്ദേശങ്ങൾ ഇൻഷിയലൈസേഷൻ, ഡിസ്കവറി, എക്സിക്യൂഷൻ, നോട്ടിഫിക്കേഷൻ ഫ്ലോകൾ കാണിക്കുന്നു
- **കോഡ് ഉദാഹരണങ്ങൾ**: MCP SDK പാറ്റേണുകൾ പ്രതിഫലിപ്പിക്കുന്ന .NET, ജാവ, പൈത്തൺ, ജാവാസ്ക്രിപ്റ്റ് ഭാഷകളിലെ മൾട്ടി-ലാംഗ്വേജ് ഉദാഹരണങ്ങൾ പുതുക്കി

#### സെക്യൂരിറ്റി (02-Security/) - സമഗ്ര സെക്യൂരിറ്റി പുനർനിർമ്മാണം

  - ടൂൾ പൊയിസണിംഗ് മെക്കാനിസങ്ങളും "റഗ് പുൾ" ആക്രമണ പാറ്റേണുകളും

  - AI Prompt Shields, അഡ്വാൻസ്ഡ് ഡിറ്റക്ഷൻ, സ്പോട്ട്ലൈറ്റിംഗ്, ഡെലിമിറ്റർ സാങ്കേതികവിദ്യകൾ
  - Azure Content Safety ഇന്റഗ്രേഷൻ പാറ്റേണുകൾ
  - GitHub Advanced Security സപ്ലൈ ചെയിൻ സംരക്ഷണത്തിന്
